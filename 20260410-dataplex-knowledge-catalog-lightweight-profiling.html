☁️ Dataplex: Knowledge Catalog への名称変更と Lightweight プロファイリングモード

Dataplex Announcement Preview
2026-04-10
Google Cloud

📄 概要

Dataplex に 2 つの重要なアップデートが発表されました。Dataplex Universal Catalog が Knowledge Catalog に名称変更され、データプロファイルスキャンに新しい Lightweight モード が Preview として追加されました。

Lightweight モードは低レイテンシーのプロファイルスキャンを提供し、数秒で結果を返却できます。AI エージェントのレスポンスのグラウンディングやインタラクティブなデータ探索に最適化されています。

🔄 Before / After 比較

❌ アップデート前

  • Standard モードのみで、スキャン完了まで一定時間が必要
  • リアルタイムなデータ探索や AI エージェントのレスポンス生成には不向き
  • 大規模プロファイル事前生成のコスト効率が限定的
  • 「Dataplex Universal Catalog」という名称が長く直感的でない

✅ アップデート後

  • Lightweight モードで数秒でプロファイル結果を取得可能
  • AI エージェントのグラウンディングやインタラクティブ探索がリアルタイムで可能
  • コスト効率の高いプロファイル事前生成でデータディスカバリーが向上
  • 「Knowledge Catalog」へ改称し、ナレッジ管理の目的が明確化

⚡ 主要機能

🏷️ 1. Knowledge Catalog への名称変更

→「Dataplex Universal Catalog」が「Knowledge Catalog」に改称
→API エンドポイント、クライアントライブラリ、gcloud CLI、IAM ロール名は全て変更なし
→既存のコード、スクリプト、IAM ポリシーはそのまま動作
→Google Cloud コンソール上の表記が順次更新される

⚡ 2. Lightweight プロファイリングモード (Preview)

→データプロファイルスキャンの新しい実行モードとして追加
→低レイテンシーで数秒以内にプロファイルスキャン結果を返却
→AI エージェントのレスポンスのグラウンディングに最適
→インタラクティブなデータ探索時の迅速な「ヘルスレポート」を提供
→大規模データディスカバリーのためのプロファイル事前生成をコスト効率よく実行可能

🔄 3. Standard モードとの使い分け

🔵 Standard モード

詳細な分析や長期的なデータ品質監視に適している

✓スコープ (全テーブル / インクリメンタル)
✓サンプリング設定
✓行フィルタ / 列フィルタ

🟢 Lightweight モード

速度とコスト効率を優先するユースケースに適している

✓数秒で結果返却
✓コスト効率の高い大規模スキャン
✗スコープ / フィルタ / サンプリング変更不可

🎯 ユースケース

🤖 ユースケース 1: AI エージェントによるデータ品質のリアルタイムグラウンディング

データ分析チームが社内向けの AI チャットボットを開発しており、ユーザーが「このテーブルのデータは信頼できるか?」と質問した場合に、AI エージェントがリアルタイムでデータの品質状況を回答する必要があるシナリオ。

Python Lightweight モードでオンデマンドスキャンを実行 
from google.cloud import dataplex_v1

# Lightweight モードでオンデマンドスキャンを実行
client = dataplex_v1.DataScanServiceClient()
request = dataplex_v1.RunDataScanRequest(
    name="projects/my-project/locations/us-central1/dataScans/my-scan"
)
response = client.run_data_scan(request=request)

# 数秒で返却されるプロファイル結果を AI エージェントのコンテキストに含める
profile_results = response.result
🚀 効果: AI エージェントが数秒以内にデータの統計特性 (NULL 率、ユニーク率、値分布など) を取得し、信頼性の高いレスポンスを生成できる。

📚 ユースケース 2: データカタログの自動プロファイル生成

数百のテーブルを持つデータウェアハウスにおいて、すべてのテーブルのプロファイルを効率的に生成し、データディスカバリーを促進したいシナリオ。

🚀 効果: Lightweight モードのコスト効率と速度を活かして、大量のテーブルに対するプロファイルの事前生成が実用的になる。データアナリストがテーブルを検索する際に、即座にプロファイル情報を確認できる。

🔍 ユースケース 3: インタラクティブなデータ探索

データサイエンティストが新しいデータセットを探索する際、まずデータの概要を素早く把握したいシナリオ。

🚀 効果: Lightweight モードにより、テーブルの基本的な統計情報を数秒で取得でき、探索的データ分析の初期段階を大幅に効率化できる。

✅ メリット

💼 ビジネス面

→AI エージェント活用の加速: リアルタイムにデータの特性を把握してレスポンスをグラウンディング
→データディスカバリーの効率化: 高速かつコスト効率よくプロファイルを事前生成
→ブランディングの明確化: 「Knowledge Catalog」への改名でサービスの価値提案が直感的に

🛠️ 技術面

→低レイテンシー応答: 数秒でプロファイル結果が返却され、同期的ワークフローに組み込みやすい
→後方互換性の維持: API、CLI、IAM 名が変更されていないため既存インテグレーションに影響なし
→インフラ不要: Google テナントプロジェクト上でスキャンを実行

⚠️ デメリット・制約事項

⚠️ 制限事項:
  • Lightweight モードは現在 Preview であり、本番ワークロードでの SLA は提供されない
  • Lightweight モードではスコープ、フィルタ、サンプリングサイズの変更ができない
  • Lightweight モードは BigQuery ビューおよび外部テーブルをサポートしていない
  • BIGNUMERIC カラム型を含むテーブルはスキャンできない
🚨 考慮すべき点:
  • 包括的な分析が必要な場合は Standard モードを使用する必要がある
  • Preview 段階の機能であるため、GA までに仕様が変更される可能性がある
  • 名称変更に伴い、ドキュメントや社内資料の「Universal Catalog」への参照を更新する運用作業が発生する可能性がある

💰 料金情報

データプロファイリングは Dataplex の Premium Processing SKU に該当し、Data Compute Unit (DCU) ベースで課金されます。課金は秒単位で、最低 1 分から計算されます。

項目 料金 (us-central1)
Premium Processing (デフォルト) $0.089 / DCU 時間
Premium Processing (BigQuery CUD 1 年) $0.0801 / DCU 時間
Premium Processing (BigQuery CUD 3 年) $0.0712 / DCU 時間
💡 コスト最適化のヒント: Standard モードではサンプリング、インクリメンタルスキャン、カラムフィルタ、行フィルタによるコスト削減が可能です。
⚠️ 注意: Standard Processing の無料枠 (月 100 DCU 時間) は Premium Processing SKU には適用されません。
