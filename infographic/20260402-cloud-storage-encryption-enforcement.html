🔐 Cloud Storage: バケットレベルの暗号化タイプ制御機能

Cloud Storage Security Feature
リリース日: 2026-04-02

📋 概要

Cloud Storage に、バケット単位で暗号化タイプを制御できる機能が追加されました。管理者は標準暗号化 (Google デフォルト暗号化)、顧客管理暗号鍵 (CMEK)、顧客指定暗号鍵 (CSEK) の各暗号化方式について、バケットごとに許可または禁止を設定できます。

組織のセキュリティポリシーやコンプライアンス要件に基づいて、データの暗号化方式をバケットレベルで厳密に管理することが可能になります。対象ユーザーはセキュリティ管理者、クラウドインフラ管理者、コンプライアンス担当者であり、特に金融・医療・公共セクターなど厳格なデータ保護要件を持つ組織に有用です。

🔄 Before / After 比較

❌ アップデート前の課題

  • バケットレベルで暗号化タイプを強制する手段がなかった
  • ユーザーが任意の暗号化方式でオブジェクトをアップロード可能
  • CSEK 使用時の鍵紛失によるデータ喪失リスク
  • コンプライアンス監査で暗号化方式の統一性を保証困難

✅ アップデート後の改善

  • 暗号化タイプをバケットごとに明示的に設定可能
  • 各方式に NotRestricted / FullyRestricted を個別指定
  • ポリシー違反リクエストが自動的に拒否される
  • gcloud CLI / JSON API / XML API 全てから設定可能

⚡ 主要機能

🔑 暗号化タイプの制御設定

  • 標準暗号化 (Google デフォルト暗号化)、CMEK、CSEK の 3 種類を個別に制御可能
  • 各暗号化タイプに NotRestricted (許可) または FullyRestricted (禁止) を設定
  • 最低 1 つの暗号化タイプを許可する必要がある

🛡️ 新規オブジェクト作成時の自動検証

  • アップロード、コピー、合成 (compose)、ソフト削除からの復元の全操作に適用
  • ポリシー違反のリクエストは自動的に拒否
  • 既存オブジェクトの暗号化タイプには影響しない

🔗 デフォルト KMS 鍵との整合性チェック

  • バケットにデフォルト KMS 鍵が設定されている場合、CMEK と CSEK の両方を同時に制限できない
  • 整合性のない設定を防止するバリデーションが組み込み

暗号化タイプと設定キー

暗号化タイプ gcloud 設定キー JSON API 設定キー 説明
標準暗号化 gmekEnforcement googleManagedEncryption
EnforcementConfig		 Google が管理する AES-256 暗号化
CMEK cmekEnforcement customerManagedEncryption
EnforcementConfig		 Cloud KMS で管理する暗号鍵
CSEK csekEnforcement customerSuppliedEncryption
EnforcementConfig		 ユーザーが各リクエストで提供する暗号鍵

制限モード

モード 値 動作
✅ 許可 NotRestricted その暗号化タイプで新しいオブジェクトを作成可能
❌ 禁止 FullyRestricted その暗号化タイプでの新しいオブジェクト作成を拒否

必要な IAM 権限

操作 必要な権限
新規バケット作成時に設定 storage.buckets.create

推奨される事前定義ロール roles/storage.admin (Storage Admin)

⚙️ 設定方法

📝 前提条件: roles/storage.admin ロールまたは storage.buckets.create / storage.buckets.update 権限を持つ IAM アカウントが必要です。

ステップ 1: 暗号化ポリシーの JSON ファイルを作成

JSON 暗号化ポリシー設定ファイル (CSEK を禁止する例) 
{
  "gmekEnforcement": {
    "restrictionMode": "NotRestricted"
  },
  "cmekEnforcement": {
    "restrictionMode": "NotRestricted"
  },
  "csekEnforcement": {
    "restrictionMode": "FullyRestricted"
  }
}

ステップ 2: 既存バケットに暗号化ポリシーを適用

Bash gcloud CLI で暗号化ポリシーを適用 
gcloud storage buckets update gs://BUCKET_NAME \
  --encryption-enforcement-file=ENCRYPTION_ENFORCEMENT_FILE
⚠️ 注意: 設定が反映されるまで最大 2 分かかる場合があります。

ステップ 3: JSON API を使用する場合

Bash JSON API での暗号化ポリシー適用 
curl -X PATCH --data-binary @encryption_policy.json \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  "https://storage.googleapis.com/storage/v1/b/BUCKET_NAME?fields=encryption"

JSON API を使用する場合は、設定キー名が異なります (googleManagedEncryptionEnforcementConfig 等)。

🎯 ユースケース

🛡️ ユースケース 1: ランサムウェア対策としての CSEK 制限

シナリオ: 攻撃者が CSEK を使用してデータを暗号化し、鍵を人質に身代金を要求するランサムウェア攻撃を防止したい。

JSON CSEK を禁止する設定 
{
  "gmekEnforcement": { "restrictionMode": "NotRestricted" },
  "cmekEnforcement": { "restrictionMode": "NotRestricted" },
  "csekEnforcement": { "restrictionMode": "FullyRestricted" }
}

→ CSEK によるオブジェクト作成が拒否されるため、攻撃者が独自の暗号鍵でデータを上書きすることを防止できます。

📋 ユースケース 2: CMEK 必須化によるコンプライアンス対応

シナリオ: 規制要件により、すべてのデータを組織管理の暗号鍵で暗号化する必要がある。標準暗号化や CSEK を禁止し、CMEK のみを許可したい。

JSON CMEK のみを許可する設定 
{
  "gmekEnforcement": { "restrictionMode": "FullyRestricted" },
  "cmekEnforcement": { "restrictionMode": "NotRestricted" },
  "csekEnforcement": { "restrictionMode": "FullyRestricted" }
}

→ Cloud KMS で管理された CMEK のみが許可されるため、鍵のライフサイクル管理、ローテーション、監査ログを一元的に制御できます。

⚖️ メリット・デメリット

✅ メリット

ビジネス面

  • PCI-DSS、HIPAA、SOC 2 などの規制要件に基づく暗号化ポリシーを技術的に強制可能
  • 組織全体で統一された暗号化基準を適用し、監査対応を簡素化
  • CSEK 制限で鍵紛失によるデータ喪失リスクを排除

技術面

  • 宣言的なセキュリティ設定で IaC との統合が容易
  • 全操作に自動適用されるため抜け漏れがない
  • 既存データへの非影響で段階的な移行が可能

⚠️ 制約事項・注意点

制限事項

  • 既存オブジェクトの暗号化タイプは変更されない (新規のみ)
  • デフォルト KMS 鍵設定時、CMEK と CSEK の同時制限不可
  • 設定反映まで最大 2 分の遅延
  • 全暗号化タイプの禁止は不可 (最低 1 つは許可必須)

考慮すべき点

  • 既存バケットへの適用前にワークロードの事前調査が必要
  • CSEK 使用中のワークロードは CMEK への移行を計画
  • gcloud/JSON API と XML API で省略時の動作が異なる点に注意

💰 料金情報

💡 ポイント: 暗号化タイプの制御設定自体には追加料金は発生しません。ただし、CMEK を使用する場合は Cloud KMS の料金が別途発生します。標準暗号化 (Google デフォルト暗号化) は無料です。

Cloud KMS の料金例

鍵の種類 月額料金 (概算)
ソフトウェア鍵 (Cloud KMS) $0.06 / 鍵バージョン
ハードウェア鍵 (Cloud HSM) $1.00 ~ $2.50 / 鍵バージョン
外部鍵 (Cloud EKM) $3.00 / 鍵バージョン

🌍 利用可能リージョン

Cloud Storage の暗号化タイプ制御機能は、Cloud Storage が利用可能なすべてのリージョンおよびマルチリージョンで使用できます。

🔗 関連サービス・機能

🔑 Cloud Key Management Service (Cloud KMS)

CMEK の鍵管理サービス。鍵の作成、ローテーション、アクセス制御を担当

🤖 Cloud KMS Autokey

CMEK の鍵作成と割り当てを自動化する機能。暗号化ポリシーの運用負荷を軽減

🏢 組織ポリシー (Organization Policy)

プロジェクト横断的な暗号化ガバナンスを実現する組織レベルの制約

📝 Cloud Audit Logs

暗号化ポリシーの変更やポリシー違反のリクエストを監査ログに記録

🛡️ VPC Service Controls

データ境界と暗号化ポリシーを組み合わせた多層防御の実現
