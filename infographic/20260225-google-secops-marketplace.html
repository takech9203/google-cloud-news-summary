🛡️ Google SecOps Marketplace: インテグレーションアップデート

Google SecOps Marketplace Feature / Change
📅 2026-02-25

📋 概要

Google SecOps (旧 Chronicle SOAR) の Marketplace において、2026 年 2 月 25 日付けで複数のインテグレーションがアップデートされた。Google Workspace (v23.0) では新しいアクション「Remove Extension」が追加され、Microsoft 365 Defender (v23.0) では Graph API のサポートが拡充された。また、Prisma Access (v3.1) では Ingestion コネクタにおいて GlobalProtect Integration ログの取り込みがサポートされた。

これらのアップデートにより、SOC チームはより広範なデータソースからのログ収集と、より柔軟なインシデント対応アクションが利用可能になる。

🔄 Before / After 比較

❌ アップデート前の課題

  • Google Workspace でブラウザ拡張機能の削除を自動化できなかった
  • Microsoft 365 Defender が旧 API に依存しており、最新の Graph API 機能を活用できなかった
  • Prisma Access で GlobalProtect ログを Ingestion コネクタで直接取り込めず、VPN 接続の可視性が制限されていた

✅ アップデート後の改善

  • Workspace v23.0「Remove Extension」でプレイブック経由の拡張機能自動削除が可能に
  • M365 Defender v23.0 で Graph API サポート追加、高度なクエリが実行可能に
  • Prisma Access v3.1 で GlobalProtect Integration ログの包括的な取り込みが可能に

⭐ 主要機能

📧 Google Workspace: Version 23.0 - Remove Extension アクション

  • 新しいアクション「Remove Extension」が追加された
  • プレイブック内でブラウザ拡張機能の削除を自動化できる
  • インシデント対応ワークフローの一部として、不正または脆弱な拡張機能を検出した際の自動修復に活用可能
  • Google SecOps の User エンティティに対して実行可能と想定される

🛡️ Microsoft 365 Defender: Version 23.0 - Graph API サポート拡充

  • 以下のアクションに Graph API サポートが追加:
    • Execute Query: 標準的なクエリ実行
    • Execute Custom Query: カスタムクエリの実行
    • Execute Entity Query: エンティティベースのクエリ実行
  • Microsoft Graph API により Azure AD サインインイベントを含む幅広いデータへアクセス可能
  • 従来の API と比較して、より豊富なデータフィールドとフィルタリングオプションを利用可能

🌐 Prisma Access: Version 3.1 - GlobalProtect Integration ログサポート

  • Ingestion コネクタにおいて GlobalProtect Integration ログの取り込みに対応
  • Palo Alto Networks の GlobalProtect VPN 経由のリモートアクセスイベントを Google SecOps に取り込み可能
  • VPN 接続 / 切断、認証イベント、ネットワークアクセスパターンなどのログを一元管理

前提条件

インテグレーションのアップグレード

Google SecOps コンソールから Marketplace にアクセスし、対象のインテグレーションをアップグレードする。

Console Marketplace アップグレード手順 
Google SecOps コンソール > Marketplace > Integrations
> 対象インテグレーションを検索
> "Upgrade to [バージョン]" をクリック
2

コネクタの更新 (Prisma Access の場合)

インテグレーションのアップグレード後、Ingestion コネクタの設定を更新する。

Console コネクタ更新手順 
Google SecOps コンソール > Settings > Ingestion > Connectors
> Prisma Access コネクタを選択
> "Update" をクリック
> GlobalProtect Integration ログの取り込み設定を確認
3

プレイブックへの組み込み (Google Workspace の場合)

新しい「Remove Extension」アクションをプレイブックに追加する。

Console プレイブック設定手順 
Google SecOps コンソール > Playbooks
> 既存のプレイブックを編集、または新規作成
> アクション追加 > Google Workspace > Remove Extension
> パラメータを設定して保存

⚖️ メリット・デメリット

✅ ビジネス面のメリット

  • SOC 運用効率の向上: 不正ブラウザ拡張機能の検出から削除までを自動化し、アナリストの手動対応時間を短縮
  • マルチクラウド環境の可視性強化: Microsoft 365 / Palo Alto Networks 環境からのデータ取り込みが強化
  • コンプライアンス対応の強化: VPN 接続ログの一元管理により監査要件への対応が容易に

✅ 技術面のメリット

  • Graph API 対応による拡張性: Azure AD サインインイベント、条件付きアクセスポリシーなどへのアクセスが可能
  • ログ取り込みの包括性向上: GlobalProtect ログにより VPN 通信パターンを SIEM で分析可能
  • プレイブック自動化の拡充: Remove Extension でエンドポイントセキュリティの自動化範囲が拡大

⚠️ 制限事項

  • アップグレード時に Ontology マッピングの上書きまたは保持を選択する必要あり。カスタムマッピングの事前バックアップを推奨
  • M365 Defender の API パーミッション変更後はクライアントシークレットの再生成と認証情報の更新が必要
  • GlobalProtect ログのフォーマット詳細は公式ドキュメントの更新を確認する必要あり

⚠️ 考慮すべき点

  • 既存コネクタ設定がある場合、アップグレード後にコネクタの手動更新が必要
  • Graph API への移行に伴い既存クエリの動作が変わる可能性があるため、テスト環境での検証を推奨
  • Prisma Access のログ量が大量の場合、Ingestion ボリュームへの影響を事前に見積もること

📝 まとめ

今回の Google SecOps Marketplace アップデートは、Google Workspace、Microsoft 365 Defender、Prisma Access の 3 つのインテグレーションを強化するものである。SOC チームは新しい Remove Extension アクションを活用したブラウザセキュリティの自動化、Graph API によるより柔軟な Microsoft 環境のクエリ、GlobalProtect ログの包括的な取り込みにより、マルチベンダー環境でのセキュリティ運用をさらに効率化できる。

📌 推奨アクション: 既存のインテグレーションを利用中の場合は、Marketplace からアップグレードを実行し、コネクタ設定の更新を忘れずに行うこと。
