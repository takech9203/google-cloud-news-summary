📋 Looker (Google Cloud core): コンソールでのバックアップ管理が可能に

gcloud CLI だけでなく、GUI から直感的にバックアップ運用を実現
2026-03-02
Feature (一般提供)
Looker (Google Cloud core)

📝 概要

Looker (Google Cloud core) インスタンスのバックアップ管理が、Google Cloud コンソールから直接実行できるようになりました。インスタンス詳細ページに新たに追加された Backups タブから、手動バックアップの作成、一覧表示、リストア、削除の操作が GUI ベースで行えます。

Looker (Google Cloud core) は Google Cloud 上でホストされるフルマネージドの BI プラットフォームであり、バックアップ機能はインスタンスの内部データベースとファイルサーバーのポイントインタイムスナップショットを提供します。

Before (これまで)
  • gcloud CLI のみで操作
  • CLI 学習コストが高い
  • 運用が属人化しやすい
  • 視覚的な管理が不可
→
After (今回の改善)
  • コンソール GUI で操作完結
  • 直感的なインターフェース
  • 運用の民主化を実現
  • Backups タブで一元管理

✨ 主要機能

  • 1
    Backups タブの追加
    インスタンス詳細ページに新しい Backups タブ。自動・手動バックアップを一元管理。ID、ステータス (ACTIVE / FAILED)、作成日時、有効期限を視覚的に確認可能。
  • 2
    手動バックアップの作成
    Backups タブからオンデマンドで作成。作成中もインスタンスのパフォーマンスに影響なし。
  • 3
    バックアップの一覧表示
    過去 30 日間の自動・手動バックアップを表形式で確認。ID、ステータス、作成日時、有効期限を表示。
  • 4
    バックアップからのリストア
    コンソールからバックアップを選択してリストア実行。リストア中はステータスが Updating に変わり、完了後 Active に復帰。
  • 5
    バックアップの削除
    自動・手動バックアップの両方をコンソールから削除可能。不要なバックアップの整理を直感的に実行。

手動バックアップ任意のタイミングで作成可能

操作必要なロール

📌 設定方法

前提条件

1.Looker (Google Cloud core) インスタンスが作成済み
2.Looker Admin (roles/looker.admin) が付与済み
3.Looker API が有効化済み
Bash Looker API 有効化 
gcloud services enable looker.googleapis.com \
  --project=PROJECT_ID
1
手動バックアップの作成: コンソール > Looker > インスタンス詳細 > Backups タブ > バックアップ作成
2
バックアップの一覧確認: Backups タブで自動・手動バックアップの一覧を確認
3
バックアップからのリストア: リストア前にバックアップ取得を推奨。Backups タブから対象を選択してリストア実行
4
バックアップの削除: Backups タブから不要なバックアップを選択して削除
Bash gcloud CLI でのバックアップ操作 
# 手動バックアップの作成
gcloud looker backups create \
  --instance=INSTANCE_NAME \
  --region=REGION

# バックアップの一覧確認
gcloud looker backups list \
  --instance=INSTANCE_NAME \
  --region=REGION

# バックアップからのリストア
gcloud looker instances restore INSTANCE_NAME \
  --backup=BACKUP_ID \
  --region=REGION \
  --async

# バックアップの削除
gcloud looker backups delete BACKUP_ID \
  --instance=INSTANCE_NAME \
  --region=REGION

🎯 メリット

ビジネス面

📊運用の民主化: CLI に不慣れな BI 管理者でも GUI から直感的にバックアップ運用を実行可能
⚡運用効率の向上: コマンドの記憶やドキュメント参照が不要に。操作を迅速に実行
🛡️ガバナンスの強化: バックアップ状態を視覚的に一覧確認。ポリシー遵守状況を容易に把握

技術面

🔄コンソールと CLI の両方をサポート: 既存のワークフローに影響なし
👁️バックアップ状態の視覚的確認: ステータス、作成日時、有効期限を即座に把握
📡リストア進行状況のモニタリング: インスタンスステータスでリアルタイムに確認

⚠️ デメリット・制約事項

制限事項

⛔異なるインスタンスへのリストアは不可 (インポート/エクスポート機能を使用)
⛔Elite System Activity のデータはバックアップに含まれない
⛔リストア操作中はインスタンスにログイン不可
⛔バックアップ・リストア操作は開始後キャンセル不可
⛔保持期間は 30 日間で変更不可

📋 考慮すべき点

📌リストアでバックアップ後のデータは上書きされ失われる
📌CMEK 使用時: バックアップ作成時の鍵バージョンがリストア時にも有効である必要あり。鍵ローテーション後は前の鍵バージョンを 45 日間有効に保つことを推奨
📌Looker reports 機能で削除されたレポートはリストアでも復元不可

💡 ユースケース

🔧 ユースケース 1: 障害発生時のデータ復旧

シナリオ: 誤った設定変更やデータ破損が発生し、以前の状態に戻す必要がある。

手順: 現在の状態をバックアップ → 過去のバックアップ一覧を確認 → 適切なバックアップを選択してリストア

効果: 障害発生前の状態に復元し、ビジネスへの影響を最小限に。CLI に不慣れな管理者でも迅速に復旧対応が可能。

🛡️ ユースケース 2: メンテナンス前の安全バックアップ

シナリオ: LookML プロジェクトの大規模変更やダッシュボード再構成の前に、変更前の状態を保存。

手順: インスタンス詳細 > Backups タブ > 手動バックアップ作成 > ACTIVE 確認 > メンテナンス実施 > 問題時はリストア

効果: 確実なロールバックポイントを確保し、安全にメンテナンスを実施。
