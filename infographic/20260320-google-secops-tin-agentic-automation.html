📋 概要

Google SecOps に 2 つの AI 駆動型セキュリティ機能 がリリースされた。Triage and Investigation Agent (TIN) の調査結果をケースサマリーに統合する機能と、AI エージェントをプレイブックに組み込む Agentic Automation により、セキュリティ運用の効率が大幅に向上する。

⚡ 主要機能

🔍 TIN ケースサマリー統合

Preview

→調査結果と判定サマリーをケースサマリー内で直接表示
→リアルタイム進捗更新でケースから離れず状況把握
→真陽性/偽陽性の自動判定と信頼度レベルを確認可能
→段階的ロールアウトとして提供中

🤖 Agentic Automation

Public Preview

→AI エージェントをプレイブックに直接埋め込み可能
→決定論的ステップ + AI 推論のハイブリッド構築
→3 状態管理: Not Run / Running / Completed
→非同期処理で実行キューを占有しない

🧰 TIN 調査ツールセット

→動的検索クエリ: SecOps 内検索を実行・精緻化
→GTI エンリッチメント: ドメイン、URL、ハッシュで IoC 強化
→コマンドライン分析: 自然言語でコマンドを解説
→プロセスツリー再構築: 関連アクティビティの全体像表示

🔀 Expression Builder ロジック分岐

→TIN の構造化結果を Expression Builder で参照
→verdict / confidence_level に基づく条件分岐設定
→True Positive / False Positive (高/低信頼度) で異なるパスを定義

📐 技術仕様

TIN 調査ワークフロー

フェーズ説明
初期評価アラートの詳細とメタデータを分析し、高信頼度の良性アクティビティを識別。低リスクと判定された場合は調査を終了
コンテキスト強化GTI エンリッチメント、ECG 分析、ネットワークコンテキスト収集、ケースメタデータ統合、プロセスツリー構築を並列実行
適応型調査前段階の結果に基づき動的に次の調査アクションを決定。GTI、ECG、コマンドライン分析、ターゲット検索を反復実行
判定出力真陽性/偽陽性の判定、信頼度レベル、サマリー説明を構造化データとして出力

Agentic Automation の構成要素

項目詳細
アクションタイプAutomatic (自動実行) / Manual (アナリストが手動実行)
失敗時のリトライトグルで設定可能 (エージェント実行自体の失敗に適用)
エラーハンドリングStop the playbook (停止) / Skip to the next step (スキップ)
エージェント状態Not Run / Running / Completed の 3 状態で管理
実行制限自動調査: 最大 5 件/時間 (Vertex AI キャパシティに依存)

必要な権限

chronicle.investigations.settings - エージェント設定の管理

🔧 設定方法

前提条件: Google SecOps Enterprise / Enterprise Plus / Google Unified Security パッケージ、シングルテナント構成、chronicle.investigations.settings 権限
  1. TIN エージェントの有効化
    Case Overview 画面の Gemini Investigation サイドメニュー設定から、テナントに対して TIN エージェントをオプトインする。
  2. AI エージェントステップをプレイブックに追加
    Response > Playbooks に移動し、Step Selection パネルの AI Agents カテゴリからエージェントをキャンバスにドラッグ&ドロップ。
  3. エージェントステップの設定
    Action type (Automatic / Manual)、Retry on failure、Error Handling を設定パネルで定義する。
  4. 条件分岐の設定
    AI Agent ステップの後に Flow ブロック (Conditional) を追加し、Expression Builder で verdict / confidence_level を評価して分岐パスを定義する。

✅ メリット

💼 ビジネス面

✓アナリストの生産性向上: ケースサマリーから離れずに TIN 判定を確認。コンテキストスイッチングコストを削減
✓SOC 運用の効率化: AI エージェントによる自動トリアージで Tier 1 アナリストの手動調査負荷を大幅軽減
✓対応速度の向上: TIN 判定に基づく後続アクション自動化で MTTR (平均修復時間) を短縮

⚙️ 技術面

✓ハイブリッドワークフロー: 決定論的自動化の予測可能性 + AI エージェントの柔軟性で未知のシナリオに対応
✓非同期処理: エージェント実行中もプレイブック実行キューを占有せず、並行処理のスループットを維持
✓構造化出力: TIN の判定結果を Expression Builder で参照し、プログラマティックな条件分岐が可能

⚠️ デメリット・制約事項

制限事項

✕TIN はシングルテナント構成のみ。マルチテナント (親テナント + 子サブテナント) 構成では利用不可
✕TIN は SIEM で取り込まれたデータのみ処理可能。SOAR コネクター経由のアラートは対象外
✕Agentic Automation の自動調査は最大 5 件/時間。Vertex AI キャパシティ制約の影響あり
✕両機能とも Pre-GA (Preview / Public Preview) であり、仕様変更の可能性がある

考慮すべき点

💡トライアル期間 (2026/4/1 - 6/30) 終了後の料金体系を確認する必要がある
💡未サポートアラートタイプで自動ステップがトリガーされた場合のエラーハンドリング設計が必要
💡AI 判定結果は参考情報であり、重要なセキュリティ判断にはアナリスト確認を推奨
💡TIN 調査完了は平均 60 秒、最大 20 分。リアルタイム性が必要な場面では時間制約を考慮

🎯 ユースケース

📌 ユースケース 1: SOC Tier 1 アナリストのアラートトリアージ自動化

シナリオ:

大量のセキュリティアラートを処理する SOC で、Tier 1 アナリストが日々数百件のアラートを手動トリアージ。偽陽性の割合が高く、真に対応が必要なアラートの発見が遅れている。

効果: TIN が自動調査・分類し、Agentic Automation で偽陽性 (高信頼度) を自動クローズ。アナリストはケースサマリーで判定結果を確認するだけで済み、真陽性インシデントに集中できる。

📌 ユースケース 2: インシデント対応のハイブリッド自動化

シナリオ:

特定の脅威パターン (不正なラテラルムーブメントなど) に対するインシデント対応プレイブックを運用しているが、アラートコンテキストの不足でプレイブックが停止する問題。

効果: Agentic Automation で TIN と決定論的ステップを組み合わせ、不足情報を AI が動的に補完。True Positive はホスト隔離やユーザーブロックを自動実行し、不確実な場合はアナリストにエスカレーション。

💰 料金

TIN トライアル: 2026年4月1日 - 6月30日まで無料で提供。Enterprise、Enterprise Plus、Google Unified Security の全顧客が対象。

トライアル期間中は時間あたりの調査件数に上限あり (Enterprise: 10件/時、Enterprise Plus / Google Unified Security: 20件/時)。

トライアル終了後の料金体系は、Google SecOps 担当カスタマーエンジニアに確認を推奨。

🔗 関連サービス

🌐 Google Threat Intelligence (GTI)

TIN が IoC エンリッチメントに使用。VirusTotal データも統合。

🤖 Vertex AI

Agentic Automation の AI 推論基盤。Gemini モデルによるアラート分析と判定。

🔄 Google SecOps SOAR

プレイブックの実行環境。Agentic Automation は SOAR のプレイブック機能を拡張。

📡 Google SecOps SIEM

アラート検知エンジン。TIN は SIEM で取り込まれたデータに基づいて調査を実行。

🕸️ Entity Context Graph (ECG)

エンティティの関係性と prevalence データを提供し、TIN の調査コンテキストを強化。

📝 まとめ

TIN ケースサマリー統合と Agentic Automation は、Google SecOps における AI 駆動型セキュリティ運用の大きな前進 である。アナリストはケースサマリーから離れることなく AI の調査結果を活用でき、プレイブック内で決定論的ステップと AI エージェントを組み合わせた適応型ワークフローを構築できる。

両機能ともプレビュー段階のため、トライアル期間 (2026/4/1 - 6/30) を活用して自組織のアラートトリアージワークフローへの適用を検証することが推奨される。
