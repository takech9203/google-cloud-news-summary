⚡ Gemini Enterprise: カスタム MCP サーバーデータストア

2026-04-28
Public Preview Gemini Enterprise MCP

📋 概要

Gemini Enterprise に、カスタム Model Context Protocol (MCP) サーバーを接続してデータストアとして利用できる新機能が Public Preview としてリリースされました。この機能により、企業が保有するプライベートデータ、社内カスタムツール、MCP 準拠のサードパーティシステムに対して、Gemini Enterprise から安全にアクセスできるようになります。

🤖 MCP とは?

Model Context Protocol (MCP) は、大規模言語モデル (LLM) や AI アプリケーションが外部データソースに接続する方法を標準化するオープンプロトコルです。MCP サーバーとして公開されたデータソースであれば、標準コネクタライブラリに含まれないものでも Gemini Enterprise と統合し、自然言語による検索やデータアクセスが可能になります。

⚠️ セキュリティ・バイ・デフォルト

この機能はデフォルトで無効化されています。利用するには Organization Policy Administrator が組織の制約を解除する必要があります。これにより、企業は外部 MCP サーバーへの接続を組織レベルで統制できます。

🔄 Before / After 比較

❌ Before (アップデート前)

  • 利用可能なデータソースは Google データソース (Drive, Gmail) と標準サードパーティコネクタ (SharePoint, Jira, Confluence) に限定
  • 社内独自 DB やカスタムツールの統合には Discovery Engine API でカスタムコネクタ開発が必要 (高い技術的ハードル)
  • MCP 準拠サードパーティシステムとの直接統合手段がなく、MCP エコシステムの拡大に対応できなかった

✅ After (アップデート後)

  • カスタム MCP サーバーを Gemini Enterprise のデータストアとして直接接続可能に (統合対象が大幅に拡大)
  • MCP プロトコルに準拠したサーバーであれば追加開発なしで接続可能
  • 組織ポリシーによる制御により、セキュリティを維持しながら柔軟なデータ統合を実現

⚡ 主要機能

🔗 カスタム MCP サーバー接続

  • MCP プロトコルに準拠した任意のサーバーを Gemini Enterprise のデータストアとして登録可能
  • 社内プライベートデータ、カスタム内部ツール、MCP 準拠サードパーティシステムへのアクセスを実現
  • データソースタイプとして custom_mcp を使用して DataConnector リソースを作成

🔒 組織ポリシーによるアクセス制御

  • 機能はデフォルトで無効化 (セキュリティ・バイ・デフォルト)
  • Organization Policy Administrator が組織の制約 (constraint) を解除することで有効化
  • カスタム制約を使用して、MCP コネクタの作成を組織/フォルダ/プロジェクトレベルで制御可能
  • 制約例: resource.dataSource == 'custom_mcp' 条件で CREATE を DENY/ALLOW

🛡️ Agent Gateway によるエグレスガバナンス

  • カスタム MCP サーバーへの通信は Agent Gateway を経由し、セキュリティポリシーを適用可能
  • Agent Registry に登録された MCP サーバーのみ通信を許可
  • Model Armor との統合により、プロンプトインジェクション攻撃からの防御が可能

📄 組織ポリシーの制約設定例

カスタム MCP コネクタの作成を制御する組織ポリシーの例です。

YAML カスタム MCP コネクタの作成を禁止する制約 
# カスタム MCP コネクタの作成を禁止する制約の例
name: organizations/ORGANIZATION_ID/customConstraints/custom.restrictCustomMcpConnector
resourceTypes:
  discoveryengine.googleapis.com/DataConnector
condition: "resource.dataSource == 'custom_mcp'"
actionType: DENY
methodTypes:
  - CREATE
displayName: Restrict custom MCP connectors
description: Custom MCP connectors cannot be created.

🛠️ 設定方法

📋 前提条件
  • Gemini Enterprise が有効化された Google Cloud プロジェクト
  • Organization Policy Administrator (roles/orgpolicy.policyAdmin) ロール
  • Discovery Engine Editor (roles/discoveryengine.editor) ロール
  • カスタム MCP サーバーが構築・稼働済みであること

組織ポリシーの制約を解除する

組織ポリシー管理者がカスタム MCP データストアの組織制約を解除します。変更の反映には最大 15 分程度かかる場合があります。

Bash 制約解除の概念例 
# 組織ポリシーの制約解除 (具体的なコマンドは公式ドキュメントに従う)
gcloud org-policies set-policy POLICY_FILE.yaml

カスタム MCP サーバーをセットアップする

カスタム MCP サーバーを Gemini Enterprise に接続するための設定を行います。公式ドキュメントの手順に従ってください。

データストアの作成と接続

Gemini Enterprise コンソールで MCP データストアを作成し、アプリケーションに接続します。

  • Google Cloud コンソールで Gemini Enterprise ページに移動
  • ナビゲーションメニューから Data Stores をクリック
  • Create Data Store をクリック
  • データソースとしてカスタム MCP サーバーを選択
  • MCP サーバーの接続情報を設定
  • データストアを作成後、既存のアプリに接続するか、新しいアプリを作成して接続

⚖️ メリット / 制約事項

✅ メリット

ビジネス面

  • 標準コネクタでカバーされない社内システムやニッチなサービスのデータを検索対象に追加可能
  • MCP エコシステムの急速な普及に伴い、対応ツールやサービスをすぐに統合利用
  • 組織ポリシーによる制御で部門/プロジェクト単位の段階的導入が可能

技術面

  • MCP はオープンスタンダードであり、ベンダーロックインリスクが低い
  • デフォルト無効で意図しない外部接続リスクを排除
  • Agent Gateway による認可ポリシー適用、トラフィック監視、Model Armor 連携
  • 従来の Discovery Engine API ベースのカスタムコネクタとの共存が可能

⚠️ 制約事項・考慮点

制限事項

  • Public Preview のため「Pre-GA Offerings Terms」が適用、サポートが限定的な場合あり
  • 利用開始前に Organization Policy Administrator による制約解除が必須
  • Agent Gateway 統合時は追加のネットワーク設定とリソースプロビジョニングが発生

考慮すべき点

  • カスタム MCP サーバーの可用性・性能は利用者側の責任 (運用体制の整備が必要)
  • データアクセスのレイテンシーはサーバー応答速度に依存
  • 組織ポリシー変更は階層全体に影響するため、事前に影響範囲を評価
  • Preview 機能のため GA 時に仕様変更の可能性あり

💡 ユースケース

🏭 ユースケース 1: 社内ナレッジベースの統合

シナリオ: 大規模な製造業の企業が、社内で運用している独自のナレッジベースシステム (Wiki、技術文書管理システム) を MCP サーバーとして公開し、Gemini Enterprise から従業員が自然言語で社内情報を検索できるようにしたい。

✨ 効果: 従業員は Gemini Enterprise の検索インターフェースから、Google Drive や SharePoint のデータに加えて、社内独自システムのデータもまとめて検索可能に。情報の分散が解消され、必要な情報へのアクセス時間が短縮されます。

💻 ユースケース 2: MCP 準拠サードパーティツールとの統合

シナリオ: ソフトウェア開発企業が、GitHub、Linear、Notion など MCP サーバーを提供するサードパーティツールを Gemini Enterprise に接続し、開発チームの生産性を向上させたい。

✨ 効果: MCP プロトコルに準拠したサードパーティツールを個別のカスタムコネクタ開発なしで接続可能。開発者は横断的にプロジェクト情報を検索し、関連するイシュー、ドキュメント、コード情報に素早くアクセスできます。

🏦 ユースケース 3: レガシーシステムの段階的な AI 統合

シナリオ: 金融機関が、オンプレミスで運用するレガシーな顧客管理システムや取引記録データベースに MCP サーバーのインターフェースを構築し、Gemini Enterprise と安全に接続したい。

✨ 効果: Agent Gateway によるエグレスガバナンスと組織ポリシーによるアクセス制御を組み合わせ、機密性の高い金融データへのアクセスをセキュアに制御しながら、AI によるデータ活用を実現。部門単位で段階的に展開し、リスクを最小化できます。

🔗 関連サービス・機能

☁️ Gemini Enterprise コネクタ Google データソースおよびサードパーティデータソースを接続する標準コネクタ群。カスタム MCP サーバーはこれを補完する位置づけ。
🔌 カスタムコネクタ (Discovery Engine API) Discovery Engine API を直接使用してカスタムデータソースを統合する既存方式。MCP サーバー方式はより標準化されたアプローチ。
🛡️ Agent Gateway Gemini Enterprise からの外部通信を制御するネットワーキングコンポーネント。MCP サーバーへの通信の認可とセキュリティを担当。
📚 Agent Registry エージェント、ツール、MCP サーバーを登録・管理する中央レジストリ。Agent Gateway が通信を許可する対象を管理。
🏢 Organization Policy Service Google Cloud リソースの制約を組織レベルで管理するサービス。カスタム MCP コネクタの利用可否を制御。
🛡️ Model Armor Agent Gateway と統合し、MCP 通信におけるプロンプトインジェクション攻撃や有害コンテンツを検出・ブロック。

📌 まとめ

Gemini Enterprise にカスタム MCP サーバーデータストア機能が Public Preview として追加されたことで、企業が保有するあらゆるデータソースを MCP プロトコル経由で Gemini Enterprise と統合する道が開かれました。セキュリティ・バイ・デフォルトの設計により組織レベルでの統制が確保されており、Agent Gateway によるガバナンスと組み合わせることで、エンタープライズ要件を満たしたデータ統合が可能です。

MCP エコシステムの拡大を背景に、今後の GA リリースに向けて早期に評価・検証を開始することを推奨します。
