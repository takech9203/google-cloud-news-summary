📋 概要

Gemini 3.1 Flash Image が Public Preview として利用可能になりました。画像理解と画像生成に最適化されたマルチモーダルモデルで、テキストと画像を入力として受け取り、テキストと画像の両方を出力できます。Google は画像生成を行う際に本モデルの使用を推奨しています。

🤖 Thinking (思考プロセス) 機能を新搭載

画像生成リクエストに対してより高度な推論を実行可能。前世代の Gemini 2.5 Flash Image ではサポートされていなかった新機能です。

⚡ 主要機能

🖼️ 高品質画像生成

テキストプロンプトから高品質な画像を生成。12 種類のアスペクト比をサポート: 1:1, 3:2, 2:3, 3:4, 4:1, 4:3, 4:5, 5:4, 8:1, 9:16, 16:9, 21:9。画像生成あたり最大 2,520 トークンを消費。

🔀 マルチモーダル入力対応

テキストと画像の同時入力で画像編集・合成が可能。最大 14 枚の入力画像、PDF・プレーンテキストドキュメントにも対応。GCS 経由で最大 30 MB のファイルを入力可能。

🧠 Thinking (思考プロセス) 機能

複雑なプロンプトに対して推論プロセスを実行。前世代では非サポートだった新機能で、より精緻で意図に沿った画像生成を実現。

📝 テキストと画像のインターリーブ出力

テキストと画像を混在させた出力が可能。画像の説明や文脈をテキストで補完しながら画像を生成。

⚙️ システムインストラクション対応

システムインストラクションによるモデル動作のカスタマイズが可能。一貫したスタイルやトーンでの画像生成を設定可能。

🔧 設定方法

前提条件

✅ Google Cloud プロジェクトの作成と課金の有効化
✅ Vertex AI API の有効化
✅ 適切な IAM 権限の設定

ステップ 1: Vertex AI Studio からの利用

1. Vertex AI Studio > Create prompt を開く
2. 「Switch model」をクリックし、gemini-3.1-flash-image-preview を選択
3. Outputs パネルで「Image and text」を選択
4. テキストプロンプトを入力して「Prompt」ボタンをクリック

ステップ 2: Python SDK での利用

Bash SDK インストール 
pip install --upgrade google-genai
Bash 環境変数設定 
export GOOGLE_CLOUD_PROJECT=YOUR_PROJECT_ID
export GOOGLE_CLOUD_LOCATION=global
export GOOGLE_GENAI_USE_VERTEXAI=True
Python 画像生成サンプルコード 
from google import genai
from google.genai.types import GenerateContentConfig, Modality
from PIL import Image
from io import BytesIO
import os

client = genai.Client()

response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=(
        "Generate an image of the Eiffel tower with fireworks in the background."
    ),
    config=GenerateContentConfig(
        response_modalities=[Modality.TEXT, Modality.IMAGE],
    ),
)

for part in response.candidates[0].content.parts:
    if part.text:
        print(part.text)
    elif part.inline_data:
        image = Image.open(BytesIO(part.inline_data.data))
        output_dir = "output_folder"
        os.makedirs(output_dir, exist_ok=True)
        image.save(os.path.join(output_dir, "generated-image.png"))

ステップ 3: REST API での利用

Bash cURL リクエスト例 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  "https://global-aiplatform.googleapis.com/v1/projects/${PROJECT_ID}/locations/global/publishers/google/models/gemini-3.1-flash-image-preview:generateContent" \
  -d '{
    "contents": [{
      "parts": [{
        "text": "Generate an image of a sunset over the ocean."
      }]
    }],
    "generationConfig": {
      "responseModalities": ["TEXT", "IMAGE"]
    }
  }'

131K トークンの入力ウィンドウにより、ブランドガイドライン、スタイルリファレンス、詳細な仕様書を含む複雑なプロンプトに対応可能。

🧠 マルチモーダル推論の強化

Thinking 機能により複雑な指示に対する理解度が向上し、意図に沿った高品質な画像を生成。

🖼️ 大幅に増加した入力画像数

最大 14 枚の入力画像を参照可能になり、スタイル転送やコンポジション合成の精度が向上。

📅 知識カットオフの更新

2025 年 1 月までの知識を保持しており、最新のトレンドやコンテキストを反映した画像生成が可能。

🎯 ユースケース

🛒 ユースケース 1: EC サイトの商品画像自動生成

EC サイト運営者が、複数の商品写真とブランドガイドラインをもとに、統一感のあるプロモーション画像を自動生成。14 枚の入力画像と 131K トークンのコンテキストウィンドウを活用し、ブランドの一貫性を保ちながら大量の商品画像を効率的に生成可能。

Python EC サイト商品画像生成の実装例 
from google import genai
from google.genai.types import GenerateContentConfig, Modality

client = genai.Client()

# 複数の商品画像とブランドガイドラインを入力
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=[
        "以下のブランドガイドラインに沿って、添付の商品画像を使用した"
        "Instagram 用プロモーション画像を生成してください。"
        "背景は白、商品を中心に配置、ブランドカラーのアクセントを追加。",
        # 商品画像やガイドラインドキュメントをここに追加
    ],
    config=GenerateContentConfig(
        response_modalities=[Modality.TEXT, Modality.IMAGE],
    ),
)

🎨 ユースケース 2: マルチターン会話による画像編集

デザイナーが対話形式で画像を段階的に編集し、理想のデザインに近づけます。Thinking 機能とインターリーブ出力により、各編集ステップで意図を正確に理解した上で画像を修正。マスク指定不要でテキスト指示のみで細部の編集が可能。

📢 ユースケース 3: コンテンツマーケティング素材の一括生成

マーケティングチームが、ブログ記事やソーシャルメディア投稿用の画像を、さまざまなアスペクト比で一括生成。Instagram (1:1, 4:5)、Twitter/X (16:9)、Pinterest (2:3)、ウェブバナー (8:1, 21:9) など、各プラットフォームに最適化された画像を単一のモデルで生成可能。

📝 まとめ

Gemini 3.1 Flash Image は、Vertex AI における画像生成モデルの最新版であり、131K トークンの大幅に拡張された入力コンテキスト、最大 14 枚の入力画像サポート、Thinking 機能の追加により、前世代の Gemini 2.5 Flash Image から大幅に進化しています。

Public Preview 段階ではありますが、Google が画像生成時の推奨モデルとして位置付けていることから、画像生成ワークロードを持つユーザーは早期に評価を開始し、GA 移行に備えることを推奨します。
