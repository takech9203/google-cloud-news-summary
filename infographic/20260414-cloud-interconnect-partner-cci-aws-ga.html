🌐 Partner Cross-Cloud Interconnect for AWS

Cloud Interconnect GA (VPC Network Peering) Preview (NCC)
2026-04-14

📋 概要

Google Cloud は、Partner Cross-Cloud Interconnect for AWS の VPC Network Peering 接続が一般提供 (GA) になったことを発表した。加えて、Network Connectivity Center (NCC) を使用した接続が Preview として利用可能になった。

Partner Cross-Cloud Interconnect for AWS は、Google Cloud と AWS 間のマルチクラウド接続をフルマネージドで提供するサービスである。従来の Cross-Cloud Interconnect と異なり、物理的なプロビジョニングやポート管理が不要であり、1 日以内での接続セットアップが可能。帯域幅は 1 Gbps から 100 Gbps まで柔軟に選択でき、冗長性がサービスレベルで組み込まれている。

🔄 Before / After 比較

❌ アップデート前の課題

  • Cross-Cloud Interconnect で 10 / 100 Gbps 単位の大容量接続のみ選択可能
  • 物理的なプロビジョニングに 1 - 4 週間を要した
  • 冗長性を確保するために手動で複数の接続を構成・管理する必要があった
  • 接続の開始は Google Cloud 側からのみで、AWS 側からの双方向プロビジョニングには非対応
  • NCC 経由のハブ & スポーク接続は未提供

✅ アップデート後の改善

  • VPC Network Peering 接続が GA となり、SLA 付きの本番利用が可能に
  • 1 Gbps - 100 Gbps の柔軟な帯域幅オプション
  • 物理プロビジョニング不要で 1 日以内にセットアップ完了
  • 冗長性がサービスに組み込まれ、手動構成が不要に
  • Google Cloud と AWS の 双方向プロビジョニングに対応
  • NCC サポート (Preview) によるハブ & スポークモデルが利用可能に

⚡ 主要機能

1. VPC Network Peering 接続 (GA)

→Google Cloud VPC と Transport Resource の Peering VPC 間を VPC Network Peering で接続
→Google Cloud / AWS どちらからでも接続を開始可能
→Activation Key を使用した双方向プロビジョニングに対応
→接続確立後、ピアリングされた VPC とルートテーブルの確認が可能

2. Network Connectivity Center (NCC) サポート (Preview)

→NCC Hub にハイブリッドスポークとして Partner Cross-Cloud Interconnect for AWS を接続
→複数の VPC / オンプレミス / 他クラウドネットワーク間のルーティングを NCC で一元管理
→サイト間データ転送による、Google バックボーンネットワークを活用した WAN 接続
→VPC スポークとハイブリッドスポークの組み合わせによる Any-to-Any 接続

3. フルマネージドな冗長性とプロビジョニング

→冗長性がサービスレベルで組み込まれており、手動での冗長構成が不要
→物理ポートやアタッチメントの管理が不要
→プロビジョニング時間は 1 日以内 (従来の Cross-Cloud Interconnect は 1 - 4 週間)
→帯域幅のオンデマンドスケールアップ/ダウンに対応

3.NCC で接続する場合: NCC Hub が作成済みであること

Step 1-2: プロファイル確認 & Transport Resource 作成

Bash プロファイル一覧表示 & Transport Resource 作成 
# 利用可能なプロファイルを一覧表示
gcloud compute interconnects partner-cci-for-aws \
  list-profiles --region=REGION

# Transport Resource を作成
gcloud compute interconnects partner-cci-for-aws \
  create TRANSPORT_NAME \
  --region=REGION \
  --profile=PROFILE_NAME \
  --bandwidth=BANDWIDTH \
  --network=VPC_NETWORK

Activation Key が生成されるので、AWS Console または API を使用して接続を完了する。

Step 3-4: VPC Network Peering の確立と確認

Bash VPC Peering 確立 & 確認 
# VPC Network Peering を確立
gcloud compute networks peerings create PEERING_NAME \
  --network=YOUR_VPC_NETWORK \
  --peer-network=TRANSPORT_PEERING_VPC

# 接続を確認
gcloud compute networks peerings list \
  --network=YOUR_VPC_NETWORK

NCC 接続 (Google Cloud から開始する場合)

Bash NCC Hub 作成 
# NCC Hub を作成
gcloud network-connectivity hubs create HUB_NAME \
  --description="Cross-cloud connectivity hub"

Transport Resource を作成し、NCC Hub にハイブリッドスポークとして接続する。

✅ メリット

💼 ビジネス面

✓ 導入期間の大幅短縮: 従来の 1-4 週間が 1 日以内に短縮。マルチクラウドプロジェクトの立ち上げを加速
✓ コスト最適化: 1 Gbps からの柔軟な帯域幅選択により、過剰な帯域幅への投資を回避
✓ 運用負荷の軽減: 物理インフラ管理やサードパーティ調整が不要。専門知識がないチームでもクロスクラウド接続を構築可能

🔧 技術面

✓ 組み込みの冗長性: サービスレベルで冗長性が確保。Google Cloud と AWS 双方で SLA が管理・抽象化
✓ 双方向プロビジョニング: Google Cloud / AWS どちらからでも接続開始可能
✓ NCC によるスケーラブルなルーティング: ハブ & スポークモデルで大規模マルチクラウド環境に対応
✓ 自動ルート交換: クラウド間のルート交換が自動処理。手動設定不要

⚠️ デメリット・制約事項

制限事項

! NCC サポートは Preview であり、本番ワークロードでの使用は推奨されない。SLA やサポートが制限される可能性
! Transport Resource のクォータはプロジェクトごと、リージョンごとにデフォルト 1 つに制限
! 対応する AWS リージョンはペアリングロケーションに限定。すべてのリージョン組み合わせが利用可能ではない
! 接続先は AWS のみ (Azure、OCI、Alibaba には使用不可)

考慮すべき点

  • VPC Network Peering と NCC の選択は、環境の複雑さと将来のスケーラビリティ要件に基づいて判断すること
  • NCC を使用する場合、Preview 段階であるため GA までに API や動作が変更される可能性がある
  • クラウド間のデータ転送料金は Google Cloud 側と AWS 側で別々に発生。双方の料金体系を確認すること

💡 ユースケース

📊 ユースケース 1: マルチクラウド分析基盤の構築

シナリオ: AWS の S3 に蓄積された大量のデータを Google Cloud の BigQuery で分析。定期的にデータ転送が発生するが、帯域幅は 5 Gbps 程度で十分。

効果: Partner Cross-Cloud Interconnect for AWS により、5 Gbps の帯域幅を選択してコストを最適化しつつ、プライベートな接続を 1 日以内にセットアップ。VPC Network Peering (GA) で安定した SLA 保護付きデータ転送が実現。

🌍 ユースケース 2: 大規模マルチクラウドネットワークの統合管理

シナリオ: グローバル企業が複数の Google Cloud VPC、オンプレミスデータセンター、AWS 環境を統合的に管理。各環境間での動的なルーティングとサイト間データ転送が必要。

効果: NCC Hub を中心としたハブ & スポークモデルにより、Partner Cross-Cloud Interconnect for AWS をハイブリッドスポークとして接続し、VPC スポークやオンプレミス接続と組み合わせた統合ネットワーク管理が可能に (Preview)。

🔗 関連サービス

🔌 Cross-Cloud Interconnect

10 / 100 / 400 Gbps の専用接続。物理プロビジョニング必要。AWS 以外 (Azure, OCI, Alibaba) にも対応。

🔗 VPC Network Peering

Google Cloud VPC 間のプライベート接続。Partner CCI for AWS の GA 接続方式として使用。

🔄 Network Connectivity Center (NCC)

ハブ & スポークモデルによるネットワーク接続のオーケストレーション。VPC / ハイブリッドスポーク / サイト間転送対応。

☁️ Partner CCI for OCI

Oracle Cloud Infrastructure 向けの同等のパートナー型クロスクラウド接続サービス。
📌 まとめ: Partner Cross-Cloud Interconnect for AWS の VPC Network Peering 接続が GA となり、Google Cloud と AWS 間のマルチクラウド接続がフルマネージドで本番利用可能に。物理プロビジョニング不要、1 日以内のセットアップ、1 Gbps からの柔軟な帯域幅選択、組み込みの冗長性により、導入と運用が大幅に簡素化された。マルチクラウド戦略を推進する企業は、まず VPC Network Peering (GA) での接続を検討し、より複雑なネットワーク要件がある場合は NCC (Preview) の評価を開始することを推奨する。
