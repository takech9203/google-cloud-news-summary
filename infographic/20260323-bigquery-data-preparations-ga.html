☁️ BigQuery: Cloud Storage / Google Drive ファイルからのデータプレパレーション

BigQuery GA Gemini Data Preparation
2026-03-23

📋 概要

BigQuery のデータプレパレーション機能において、Cloud Storage および Google Drive に保存されたファイルを直接データソースとして利用できるようになりました。この機能が一般提供 (GA) となり、本番環境での利用が正式にサポートされます。

🤖 Gemini AI 支援
データプレパレーションは Gemini in BigQuery を活用した AI 支援型のデータ準備機能です。BigQuery Studio 上のエディタで、ファイルに対するクリーニング、変換、エンリッチメント操作を行えます。Gemini がデータとスキーマを分析し、文脈に応じた変換提案を自動生成します。

対象ユーザー: データエンジニア、データアナリスト、ビジネスユーザーなど、BigQuery にデータを取り込む前の前処理が必要なすべてのユーザー

🔄 Before / After

❌ Before

  • Cloud Storage / Google Drive のファイルを分析するには手動で ETL パイプラインを構築するか、外部テーブルを作成してから別途クリーニングが必要
  • フォーマット不統一、欠損値、データ型不一致などの品質問題を SQL やスクリプトで個別に対処
  • Preview 段階で本番ワークロードでの利用に制約あり

✅ After

  • Cloud Storage / Google Drive のファイルをデータプレパレーションのソースとして直接利用可能 (GA)
  • Gemini AI 支援でクリーニング・変換・エンリッチメント提案が自動生成、コーディング不要
  • GA により SLA 対象、本番ワークロードでの安定運用が保証

⭐ 主要機能

☁️ Cloud Storage ファイルからのデータプレパレーション

✓ Cloud Storage バケット内のファイルをソースとして選択可能
✓ サポート形式: Avro, CSV, JSONL, ORC, Parquet
✓ DAT, TSV, TXT などは CSV として読み取り可能
✓ ワイルドカード検索 (*.csv など) に対応
✓ ファイル形式の自動検出

📁 Google Drive ファイルからのデータプレパレーション

✓ Google Drive のファイルをソースとして直接利用可能
✓ サポート形式: CSV, JSONL, Avro, Google Sheets

🤖 Gemini による AI 支援変換

✓ データとスキーマを分析して文脈に応じた変換提案を自動生成
✓ 型変換 (CAST)、文字列関数 (SUBSTR, CONCAT, REPLACE, UPPER, LOWER, TRIM)
✓ 日時関数 (PARSE_DATE, TIMESTAMP, EXTRACT, DATE_ADD)
✓ JSON 関数 (JSON_VALUE, JSON_QUERY)
✓ セル値の編集で提案品質を改善、自然言語でカスタマイズ可能

🔧 データプレパレーションステップ

→ Transformation: SQL 式によるデータクリーニングと変換
→ Filter: WHERE 句によるレコード除外
→ Deduplicate: 選択キーに基づく重複行の除去
→ Validation: 条件未達行をエラーテーブルに分離
→ Join: Inner / Left / Right / Full Outer / Cross Join
→ Destination: デスティネーションテーブルの定義

🛠️ 設定方法

💡 前提条件: Gemini in BigQuery が有効化されていること / Cloud Storage バケットまたは Google Drive ファイルへのアクセス権限 / BigQuery Studio の利用可能なプロジェクト
1
データプレパレーションの作成
Google Cloud コンソールで BigQuery ページを開き、「Create new」リストから「Data preparation」を選択
2
データソースの選択
「Google Cloud Storage」または「Google Drive」を選択し、対象ファイルを指定 (例: BUCKET_NAME/FILE_NAME.csv)
3
ステージングテーブルの定義
プロジェクト、データセット、テーブル名を指定。スキーマは自動検出され、Gemini がカラム名を推定
4
データの準備と変換
データプレパレーションエディタで、Gemini 提案の変換ステップを確認・適用。カラムやセルをクリックすると追加の提案が生成
5
デスティネーションテーブルの設定と実行
出力先テーブルを設定し、データプレパレーションを実行。必要に応じてスケジュール実行を設定

✅ メリット

💼 ビジネス面

✓ データ準備時間の短縮: Gemini AI 支援で手動クリーニング・変換作業を大幅削減
✓ ノーコードでのデータ前処理: SQL やプログラミング知識不要でGUI ベースの品質改善
✓ 本番環境での信頼性: GA により SLA に基づく安定した運用が可能

⚙️ 技術面

✓ 多様なファイル形式対応: Avro, CSV, JSONL, ORC, Parquet など主要形式をサポート
✓ CI/CD 対応: Dataform 連携によりバージョン管理とパイプライン自動化
✓ スケジュール実行: 定期実行で継続的なデータパイプラインを構築
✓ インクリメンタル処理: 差分処理に対応し、効率的な処理が可能

⚠️ デメリット・制約事項

⚠️ 制限事項
  • ソースとデスティネーションのデータセットは同一ロケーションに配置する必要がある
  • パイプライン編集中のデータとインタラクションは Gemini データセンターに送信されて処理される
  • Gemini in BigQuery は Assured Workloads に非対応
  • リポジトリ外のデータプレパレーションではバージョンの比較・復元機能が利用不可
  • データプレパレーション独自の API は提供されていない
💡 考慮すべき点
  • Gemini の提案はサンプルに基づくため、全データに対する最適な変換とは限らない
  • Gemini in BigQuery は BigQuery 本体と同一のコンプライアンス・セキュリティオファリングに対応していない
  • VPC Service Controls 使用時は Dataform と BigQuery の両方を対象としたペリメータ設定が必要

💡 ユースケース

Use Case 1

☁️ Cloud Storage からの定期データ取り込みパイプライン

シナリオ: 外部パートナーから毎日 Cloud Storage に CSV ファイルがアップロードされるが、フォーマットの不統一 (日付形式の違い、全角/半角混在、NULL 値の表現揺れなど) があり、分析前にクリーニングが必要。

効果: データプレパレーションでクリーニングルールを定義し、Dataform と連携したスケジュール実行を設定することで、毎日自動的にクリーンなデータが BigQuery テーブルに取り込まれる。

Use Case 2

📄 Google Drive のスプレッドシートデータの統合分析

シナリオ: 各部門が Google Sheets で管理している売上データや在庫データを BigQuery に集約して分析したいが、部門ごとにスキーマやデータ形式が異なる。

効果: Google Drive ファイルからデータプレパレーションを作成し、Gemini の提案に従ってスキーママッピングとデータ型変換を実施。統一フォーマットで BigQuery テーブルに出力し、横断的な分析が可能に。

🌍 利用可能リージョン

データプレパレーションはサポートされているすべての BigQuery ロケーションで利用可能です。データ処理ジョブはソースデータセットのロケーションで実行・保存されます。コードアセットの保存リージョンはジョブ実行リージョンと異なる場合があります。

詳細は BigQuery Studio ロケーションを参照してください。
