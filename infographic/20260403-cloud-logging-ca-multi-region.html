Cloud Logging: "ca" Multi-Region

Cloud Logging Announcement
2026-04-03

Overview

Cloud Logging に新たな "ca" (Canada) Multi-Region のサポートが追加された。 カナダ国内のデータセンターにログデータを保存するためのロケーションオプションであり、 カナダのデータレジデンシー要件を持つ組織にとって重要なアップデートとなる。

これまでマルチリージョンオプションは global、eu、us の 3 種類のみであったが、 今回 ca が追加されたことで、 PIPEDA (カナダ個人情報保護法) や Law 25 (ケベック州法 25 号) などのデータ主権規制に準拠しながら、 マルチリージョンの利便性を活用できるようになった。

Setup Guide

Prerequisites:
✓Cloud Logging API が有効化されていること
✓logging.buckets.create 権限を持つ IAM ロールが付与されていること

Step 1: "ca" マルチリージョンにログバケットを作成

Bash 
gcloud logging buckets create BUCKET_ID \
  --location=ca \
  --project=PROJECT_ID

Step 2: _Default シンクのルーティング先を変更

Bash 
gcloud logging sinks update _Default \
  --project=PROJECT_ID \
  --log-bucket=projects/PROJECT_ID/locations/ca/buckets/BUCKET_ID

Step 3: ログバケットの確認

Bash 
gcloud logging buckets list --project=PROJECT_ID

Limitations & Considerations

Limitations:
⚠マルチリージョンのログバケットは、リージョナルバケットと比較して追加の冗長性保証を提供しない
⚠ログバケットの作成後にロケーションを変更することはできない。別のロケーションが必要な場合は新しいバケットを作成しシンクをリダイレクトする必要がある
⚠global ロケーションでは Observability Analytics のデータ分析場所を選択できないため、"ca" マルチリージョンの使用が推奨される
Considerations:
💡既存の _Default / _Required バケットは global ロケーションで自動作成される。カナダへのデータレジデンシーが必要な場合は組織レベルでデフォルトストレージロケーションを "ca" に設定するか、新バケット作成 + シンク変更が必要
💡厳密なリージョン指定が必要な場合は個別リージョン (northamerica-northeast1 / northamerica-northeast2) の使用を検討すること

Use Cases

Use Case 1: Canadian Financial Institution Compliance

カナダの金融機関が規制要件により、すべてのログデータをカナダ国内に保存する必要があるシナリオ。

Bash 
# Create log bucket in Canada multi-region
gcloud logging buckets create compliance-logs \
  --location=ca \
  --project=my-finance-project \
  --retention-days=365

# Update _Default sink routing
gcloud logging sinks update _Default \
  --project=my-finance-project \
  --log-bucket=projects/my-finance-project/locations/ca/buckets/compliance-logs

Effect: カナダ国内のデータレジデンシー要件を満たしつつ、マルチリージョンの利便性を享受できる。

Use Case 2: Multi-Region Application in Canada

Montreal と Toronto の両方のリージョンにワークロードを展開しているアプリケーションで、統合的なログ管理を行いたいシナリオ。

Effect: "ca" マルチリージョンを使用することで、両リージョンのログを単一のマルチリージョンバケットに集約し、Logs Explorer や Observability Analytics で横断的に分析できる。
