📊 CCaaS Advanced Reporting Dashboards 4.0

CCaaS CCAI Platform Feature Fixed
2026-03-14

☁️ 概要

Google Cloud Contact Center as a Service (CCaaS) の Advanced Reporting Dashboards がバージョン 4.0 にメジャーアップデート。Queue Performance ダッシュボード、Voicemails ダッシュボード、Virtual Agent 待ち時間メトリクスなど複数の新機能が追加されたほか、フランス語 (カナダ) 対応による多言語サポートの拡充、Channel Interval / Queue Interval ダッシュボードの分析機能強化が含まれています。

さらに、17 件以上のバグ修正が実施され、レポーティング基盤の信頼性が大幅に向上しました。

🔄 Before / After

❌ Before (v3.x)

  • キューごとのパフォーマンスメトリクスを一元的に確認する専用ダッシュボードが存在しなかった
  • ボイスメールの量、エージェントのアクセス状況に関するインサイトが限定的
  • フランス語 (カナダ) 未対応
  • Virtual Agent のキュー内待機時間メトリクスが不足
  • Agent Performance のフィルタ不具合、時刻フォーマット不整合が発生
  • 無効化されたユーザーがダッシュボードに表示される問題

✅ After (v4.0)

  • Queue Performance - Calls / Chats ダッシュボードでキューごとの統合分析が可能に
  • Voicemails ダッシュボードでボイスメール運用分析が可能に
  • フランス語 (カナダ) 対応でネイティブ言語利用可能
  • Virtual Agent に In-Queue Interactions / Time メトリクス追加
  • 17 件以上のバグ修正でフィルタ・メトリクス計算の正確性が改善
  • Agent Performance ダッシュボードのパフォーマンス向上

🆕 新規ダッシュボード

📊 Queue Performance - Calls

キューごとのパフォーマンスメトリクスを提供する新しいダッシュボード。インタラクション量、放棄数、対応時間 (Handle Time)、コールバック、センチメント、CSAT を一画面で確認可能。

→ ドキュメント

💬 Queue Performance - Chats

チャットセッション用のキューパフォーマンスダッシュボード。インタラクション量、放棄数、Handle Time、センチメント、CSAT をチャットチャネル単位で確認可能。

→ ドキュメント

📧 Voicemails

ボイスメールの量、エージェントによるアクセス状況、平均エージェント応答時間に関するインサイトを提供。ボイスメール運用の効率化と応答品質の改善に活用可能。

→ ドキュメント

⬆️ 機能強化

🤖 Virtual Agent - 待ち時間メトリクス

新しいタイルが追加:

  • Total VA In-Queue Interactions
  • Total VA In-Queue Time
  • Avg VA In-Queue Time (通話のみ)

Virtual Agent のキュー内待機時間を定量的に把握可能に。

→ ドキュメント

📈 Queue / Channel Interval ドリルダウン

バーチャートやデータポイントをクリックすることで、インターバルごとの詳細な履歴メトリクスを確認可能に。データのドリルダウン操作により、問題の根本原因の特定が容易になりました。

🌐 フランス語 (カナダ) 対応

Advanced Reporting ダッシュボードがフランス語 (カナダ) で利用可能に。インスタンスのロケーションと言語を設定することで有効化。

→ 設定方法

⚙️ 設定方法

前提条件

✓ CCAI Platform インスタンスが対応リージョンに存在すること
✓ Advanced Reporting 拡張機能が有効化されていること
✓ 適切な IAM 権限が付与されていること
1
Advanced Reporting 拡張機能の有効化
Google Cloud コンソール → CCAI Platform → 対象インスタンス → Edit → Configure extensions → Advanced reporting チェックボックスを選択 → Save
2
フランス語 (カナダ) の設定 (必要に応じて)
インスタンスのロケーションと言語設定でフランス語 (カナダ) を選択する。
3
新規ダッシュボードへのアクセス
CCAI Platform ポータル → Dashboard → Advanced Reporting → Landing Page から目的のダッシュボードを選択 → フィルタ設定 → Update
⚠️ 注意: Advanced Reporting を有効化すると、レガシーの CCAI Platform ダッシュボードは使用できなくなります (不可逆的な変更)。

⚠️ デメリット・制約事項

制限事項
  • Advanced Reporting は 7 リージョンでのみ利用可能。対応リージョン外のインスタンスでは有効化できない
  • Advanced Reporting を有効化すると、レガシーの CCAI Platform ダッシュボードは利用不可 (不可逆)
  • ダッシュボードの日付範囲指定は最大 45 日間に制限
考慮すべき点
  • レガシーダッシュボードからの移行計画を事前に策定すること。カスタムレポートやワークフローの影響を確認する必要がある
  • v4.0 へのアップグレードに伴い、既存のカスタムダッシュボードや Look の動作確認を推奨

💡 ユースケース

📊 ユースケース 1: キューパフォーマンスの最適化

シナリオ: コンタクトセンターのスーパーバイザーが、特定のキューで放棄率が高く、顧客満足度が低下している原因を調査したい。

→ Queue Performance ダッシュボードにより、キューごとのインタラクション量、放棄数、平均対応時間、CSAT を一覧で確認し、問題のあるキューを特定。データに基づくスタッフ再配置やルーティング変更により、放棄率の低減と CSAT の向上を実現。

🤖 ユースケース 2: Virtual Agent の効果測定

シナリオ: Virtual Agent を導入したが、キュー内での待機時間に与える影響を定量的に把握できていない。

→ Virtual Agent ダッシュボードの新しい待ち時間メトリクス (Total VA In-Queue Interactions、Avg VA In-Queue Time) により、Virtual Agent がキュー内待機時間の削減にどの程度貢献しているかを数値で把握。導入効果の報告や追加投資の判断材料として活用可能。

🌐 ユースケース 3: バイリンガルコンタクトセンターの運用

シナリオ: カナダのコンタクトセンターで英語とフランス語の両方で運用しており、フランス語圏のスーパーバイザーがダッシュボードを母国語で利用したい。

→ フランス語 (カナダ) 対応により、フランス語圏のスーパーバイザーやアナリストがネイティブ言語でダッシュボードを操作でき、レポート解釈の正確性と業務効率が向上。
