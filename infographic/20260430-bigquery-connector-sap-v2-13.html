🔌 BigQuery Connector for SAP v2.13

パーティションテーブル + スキーマキャッシュの不具合修正

📋 概要

BigQuery Connector for SAP バージョン 2.13 がリリースされました。テーブルスキーマキャッシュ機能有効時にパーティションテーブルのデータレプリケーションが失敗する問題が修正されています。

BigQuery Connector for SAP は、SAP LT Replication Server にインストールされ、SAP アプリケーションデータを BigQuery にほぼリアルタイムでレプリケーションするコネクタです。

❌ Before (v2.12 以前の問題)

  • スキーマキャッシュ有効時にパーティションテーブルへのレプリケーションが失敗
  • キャッシュ無効化が必要で API クォータ制限に抵触するリスク
  • 高頻度レプリケーション環境で tables.get 呼び出しが過剰化

✅ After (v2.13)

  • パーティションテーブルでもスキーマキャッシュが正常動作
  • 全環境でスキーマキャッシュ活用可能、API クォータ消費削減
  • パーティション/非パーティション混在環境でも一貫設定で最適化可能

🔧 主要機能

1. パーティションテーブル向けスキーマキャッシュの不具合修正

SCH_CACHE_ENABLE 有効時にパーティションテーブルへのレプリケーションが失敗する問題を解消。パーティション情報を含むスキーマ定義がキャッシュから正しく取得されるように修正されました。

2. テーブルスキーマキャッシュ機能 (v2.12 導入)

SAP ABAP Shared Objects メモリにテーブルスキーマ情報を格納し、スキーマに変更がない場合は BigQuery API の tables.get 呼び出しをバイパス。BigQuery API クォータ制限 (TableService.getTable) の超過を防止します。

3. 3 つのレプリケーションパスのサポート

➡CDC レプリケーション (Pub/Sub 経由): BigQuery テーブルを SAP ソースと同期
➡ストリーミングデータレプリケーション: 全変更をアペンドオンリーで記録
➡ファイルベースレプリケーション (Cloud Storage): JSON ファイルとして配信

🛠️ 設定方法

前提条件

✅BigQuery Connector for SAP バージョン 2.13 がインストールされていること
✅SAP LT Replication Server が稼働していること
✅BigQuery データセットが作成済みであること
✅IAM サービスアカウントに BigQuery Data Editor / Job User ロールが付与されていること

設定手順

  1. BigQuery Connector for SAP をバージョン 2.13 に更新
  2. SAP GUI で /n/GOOG/SLT_SETTINGS トランザクションを実行
  3. Settings Table ドロップダウンメニューから「Parameters」を選択
  4. 行挿入アイコンをクリック
  5. Parameter Name に「SCH_CACHE_ENABLE」、Value に「X」を入力
  6. 保存をクリック
  7. パーティションテーブルへのレプリケーション動作を確認

📈 メリット

💼 ビジネス面

運用の安定性向上: パーティションテーブル環境でスキーマキャッシュによるパフォーマンス最適化を安心して利用可能

API コスト最適化: BigQuery API 呼び出し回数の削減により、大規模レプリケーション環境でのクォータ消費を抑制

⚙️ 技術面

パフォーマンス改善: tables.get 呼び出し削減でスループット向上

クォータ制限回避: 高頻度シナリオで TableService.getTable の超過防止

一貫した設定管理: 全テーブルで同一のキャッシュ設定を使用可能

⚠️ 制約事項・注意点

⚠️ 制限事項:
  • スキーマキャッシュはストリーミングデータレプリケーション (insert-only) パスでのみ利用可能
  • テーブルスキーマを変更した場合、キャッシュの手動クリアが必要になる場合がある
  • CDC レプリケーション (Pub/Sub 経由) では別途 Cache Validations 機能を使用する必要あり
💡 考慮すべき点:
  • スキーマ変更を頻繁に行う環境では、キャッシュの無効化タイミングに注意
  • v2.13 へのアップデート後、既存のレプリケーション設定の動作確認を推奨
  • SAP LT Replication Server の ABAP Shared Objects メモリの利用量が増加する可能性

🎯 ユースケース

🏭 大規模 SAP データウェアハウスのリアルタイム分析

シナリオ: 製造業の企業が、SAP ERP の販売データ・在庫データをパーティションテーブル (日付パーティション) で BigQuery に格納。高頻度レプリケーションにより BigQuery API クォータ制限に達しており、スキーマキャッシュを有効にしたいがパーティションテーブルの問題により有効化できなかった。

✨ 効果: v2.13 に更新することで、パーティションテーブル環境でもスキーマキャッシュを安全に有効化でき、API クォータ問題を解消しつつ安定したリアルタイムデータ連携を実現。

🛒 マルチテーブルレプリケーション環境での統一設定

シナリオ: 小売業の企業が、数百の SAP テーブルを BigQuery にレプリケーション中。一部テーブルのパーティション化により、テーブルごとに個別のキャッシュ設定を管理する必要があった。

✨ 効果: v2.13 により、すべてのテーブルに対して統一的にスキーマキャッシュを有効化でき、設定管理の複雑さが解消。

📝 まとめ

BigQuery Connector for SAP v2.13 は、テーブルスキーマキャッシュとパーティションテーブルの組み合わせで発生していた不具合を修正する重要なバグフィックスリリースです。

📌 アクション推奨: パーティションテーブルを使用して SAP データを BigQuery にレプリケーションしている環境では、本バージョンへの早期アップデートを推奨します。更新後にスキーマキャッシュを有効化することで、API クォータ消費の削減とレプリケーションパフォーマンスの向上が期待できます。
