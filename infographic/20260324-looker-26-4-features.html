☁️ 概要

Looker 26.4 は、Gemini を活用した AI 機能の拡充、データガバナンスの強化、運用管理の改善を含む大規模なリリースです。BI プラットフォームとしての Looker に AI ファーストのアプローチを本格的に導入し、データ分析の民主化を加速させます。

🤖 Gemini 統合のポイント: Expression Assistant、Enhanced Search、AI-assisted Quick Starts の 3 つの Gemini 連携機能が Preview として追加され、Looker の学習曲線を下げ、非技術ユーザーでもデータ分析が可能になります。

🤖 主要機能: Gemini AI 連携 (Preview)

✏️ Expression Assistant

Gemini を使用して、自然言語でテーブル計算やカスタムフィールドの Looker 式を生成します。

→「Help me write an expression with Gemini」ボタンから利用
→生成された式を「Refine」で修正、「Apply」で適用
→create_table_calculations または create_custom_fields 権限 + Gemini ロールが必要
⚠️ 注意: 英語 (アメリカ英語) での利用が推奨されています。多言語対応は限定的です。

🔍 Enhanced Search

Gemini によるセマンティック検索で、キーワードマッチングを超えた概念的な意味の理解を実現します。

→ビジネス用語や分析的な質問 (例: 「total customer acquisition cost」) で検索可能
→Boards、Dashboards、Explores、Folders、Looks、LookML ダッシュボードを検索
→人気度、閲覧頻度、認証ステータスに基づくランキング
→コンテンツタイプ、フォルダ、作成者、日付、認証ステータスでフィルタリング

⚡ AI-assisted Quick Starts for Explores

Gemini が Explore を分析し、Quick Start 分析を自動生成します。

→Quick Start 未定義の Explore で「Try AI Quick Starts」オプション表示
→クリックでクエリ自動実行、ビジュアライゼーション表示
→develop 権限 + Gemini ロールが必要

✅ 主要機能: GA 昇格

📊 Self-service Explores GA

CSV、Excel、Google Sheets ファイルをアップロードしてデータを探索可能。

→26.4 で OAuth 経由の Google Sheets アップロードに対応
→Google Drive ナビゲーションでシートを直接選択
→アップロードデータは BigQuery に永続化、LookML モデル自動生成
→upload_data 権限が必要

🏅 Content Certification GA

ダッシュボード、Look、Explore に信頼性を示す認証バッジを付与。

→LookML Explores の認証が新たに可能に
→LookML ダッシュボードと Explores の自動認証に対応
→Enhanced Search で認証ステータスによるソート/フィルタリング
→「Revoke Certification on Edit」で編集時に認証自動取り消し

🔧 主要機能: 運用管理改善

🔌 Connection Disable オプション

Connection Settings ページに新しい「Disable Connection」トグルを追加。

→データベース障害時、管理者が接続をワンクリックで一時無効化
→無効化中は Looker がクエリを送信せず、ユーザーにエラーメッセージを返す
→問題解決後、トグルをオフにして即座に復旧

📈 Increased Row Limit Preview

マップ、散布図、テーブルチャートの行数上限を最大 50,000 行/データポイントまで設定可能。

→Content guardrails の「Visualization limits」で設定
→デフォルトは無効、管理者が有効化
→既存のダッシュボードタイルは自動反映されず、手動編集が必要

🛠️ 設定方法

💡 前提条件: Looker インスタンスが 26.4 以降にアップデート済みであること。Gemini 機能を利用する場合、Gemini in Looker が管理者により有効化されていること。

ステップ 1: Gemini in Looker の有効化

Looker (original) の場合:

→Admin パネル > Platform > Gemini in Looker で各機能を個別に有効化
→Expression Assistant、Semantic Search、AI-Assisted Quick Starts をそれぞれトグルでオン

Looker (Google Cloud core) の場合:

→Google Cloud console で Gemini in Looker を有効化

ステップ 2: Self-service Explores の OAuth 設定

1.Google Cloud console で OAuth クライアントを作成 (Web application タイプ)
2.Authorized JavaScript origins に Looker インスタンスの URL を追加
3.Client ID と Client secret を Looker の Admin > Self-service Explores ページに入力

ステップ 3: Increased Row Limit の有効化

1.Admin パネル > Preview Features で「Increased Row Limit」を有効化
2.Admin パネル > Performance Center > Content guardrails で Visualization limits を設定
3.既存のダッシュボードタイルを編集して新しい行数上限を適用

ステップ 4: Connection Disable の使用

1.Admin パネル > Database > Connections で対象の接続を編集
2.「Disable Connection」トグルを有効化
3.データベースの問題解決後、トグルを無効化して接続を復旧

💡 メリット

ビジネス面

✓データ分析の民主化: Gemini Expression Assistant と AI Quick Starts により、Looker 式の知識がなくてもデータ分析が可能に
✓データガバナンスの強化: Content Certification の GA 化と LookML Explores 認証で本格運用可能
✓運用コストの削減: Connection Disable で障害対応が簡素化、対応時間短縮

技術面

✓セマンティック検索: コンテンツの発見性が大幅に向上、既存分析の再利用促進
✓大規模データの可視化: 行数上限が 50,000 に拡大、より詳細な可視化が可能
✓自動認証: LookML ダッシュボードと Explores の自動認証でガバナンスワークフロー自動化

⚠️ 制限事項・注意点

!Gemini 機能 (Expression Assistant、Enhanced Search、AI Quick Starts) は Preview 段階であり、サポートが限定的
!Expression Assistant は英語 (アメリカ英語) 推奨、多言語対応は限定的
!Increased Row Limit を有効化すると、インスタンスのパフォーマンスに影響する可能性 (データベース負荷、ブラウザ負荷、ネットワーク負荷)
!Self-service Explores のデータは BigQuery に永続化されるため、ストレージコストが発生
!Connection Disable 中は全てのクエリがブロックされるため、影響範囲を事前に確認する必要あり
!PDF ダウンロード時、テーブルチャートはダッシュボードあたり 200,000 セルの合計上限
