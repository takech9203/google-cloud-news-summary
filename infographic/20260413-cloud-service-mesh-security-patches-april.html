📋 概要

Google Cloud は Cloud Service Mesh のインクラスター版およびマネージド版に対して、複数のセキュリティパッチをリリースしました。今回のパッチでは、Critical (CVSS 9.1) から Low (3.4) まで合計 7 件の CVE が修正されています。

特に CVE-2026-33186 は CVSS スコア 9.1 の Critical 脆弱性であり、早急な対応が推奨されます。インクラスター版では 3 つのメジャーバージョンライン、マネージド版では全 3 チャネルにパッチが提供されています。

🚨 重要: CVE-2026-33186 (Critical 9.1) はリモートからの攻撃に悪用される可能性があり、RBAC バイパス、クラッシュ、情報漏洩のリスクがあります。インクラスター版をご利用の場合は速やかにアップグレードしてください。

🔧 設定方法

💡 前提条件: 現在使用中の Cloud Service Mesh のバージョンとデプロイメントモデル (インクラスター / マネージド) を確認し、gcloud CLI が最新バージョンであること、クラスタに対する適切な IAM 権限を保持していることを確認してください。

Step 1: 現在のバージョンを確認

Bash バージョン確認コマンド 
# インクラスター版の場合: istiod のバージョンを確認
kubectl get pods -n istio-system -l app=istiod \
  -o jsonpath='{.items[*].spec.containers[*].image}'

# マネージド版の場合: コントロールプレーンのリビジョンを確認
kubectl get controlplanerevision -n istio-system

Step 2: インクラスター版のアップグレード

Bash asmcli を使用したアップグレード (例: 1.28.5-asm.12) 
./asmcli install \
  --project_id PROJECT_ID \
  --cluster_name CLUSTER_NAME \
  --cluster_location CLUSTER_LOCATION \
  --fleet_id FLEET_PROJECT_ID \
  --output_dir OUTPUT_DIR \
  --enable_all
✅ 推奨: 公式のアップグレードガイドに従い、カナリアアップグレード方式を推奨します。

Step 3: マネージド版の場合

マネージド版を使用している場合、Google が自動的にローリングアップデートを適用します。追加の操作は不要ですが、以下のコマンドでロールアウト状況を確認できます。

Bash ロールアウト状況の確認 
# マネージド版のコントロールプレーンステータスを確認
kubectl get controlplanerevision -n istio-system -o yaml

Step 4: サイドカープロキシの再起動

Bash ワークロードのローリングリスタート 
# 名前空間内のワークロードをローリングリスタートしてサイドカープロキシを更新
kubectl rollout restart deployment -n NAMESPACE

⚠️ デメリット・制約事項

🚫 EOL バージョンの制限

Cloud Service Mesh v1.25 以前のバージョンはサポート終了 (EOL) であり、今回の CVE 修正はバックポートされません。v1.26 以降への早急なバージョンアップが必要です。

🔧 手動アップグレード作業

インクラスター版では手動でのアップグレードが必要であり、大規模クラスタでは計画的なメンテナンスウィンドウが必要です。

🔄 ローリングリスタートの影響

サイドカープロキシの更新にはワークロードのローリングリスタートが必要であり、一時的にリクエスト処理能力が低下する可能性があります。

⏳ 展開完了までの時間

マネージド版のローリングアップデートは段階的に適用されるため、全チャネルへの展開完了まで数週間かかる場合があります。

💡 考慮事項: カナリアアップグレード方式の場合、一時的に複数バージョンのコントロールプレーンが共存するため、リソース使用量が増加します。

🎯 ユースケース

🏢 本番環境の緊急パッチ適用

Critical CVE への対応として、インクラスター版のカナリアアップグレードを実施。段階的にパッチを適用し、サービス影響を最小限に抑えながらセキュリティを確保。

☁️ マネージド版の自動保護

マネージド版を利用しているチームは、自動ローリングアップデートにより追加作業なくセキュリティパッチが適用。ロールアウト状況の監視のみで対応完了。

📋 コンプライアンス監査への対応

セキュリティ監査で CVE 対応を求められた際に、全バージョンラインへのパッチ適用状況をエビデンスとして提出。GCP-2026-013 などのセキュリティ情報と合わせて報告。
