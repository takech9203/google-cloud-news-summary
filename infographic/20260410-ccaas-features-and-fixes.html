Contact Center as a Service

📋 概要

Google Cloud Contact Center as a Service (CCaaS) に複数の新機能とバグ修正がリリースされました。 ダイレクトコールの言語選択、 仮想エージェントから特定ヒューマンエージェントへの転送、 予測キャンペーンのセルフサービス化が主要なアップデートです。 これらの機能強化により、多言語対応、パーソナライズされた顧客体験、キャンペーン運用の自律性が大幅に向上します。

❌ Before

✗ ダイレクトコールで言語選択ができず、デフォルト言語のみ
✗ 予測キャンペーンのコントロールに Google アカウントチームの支援が必要
✗ 仮想エージェントからの転送はキュー単位のみ
✗ チャットのバックスラッシュ表示やトランスクリプトに不具合

✅ After

✓ 通話冒頭で希望言語を選択可能に
✓ 管理者がセルフサービスでキャンペーンコントロールを設定
✓ agent_extension / agent_id で特定エージェントに転送可能
✓ チャット表示、トランスクリプト、メタデータのバグを修正

🏗️ アーキテクチャ図

仮想エージェントから特定のヒューマンエージェントへの通話転送フローを示します。 仮想エージェントはカスタムペイロードで agent_extension または agent_id を指定し、 CCAI Platform が該当エージェントの可用性を確認した上で転送を実行します。 指定エージェントが利用不可の場合は、同一キューの利用可能なエージェントにフォールバックします。

        end
    end

⚡ 主要機能

🌐 ダイレクトコール (直通電話) の言語選択サポート

エンドユーザーがエージェントの直通電話番号に発信した際、通話の冒頭で言語を選択できるようになりました。IVR 経由のコールと同様に多言語対応が可能になり、ダイレクトコールのセッションタイプは「Voice Inbound (Direct)」として記録されます。

設定は CCAI Platform ポータルの Settings > Languages & Messages で管理される言語選択メッセージ設定と連動します。

👤 仮想エージェントから特定ヒューマンエージェントへの転送

仮想エージェントが agent_extension または agent_id フィールドを使用して、特定のヒューマンエージェントに通話を転送できるようになりました。Dialogflow CX のカスタムペイロードで転送先エージェントを指定します。従来のキュー単位のエスカレーション (menu_id) も引き続き利用可能です。

📈 予測キャンペーンの改善コントロールのセルフサービス化

Max Calls Per Agent (エージェント 1 人あたりの最大発信数) と Target Agent Occupancy (エージェント稼働率の目標パーセンテージ) が Google アカウントチームの支援なしで利用可能になりました。過剰ダイヤルによる通話放棄リスクを低減しつつ、ダイヤルレートを自動調整できます。

🔧 バグ修正 (3 件)

チャットでバックスラッシュ (\) が正しく表示されない問題、仮想エージェントのチャットトランスクリプトに関する不具合、セッションメタデータにおける通話時間の不正確さがそれぞれ修正されました。

📣 次期バージョンのプレリリースノート公開

CCaaS の次期バージョンに関するプレリリースノートが公開されました。

📝 設定方法

💡 前提条件: CCAI Platform へのアクセス権限、管理者ロールまたは適切な権限が必要です。言語選択機能には複数言語の設定、予測キャンペーンにはキャンペーンマネージャーへのアクセス権限が必要です。

ステップ 1: ダイレクトコールの言語選択を設定

1

言語選択メッセージの設定

CCAI Platform ポータルで Settings > Languages & Messages を選択し、Language Selection Message セクションで言語選択メッセージを設定します。テキスト読み上げ (Text-to-speech) の入力、または事前録音した音声ファイルのアップロードが可能です。

2

言語ステータスの確認

ダイレクトコールに使用する言語が Live ステータスになっていることを確認し、Save Languages をクリックします。

ステップ 2: 予測キャンペーンの改善コントロールを設定

1

キャンペーンの作成

Campaigns から Add Campaign をクリックし、Mode で Predictive を選択します。

2

コントロールの設定

Max Calls Per Agent (最大発信数) と Target Agent Occupancy (目標稼働率) を設定します。必要に応じて Max Abandonment で放棄率の上限も設定できます。

ステップ 3: 仮想エージェントから特定ヒューマンエージェントへの転送を設定

1

カスタムペイロードの設定

Dialogflow CX コンソールで対象の仮想エージェントを開き、エスカレーションフローのフルフィルメントでカスタムペイロードを設定します。ペイロード内で agent_extension または agent_id フィールドに転送先のヒューマンエージェントの内線番号または ID を指定します。

2

フォールバック設定の確認

指定エージェントが利用不可の場合のフォールバック用キュー設定を確認します。

⚠️ デメリット・制約事項

制限事項

✗ ダイレクトコールの言語選択を利用するには、使用する言語が事前に CCAI Platform で設定され「Live」ステータスになっている必要がある
✗ 仮想エージェントからの特定エージェントへの転送は、指定エージェントが利用不可の場合にキュー内の他エージェントにフォールバックするが、全エージェントが利用不可の場合は通話が終了する可能性がある
✗ 予測キャンペーンの Max Abandonment 設定は法規制 (国ごとに異なる放棄率制限) に準拠する必要がある

考慮すべき点

⚠️ エスカレーションフローとの整合性: 仮想エージェントの転送ペイロードを agent_extension や agent_id に変更する際は、既存のエスカレーションフローとの整合性を確認すること。

⚠️ 音声ガイダンスの準備: ダイレクトコールの言語選択メッセージを設定する際は、全てのサポート言語に対応する音声ガイダンスを準備する必要がある。

💡 ユースケース

🏦 ユースケース 1: VIP 顧客の専任エージェントへのダイレクト転送

シナリオ: 金融サービス企業のコンタクトセンターで、VIP 顧客が仮想エージェントと対話後、専任のアカウントマネージャーに転送されるフローを構築する。

JSON 
{
  "ujet": {
    "type": "action",
    "action": "escalation",
    "escalation_reason": "by_virtual_agent",
    "agent_id": "VIP_ACCOUNT_MANAGER_ID",
    "language": "ja"
  }
}

効果: VIP 顧客が仮想エージェントで用件を伝えた後、自動的に専任エージェントに転送される。顧客は一般キューで待機する必要がなく、パーソナライズされたサービスを受けられる。

🌐 ユースケース 2: 多言語コンタクトセンターのダイレクトコール運用

シナリオ: グローバル展開している企業のコンタクトセンターで、各国の顧客が担当エージェントの直通番号に電話をかける際に、自分の言語でサポートを受けたい。

効果: ダイレクトコールの言語選択機能により、エンドユーザーは通話の冒頭で希望言語を選択でき、IVR 経由の通話と同等の多言語対応体験を得られる。レポーティングでも言語別のデータが正確に記録される。

📊 ユースケース 3: 予測キャンペーンの放棄率最適化

シナリオ: テレマーケティング企業が予測キャンペーンの過剰ダイヤルによる通話放棄率を低減したい。米国の規制では 30 日間の放棄率を 3% 以下に抑える必要がある。

効果: Max Calls Per Agent と Target Agent Occupancy コントロールを管理ポータルから直接設定することで、Google アカウントチームへの依頼を待たずにダイヤルレートを迅速に調整できる。規制準拠を維持しつつ、エージェント稼働率を最適化できる。
