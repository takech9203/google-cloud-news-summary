📋 概要

BigQuery ML において、Vertex AI Model Garden や Hugging Face のオープンモデルを CREATE MODEL ステートメントだけで Vertex AI エンドポイントに自動デプロイできる機能がリリースされました。Vertex AI リソースの作成・管理を BigQuery が自動的に行い、SQL のみで生成 AI 機能を活用できるようになります。

🎯 対象ユーザー: BigQuery ML でテキスト生成やエンベディング生成を行う ML エンジニア、データサイエンティスト、BigQuery ベースの AI パイプラインを構築するソリューションアーキテクト

⚡ 主要機能

🔧 Vertex AI リソースの自動管理

→CREATE MODEL で指定したオープンモデルが Vertex AI 共有パブリックエンドポイントに自動デプロイ
→Vertex AI リソース ID は bq-managed- プレフィックスで管理
→BigQuery ML モデルの ALTER や DELETE に連動して Vertex AI リソースも自動変更・削除

🖥️ Compute Engine リザベーションとの連携

→リザベーションで推論用リソース (VM) の可用性を確保
→RESERVATION_AFFINITY_TYPE、RESERVATION_AFFINITY_KEY、RESERVATION_AFFINITY_VALUES で設定
→安定した推論パフォーマンスとバースト的なワークロードへの対応

💰 自動・即時アンデプロイによるコスト管理

→ENDPOINT_IDLE_TTL でアイドル時間しきい値を設定し、自動アンデプロイ
→ALTER MODEL ... DEPLOY_MODEL = FALSE で即座にアンデプロイし課金停止
→アンデプロイ後もモデルオブジェクトとメタデータは保持、DEPLOY_MODEL = TRUE で再デプロイ可能

🛠️ 設定方法

📋 前提条件:
  • BigQuery Admin ロール (roles/bigquery.admin) + Vertex AI Administrator ロール (roles/aiplatform.admin)
  • BigQuery Connection API + Vertex AI API が有効化済み
  • BigQuery 接続 (Connection) が作成済み (DEFAULT 接続も使用可能)

Step 1: Model Garden モデルの自動デプロイ

SQL Gemma 3 モデルを自動デプロイ (8時間アイドル後に自動アンデプロイ) 
CREATE MODEL `project_id.mydataset.my_model_garden_model`
REMOTE WITH CONNECTION DEFAULT
OPTIONS (
  model_garden_model_name = 'publishers/google/models/gemma3@gemma-3-1b-it',
  endpoint_idle_ttl = INTERVAL 8 HOUR
);

Step 2: Hugging Face モデルの自動デプロイ

SQL Hugging Face モデルを 10 レプリカで自動デプロイ 
CREATE MODEL `project_id.mydataset.my_hugging_face_model`
REMOTE WITH CONNECTION DEFAULT
OPTIONS (
  hugging_face_model_id = 'intfloat/multilingual-e5-small',
  max_replica_count = 10
);

Step 3: 推論の実行

SQL テキスト生成の推論 
SELECT * FROM AI.GENERATE_TEXT(
  MODEL `project_id.mydataset.my_model_garden_model`,
  (SELECT 'What is the purpose of dreams?' AS prompt)
);

Step 4: コスト管理のためのアンデプロイ

SQL 即座にアンデプロイ / 再デプロイ 
-- 推論完了後に即座にアンデプロイしてコストを停止
ALTER MODEL `project_id.mydataset.my_model_garden_model`
SET OPTIONS (deploy_model = FALSE);

-- 再度使用する際に再デプロイ
ALTER MODEL `project_id.mydataset.my_model_garden_model`
SET OPTIONS (deploy_model = TRUE);

✅ メリット

💼 ビジネス面

運用コストの削減

Vertex AI リソースの手動管理が不要になり、ML エンジニアの運用負荷が大幅に軽減。アイドル時の自動アンデプロイで不要コストを自動抑制。

ML 導入の加速

BigQuery SQL のみでオープンモデル活用が可能。Vertex AI の専門知識がなくてもデータアナリストが生成 AI を利用可能に。

🔧 技術面

リソース管理の自動化

BigQuery ML モデルと Vertex AI リソースのライフサイクルが自動連動。削除忘れによる不必要な課金を防止。

安定した推論パフォーマンス

Compute Engine リザベーションで VM 可用性を確保。バースト的な推論ワークロードにも対応。

SQL ベースの一元管理

デプロイ、推論、アンデプロイを SQL で完結。CI/CD パイプラインへの組み込みが容易。

⚠️ 制限事項・注意点

⚠️ 制限事項:
  • オープンモデルはテキストデータのみ処理可能。マルチモーダルデータは非サポート
  • モデル作成・更新ジョブをキャンセルしても、Vertex AI 側のリソースはクリーンアップされない
  • タイムトラベルは自動デプロイモデルで非サポート。Vertex AI リソースは復元されない
  • データセット削除前にモデルを先に削除すること。そうしないと Vertex AI リソースがアクティブなまま残る
🚨 考慮すべき点:
  • BigQuery が管理する Vertex AI リソース (bq-managed-*) を Vertex AI 側から直接変更しないこと。推論中の手動アンデプロイはクエリ失敗の原因に
  • マシン時間単位で課金。エンドポイント設定完了後から課金開始。ENDPOINT_IDLE_TTL の適切な設定が重要
  • Preview ステータス。Pre-GA Offerings Terms が適用。本番ワークロードでの利用時はサポート範囲を確認

💡 ユースケース

📝 ユースケース 1: バッチテキスト生成パイプライン

シナリオ: 毎日定時に BigQuery 内の顧客フィードバックデータに対してテキスト生成 (要約・分類) を実行し、完了後にモデルをアンデプロイしてコストを最適化。

SQL バッチ推論パイプライン例 
-- モデル作成 (初回のみ)
CREATE MODEL IF NOT EXISTS `project.dataset.feedback_summarizer`
REMOTE WITH CONNECTION DEFAULT
OPTIONS (
  model_garden_model_name = 'publishers/google/models/gemma3@gemma-3-1b-it',
  endpoint_idle_ttl = INTERVAL 2 HOUR
);

-- バッチ推論
SELECT feedback_id, ml_generate_text_result
FROM AI.GENERATE_TEXT(
  MODEL `project.dataset.feedback_summarizer`,
  TABLE `project.dataset.daily_feedback`
);

効果: 推論完了後 2 時間で自動アンデプロイ。バッチ処理に必要な時間のみ課金が発生。

🔍 ユースケース 2: 大規模エンベディング生成によるベクトル検索基盤構築

シナリオ: BigQuery に格納された大量のドキュメントデータに対してエンベディングを生成し、ベクトル検索機能と組み合わせてセマンティック検索基盤を構築。

SQL 大規模エンベディング生成例 
-- Hugging Face エンベディングモデルの自動デプロイ
CREATE MODEL `project.dataset.doc_embedder`
REMOTE WITH CONNECTION DEFAULT
OPTIONS (
  hugging_face_model_id = 'intfloat/multilingual-e5-small',
  max_replica_count = 10
);

-- 大規模エンベディング生成
SELECT * FROM AI.GENERATE_EMBEDDING(
  MODEL `project.dataset.doc_embedder`,
  TABLE `project.dataset.documents`
);

効果: 10 レプリカの並列処理で大規模データのエンベディング生成を高速完了。BigQuery のベクトル検索と組み合わせたセマンティック検索が SQL のみで実現。

💲 料金情報

自動デプロイされたオープンモデルは、Vertex AI のマシン時間単位で課金されます。エンドポイントが完全にセットアップされた時点から課金が開始され、モデルがアンデプロイされるまで継続します。

課金項目説明

🔗 関連サービス

Vertex AI Model Garden - Gemma、Llama、DeepSeek 等のオープンモデルを提供するモデルカタログ。自動デプロイの対象モデルはここから選択。
Vertex AI Model Registry - BigQuery ML モデルの登録・バージョン管理・デプロイを行うサービス。自動デプロイは Model Registry を経由した管理とは別のアプローチ。
Compute Engine リザベーション - 推論用 VM の可用性を確保するためのリソース予約サービス。自動デプロイモデルと連携して安定した推論を実現。
BigQuery AI 関数 - AI.GENERATE_TEXT、AI.GENERATE_EMBEDDING 等の SQL 関数。自動デプロイモデルへの推論リクエスト送信に使用。
