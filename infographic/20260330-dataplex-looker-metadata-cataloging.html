📊 Dataplex: Looker メタデータの自動カタログ化と BigQuery データリネージ取り込み

Dataplex Universal Catalog Looker BigQuery Preview
2026-03-30

☁️ 概要

Dataplex Universal Catalog に Looker (Google Cloud core) のメタデータ自動カタログ化とBigQuery ソースからのデータリネージ自動取り込み機能がプレビューとして追加されました。これにより、BI レイヤーからデータウェアハウスまでのエンドツーエンドのデータガバナンスが実現します。

💡 ポイント: Looker で定義された Explore、View、Dashboard などの BI メタデータが Dataplex Universal Catalog に自動登録され、組織全体のデータカタログから一元的に検索・管理が可能に。BigQuery から Looker へのデータフローも自動追跡されます。

🔄 Before / After 比較

❌ Before (これまでの課題)
  • Looker のメタデータは Looker 内部でのみ管理され、Dataplex から検索不可
  • BigQuery → Looker のリネージ情報を手動で追跡・文書化する必要があった
  • BI レイヤーとデータウェアハウスのメタデータが分断、組織横断的なガバナンスが困難
✅ After (改善後)
  • Looker メタデータが Dataplex Universal Catalog に自動カタログ化、統合検索で発見可能に
  • BigQuery → Looker のデータリネージが自動取り込み、エンドツーエンドの可視化
  • データウェアハウスから BI ダッシュボードまでの完全なメタデータ管理とガバナンスを実現

⚡ 主要機能

🗂️ Looker メタデータの自動カタログ化

✓ LookML モデル、Explore、View、ダッシュボードのメタデータを自動登録
✓ Dataplex の統合検索 (キーワード / 自然言語) で発見可能に
✓ ビジネスコンテキスト (アスペクト、ビジネス用語集) を付与してメタデータ拡充

🔗 BigQuery ソースからのデータリネージ取り込み

✓ BigQuery テーブル参照の Looker モデルでデータの流れを自動記録
✓ テーブルレベルのリネージ追跡 (上流 BigQuery → 下流 Looker ダッシュボード)
✓ Google Cloud コンソールのリネージグラフ機能で視覚的に確認可能

🛡️ 統合データガバナンス

✓ Dataplex の既存ガバナンス機能 (IAM、データ品質、メタデータ変更フィード) が Looker にも適用
✓ インパクト分析: BigQuery テーブルの変更が Looker ダッシュボードに与える影響を事前評価

🔧 設定方法

前提条件:
1. Looker (Google Cloud core) インスタンスが設定済みであること
2. Dataplex API および Data Lineage API が有効化されていること
3. 適切な IAM ロールが付与されていること

Step 1: API の有効化

Bash 
# Dataplex API と Data Lineage API を有効化
gcloud services enable dataplex.googleapis.com
gcloud services enable datalineage.googleapis.com

Step 2: Looker メタデータカタログ化の確認

Bash 
# Dataplex Catalog で Looker エントリを検索
gcloud dataplex entries search "system=looker" \
  --project=PROJECT_ID \
  --location=LOCATION

Step 3: データリネージの確認

Bash 
# BigQuery テーブルのリネージ情報を確認
gcloud dataplex lineage search-links \
  --project=PROJECT_ID \
  --location=LOCATION \
  --target="bigquery:PROJECT_ID.DATASET.TABLE"

🎯 ユースケース

🏦 ユースケース 1: 規制対応のためのデータリネージ追跡

シナリオ: 金融機関のデータガバナンスチームが、個人情報を含む BigQuery テーブルのデータが最終的にどの Looker ダッシュボードで表示されているかを追跡する必要がある。

Bash 
# 個人情報テーブルから下流のリネージを検索
gcloud dataplex lineage search-links \
  --project=my-project \
  --location=us-central1 \
  --source="bigquery:my-project.customer_data.pii_table"

効果: 個人情報の流れをエンドツーエンドで可視化し、GDPR やプライバシー規制への対応を効率化。データ侵害時のインパクト範囲を迅速に特定可能。

🔍 ユースケース 2: BI アセットの発見と再利用

シナリオ: 新規プロジェクトのアナリストが、既存の Looker ダッシュボードや Explore を検索して、類似の分析要件に対して既存リソースを再利用したい。

効果: Dataplex Universal Catalog の自然言語検索を使用して「売上分析」のような検索クエリで関連する Looker アセットを即座に発見でき、ゼロからの構築を回避して開発時間を大幅に短縮。

📝 まとめ

今回のアップデートにより、Dataplex Universal Catalog が Looker (Google Cloud core) のメタデータを自動的にカタログ化し、BigQuery から Looker へのデータリネージを追跡できるようになりました。これは、データウェアハウスから BI レイヤーまでのエンドツーエンドのデータガバナンスを実現する重要な一歩です。

💡 推奨アクション: プレビュー段階ではありますが、データガバナンスの強化を検討している組織は早期に検証を開始し、GA 時にスムーズに導入できるよう準備することを推奨します。
