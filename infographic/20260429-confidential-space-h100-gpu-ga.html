🔒 Confidential Space: H100 GPU サポート GA

Confidential Computing GA a3-highgpu-1g
2026-04-29

📋 概要

Confidential Space イメージ (260400) がリリースされ、H100 GPU (a3-highgpu-1g) 上での Confidential Space の実行が一般提供 (GA) となった。NVIDIA H100 GPU のハードウェアベースの機密コンピューティング機能を活用し、GPU ワークロードをハードウェアレベルで分離した信頼実行環境 (TEE) 上で実行可能に。

複数の当事者が所有するセンシティブなデータを機密性を保ちながら処理するための隔離環境を提供。大規模言語モデル (LLM) の推論やファインチューニングなど、GPU を必要とする機密ワークロードを本番環境で安心して実行できる。

対象ユーザー: 医療データ・金融データ・知的財産などの機密データを GPU で処理する必要がある組織、マルチパーティでのデータ共同分析を行う企業、AI/ML モデルの機密保護を必要とするプロバイダー

Before (課題)

  • GPU ワークロードはプレビュー段階で本番環境利用が保証されていなかった
  • 機密 AI/ML ワークロードにカスタムセキュリティソリューションが必要
  • ハードウェアレベルのアテステーションが GA として未提供

After (改善)

  • H100 GPU 上の Confidential Space が GA、SLA 付きサポート提供
  • NVIDIA Confidential Computing で GPU メモリとワークロードをハードウェア保護
  • Intel TDX + NVIDIA H100 の包括的な機密保護を実現

⚡ 主要機能

🖥️ NVIDIA Confidential Computing on H100 GPU
a3-highgpu-1g マシンタイプ (26 vCPU, 234 GB メモリ, 1x H100 80GB HBM3) で、Intel TDX による CPU/メモリ保護と NVIDIA H100 のハードウェアベース GPU 保護を組み合わせ。GPU アテステーションによるワークロードの整合性検証を提供。
💿 Confidential Space イメージ 260400
Container-Optimized OS ベースのハードニングされた新イメージ。本番用 (confidential-space) とデバッグ用 (confidential-space-debug) のイメージファミリーを提供。
🔑 リモートアテステーション
Google Cloud Attestation による OIDC トークンの発行。vTPM (仮想 Trusted Platform Module) による測定ベースのブートシーケンス検証と、ワークロードコンテナイメージのハッシュ検証を実施。

🛠️ 設定方法

前提条件

✓NVIDIA Confidential Computing に対応した GPU クォータの確保
✓Confidential Space 用のサービスアカウントの作成
✓ワークロードコンテナイメージの Artifact Registry への登録
✓ブートディスクサイズ 30 GB 以上の確保 (NVIDIA ドライバー + CUDA ツールキット用)

ステップ 1: Spot VM による Confidential Space インスタンスの作成

Bash Confidential Space インスタンス作成 
gcloud compute instances create INSTANCE_NAME \
  --provisioning-model=SPOT \
  --confidential-compute-type=TDX \
  --machine-type=a3-highgpu-1g \
  --maintenance-policy=TERMINATE \
  --shielded-secure-boot \
  --image-project=confidential-space-images \
  --image-family=confidential-space \
  --metadata="^~^tee-image-reference=us-docker.pkg.dev/PROJECT_ID/REPO/WORKLOAD:latest~tee-install-gpu-driver=true" \
  --service-account=SA_NAME@PROJECT_ID.iam.gserviceaccount.com \
  --scopes=cloud-platform \
  --boot-disk-size=30G \
  --zone=us-central1-a \
  --project=PROJECT_ID
💡 ポイント: tee-install-gpu-driver=true メタデータを設定することで、NVIDIA Confidential Computing に必要なドライバーが自動的にインストールされる。

ステップ 2: ワークロード ID プールの設定

Bash Workload Identity Pool 設定 
gcloud iam workload-identity-pools create POOL_NAME \
  --location="global" \
  --display-name="Confidential Space Pool"

gcloud iam workload-identity-pools providers create-oidc PROVIDER_NAME \
  --location="global" \
  --workload-identity-pool=POOL_NAME \
  --issuer-uri="https://confidentialcomputing.googleapis.com" \
  --attribute-mapping="google.subject=assertion.sub"

データオーナーが Workload Identity Pool を設定し、アテステーション条件に基づいてデータへのアクセスを制御する。

✅ メリット

ビジネス面

✓コンプライアンス強化: HIPAA, PCI DSS 等の規制要件を満たしつつ GPU ワークロード実行可能
✓マルチパーティ協業: 複数組織間で機密データを共有せずに GPU ベースの共同分析が可能
✓知的財産保護: AI/ML モデルの重みやアルゴリズムをオペレーターからも保護

技術面

✓二重保護: Intel TDX (CPU) + NVIDIA CC (GPU) によるハードウェアベースの分離
✓リモートアテステーション: ハードウェアレベルで暗号学的にワークロード整合性を検証
✓GA 安定性: 本番環境向け SLA とサポートの提供

⚠️ 制約事項・注意点

制限事項
✕マルチノードワークロード用のクラスター作成は非サポート
✕リザベーション (予約) は非サポート
✕Hyperdisk Extreme は非サポート
✕ライブマイグレーション非対応 (メンテナンスポリシーは TERMINATE のみ)
✕Spot VM または Flex-start VM でのプロビジョニングが必須
既知の問題
  • GPU アテステーションがインデックス 9 の測定レコード不一致で失敗する問題あり (VM の停止/再起動で回避可能)
  • サポートゾーンが 3 つに限定
  • 推奨 OS イメージは ubuntu-2204-lts または cos-tdx-113-lts に限定

🎯 ユースケース

🏥 ユースケース 1: マルチパーティ機密 AI/ML トレーニング

シナリオ: 複数の医療機関が患者データを共有せずに、共同で疾病予測 AI モデルをファインチューニング

Bash 医療 AI トレーニング例 
# 各医療機関のデータは暗号化されて Cloud Storage に保存
# Confidential Space ワークロードのみがアテステーション後にアクセス可能

gcloud compute instances create medical-ai-training \
  --provisioning-model=SPOT \
  --confidential-compute-type=TDX \
  --machine-type=a3-highgpu-1g \
  --image-project=confidential-space-images \
  --image-family=confidential-space \
  --metadata="^~^tee-image-reference=us-docker.pkg.dev/ml-project/repo/federated-training:v1~tee-install-gpu-driver=true" \
  --boot-disk-size=30G \
  --zone=us-central1-a

効果: 患者データの機密性を保ちながら、H100 GPU の高速処理で大規模モデルのファインチューニングが可能。各医療機関は自身のデータが他者に公開されないことをアテステーションで検証可能。

🏦 ユースケース 2: 機密推論サービス (Confidential Inference)

シナリオ: 金融機関が独自の不正検知 AI モデルを第三者のインフラ上で推論サービスとして提供したいが、モデルの重みを公開したくない

効果: モデルの知的財産 (重み、アーキテクチャ) がハードウェアレベルで保護され、インフラオペレーターからも参照不可。H100 の 80GB HBM3 により大規模モデルの効率的な推論が可能。

💰 料金

Confidential Space on H100 GPU の料金は、a3-highgpu-1g インスタンスの Spot VM 料金に基づく。Confidential Computing の機能自体に追加料金は発生しないが、対象マシンタイプの利用料金が適用される。

Cloud KMS 暗号鍵の管理。アテステーション条件で鍵アクセスを制御可能

📝 まとめ

Confidential Space on H100 GPU の GA は、機密 AI/ML ワークロードを本番環境で実行するための重要なマイルストーンである。Intel TDX と NVIDIA Confidential Computing の組み合わせにより、CPU と GPU の両方でハードウェアレベルの保護が提供され、医療・金融・知的財産保護など、高いセキュリティ要件を持つ GPU ワークロードの実行が可能になった。

推奨事項: 対象ゾーンが限定的であること、Spot/Flex-start VM でのプロビジョニングが必須であることを考慮し、ワークロードの可用性設計を行うことを推奨。
