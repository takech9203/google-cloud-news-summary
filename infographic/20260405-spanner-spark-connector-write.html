DataFrame 書き込みサポート

📋 概要

Spark Spanner コネクタに、Spark DataFrame を Spanner テーブルに書き込む機能が追加されました。これにより、Apache Spark で処理したデータを Spark data source API 経由で直接 Cloud Spanner に保存できます。

これまで読み取り専用だったコネクタが双方向データ連携に対応し、ETL パイプラインの最終段階として Spanner への直接データ投入が可能になりました。

❌ Before

  • コネクタは読み取り専用
  • 書き込みにはカスタムコードや別ツールが必要
  • Spanner クライアントライブラリの直接使用が必須
  • 読み取り/書き込みで異なるインターフェース

✅ After

  • 標準 Spark API で書き込み可能
  • df.write.format('cloud-spanner') で完結
  • PySpark / Scala 両対応
  • 同一コネクタで読み書き統一

⚡ 主要機能

📝 Spark data source API による書き込み

df.write.format('cloud-spanner') で標準的な Spark の書き込み API を使用。mode("append") による追記書き込みをサポート。

🔗 接続パラメータの統一

projectId、instanceId、databaseId、table の各オプションで書き込み先を指定。読み取り時と同じパラメータ体系。

🐍 PySpark / Scala 両対応

Python (PySpark) と Scala の両方から同じインターフェースで書き込み可能。Dataproc ジョブおよび spark-submit の両方に対応。

💻 設定方法・コードサンプル

前提条件

💡 必要な準備:
  • Dataproc クラスタ (イメージバージョン 2.1 以降)
  • 書き込み先の Spanner インスタンス / データベース / テーブルが作成済み
  • IAM ロール: roles/dataproc.worker + roles/spanner.databaseUser

ステップ 1: PySpark スクリプトの作成

Python 
"""Spanner PySpark write example."""
from pyspark.sql import SparkSession

spark = SparkSession.builder.appName('Spanner Write App').getOrCreate()

columns = ['id', 'name', 'email']
data = [
    (1, 'John Doe', 'john.doe@example.com'),
    (2, 'Jane Doe', 'jane.doe@example.com')
]

df = spark.createDataFrame(data, columns)

df.write.format('cloud-spanner') \
    .option("projectId", "PROJECT_ID") \
    .option("instanceId", "INSTANCE_ID") \
    .option("databaseId", "DATABASE_ID") \
    .option("table", "TABLE_NAME") \
    .mode("append") \
    .save()

🎯 ユースケース

1. バッチ ETL パイプライン

Cloud Storage 上の CSV / Parquet データを Spark で加工し、正規化・クレンジングした結果を Spanner のトランザクションテーブルに投入。中間ストレージを経由せず、パイプラインのレイテンシと複雑性を大幅に削減。

Python 
# Cloud Storage からデータを読み込み
raw_df = spark.read.parquet("gs://my-bucket/raw-data/")

# データ加工
processed_df = raw_df.filter(raw_df.status == "active") \
    .select("user_id", "name", "email")

# Spanner に書き込み
processed_df.write.format('cloud-spanner') \
    .option("projectId", "my-project") \
    .option("instanceId", "my-instance") \
    .option("databaseId", "my-database") \
    .option("table", "Users") \
    .mode("append") \
    .save()

2. 機械学習の推論結果の保存

Spark MLlib で実行した推論結果 (スコアリング、分類結果など) を、アプリケーションが参照する Spanner テーブルに書き込み、リアルタイムサービングに活用。推論パイプラインと結果の永続化が単一の Spark ジョブ内で完結。

✅ メリット

ビジネス面

✓パイプラインの簡素化 - 中間ストレージや追加 ETL ツールが不要、運用コスト削減
✓開発速度の向上 - 標準 Spark API を使用、既存スキルで即活用可能

技術面

✓統一的な API - 読み書き両方を cloud-spanner フォーマットで統一
✓Spark エコシステム統合 - data source API 準拠、他コネクタと同様のパターン

⚠️ 制約事項・注意点

制限事項:
  • Dataproc クラスタ上での実行が前提 (ローカル環境にはクラスタ接続設定が必要)
  • 書き込み先の Spanner テーブルは事前に作成しておく必要あり
考慮すべき点:
  • 大量データ書き込み時は Spanner のスループットとノード数に応じたパフォーマンスチューニングが必要
  • IAM ロール (roles/spanner.databaseUser) の付与が必要。最小権限の原則に基づく権限管理を推奨

🔗 関連サービス

Dataproc

Cloud Spanner

📌 まとめ: Spark Spanner コネクタに書き込み機能が追加され、Dataproc 上の Spark ジョブから標準 API で Spanner テーブルへのデータ投入が可能に。読み取りと書き込みの双方向データ連携が統一インターフェースで実現し、ETL パイプライン設計がよりシンプルになります。Dataproc + Spanner を運用しているチームは、既存の書き込みロジックのコネクタベースへの移行を検討してください。
