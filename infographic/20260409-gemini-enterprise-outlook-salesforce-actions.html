🤖 Gemini Enterprise: Outlook / Salesforce 向け新アクション

Gemini Enterprise Public Preview Microsoft Outlook Salesforce
2026-04-09
Google Cloud Release Notes

📋 概要

Gemini Enterprise において、Microsoft Outlook および Salesforce のデータストアに対する新しいアクション機能が Public Preview として利用可能になりました。 自然言語コマンドを使用して、Gemini Enterprise のアシスタントからこれらのサードパーティアプリケーション上で直接操作を実行できます。

2025 年 12 月に統合データソース作成フローの一部として初めて導入されたアクション機能が、今回のアップデートで Microsoft Outlook と Salesforce に拡大されました。 営業チーム、カスタマーサポート、および日常的にこれらのツールを使用するビジネスユーザーにとって、アプリケーション切り替えなしで操作が完結する大きな効率化が期待できます。

🔄 Before / After

❌ Before

  • Outlook / Salesforce のデータは検索(フェデレーション検索・データ取り込み)のみ可能で、書き込み操作は不可
  • メール送信やカレンダー操作には Outlook アプリへの切り替えが必要
  • Salesforce データ操作は Salesforce 管理画面から直接行う必要がありワークフローが分断

✅ After

  • Gemini Enterprise アシスタントから自然言語で Outlook アクション(メール送信、イベント作成・更新、連絡先管理、添付ファイルDL)を実行可能
  • Salesforce データストアに対してもアクションが利用可能になり、検索+データ操作が可能
  • 単一インターフェースから複数サードパーティサービスへの操作をシームレスに実現

⚡ 主要機能

📧 Microsoft Outlook アクション

自然言語コマンドで Outlook 操作を実行可能。フェデレーション検索・データ取り込みの両方でアクションをサポート。Microsoft Graph API v1.0 を使用した安全な操作。

☁️ Salesforce アクション

Salesforce データストアに対するアクション機能が新たに利用可能。フェデレーション検索と組み合わせたデータ操作。Salesforce V2 コネクタ(推奨)を使用。

⚙️ 統合アクション管理

データストア作成時または作成後にアクションを追加可能。Google Cloud コンソールの「Actions」ページからアクションの有効化・無効化を管理。再認証機能にも対応。

🛠️ 設定方法

前提条件

✓Gemini Enterprise ライセンスが有効 (Standard, Plus, Business)
✓Google Cloud プロジェクトで Gemini Enterprise が有効化済み
✓Discovery Engine Editor ロール (roles/discoveryengine.editor) が付与済み
✓Outlook: Microsoft Entra ID でのアプリケーション登録 + API 権限設定
✓Salesforce: OAuth 認証情報 (Client ID, Client Secret) の取得

データストアの作成

Google Cloud コンソールで Gemini Enterprise ページに移動し、「Data Stores」から「Create Data Store」をクリック。データソース選択画面で「Microsoft Outlook」または「Salesforce V2」を選択します。

認証設定

Microsoft Outlook: Client ID, Client Secret, Tenant ID を入力し、「Login」で Microsoft Outlook にサインインして認証を完了。

Salesforce: Instance URL, OAuth Client ID, Client Secret を入力し、「Login」で Salesforce にサインインして認証を完了。

アクションの有効化

データストア作成フローの「Actions」セクションで有効にするアクションを選択し、「Enable actions」をクリック。フェデレーション検索モード / データ取り込みモードそれぞれで設定方法が異なります。

アプリとの接続

アプリを作成し、作成したデータストアを接続します。ユーザーがアクションを使用する前に、各サービスへのアクセスを承認する必要があります。

✅ メリット

💼 ビジネス面

→ワークフローの統合: 単一インターフェースから Outlook / Salesforce を操作し、アプリ間の切り替え不要で業務効率向上
→生産性の向上: 自然言語でのコマンド実行により、技術知識がなくてもデータ操作が可能
→営業プロセス効率化: Salesforce CRM データ参照と Outlook メール送信・ミーティング設定をシームレスに連携

🔧 技術面

→API 統合の簡素化: Microsoft Graph API / Salesforce API への直接プログラミングが不要。自然言語で操作に集約
→権限管理の一元化: Google Cloud IAM + 各サービスの OAuth 認証を組み合わせた統一的アクセス管理
→拡張性: 既存データストアに対してデータストア作成後でもアクションを追加可能な柔軟な設計

⚠️ デメリット・制約事項

制限事項

⚠️Public Preview のため「Pre-GA Offerings Terms」が適用。限定的サポートの可能性あり
⚠️Salesforce データ取り込みモードは Private Preview (許可リストへの追加が必要)
⚠️Salesforce データストアは Global / US / EU ロケーションのみサポート
⚠️既存 Salesforce データストアへの VPC Service Controls 適用は非サポート (データストア削除・再作成が必要)
⚠️1 つのアプリに同一コネクタタイプの複数データストア(アクション付き)の関連付けは非推奨

考慮すべき点

🔒Outlook アクション使用には Microsoft Entra ID での適切な API 権限設定が必須
🔒フェデレーション検索時、クエリはサードパーティの検索バックエンドに送信されるため、データ処理に関する利用規約・プライバシーポリシーへの準拠が必要
🔒LLM によるクエリ書き換えで、セッション履歴の一部がサードパーティ API に送信される可能性あり
