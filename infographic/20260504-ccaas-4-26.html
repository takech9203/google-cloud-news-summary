🎯 概要

Google Cloud CCaaS バージョン 4.26 がリリースされました。チャットルーティングの最適化、コールスケジューリングの大幅改善、HubSpot CRM 連携の強化、アクセシビリティ対応の拡充など、コンタクトセンター運用の効率化と品質向上に寄与する多数の機能が追加されています。

CCaaS は Gemini Enterprise for CX の一部として、AI ベースのオムニチャネルルーティング、バーチャルエージェント、Agent Assist、インサイト機能を提供するフルスタックコンタクトセンタープラットフォームです。

  • レコード自動作成のスキップが可能
  • 転送時も Agent Assist が正しく表示
  • WCAG 準拠・RTL・スクリーンリーダー対応

⚡ 主要機能

🔀 Fewest Agents Routing (最少エージェントルーティング)

新しいチャットルーティングロジックにより、できるだけ少ないエージェントにチャットを集中的に割り当て。エージェントの稼働率を最大化し、遊休エージェント数を最小化します。キューレベルで有効/無効を切り替え可能。

📅 コールスケジューリングの改善

タイムスロットの粒度を細かく設定可能に。顧客による再スケジュールとキャンセルに対応。キューレベルでの設定が可能。Headless Web SDK に fetchTimeSlotAvailability メソッドを追加。

🔔 Breakthrough for Wrap-up (ラップアップ中の割り込み応答)

エージェントがラップアップ (後処理) 中でも新規着信コールに応答可能。応答待ち時間の短縮とエージェント稼働率の向上に貢献。管理者がキューレベルで制御可能。

🤖 Agent Assist の転送時可視性向上

ダイレクト着信コールの転送時にも Agent Assist が正しく動作。転送先エージェントがリアルタイムで AI アシスト情報を確認可能。顧客との会話コンテキストが転送後も維持されます。

🔗 HubSpot CRM 連携の強化

レコード自動作成のスキップオプション追加。SDK カスタムデータから HubSpot フィールドへのカスタムマッピングに対応。

♿ アクセシビリティ改善

WCAG 準拠の強化、RTL 言語サポート追加、スクリーンリーダー対応の改善。

ビジネス面

→デバッグ効率向上: getLogs メソッドでトラブルシュート改善

⚠️ 制約事項・注意点

⚠️ デプロイメントスケジュールに依存するため、即座にすべてのインスタンスに適用されるわけではない
⚠️ Fewest Agents Routing は現時点でチャットチャネルのみ対象 (音声チャネルは未対応)
⚠️ HubSpot カスタムフィールドマッピングは SDK カスタムデータに限定
⚠️ Breakthrough 機能はラップアップ品質とのトレードオフを考慮する必要あり
⚠️ 30 件以上のバグ修正が含まれるため、アップデート後の十分な動作検証を推奨

💡 ユースケース

🛒 ユースケース 1: 大規模チャットサポートの効率化

シナリオ: EC サイトのカスタマーサポートで、繁忙期に 100 名以上のエージェントがチャット対応。従来のラウンドロビンでは一部エージェントが遊休状態となり、人件費効率が低下。

✔ 効果: Fewest Agents Routing でアクティブなエージェント数を最小化し、残りを他業務に割り当て可能。稼働率 15-20% 向上が期待される。

🏢 ユースケース 2: 保険会社のコールバック予約システム

シナリオ: 保険会社のコールセンターで、顧客が Web サイトからコールバック予約を行う。従来はタイムスロットの変更やキャンセルが電話でしかできなかった。

✔ 効果: 顧客が自身で再スケジュール・キャンセルを Web 上で完結可能。fetchTimeSlotAvailability でリアルタイムの空き状況を表示。

💻 ユースケース 3: HubSpot を利用する SaaS 企業のサポート

シナリオ: SaaS 企業が HubSpot CRM と CCaaS を連携。チャットボットとの会話でも CRM レコードが自動作成され、データベースが不要レコードで肥大化。

✔ 効果: レコード自動作成のスキップ設定により、人的対応時のみレコード作成。カスタムフィールドマッピングで適切なフィールドに自動マッピング。
