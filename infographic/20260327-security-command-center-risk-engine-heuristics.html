📋 概要

Security Command Center の Risk Engine において、デフォルト高価値リソースセットの識別精度を向上させる強化ヒューリスティクスがリリースされました。本番環境以外のリソースをより正確に識別し、攻撃エクスポージャスコアの計算優先順位を最適化します。

💡 対象ユーザー: Security Command Center Premium および Enterprise ティアを組織レベルで有効化しているユーザーが対象です。カスタムの高価値リソースセットを定義している場合は影響を受けません。

🔄 Before / After 比較

❌ アップデート前

  • 本番環境と非本番環境のリソースが同等に扱われていた
  • 非本番リソースにも同じ優先度でスコアが計算され、注意が分散
  • 攻撃エクスポージャスコアが実際のビジネスリスクを正確に反映していない場合があった

✅ アップデート後

  • 強化ヒューリスティクスにより非本番リソースを自動的に識別
  • 本番環境のリソースに対するスコア計算が優先され、重要なアセットから先にスコアを算出
  • セキュリティチームが真にリスクの高いリソースに集中でき、対応効率が向上

⚡ 主要機能

🔍 1. 非本番リソースの自動識別

Risk Engine がヒューリスティクスを使用して、デフォルト高価値リソースセット内の非本番用途のアセットを自動的に識別します。開発環境、テスト環境、ステージング環境などのリソースが対象です。

📊 2. スコア計算の優先順位最適化

本番環境のリソースに対する攻撃エクスポージャスコアが先に計算されます。非本番リソースのスコアは後順位で計算されるため、重要なアセットの情報が確実に提供されます。

🎯 3. エクスポージャスコアの精度向上

検出結果 (Findings)、リソース、およびイシューのスコアがより正確にビジネスリスクを反映。Toxic Combination や Chokepoint の検出精度も間接的に向上します。

⚙️ 設定方法

📌 前提条件: Security Command Center Premium または Enterprise ティアが組織レベルで有効化されていること。適切な IAM ロールが付与されていること。
1

現在の高価値リソースセットの確認

Google Cloud コンソールで Security Command Center の設定ページを開き、Attack path simulation タブから現在の高価値リソースセットを確認します。デフォルトセットを使用している場合、今回のヒューリスティクス強化の影響を受けます。

2

エクスポージャスコアの変動確認

シミュレーション実行後、Google Cloud コンソールの Security Command Center > Assets > High value resource set タブで攻撃エクスポージャスコアの変動を確認します。

3

カスタム高価値リソースセットの定義 (推奨)

Security Command Center の Settings > Attack path simulation タブで Create new configuration をクリックし、リソースタイプ、スコープ、タグ、リソース値を指定してカスタム構成を作成します。

⚖️ メリット / 制約事項

✅ メリット

ビジネス面

  • セキュリティ対応の効率化 -- 本番環境の重要リソースに対するスコアが優先的に計算され、迅速な対応が可能
  • リスク可視化の精度向上 -- 非本番リソースによるノイズが低減され、経営層への報告がより正確に

技術面

  • 自動的な優先順位付け -- ヒューリスティクスによる自動判定で手動構成変更が不要
  • スコア信頼性の向上 -- Toxic Combination や Chokepoint の対応判断が改善

⚠️ 制約事項・考慮点

  • デフォルト高価値リソースセット使用時のみ影響 (カスタム構成定義済みの場合は無関係)
  • 組織レベルでの SCC 有効化が必須 (プロジェクトレベルでは利用不可)
  • ヒューリスティクスの判定基準は非公開のため、分類の事前予測は困難
  • スコア変動により、既存のアラートしきい値やダッシュボードの見直しが必要になる場合がある
  • 重要なリソースが非本番と誤判定されていないか確認が推奨
  • より正確なスコアにはカスタム高価値リソースセットの定義が推奨

💼 ユースケース

🏢 ユースケース 1: 大規模組織でのセキュリティ優先順位付け

📝 シナリオ: 数千の Compute Engine インスタンスと Cloud Storage バケットを持つ大規模組織で、開発・テスト・本番環境が混在。デフォルト高価値リソースセットを使用しているが、非本番リソースのスコアがノイズとなり、本番環境の重要な脆弱性への対応が遅れていた。
✅ 効果: 強化ヒューリスティクスにより、本番環境のリソースに対するスコアが優先的に計算され、セキュリティチームは最も重要な脆弱性から順に対応できるようになる。

🔗 ユースケース 2: Toxic Combination 検出の精度向上

📝 シナリオ: 複数のクラウドサービスを利用する組織で、Toxic Combination (複合的な脆弱性の組み合わせ) の検出結果が多数報告されている。非本番リソースに関連する検出結果も含まれており、優先度の判断が難しかった。
✅ 効果: 本番環境に関連する Toxic Combination のスコアが優先的に計算されるため、実際にビジネスインパクトの大きい複合脆弱性への対応を効率的に進められる。
