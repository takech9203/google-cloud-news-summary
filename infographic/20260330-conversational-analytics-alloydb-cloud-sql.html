📋 概要

AlloyDB for PostgreSQL、Cloud SQL for MySQL、Cloud SQL for PostgreSQL の 3 サービスに対して、Conversational Analytics 機能が Preview として同時提供開始されました。

Conversational Analytics API (geminidataanalytics.googleapis.com) は Gemini for Google Cloud を活用した AI 搭載のデータエージェント基盤です。従来は BigQuery、Looker、Looker Studio 向けに提供されていましたが、今回のアップデートで QueryData メソッドを通じて運用データベースへの自然言語クエリが可能になりました。

🤖 ポイント: SQL の専門知識がないビジネスユーザーでも、運用データベースから直接インサイトを引き出せるようになります。

⚡ 主要機能

💬 自然言語データクエリ (QueryData)

自然言語の質問を SQL に自動変換し実行。AlloyDB、Cloud SQL for MySQL/PostgreSQL をサポート。

POST /v1beta/.../conversations:queryData

🤖 データエージェント機能

ビジネス固有のコンテキストや指示を設定できるエージェントを作成。ゴールデンクエリの登録で精度向上。

🐘 AlloyDB AI Natural Language API

AlloyDB 固有の高度な機能群:
get_sql() - SQL 生成
execute_nl_query() - 直接回答
get_sql_summary() - 結果要約
スキーマコンテキスト自動生成、セキュアビューによるアクセス制御

🔧 マルチツール対応

SQL、Python、可視化ライブラリなどのツールを利用可能。テキスト、データ、チャートなど複数形式で回答出力。

🔧 設定方法

前提条件

✓Google Cloud プロジェクトで課金が有効化されていること
✓Conversational Analytics API (geminidataanalytics.googleapis.com) が有効化されていること
✓AlloyDB / Cloud SQL インスタンスが作成済みであること
✓適切な IAM ロール (cloudaicompanion.topicAdmin など) が付与されていること

Step 1: API の有効化

Bash Conversational Analytics API を有効化 
gcloud services enable geminidataanalytics.googleapis.com \
  --project=YOUR_PROJECT_ID

Step 2: 認証とアクセストークンの取得

Bash アクセストークン取得と API リクエスト 
# アクセストークンの取得
ACCESS_TOKEN=$(gcloud auth print-identity-token)

# API へのリクエスト例
curl -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  https://geminidataanalytics.googleapis.com/v1beta/projects/YOUR_PROJECT_ID/locations/global/dataAgents

Step 3: QueryData メソッドによるデータクエリ

Bash 自然言語でデータベースにクエリ 
curl -X POST \
  -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  "https://geminidataanalytics.googleapis.com/v1beta/projects/YOUR_PROJECT_ID/locations/global/conversations:queryData" \
  -d '{
    "query": "先月の売上が最も高かった製品を教えてください"
  }'

Step 4: AlloyDB 固有の設定

SQL alloydb_ai_nl 拡張のインストールと設定 
-- alloydb_ai_nl 拡張のインストール
ALTER SYSTEM SET alloydb_ai_nl.enabled = on;
SELECT pg_reload_conf();
CREATE EXTENSION alloydb_ai_nl CASCADE;

-- スキーマコンテキストの自動生成と適用
SELECT alloydb_ai_nl.generate_schema_context('my_config', TRUE);
SELECT alloydb_ai_nl.apply_generated_schema_context('my_config', TRUE);

-- 自然言語から SQL を生成
SELECT alloydb_ai_nl.get_sql('my_config', '先月の売上トップ3の製品は?') ->> 'sql';

-- 自然言語の質問に直接回答
SELECT alloydb_ai_nl.execute_nl_query('my_config', '東京のお客様の数は?');

-- 結果の要約を取得
SELECT alloydb_ai_nl.get_sql_summary(
  nl_config_id => 'my_config',
  nl_question => '最も売上が多いブランドはどれですか?'
);

alloydb_ai_nl 拡張でスキーマコンテキスト自動生成やセキュアビューなど高度な機能を提供。

⚠️ デメリット・制約事項

⚠️ 制限事項
  • Preview ステータス - 本番環境での使用は推奨されない (Pre-GA Offerings Terms が適用)
  • QueryData メソッドは BigQuery や Looker のデータソースをサポートしない (逆に Chat/DataAgent メソッドは AlloyDB/Cloud SQL をサポートしない)
  • AI 生成の出力は事実と異なる可能性があるため、すべての出力の検証が推奨される
  • Preview 期間中は限定的なサポートのみ
💡 考慮すべき点
  • スキーマコンテキストやゴールデンクエリの適切な設定が回答精度の向上に重要
  • 個人データや法的・規制コンプライアンス要件の対象データは、テスト環境でのみ使用すべき
  • Vertex AI の利用料金が別途発生する可能性がある (AlloyDB AI Natural Language 使用時)

🎯 ユースケース

📈 ユースケース 1: セールスアナリティクス

シナリオ: 営業マネージャーが AlloyDB の CRM データに自然言語で問い合わせ、営業パフォーマンスを把握。

SQL AlloyDB AI Natural Language API の使用例 
-- 営業トップ3の取得
SELECT alloydb_ai_nl.get_sql(
  'sales_config',
  '先月の東京オフィスの売上トップ3の営業担当者は誰ですか?'
) ->> 'sql';

-- 四半期トレンドの要約
SELECT alloydb_ai_nl.get_sql_summary(
  nl_config_id => 'sales_config',
  nl_question => '四半期ごとの売上トレンドを教えてください'
);

効果: 営業チームが SQL なしでリアルタイムの営業データを分析し、迅速な意思決定が可能に。

🎧 ユースケース 2: カスタマーサポート分析

シナリオ: サポート部門マネージャーが Cloud SQL for MySQL のチケットデータに自然言語で問い合わせ、サポート品質のトレンドを把握。

効果: 問い合わせカテゴリの傾向や対応時間の分析をセルフサービスで実行し、サービス品質改善に直結。

🚚 ユースケース 3: サプライチェーン最適化

シナリオ: ロジスティクス担当者が Cloud SQL for PostgreSQL の出荷データに「3日以上遅延している出荷を表示して」のような自然言語クエリを実行。

効果: リアルタイムの運用データに基づき、配送遅延の早期発見と対応が可能に。

📝 まとめ

3 サービスで Conversational Analytics 機能が Preview 提供されたことは、Google Cloud のデータベースサービスにおける AI 統合の大きな進展です。QueryData メソッドで運用 DB への自然言語クエリが統一的に実現され、特に AlloyDB では alloydb_ai_nl 拡張によるスキーマ認識型の高度な SQL 生成機能も利用できます。
