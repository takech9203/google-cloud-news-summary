Buildpacks Cloud Run GA
2026-02-11

📋 概要

Google Cloud は、Buildpacks と Cloud Run の両サービスにおいて osonly24 ランタイムの一般提供 (GA) を発表しました。osonly24 は言語に依存しない汎用ランタイムで、Ubuntu 24 LTS ベースの最小構成 OS イメージのみを含み、言語固有のランタイムコンポーネントを含みません。

Go アプリケーションのソースからのデプロイや、Dart / Go などのコンパイル済みバイナリのデプロイが可能になります。Cloud Run の自動ベースイメージ更新に対応しており、OS レベルのセキュリティパッチが自動的に適用されます。2025 年 12 月 18 日に Preview として公開され、今回の GA 昇格で本番利用が正式サポートされました。

❌ Before (GA 以前の課題)

  • Go / Dart バイナリのデプロイには Dockerfile の自前作成が必要
  • 言語固有ランタイム使用時に不要なコンポーネントが含まれ、イメージサイズが増大
  • Preview 段階のため SLA が適用されず本番利用に不安

✅ After (GA による改善)

  • --base-image=osonly24 指定だけで最小構成デプロイが可能
  • GA ステータスで SLA 適用、本番環境の正式サポート
  • 自動ベースイメージ更新で OS セキュリティパッチがゼロダウンタイム適用

⭐ 主要機能

🐧 OS のみの最小ランタイム

Ubuntu 24 LTS ベースの google-24 スタック上で動作。言語固有のランタイムコンポーネントを含まず、OS パッケージのみを提供します。

→ ランタイム ID: osonly24
→ ベースイメージ: google-24/osonly24

📦 コンパイル済みバイナリのデプロイ

Go や Dart などのコンパイル済みバイナリを直接デプロイ可能。Linux OS をターゲットにバイナリをビルドする必要があります。

→ 例: GOOS="linux" GOARCH=amd64 go build

🔄 2 つのデプロイ方式

ビルド付きデプロイ (GA): Buildpacks 経由でソースからビルドしてデプロイ。ビルドなしデプロイ (Preview): Cloud Build をスキップして直接デプロイ。

🛡️ 自動ベースイメージ更新

Cloud Run の自動ベースイメージ更新機能に対応。特定の言語ランタイムなしでも OS レベルのセキュリティパッチがゼロダウンタイムで自動適用されます。

⚙️ 設定方法

📝 前提条件: Google Cloud プロジェクトが作成済み / Cloud Run API が有効化済み / gcloud CLI が最新バージョンにインストール済みであること

ステップ 1: Go アプリケーションをソースからデプロイ (ビルド付き)

Bash ビルド付きデプロイ 
gcloud run deploy SERVICE \
  --source . \
  --base-image=osonly24 \
  --project PROJECT_ID \
  --automatic-updates

--automatic-updates フラグにより、自動ベースイメージ更新が有効になります。

ステップ 2: コンパイル済みバイナリをデプロイ (ビルドなし)

Go バイナリを事前にコンパイルします。

Bash Go バイナリのクロスコンパイル 
GOOS="linux" GOARCH=amd64 go build main.go

ビルドなしでデプロイします。

Bash ビルドなしデプロイ 
gcloud beta run deploy SERVICE \
  --source . \
  --no-build \
  --base-image=osonly24 \
  --project PROJECT_ID \
  --command ./main

--no-build フラグにより Cloud Build ステップがスキップされ、デプロイ時間が大幅に短縮されます。

⚠️ デメリット・制約事項

制限事項

  • ビルドなしデプロイはまだ Preview 段階であり、GA ではない
  • コンパイル済みバイナリは x86 (linux/amd64) と互換性が必要
  • ビルドなしデプロイ時のソースアーカイブサイズは 250 MiB 以下に制限
  • 自動ベースイメージ更新ではコンパイル言語のアプリケーションは再コンパイルされない (言語レベルのセキュリティ修正には再ビルドが必要)
  • Dockerfile を使用したデプロイには非対応

考慮すべき点

  • コンパイル済みバイナリ使用時は Linux OS をターゲットにしたクロスコンパイルが必要
  • ビルドなしデプロイでは依存関係を事前にローカルでインストールする必要がある
  • osonly24 の非推奨予定日は 2029 年 4 月 30 日であり、長期的な移行計画を考慮すること

🎯 ユースケース

🐹 ユースケース 1: Go マイクロサービスの高速デプロイ

シナリオ: Go で記述された複数のマイクロサービスを Cloud Run で運用しており、デプロイサイクルを高速化したい場合。

Bash Go マイクロサービスのデプロイ例 
# Go バイナリをクロスコンパイル
GOOS="linux" GOARCH=amd64 go build -o myservice main.go

# Cloud Build をスキップして直接デプロイ
gcloud beta run deploy my-go-service \
  --source . \
  --no-build \
  --base-image=osonly24 \
  --region=asia-northeast1 \
  --command=./myservice
✅ 効果: Cloud Build ステップを省略し、デプロイ時間が大幅に短縮。CI/CD パイプラインでビルドはローカル / CI 環境で実行し、デプロイのみを Cloud Run に委譲するワークフローが実現。

🎯 ユースケース 2: Dart アプリケーションのサーバーレスデプロイ

シナリオ: Dart で構築したバックエンド API サーバーを、言語固有のランタイムなしで Cloud Run にデプロイしたい場合。

Bash Dart アプリケーションのデプロイ例 
# Dart アプリケーションをコンパイル
dart compile exe bin/server.dart -o server

# osonly24 ランタイムでデプロイ
gcloud beta run deploy my-dart-api \
  --source . \
  --no-build \
  --base-image=osonly24 \
  --region=asia-northeast1 \
  --command=./server
✅ 効果: Dart 専用のランタイムが不要で、コンパイル済みバイナリを最小構成の OS 上で実行可能。

💰 料金

osonly24 ランタイム自体には追加料金は発生しません。Cloud Run の標準料金体系が適用されます。

✅ ビルド付きデプロイ

Cloud Run 料金 + Cloud Build 料金

✅ ビルドなしデプロイ

Cloud Run 料金のみ (Cloud Build 不要)

詳細は Cloud Run の料金ページ を参照してください。

🌍 利用可能リージョン

osonly24 ランタイムは Cloud Run が利用可能なすべてのリージョン (40 以上) で使用できます。主要なリージョンは以下の通りです。

🗾 asia-northeast1 (東京) 🗾 asia-northeast2 (大阪) 🇺🇸 us-central1 (アイオワ) 🇺🇸 us-east1 (サウスカロライナ) 🇪🇺 europe-west1 (ベルギー)

全リージョンの一覧は Cloud Run のロケーション を参照してください。

📝 まとめ

osonly24 ランタイムの GA により、Go や Dart などのコンパイル済みバイナリを最小構成の OS ランタイム上で本番環境にデプロイできるようになりました。自動ベースイメージ更新との互換性により、言語固有のランタイムを使用しない場合でも OS レベルのセキュリティが自動的に維持されます。Go や Dart でマイクロサービスを構築しているチームは、osonly24 ランタイムの採用を検討することを推奨します。
