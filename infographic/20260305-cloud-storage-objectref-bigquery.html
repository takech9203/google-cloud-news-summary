☁️ Storage Insights ObjectRef Functions for BigQuery

Cloud Storage BigQuery Preview
2026-03-05

📋 概要

Cloud Storage の Storage Insights データセットに BigQuery ObjectRef 関数のサポートが追加されました。データセット内の ref カラムを使い、BigQuery から Cloud Storage オブジェクトのコンテンツを直接分析できます。

🤖 AI 関数との統合: AI.GENERATE や AI.GENERATE_TEXT と組み合わせることで、SQL クエリだけで機密情報検出や画像説明文生成などの高度なコンテンツ分析が可能になります。

🔄 Before / After

❌ Before

✕メタデータ (サイズ、ストレージクラスなど) の分析のみ可能
✕機密情報検出には DLP API と専用パイプラインが必要
✕画像説明文生成には複数サービスの組み合わせが必要
✕コンテンツ分析にはオブジェクトのダウンロードが必要

✅ After

✓メタデータ + コンテンツの両方を BigQuery から分析可能
✓SQL クエリだけで機密情報検出が実現
✓AI 関数と ObjectRef の組み合わせで画像説明文を自動生成
✓データパイプライン不要、統一的な SQL インターフェース

Storage Insights データセットが Cloud Storage オブジェクトのメタデータを収集し、BigQuery リンクドデータセットとして公開します。ObjectRef 関数により BigQuery から Cloud Storage オブジェクトのコンテンツに直接アクセスし、AI 関数と組み合わせた高度な分析が可能になります。

⚡ 主要機能

🔗 Storage Insights データセットの ref カラム

オブジェクトメタデータテーブルの ref カラムが各行の Cloud Storage オブジェクトへの ObjectRef 参照を保持。メタデータだけでなくオブジェクトコンテンツにもアクセス可能。

⚙️ ObjectRef 関数によるオブジェクトアクセス

→OBJ.MAKE_REF: Cloud Storage オブジェクトの URI と接続情報から ObjectRef 値を作成
→OBJ.FETCH_METADATA: ObjectRef 値から Cloud Storage メタデータ (content_type, md5_hash, size, updated) を取得
→OBJ.GET_ACCESS_URL: ObjectRef から署名付き読み取り/書き込み URL を生成 (有効期限は最大 6 時間)

🤖 AI 関数との統合によるコンテンツ分析

AI.GENERATE、AI.GENERATE_TEXT などの生成 AI 関数に ObjectRefRuntime 値を入力として渡すことで、Gemini モデルを使用したコンテンツ分析が可能。画像説明文の生成、ドキュメントの要約、機密情報の検出などを SQL クエリで実行できます。

📐 技術仕様

ObjectRef 値の構造

項目詳細
uriCloud Storage オブジェクト URI (例: gs://mybucket/path/to/file.jpg)
versionCloud Storage オブジェクトバージョン
authorizerBigQuery がオブジェクトにアクセスする際に使用する Cloud リソース接続
detailsCloud Storage が管理するオブジェクトメタデータ (JSON)

ObjectRefRuntime 値

項目詳細
obj_ref元の ObjectRef 値 (uri, version, authorizer, details)
access_urls.read_url読み取り専用の署名付き URL
access_urls.write_url書き込み可能な署名付き URL
access_urls.expiry_timeURL の有効期限 (YYYY-MM-DD'T'HH:MM:SS'Z' 形式)

🛠️ 設定方法

前提条件

✓Storage Intelligence が有効化されていること
✓Storage Insights データセットが作成・設定済みであること
✓データセットが BigQuery にリンクされていること
✓Cloud リソース接続が作成されていること

Step 1: Storage Insights データセットのリンク確認

Bash 
# データセット設定の確認
gcloud storage insights dataset-configs list --location=us-central1

# BigQuery へのリンク作成 (未リンクの場合)
gcloud storage insights dataset-configs create-link my_config --location=us-central1

Step 2: 画像説明文の生成例

SQL 
-- Storage Insights データセットの ref カラムから ObjectRefRuntime を生成し、
-- AI 関数で画像説明文を生成する例
SELECT
  obj_metadata.name,
  AI.GENERATE_TEXT(
    'gemini-2.0-flash',
    'Describe this image in detail',
    OBJ.GET_ACCESS_URL(obj_metadata.ref, 'r', INTERVAL 1 HOUR)
  ) AS image_description
FROM
  `project.dataset.objects_metadata` AS obj_metadata
WHERE
  obj_metadata.content_type LIKE 'image/%';

Step 3: 機密情報の検出

SQL 
-- ドキュメント内の機密情報を検出するクエリ例
SELECT
  obj_metadata.name,
  obj_metadata.bucket,
  AI.GENERATE(
    'gemini-2.0-flash',
    CONCAT(
      'Analyze this document for sensitive information such as ',
      'PII, credit card numbers, or API keys. ',
      'Return a JSON with fields: has_sensitive_data (boolean), ',
      'types (array of strings), confidence (float).'
    ),
    OBJ.GET_ACCESS_URL(obj_metadata.ref, 'r', INTERVAL 1 HOUR)
  ) AS sensitivity_analysis
FROM
  `project.dataset.objects_metadata` AS obj_metadata
WHERE
  obj_metadata.content_type IN ('application/pdf', 'text/plain', 'text/csv');

✅ メリット

💼 ビジネス面

→データガバナンスの強化: SQL クエリだけで組織全体の機密情報一括検出が可能。コンプライアンス対応が効率化。
→運用コスト削減: 専用データパイプラインの構築・運用が不要になり、インフラコストと開発工数を削減。

🔧 技術面

→統一的なインターフェース: BigQuery SQL からメタデータとコンテンツの両方を分析可能。
→スケーラブルな分析: BigQuery のサーバーレスアーキテクチャで大量オブジェクトを並列処理。
→AI 関数直接統合: Gemini モデル活用の高度な分析を追加インフラなしで利用可能。

⚠️ デメリット・制約事項

制限事項

✕ObjectRef 関数は Preview ステータスであり、本番環境での SLA が限定的な可能性がある
✕1 プロジェクトあたり ObjectRef/ObjectRefRuntime を参照する接続は最大 20 個に制限
✕ObjectRefRuntime のアクセス URL は最大 6 時間で失効。長時間ワークフローでは再生成が必要

考慮すべき点

!Storage Intelligence サブスクリプションが前提条件 (追加コスト発生)
!AI 関数の呼び出しには BigQuery 処理コスト + Gemini 推論コストが発生
!大量オブジェクトのコンテンツ分析では、署名付き URL 生成と AI 推論のコストが累積

🎯 ユースケース

🏦 ユースケース 1: 組織全体のドキュメント機密情報スキャン

シナリオ: 金融機関が複数のプロジェクトにまたがる Cloud Storage バケットに保存された PDF やテキストファイルに対して、PII (個人識別情報) や機密データが含まれていないかを定期的にスキャンしたい。

SQL 
SELECT
  obj.name,
  obj.bucket,
  obj.project,
  AI.GENERATE(
    'gemini-2.0-flash',
    'Detect any PII such as names, addresses, SSNs, or credit card numbers. Return JSON with detected_pii array.',
    OBJ.GET_ACCESS_URL(obj.ref, 'r', INTERVAL 1 HOUR)
  ) AS pii_scan_result
FROM
  `project.insights_dataset.objects_metadata` AS obj
WHERE
  obj.content_type IN ('application/pdf', 'text/plain')
  AND obj.size < 10485760  -- 10MB 以下のファイルに限定

効果: DLP API 連携の専用パイプラインを構築することなく、BigQuery SQL のみで組織全体の機密情報スキャンを定期的に実行でき、コンプライアンス監査の工数を大幅に削減。

🛒 ユースケース 2: 商品画像の自動説明文生成

シナリオ: EC サイトを運営する企業が Cloud Storage に保存された商品画像に対して、アクセシビリティ対応や SEO 最適化のために画像の説明文 (alt テキスト) を自動生成したい。

効果: 画像処理パイプラインを別途構築することなく、BigQuery のスケジューリング機能と組み合わせて定期的に新規画像の説明文を自動生成し、結果をテーブルに保存して下流のアプリケーションから参照可能。
