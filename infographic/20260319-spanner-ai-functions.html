📋 概要

Spanner に新たに AI 関数 が追加されました。AI.CLASSIFY、AI.IF、AI.SCORE の 3 つの関数により、SQL クエリ内で大規模言語モデル (LLM) を活用したセマンティック操作を直接実行できます。

モデル登録やプロンプト設計が不要になり、簡潔な SQL 構文で自然言語入力に対する分類、条件評価、スコアリングをデータベースレベルで実行可能です。Spanner Enterprise / Enterprise Plus エディションで利用できます。

🤖 ポイント: データをアプリケーション側に移動させることなく、SQL だけで LLM を活用したテキスト分析が可能になります。

⚡ 主要機能: 3 つの AI 関数

AI.CLASSIFY 分類

ユーザー定義のカテゴリリストに基づいて、自然言語テキストを分類する関数。顧客フィードバック分類、サポートチケットのルーティング、ドキュメント分類などに活用可能。

入力: テキスト + カテゴリ配列 出力: STRING (カテゴリ名) エラー時: NULL
AI.IF 条件評価

自然言語で記述された条件に対して、テキストが合致するかどうかを真偽値で返す関数。WHERE 句でのフィルタリングに直接使用可能。

入力: テキスト + 自然言語条件 出力: BOOL エラー時: NULL
AI.SCORE スコアリング

自然言語入力に対して数値スコアを割り当てる関数。プロンプトの自動最適化によりスコアリングルーブリックが自動生成される。

入力: テキスト + プロンプト 出力: FLOAT64 自動最適化: ルーブリック自動生成

💻 設定方法・コードサンプル

📋 前提条件: Spanner Enterprise / Enterprise Plus インスタンス、Vertex AI API の有効化、Spanner から Vertex AI エンドポイントへのアクセス権限が必要です。

ステップ 1: AI.CLASSIFY によるテキスト分類

SQL 顧客フィードバックを 6 カテゴリに自動分類 
SELECT
  feedback_id,
  feedback_text,
  AI.CLASSIFY(
    feedback_text,
    categories => ['請求に関する問題', 'アカウントアクセス',
                    '製品バグ', '機能リクエスト',
                    '配送遅延', 'その他']
  ) AS topic
FROM
  customer_feedback;

ステップ 2: AI.IF による条件フィルタリング

SQL セマンティック条件で行をフィルタリング 
SELECT
  review_id,
  review_text
FROM
  customer_reviews
WHERE
  AI.IF(('このレビューは製品のセットアップの
         困難さについて言及していますか？
         レビュー: ', review_text));

ステップ 3: AI.SCORE によるスコアリングとランク付け

SQL セマンティック類似度でランク付け 
SELECT
  review_id,
  review_text,
  AI.SCORE(
    ('製品セットアップの困難さとの類似度を
      評価してください。スコアが高いほど
      類似度が高いことを示します。
      レビュー: ', review_text)
  ) AS setup_difficulty_score
FROM
  customer_reviews
ORDER BY
  setup_difficulty_score DESC
LIMIT 10;

⚠️ デメリット・制約事項

制限事項
  • Enterprise / Enterprise Plus エディション限定 (Standard は非対応)
  • LLM 呼び出しに伴うレイテンシ発生 (大量行ではクエリ時間が長くなる可能性)
  • エラー時は NULL が返却されるため、結果の検証が必要
考慮すべき点
  • Vertex AI の利用料金が別途発生 (大規模データでの頻繁な呼び出しはコスト増)
  • LLM の応答は確率的であり、同一入力で結果が異なる場合がある
  • Vertex AI エンドポイントと Spanner を同一リージョンに配置することを推奨

🎯 ユースケース

📦 ユースケース 1: 顧客フィードバックの自動分類とトレンド分析

EC サイトの運営チームが、大量の顧客レビューを自動的にカテゴリ分類し、問題の傾向を把握したい。

SQL カテゴリ別集計クエリ 
SELECT
  AI.CLASSIFY(
    review_text,
    categories => ['請求に関する問題', 'アカウントアクセス',
                    '製品バグ', '機能リクエスト',
                    '配送遅延', 'その他']
  ) AS topic,
  COUNT(*) AS review_count
FROM customer_feedback
GROUP BY topic
ORDER BY review_count DESC;

効果: カスタム分類モデルの構築・デプロイ不要。SQL のみで顧客フィードバックのカテゴリ別集計が可能。

🛡️ ユースケース 2: コンテンツモデレーションの自動化

コミュニティプラットフォームで、投稿が利用規約に違反しているかを自動判定し、違反の度合いをスコアリングしたい。

SQL AI.IF + AI.SCORE の組み合わせ 
SELECT
  post_id, post_content,
  AI.IF(('この投稿は利用規約に違反する攻撃的な
         内容を含んでいますか？ 投稿: ',
         post_content)) AS is_violation,
  AI.SCORE(('この投稿の不適切さの度合いを
            評価してください。投稿: ',
            post_content)) AS violation_score
FROM community_posts
WHERE AI.IF(('この投稿は利用規約に違反する
             攻撃的な内容を含んでいますか？
             投稿: ', post_content))
ORDER BY violation_score DESC;

効果: AI.IF でフィルタリング + AI.SCORE で重大度を定量化。モデレーションチームが優先対応すべきコンテンツを効率的に特定。

📝 まとめ

Spanner の AI 関数 (AI.CLASSIFY, AI.IF, AI.SCORE) は、SQL クエリ内で LLM を活用したセマンティック操作を簡潔に実行できる機能です。モデル登録やプロンプト設計の複雑さを抽象化し、データエンジニアやアナリストが SQL の知識のみ でテキスト分類、条件評価、スコアリングを実行できるようになります。

💡 推奨アクション: Spanner Enterprise エディション以上を利用している場合は、既存のデータに対してこれらの AI 関数を試し、顧客フィードバック分析やコンテンツモデレーションなどのユースケースでの効果を検証することを推奨します。
