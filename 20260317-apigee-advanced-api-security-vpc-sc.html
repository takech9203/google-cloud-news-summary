📋 概要

Apigee Advanced API Security の不正利用検出 (Abuse Detection) が更新され、VPC Service Controls (VPC-SC) 環境での完全サポートが実現した。VPC-SC を有効化している組織でも、Advanced Anomaly Detection ML モデルによる不正利用検出および検出除外リストの機能が利用可能になった。

これまで VPC-SC を有効化した Apigee 組織では、2024 年 6 月導入の Vertex AI ベース ML モデル、2025 年 10 月導入の検出除外リスト、インシデントモデルの改善など主要機能が利用不可であった。今回のリリースにより、セキュリティ要件の高い環境でもこれらの機能をフルに活用できるようになった。

✗ Before

  • VPC-SC 環境で Advanced Anomaly Detection ML モデルが利用不可
  • VPC-SC 環境で検出除外リスト (Exclusion Lists) が利用不可
  • VPC-SC 環境でインシデントモデルの改善 (低ノイズ/高精度) が未適用
  • セキュリティ要件の高い組織ほど高度な検出機能を必要としていた

✓ After

  • VPC-SC 環境で ML ベースの不正利用検出が完全に利用可能
  • VPC-SC 環境で検出除外リストを作成・管理可能
  • セキュリティ境界を維持しながら高精度な異常検出を活用
  • 金融・医療・公共機関など規制業界での API セキュリティ強化

🔑 主要機能

1. VPC-SC 対応の Advanced Anomaly Detection ML モデル

-> Vertex AI ベースのカスタム機械学習モデルが VPC-SC 環境で動作
-> 組織固有の API トラフィックパターンで学習し、精度が向上
-> オプトインによるモデル学習への同意が必要 (データは他の顧客と共有されない)
-> オプトイン後、約 6 時間で異常検出が開始

2. VPC-SC 対応の検出除外リスト (Exclusion Lists)

-> CIDR レンジおよび IP アドレスを指定して安全なトラフィックを除外
-> 複数の除外リストを作成・管理可能
-> 自動テストや既知の安全なリクエストを除外しノイズを低減
-> Terraform による管理にも対応済み

3. VPC-SC 環境での完全な Abuse Detection 体験

-> インシデントの検出と詳細表示
-> 検出されたトラフィックの分析
-> Security Actions によるブロック・フラグ付与
-> Security Alerts による通知

💡 ユースケース

ユースケース 1: 金融機関の API セキュリティ強化

シナリオ: 金融機関が VPC-SC を使用してデータ漏洩防止を実現している環境で、オープンバンキング API への不正アクセスや悪用を検出したい。

効果: VPC-SC のセキュリティ境界を維持しながら、ML ベースの異常検出により通常の API 利用パターンからの逸脱をリアルタイムに検出し、Security Actions で自動的にブロックできる。

ユースケース 2: 大規模テスト環境のノイズ除去

シナリオ: CI/CD パイプラインから大量の自動テストリクエストが発生する環境で、テストトラフィックを不正利用として誤検知しないようにしたい。

効果: 検出除外リストにテスト環境の CIDR レンジを登録することで、正当な自動テストトラフィックを除外し、実際の不正利用のみをインシデントとして検出できる。
