📊 BigQuery: MCP Server GA & Conversational Analytics Preview

GA Preview BigQuery 2026-04-20

☁️ 概要

BigQuery に関する 2 つの重要なアップデートが発表されました。BigQuery MCP Server の GA (一般提供) と Conversational Analytics エージェントの Gemini Enterprise Preview です。

🔗 BigQuery MCP Server (GA)

Model Context Protocol (MCP) を通じて、Gemini CLI、Claude、ChatGPT などの主要 AI アプリケーションから BigQuery のリソース操作、SQL クエリの生成・実行、結果の解釈をセキュアに実行できます。標準化されたプロトコルによる統一的な接続が可能に。

💬 Conversational Analytics in Gemini Enterprise (Preview)

BigQuery で作成したデータエージェントを Gemini Enterprise に公開し、ビジネスユーザーが自然言語でデータに質問できる環境を提供。Agent Gallery でのブラウズ、@メンションでの呼び出し、自動ルーティングに対応。

🔄 Before / After

❌ アップデート前の課題

  • ❌ AI アプリケーションから BigQuery を利用するには各アプリ固有の API 統合や SDK 実装が必要
  • ❌ AI アプリがスキーマやメタデータを理解するには手動でコンテキストを提供する必要あり
  • ❌ データエージェントは BigQuery Studio 内でしか利用できず、Gemini Enterprise ユーザーに公開不可
  • ❌ ビジネスユーザーは BigQuery UI にアクセスしないとデータ分析不可

✅ アップデート後の改善

  • ✅ MCP 標準プロトコルで Gemini CLI、Claude、ChatGPT から統一的に BigQuery 接続可能
  • ✅ MCP Server がリソース探索、SQL 生成、クエリ実行、結果解釈をツールとして公開
  • ✅ Conversational Analytics エージェントを Gemini Enterprise に公開、Agent Gallery・@メンション・自動ルーティングで利用可能
  • ✅ IAM、OAuth 2.0、Model Armor によるエンタープライズグレードのセキュリティ

🔧 主要機能 1: BigQuery MCP Server (GA)

✅ GA (一般提供): BigQuery API を有効化するだけで MCP Server が利用可能に。グローバルエンドポイント https://bigquery.googleapis.com/mcp を提供。
→標準プロトコル対応: HTTP トランスポートで通信し、OAuth 2.0 と IAM で認証・認可
→6 つの MCP ツール: list_dataset_ids、get_dataset_info、list_table_ids、get_table_info、execute_sql_readonly、execute_sql
→Model Armor 連携: プロンプトインジェクション防御と機密データ保護に対応
→主要 AI アプリ対応: Gemini CLI、Claude、ChatGPT などから標準化された方法で接続可能

🤖 Preview: BigQuery で作成したデータエージェントを Gemini Enterprise に公開し、ビジネスユーザーが自然言語でデータ分析を実行可能に。

🔒 エンタープライズセキュリティ

🛡️IAM deny ポリシー: BigQuery MCP Server にツールアクセスの細粒度制御を適用可能。execute_sql への書き込みアクセスを deny ポリシーで制限。
🔐Model Armor Floor Settings: MCP 通信のセキュリティスキャン (プロンプトインジェクション検出、機密データ保護)。
👤Conversational Analytics IAM ロール: dataAgentUser (チャット権限) / dataAgentEditor (編集権限) / dataAgentViewer (閲覧権限) で制御。

ツール名 機能 読み取り専用 API メソッド

Step 1: BigQuery MCP Server の有効化

Bash BigQuery API と MCP Server の有効化 
# BigQuery API を有効化 (新規プロジェクトでは自動的に有効)
gcloud services enable bigquery.googleapis.com

# MCP Server を有効化
gcloud beta api-registry mcp enable bigquery.googleapis.com

Step 2: MCP クライアントの設定

JSON MCP クライアント設定ファイル 
{
  "mcpServers": {
    "bigquery": {
      "name": "BigQuery MCP server",
      "url": "https://bigquery.googleapis.com/mcp",
      "transport": "http",
      "authentication": {
        "type": "oauth2",
        "scopes": ["https://www.googleapis.com/auth/bigquery"]
      }
    }
  }
}

✅ メリット

ビジネス面

📊 データアクセスの民主化

Gemini Enterprise を通じてビジネスユーザーが SQL を書くことなく自然言語でデータ分析を実行可能。テーブル名やフィールド名を知らなくても、ビジネス用語 (「売上」「トップパフォーマー」など) で質問できます。

🚀 AI アプリケーション開発の加速

MCP 標準プロトコルにより、BigQuery との統合に必要な開発工数が大幅に削減。独自の API 統合を構築する代わりに、MCP クライアント設定のみで接続可能。

技術面

🔗 標準化されたプロトコル

MCP は Anthropic が開発したオープンソースプロトコル。BigQuery 固有の知識がなくてもツールを通じて適切な操作が可能。

🛡️ 多層的なセキュリティ

OAuth 2.0 認証、IAM による細粒度の認可、Model Armor によるプロンプトインジェクション防御と機密データ保護を組み合わせた多層防御。

📏 クォータ管理の透明性

MCP Server 固有のクォータはなく、既存の BigQuery API クォータに準拠。既存のコスト管理・クォータ設定がそのまま有効。

⚠️ デメリット・考慮事項

⚠️クエリ結果は最大 3,000 行に制限。大量データの取得には不向き。
⚠️クエリ処理時間はデフォルト 3 分で自動キャンセル。長時間実行クエリには非対応。
⚠️Google Drive 外部テーブルへのクエリは execute_sql / execute_sql_readonly の両方で非対応。
⚠️Conversational Analytics の Gemini Enterprise 公開は Preview。アローリストへの申請が必要。
⚠️Conversational Analytics API の公式サポート言語は英語のみ (日本語は非公式)。DRZ にも非対応。
💡 考慮すべき点:
  • MCP Server を通じたクエリは通常の BigQuery クエリと同様に課金 (オンデマンドの場合は処理バイト数に基づく)
  • execute_sql ツールは書き込み操作を許可するため、IAM deny ポリシーで制限を推奨
  • Model Armor の統合はオプションだが、セキュリティベストプラクティスとして有効化を推奨

💡 ユースケース

🔍 ユースケース 1: AI アプリケーションからのデータ探索と分析

シナリオ: データサイエンティストが Gemini CLI や Claude を使って BigQuery のデータを対話的に探索し、SQL クエリを生成・実行する。

Bash Gemini CLI からの BigQuery MCP Server 利用例 
# ユーザーの質問: 「sales データセットにはどんなテーブルがある?」
# → MCP ツール list_table_ids が呼ばれ、テーブル一覧を取得

# ユーザーの質問: 「先月の売上トップ10の商品を教えて」
# → MCP ツール get_table_info でスキーマ確認後、
#   execute_sql_readonly で適切な SQL を生成・実行

効果: SQL の知識がなくても自然言語でデータにアクセスでき、AI アプリケーションがスキーマを自動的に理解して適切なクエリを生成。

🏢 ユースケース 2: ビジネスユーザー向けデータエージェントの組織展開

シナリオ: データアナリストが売上分析用のデータエージェントを BigQuery で作成し、Gemini Enterprise に公開。営業部門のビジネスユーザーが自然言語で売上データに質問可能に。

効果: ビジネスユーザーが「今四半期の売上トレンドは?」「トップパフォーマーは誰?」といった質問を自然言語で行え、テキスト・チャート・テーブル形式で結果を取得。データエージェントにビジネスロジックや用語の定義を設定し、組織固有の文脈を反映した正確な回答が得られます。

💰 料金

🔗 BigQuery MCP Server

MCP Server 自体には追加料金なし。クエリは通常の BigQuery クエリと同様に課金。

  • オンデマンド: 処理バイト数に基づく従量課金
  • キャパシティ: スロット予約に基づく課金
  • 無料枠: 毎月 1 TB のクエリ処理が無料
💡 MCP 経由のクエリにはジョブラベル goog-mcp-server: true が自動設定され、利用量を追跡可能。

💬 Conversational Analytics

Preview 期間中は追加料金なし。データエージェントの作成・利用に対する追加課金なし。

⚠️ データエージェントとの会話で実行される BigQuery クエリには通常の BigQuery 料金が発生。
