📊 Compute Engine: MIG オートスケーリング モニタリングチャートの強化

Compute Engine GA
2026-03-10

☁️ 概要

Managed Instance Group (MIG) のオートスケーリング利用時に、構成されたグループサイズとオートスケーラーの推奨サイズをチャート上で並べて監視できるようになりました。これにより、オートスケーラーの判断プロセスの可視性が向上し、リアルタイムでの差異把握が可能になります。

❌ Before

✕スケールイン/アウトのタイミングと推奨サイズの変動の関連を把握しにくい

✓スケーリングポリシーの効果をリアルタイムで確認しチューニングに活用

⚡ 主要機能

📈 1. Group Size チャートの強化

→構成されたグループサイズ (実際の VM インスタンス数) と推奨サイズを同一チャート上に表示
→時系列での推移を視覚的に比較可能
→時間範囲セレクターによるカスタマイズに対応

🎯 2. オートスケーラー推奨サイズの可視化

→安定化期間 (過去 10 分間または初期化期間のいずれか長い方) のピーク負荷に基づいて算出
→スケールインコントロール設定時はその制約も反映
→オートスケーラーモード (ON / ONLY_SCALE_OUT / OFF) に関わらず推奨サイズを算出

🔗 3. 既存モニタリング機能との統合

→Autoscaler utilization、CPU utilization、Disk I/O、Network metrics チャートと同じ Monitoring タブで閲覧
→ログパネルとの連携でスケーリングイベントとサイズ変動の相関分析が可能
→チャート上のドラッグ操作でズームイン分析に対応

🔧 技術仕様

推奨サイズの算出ロジック

項目詳細
算出基準安定化期間中のピーク負荷に基づく推奨 VM 数
安定化期間過去 10 分間または初期化期間のいずれか長い方
制約条件minNumReplicas / maxNumReplicas、スケールインコントロール
予測的オートスケーリング有効時は過去の CPU 使用率パターンから将来負荷を予測
再計算頻度常時再計算

オートスケーラーモードと推奨サイズの関係

モードターゲットサイズへの反映
ONMIG のターゲットサイズを推奨サイズに設定
ONLY_SCALE_OUT推奨サイズが増加した場合のみターゲットサイズを増加
OFFターゲットサイズは変更されないが、推奨サイズは算出される

🛠️ 設定方法

📋 前提条件: Managed Instance Group (MIG) が作成されていること、オートスケーリングポリシーが構成されていること。
  1. インスタンスグループページを開く
    Google Cloud Console > Compute Engine > Instance groups から対象の MIG をクリック
  2. Monitoring タブを選択
    MIG 詳細ページで Monitoring タブを選択すると、Group Size チャートに構成サイズと推奨サイズが表示される
  3. チャートの分析
    時間範囲セレクターで分析期間を調整、ドラッグでズームイン、ログパネルでスケーリングイベントの詳細を確認

⚠️ デメリット・制約事項

制限事項
✕オートスケーリングは Unmanaged Instance Group、Stateful MIG、VM 修復が無効な MIG では使用不可
✕GKE 所有の MIG では Compute Engine オートスケーリングではなく Cluster Autoscaler を使用する必要あり
✕予測的オートスケーリングのチャートは CPU 使用率ベースのスケーリングメトリクスでのみ利用可能
💡 考慮すべき点
→チャートは Google Cloud コンソールでのみ閲覧可能 (API やコマンドラインでの直接取得ではない)
→詳細な分析には Cloud Monitoring のカスタムダッシュボード併用が推奨

🎯 ユースケース

🛒 ユースケース 1: スケーリングポリシーの最適化

シナリオ: EC サイトのバックエンドで MIG オートスケーリングを使用しているが、ピーク時にレスポンスが遅延する。推奨サイズが実際のサイズを常に上回っている場合、maxNumReplicas の引き上げやターゲット使用率の引き下げを検討。

効果: チャートで推奨サイズの傾向を視覚的に確認し、適切なスケーリングパラメータを迅速に特定できる。

⏱️ ユースケース 2: スケールインコントロールの検証

シナリオ: 初期化に時間がかかるアプリケーションで、スケールインコントロールを設定した後、推奨サイズの変動が適切に制約されているかを確認。

効果: 推奨サイズの変動と実際のスケールイン速度の関係をチャートで確認し、trailing time window や maximum allowed reduction の設定が適切かを判断できる。

💰 料金

オートスケーリングポリシーの構成自体に追加料金は発生しません。オートスケーラーが動的に VM インスタンスを追加・削除するため、MIG が使用するリソースに対してのみ課金されます。モニタリングチャートの利用にも追加料金は発生しません。

💡 コスト制御: minNumReplicas と maxNumReplicas の設定でコストを制御可能。

🌍 利用可能リージョン

Compute Engine MIG オートスケーリングは、すべての Compute Engine 利用可能リージョンで使用可能です。ゾーン MIG およびリージョナル MIG の両方で利用できます。
