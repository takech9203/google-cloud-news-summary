📋 概要

Cloud SQL for SQL Server に SQL Server 2025 のサポートと Microsoft Entra ID 統合の GA という 2 つの重要なアップデートが発表されました。最新のデータベースエンジンをマネージド環境で利用でき、既存の Microsoft ID 基盤をそのままデータベース認証に活用できるようになります。

🔄 Before / After

❌ Before

  • 最新バージョンは SQL Server 2022 まで -- 2025 の新機能を活用できなかった
  • Entra ID 統合はプレビュー段階で SLA 保証なし
  • SQL Server 固有のログインとパスワードを個別管理 -- ID 基盤との統合が限定的

✅ After

  • SQL Server 2025 (Standard / Enterprise / Express) が GA で本番利用可能
  • Entra ID 統合が GA -- SLA 保証付きで一元認証を実現
  • MFA / 条件付きアクセス (CA) を DB レベルで適用可能

⭐ 主要機能

🖥️ SQL Server 2025 データベースバージョンのサポート

→SQL Server 2025 Standard: 標準的なワークロード向けエディション
→SQL Server 2025 Enterprise: 高可用性やパフォーマンスを重視するエンタープライズ向け
→SQL Server 2025 Express: 開発・テスト用途や小規模ワークロード向け軽量版
→Cloud SQL Enterprise Plus / Enterprise の両方で利用可能

🔐 Microsoft Entra ID 統合 (GA)

✓一元的な認証: 既存の Entra ID の ID 情報で Cloud SQL にサインイン
✓セキュリティ強化: MFA や条件付きアクセス (CA) ルールを DB レベルで適用
✓ユーザー管理の簡素化: Entra ID アカウント無効化/削除時に DB アクセスが自動取り消し

📝 T-SQL による Entra ID ログイン管理

→CREATE LOGIN [...] FROM EXTERNAL PROVIDER 構文で Entra ID ユーザーのログイン作成
→SSMS などの既存ツールとの互換性を確保
→ALTER ANY LOGIN 権限で管理者が他の Entra ID ログインを管理可能

🛠️ 設定方法

前提条件

1.Google Cloud プロジェクトの作成と Cloud SQL Admin API の有効化
2.Microsoft Azure ポータルで Entra ID テナント ID の確認
3.Azure ポータルでアプリケーション登録の作成と API 権限の付与

Step 1: SQL Server 2025 インスタンスの作成

Bash 
gcloud sql instances create my-sql2025-instance \
  --database-version=SQLSERVER_2025_STANDARD \
  --tier=db-custom-2-3840 \
  --region=us-central1 \
  --assign-ip \
  --root-password=YOUR_SECURE_PASSWORD
💡 ヒント: --database-version には SQLSERVER_2025_STANDARD、SQLSERVER_2025_ENTERPRISE、SQLSERVER_2025_EXPRESS のいずれかを指定します。

Step 2: Microsoft Entra ID 認証の有効化

Bash 
gcloud sql instances patch my-sql2025-instance \
  --entra-id-tenant-id=YOUR_TENANT_ID \
  --entra-id-application-id=YOUR_APPLICATION_ID
⚠️ 注意: この操作によりインスタンスが再起動されます。

Step 3: Entra ID ログインの作成

SQL 
-- 初回ログインの作成 (sqlcmd または SSMS で実行)
CREATE LOGIN [entra_user@yourdomain.com] FROM EXTERNAL PROVIDER

-- 追加ログインを管理するための権限付与
GRANT ALTER ANY LOGIN TO [entra_user@yourdomain.com] AS CustomerDbRootRole

✓マネージドサービスの利点: パッチ適用やバージョンアップが自動実施
✓柔軟な構成: Enterprise Plus / Enterprise 両エディションで利用可能

⚠️ 制限事項・注意点

❌ 制限事項
  • Entra ID 認証は SQL Server 2022 以降のみ対応 (SQL Server 2017 / 2019 は非対応)
  • Entra ID 設定変更 (有効化/変更/無効化) 時にインスタンス再起動が発生
  • プライベート IP のみのインスタンスでは Entra ID エンドポイント (login.microsoftonline.com、graph.microsoft.com) への外部接続が必要 -- Cloud NAT 等のネットワーク構成が必要
📝 考慮すべき点
  • インスタンスごとに固有の Entra ID アプリケーション登録を作成することを推奨 (セキュリティ分離のため)
  • 各エディションのマシンタイプ制限や利用可能リージョンは公式ドキュメントで確認が必要
  • SQL Server 2022 から 2025 へのメジャーバージョンアップグレード手順は別途確認が必要

💡 ユースケース

🏢 ユースケース 1: エンタープライズ環境でのハイブリッド ID 管理

シナリオ: 大規模企業が既に Microsoft Entra ID を全社的な ID 基盤として利用し、Google Cloud 上の SQL Server データベースにも同じ ID 基盤でアクセスを管理したい。

Bash 
# Entra ID 対応の SQL Server 2025 Enterprise インスタンスを作成
gcloud sql instances create production-db \
  --database-version=SQLSERVER_2025_ENTERPRISE \
  --tier=db-custom-8-32768 \
  --region=us-central1 \
  --network=projects/my-project/global/networks/my-vpc \
  --root-password=SECURE_PASSWORD \
  --entra-id-tenant-id=7e281aab-e994-4c83-88ed-d1674477a39c \
  --entra-id-application-id=4c5ed2da-0478-4aaa-ab65-6dfd33ba8bfd
✅ 効果: 社員の入退社や異動時に Entra ID 側の変更がデータベースアクセスに自動反映。管理工数の削減とセキュリティリスクの低減を実現。

🛠️ ユースケース 2: 最新 SQL Server 機能を活用した開発環境

シナリオ: 開発チームが SQL Server 2025 の新機能を検証するためのマネージド開発環境を迅速に構築したい。

✅ 効果: Cloud SQL のマネージドサービスを活用し、インフラ構築の手間なく SQL Server 2025 の新機能を即座に評価・検証可能。Express エディションでコストを抑えた開発環境を構築。

🎯 まとめ

Cloud SQL for SQL Server は最新の SQL Server 2025 エンジンをサポートし、Microsoft Entra ID 統合 も GA となりました。エンタープライズ環境における最新のデータベース機能の活用と、既存の ID 基盤を活用したセキュアなアクセス管理が Google Cloud 上で実現できます。

SQL Server ワークロードを Google Cloud で運用している組織は、新規インスタンスでの SQL Server 2025 の採用と、既存インスタンスでの Entra ID 認証の有効化を検討することを推奨します。
