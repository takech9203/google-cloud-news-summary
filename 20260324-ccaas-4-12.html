  • コールバック履行時間の設定でロールオーバー/キャンセルを制御
  • CX Agent Studio で音声+チャット両チャネルの仮想エージェント管理

🚀 主要機能

📊 1. 予測キャンペーンの制御強化

→Max Calls Per Agent: エージェントあたりの最大同時通話数を設定。オーバーダイヤルによる通話放棄リスクを軽減
→Target Agent Occupancy: 目標エージェント稼働率を設定。ダイヤルレートをより自然かつ一貫して調整
→Max Abandonment % のオプション化: 最大放棄率の設定が任意になり、放棄率上限が不要なキャンペーンに対応
設定パス: Campaigns > Add Campaign > Mode > Predictive

💬 2. チャットセッション再開エンドポイント (Resume Chat)

→新しい chats/CHAT_ID/resume エンドポイントで dismissed / va_dismissed ステータスのチャットを再開
→再開時にエンドユーザーとエージェントの両方にチャット履歴が表示される
→Web SDK の resumeChat(chatId) メソッドからも利用可能

🕐 3. コールバック履行時間の設定

→コンタクトセンターがコールバックを履行する時間帯を設定可能
→営業時間外のコールバックを翌日にロールオーバーまたはキャンセル
⚠️ 注意: デフォルトでは無効。利用するには Google 担当者にインスタンスでの有効化を依頼する必要があります。

🤖 4. CX Agent Studio によるチャット仮想エージェント作成

→CX Agent Studio でチャット仮想エージェントの作成をサポート
→既存の音声仮想エージェント対応に加え、チャットチャネルにも拡張
→音声とチャットの両チャネルを同一プラットフォームで統一管理

🔍 5. エージェント内線番号検索の改善

→複数エージェントがマッチした場合、8 件ずつグループで読み上げ
→新メッセージ追加: Multiple agents found / Search results next page / End of search results

🔗 6. HubSpot 連携の拡充

→Company プロファイルルックアップ: Contacts と Companies の両方でエンドユーザーを検索可能
→携帯電話番号検索: Phone number と Mobile phone number の両フィールドを自動検索

🔧 技術仕様

新規 API エンドポイント

項目 詳細
エンドポイント POST chats/{CHAT_ID}/resume
対象ステータス dismissed, va_dismissed
レスポンス 再開されたチャットセッション (履歴付き)
SDK メソッド client.resumeChat(chatId)

予測キャンペーン新規制御パラメータ

パラメータ 説明
Max Calls Per Agent エージェントあたりの最大同時通話数
Target Agent Occupancy 目標エージェント稼働率 (%)
Max Abandonment % 最大放棄率 (オプション化)

HubSpot 連携設定

機能 設定パス
Company プロファイルルックアップ CRM 設定 > Primary/Secondary Lookup Objects
携帯電話番号検索 Settings > Developer Settings > CRM > Phone Number Lookup

💻 実装例

チャットセッション再開 (Web SDK)

JavaScript Web SDK によるチャット再開 
// チャットセッションの再開
const chatId = "CHAT_ID_HERE";

// Web SDK メソッドを使用
const resumedSession = await client.resumeChat(chatId);

// 再開されたセッションにはチャット履歴が含まれる
console.log("Chat resumed:", resumedSession);
console.log("History:", resumedSession.messages);

REST API によるチャット再開

Bash cURL によるチャット再開リクエスト 
# チャットセッションの再開
curl -X POST \
  "https://ccaas.googleapis.com/v1/chats/{CHAT_ID}/resume" \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json"

🎯 ユースケース

📞 ユースケース 1: アウトバウンドセールスキャンペーンの最適化

シナリオ: 大規模なアウトバウンドセールスチームが予測キャンペーンを運用しているが、オーバーダイヤルによる通話放棄率が高く、規制コンプライアンスに懸念がある。
✅ 効果: Max Calls Per Agent と Target Agent Occupancy を設定することで、ダイヤルレートをエージェントの処理能力に合わせて自然に調整でき、通話放棄率を抑えつつキャンペーン効率を維持できる。

💬 ユースケース 2: カスタマーサポートのチャット継続性向上

シナリオ: EC サイトのカスタマーサポートで、エンドユーザーがチャット中に一時的に離席し、セッションがタイムアウトで dismissed になるケースが頻発している。
✅ 効果: Resume chat エンドポイントを活用することで、エンドユーザーが戻った際にチャット履歴付きでセッションを再開でき、同じ問題を最初から説明し直す必要がなくなる。
