📋 概要

Gemini Code Assist のファイルアウトライン (File Outline) と変更完了 (Finish Changes) の 2 つの機能が、VS Code で一般提供 (GA) となりました。2025 年 12 月に IntelliJ 向けにプレビューとして初めてリリースされ、2026 年 2 月に IntelliJ で GA となっていましたが、今回のアップデートにより VS Code ユーザーもこれらの AI 支援機能を本番環境で利用できるようになります。

ファイルアウトラインは、コードファイルの構造的な概要を AI が自動生成し、コードブロックごとの短い要約を提供します。変更完了は、AI ペアプログラマーとして作業中のコードを観察し、擬似コード、#TODO コメント、書きかけのコードを検出して自動的に補完します。

🤖 対応エディション: Gemini Code Assist Standard / Enterprise / For Individuals のいずれのエディションでも利用可能。追加料金なし。

⚡ 主要機能

📄 ファイルアウトライン (File Outline)

→フォーカス中のファイルに対して AI がコードブロックごとの短い英語の要約を自動生成
→Gemini Code Assist チャットペインの「Outline」タブで確認可能
→Eye アイコンでインラインコード内に直接表示可能
→手動で「Refresh outline」を実行して更新
→設定から自動アウトライン生成の有効/無効を切替可能

🔧 変更完了 (Finish Changes)

→擬似コード、#TODO コメント、書きかけコードを AI が検出し補完候補を生成
→右クリック「Gemini > Finish changes」またはキーボードショートカットで起動
→各コード補完候補に個別に「Accept」/「Reject」を選択可能
→「Accept all」/「Reject all」による一括操作にも対応
→複雑なプロンプト不要でコーディングフローを維持

⚠️ デメリット・制約事項

⚠️ 制限事項
  • アウトラインはセッション間で永続化されず、新しい IDE セッション開始時に再生成が必要
  • ファイル変更後、アウトラインは自動更新されず手動で「Refresh outline」を実行する必要あり
  • アウトラインの要約は現時点では英語のみで生成される
  • 変更完了機能は現在のファイル内のみで動作し、複数ファイルにまたがる変更の自動補完には非対応
💡 考慮すべき点
  • AI による補完結果は常に正確とは限らないため、生成コードのレビューと検証を推奨
  • 大規模ファイルではアウトライン生成や変更完了の処理に時間がかかる場合がある
  • ファイアウォール環境では cloudaicompanion.googleapis.com への通信許可が必要
