📄 概要

Cloud Composer の 2026 年 3 月リリースでは、Cloud Composer 3 および Cloud Composer 2 の両方で新しいビルドとイメージが提供されました。最も注目すべき点は、Airflow 3.1.7 に対応した新ビルド composer-3-airflow-3.1.7-build.1 が追加され、Airflow ワーカーがデータベースに直接アクセスできなくなるという重大な Breaking Change が含まれていることです。

また、apache-airflow-providers-cncf-kubernetes が v10.13.0 に、apache-airflow-providers-google が v20.0.0 にアップグレードされ、複数の旧ビルドおよびイメージが非推奨化されました。

⚠️ Breaking Change: composer-3-airflow-3.1.7-build.1 以降、Airflow ワーカーからメタデータデータベースへの直接アクセスが遮断されます。DAG 内でデータベースへの直接クエリを実行しているユーザーは、アップグレード前にコードの修正が必要です。

🔄 Before / After 比較

❌ Before (Airflow 2.x)

  • ワーカーが Airflow メタデータ DB に直接アクセス可能
  • DAG コードから直接 SQL クエリを実行する慣行が存在
  • 意図しないデータ漏洩やデータベース負荷のリスク
  • 旧バージョンのプロバイダーパッケージによる互換性制約

✅ After (Airflow 3.1.7)

  • ワーカーからのデータベース直接アクセスが遮断
  • ワーカーは Airflow API 経由でメタデータにアクセス
  • 関心の分離が徹底され、セキュリティが強化
  • 最新の Google Cloud サービスとの統合が改善

📑 アーキテクチャ図

👍 メリット

🔒 セキュリティ面

✅データベースアクセスの制限: Airflow 3.1.7 でワーカーからのデータベース直接アクセスが遮断され、DAG コードによる意図しないデータベース操作のリスクが低減
✅関心の分離: ワーカーが API 経由でのみメタデータにアクセスする設計により、各コンポーネントの責任範囲が明確化

⚡ 技術面

✅最新プロバイダーパッケージ: apache-airflow-providers-google v20.0.0 と apache-airflow-providers-cncf-kubernetes v10.13.0 により、最新の Google Cloud サービスおよび Kubernetes との統合が改善
✅継続的なバージョンサポート: 新しいビルドとイメージの定期リリースにより、セキュリティパッチや機能改善が継続的に提供

⚠️ デメリット / 制約事項

🚫 制限事項:
  • Airflow 3.1.7 の Breaking Change により、DAG 内でデータベースに直接クエリを実行しているコードは動作しなくなる
  • apache-airflow-providers-google v20.0.0 はメジャーバージョンアップであり、非推奨化されたオペレーターが削除されている可能性がある
  • 非推奨化されたビルド / イメージを使用している環境は、サポート終了前にアップグレードが必要
💡 考慮すべき点:
  • Airflow 3.1.7 へのアップグレード前に、全ての DAG コードを精査し、データベースへの直接アクセスパターンを特定 / 修正する必要がある
  • プロバイダーパッケージのメジャーアップグレードに伴い、使用しているオペレーターの互換性を確認する必要がある
  • Cloud Composer 2 の composer-2.11.4-* および composer-2.11.5-* はサポート終了となるため、早期のアップグレード計画が推奨される

🎯 ユースケース

🔍 ユースケース 1: Airflow 3.1.7 へのアップグレード前のコード監査

シナリオ: 本番環境で Cloud Composer 3 (Airflow 2.x) を運用しており、Airflow 3.1.7 へのアップグレードを計画しているチーム。

効果: Breaking Change による本番障害を未然に防止し、Airflow 3.x のセキュリティモデルに準拠したコードベースを維持できる。

Python NG: Airflow 3.1.7 以降では動作しないパターン 
# NG: ワーカーからデータベースへの直接クエリ
from airflow.models import DagRun
from airflow.utils.session import provide_session

@provide_session
def query_dag_runs(session=None):
    runs = session.query(DagRun).filter(...).all()
    return runs
Python OK: Airflow API を使用するパターン 
# OK: Airflow API 経由でメタデータにアクセス
import requests

def query_dag_runs_via_api():
    response = requests.get(
        "http://airflow-webserver:8080/api/v1/dags/my_dag/dagRuns",
        headers={"Authorization": "Bearer <token>"}
    )
    return response.json()

📋 まとめ

今回の Cloud Composer 2026 年 3 月リリースで最も重要な変更は、Airflow 3.1.7 におけるワーカーからのデータベース直接アクセス遮断です。この Breaking Change は、Airflow 3.0 以降のセキュリティ改善の一環であり、全てのユーザーは Airflow 3.1.7 へのアップグレード前に DAG コードを精査し、データベースへの直接クエリパターンを API 経由のアクセスに移行する必要があります。
