🗄️ AlloyDB for PostgreSQL: リモート MCP サーバー GA & Database Insights MCP サーバー GA

AlloyDB for PostgreSQL GA MCP
📅 2026-04-20

📋 概要

AlloyDB for PostgreSQL のリモート MCP (Model Context Protocol) サーバーに関する 2 つの GA リリースが発表されました。 Database Insights リモート MCP サーバーが GA となり、AI アプリケーションから AlloyDB のパフォーマンスおよびシステムメトリクスを PromQL クエリで分析できるようになりました。 また、AlloyDB リモート MCP サーバーのグローバルエンドポイントが GA に昇格し、読み取り専用 SQL 実行、バックアップ、データエクスポート/インポートなど幅広い運用操作が追加されました。

これにより、Gemini CLI、Claude、ChatGPT などの AI アプリケーションから AlloyDB クラスタの管理・監視を自然言語で行える基盤が本番環境向けに整備されました。

🔄 Before / After 比較

❌ アップデート前

  • MCP サーバーはプレビュー段階、本番ワークロードに SLA なし
  • AI アプリからパフォーマンスメトリクスを直接分析する MCP 手段がなかった
  • 読み取り専用 SQL、エクスポート/インポート、バックアップ/リストアが MCP 経由で利用不可
  • Public IP / PSC エンドポイントの自動化が MCP 未サポート

✅ アップデート後

  • グローバルエンドポイントが GA、SLA 付きで本番環境サポート
  • Database Insights MCP で PromQL ベースのパフォーマンス分析が可能
  • execute_sql_read_only / create_backup / restore_cluster が MCP 経由で利用可能
  • PSC エンドポイント自動作成 / Public IP 接続を MCP ツールからサポート

📊 主要機能 1: Database Insights リモート MCP サーバー (GA)

AI アプリケーションから AlloyDB のパフォーマンスおよびシステムメトリクスを PromQL クエリで分析可能に。

🔍 get_query_metrics

クエリ関連のテレメトリデータを PromQL クエリで取得。クエリレイテンシ、データベース負荷などを分析。

📈 get_system_metrics

システム関連のテレメトリデータを PromQL クエリで取得。CPU 使用率、メモリ、接続数などを分析。

💡 分析可能なメトリクス: CPU 使用率、メモリ、クエリレイテンシ、データベース負荷、接続数など

⚙️ 主要機能 2: AlloyDB リモート MCP サーバー グローバルエンドポイント (GA)

⚠️ デメリット・制約事項

制限事項:
  • create_user はパスワードベースのビルトイン認証ユーザーの作成をサポートしない (IAM 認証のみ)
  • execute_sql のレスポンスが 10 MB を超える場合、レスポンスが切り捨てられる可能性あり
  • execute_sql_read_only は PostgreSQL バージョン 17 以降でのみサポート
  • alloydb.instances.executeSqlReadOnly 権限は Google Cloud コンソールに表示されない
  • リージョナルエンドポイントはまだ Preview (GA はグローバルのみ)
  • ChatGPT ユーザーは AlloyDB MCP サーバーのツールセットを使用できない (既知の問題)
⚠️ 考慮すべき点:
  • MCP サーバーは API 有効化と同時に利用可能になるため、IAM ポリシーの適切な設定が重要
  • エージェント用の専用 ID を作成し、リソースへのアクセスを制御・監視することを推奨
  • execute_sql は DDL/DML を含むすべての SQL を実行できるため、本番環境では execute_sql_read_only の使用を検討
