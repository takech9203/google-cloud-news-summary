📋 概要

Google Distributed Cloud (software only) の バージョン 1.33.600-gke.39 が VMware 版およびベアメタル版の両方でリリースされました。本リリースは Kubernetes v1.33.5-gke.2200 上で動作し、複数の重要なバグ修正とセキュリティ脆弱性の修正を含んでいます。

オンプレミスや分散環境で Google Distributed Cloud を運用しているインフラチームや SRE チームにとって、クラスタのアップグレードと安定運用に直結する重要なパッチリリースです。

🔄 Before / After 比較

❌ アップデート前の課題

  • node-problem-detector が非 Advanced (V1) クラスタに誤デプロイされ、containerd が連続再起動
  • stackdriver.enableVPC が Advanced Cluster へのアップグレードをブロック
  • gkectl upgrade admin リトライ時に AlreadyExists エラー
  • プロキシ設定の余分な空白でクラスタ作成/アップグレード失敗
  • レジストリミラーのカスタム CA でシステム証明書プールが無視
  • ベアメタル版で CA ローテーションがスタック

✅ アップデート後の改善

  • node-problem-detector が Advanced Cluster のみに正しくデプロイ
  • stackdriver.enableVPC に関わらずアップグレード正常実行
  • gkectl upgrade admin のリトライが正常動作
  • プロキシ設定の空白が自動トリミング
  • カスタム CA とシステム証明書プールの両方を正しく参照
  • CA ローテーションが正常に完了

🔧 主要な修正内容

1. node-problem-detector のデプロイ修正 (VMware)

→非 Advanced (V1) VMware クラスタに node-problem-detector が誤ってデプロイされる問題を修正
→誤デプロイにより containerd ランタイムが連続再起動し、ETCD/CRI 障害が発生していた
→クラスタのアップグレード失敗も解消

2. Advanced Cluster アップグレードのブロック解除 (VMware)

→非推奨の stackdriver.enableVPC フィールドが true の場合、バリデーションエラーでアップグレードがブロックされていた
→非推奨フィールドの設定値に関わらずアップグレードが正常に実行可能に

3. gkectl upgrade admin リトライの安定化 (VMware)

→アップグレード失敗後の再試行で AlreadyExists エラーが発生していた
→リソースの存在チェックと冪等性が改善され、再試行が安全に実行可能に

4. プロキシ設定の空白文字処理 (VMware)

→proxy / noProxy 設定に余分な空白があるとクラスタ作成・アップグレードが失敗
→空白文字のトリミング処理が追加され、設定の堅牢性が向上

5. レジストリミラーの証明書処理修正 (VMware)

→カスタム CA 証明書設定時にシステム証明書プールが無視される問題を修正
→カスタム CA とシステム証明書プールの両方が正しくマージされるように改善

6. CA ローテーションの修正 (ベアメタル)

→セルフマネージドクラスタで CA ローテーションがスタックする問題を修正
→証明書管理の信頼性が向上

7. セキュリティ脆弱性の修正 (共通)

→VMware 版・ベアメタル版の両方で既知のセキュリティ脆弱性が修正

💡 ユースケース

🖥️ ユースケース 1: 非 Advanced クラスタの安定化

シナリオ: VMware 上で非 Advanced (V1) クラスタを運用しており、containerd の頻繁な再起動や ETCD 障害に悩まされている環境。

効果: 本バージョンにアップグレードすることで、node-problem-detector の誤デプロイが解消され、containerd の連続再起動が停止。ETCD/CRI 障害も解消され、クラスタの安定稼働が回復する。

🚀 ユースケース 2: Advanced Cluster への移行パスの確保

シナリオ: 過去の設定ファイルで stackdriver.enableVPC: true が設定されたままの VMware クラスタを運用しており、Advanced Cluster へのアップグレードがブロックされている。

効果: 非推奨フィールドによるブロックが解除され、Advanced Cluster への移行が可能に。GDC 1.33 以降ではすべてのクラスタが Advanced Cluster に自動変換されるため、この修正は移行パスの確保として重要。

🔐 ユースケース 3: ベアメタルクラスタの証明書ローテーション

シナリオ: ベアメタル上でセルフマネージドクラスタを運用しており、定期的な CA ローテーションを実施しているが、プロセスがスタックして完了しない。

効果: CA ローテーションが正常に完了するようになり、証明書の定期更新が確実に行える環境が整う。

🔗 関連サービス・機能

☸️Google Kubernetes Engine (GKE) - GDC は GKE のオンプレミス拡張であり、同一の Kubernetes バージョン (v1.33.5-gke.2200) を基盤としている
🔍Node Problem Detector - Kubernetes のオープンソースコンポーネントで、ノードの健全性を監視。本リリースでは VMware 版でのデプロイ対象が修正された
🌐GKE On-Prem API - Google Cloud コンソール、gcloud CLI、Terraform を通じたクラスタライフサイクル管理を提供。本リリースは 7-14 日後に API クライアントでも利用可能
📊Stackdriver (Cloud Operations) - 非推奨の stackdriver.enableVPC フィールドに関連。現在は DNS ベースの Private Google Access 設定が推奨
