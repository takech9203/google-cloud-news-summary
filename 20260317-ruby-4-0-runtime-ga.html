📋 概要

Google Cloud は 2026 年 3 月 17 日、Ruby 4.0 ランタイムのサポートが App Engine flexible environment、App Engine standard environment、Cloud Run、Cloud Run functions の 4 つのサーバーレスサービスにおいて一般提供 (GA) となったことを発表しました。

Ruby 4.0 は Ruby 言語のメジャーバージョンアップであり、パフォーマンス改善や新機能が多数含まれています。GA により、SLA 付きで本番ワークロードの実行が正式にサポートされます。

🔄 Before / After

❌ Before (アップデート前)

  • Ruby 4.0 ランタイムは Preview 段階で SLA なし
  • Ruby 3.4 以前では最新の新機能やパフォーマンス改善を活用不可
  • Preview 段階は予告なしの破壊的変更リスクがあり本番利用が困難

✅ After (アップデート後)

  • 4 つのサーバーレスサービスすべてで GA、SLA 付き本番サポート
  • Ruby 4.0 の最新機能とパフォーマンス改善を安心して活用可能
  • 全サービスで Ruby 4.0 を統一的に利用可能

⚡ 主要機能

🟦 App Engine Standard Environment

  • app.yaml に runtime: ruby40 を指定するだけでデプロイ可能
  • 自動スケーリング (ゼロインスタンスへのスケールダウン対応)
  • インスタンス起動時間は数秒と高速
  • 自動セキュリティパッチ適用

🟩 App Engine Flexible Environment

  • Gemfile に ruby "4.0.x"、runtime_config で ubuntu24 を設定
  • Docker コンテナベースでカスタム C ライブラリ拡張に対応
  • SSH デバッグ・バックグラウンドプロセス実行が可能

🚀 Cloud Run

  • ランタイム ID ruby40 で google-24-full スタック上で提供
  • ソースデプロイ (--base-image ruby40) で簡易デプロイ
  • 自動ベースイメージ更新によるセキュリティ維持

⚙️ Cloud Run Functions

  • ランタイム ID ruby40 で Functions Framework を使用
  • HTTP 関数および CloudEvent 関数に対応
  • gcloud run deploy で --base-image ruby40 を指定

🛠️ 設定方法

ステップ 1: App Engine Standard へのデプロイ

YAML app.yaml 
# app.yaml
runtime: ruby40
entrypoint: bundle exec rails server -p $PORT
Bash デプロイコマンド 
gcloud app deploy

ステップ 2: Cloud Run へのソースデプロイ

Bash Cloud Run デプロイ 
gcloud run deploy my-ruby-service \
  --source . \
  --base-image ruby40 \
  --region asia-northeast1

ステップ 3: Cloud Run Functions へのデプロイ

Bash Cloud Run functions デプロイ 
gcloud run deploy my-ruby-function \
  --source . \
  --function my_function_entrypoint \
  --base-image ruby40 \
  --region asia-northeast1

⚠️ デメリット・制約事項

制限事項
  • Ruby 4.0 ランタイムは google-24-full スタック上でのみ提供 (旧スタック google-22、google-18-full では利用不可)
  • App Engine standard ではファイルシステム書き込みが /tmp ディレクトリに制限
  • Cloud Run functions のイベント駆動関数は CloudEvent 関数のみサポート
考慮すべき点
  • Ruby 3.x から 4.0 へのメジャーバージョンアップで非互換性が発生する可能性 -- 移行前に十分なテストが必要
  • Ruby 4.0 で非推奨・削除された機能がある場合、Gem の互換性確認が必要
  • App Engine flexible は最小 1 インスタンス常時起動のため、低トラフィック時のコストに注意

💡 ユースケース

🚂 ユースケース 1: Rails アプリケーションの App Engine デプロイ

シナリオ: Ruby on Rails で構築した Web アプリケーションを、Ruby 4.0 の最新機能を活用しつつ App Engine standard environment で運用したい場合。

YAML app.yaml (Rails) 
# app.yaml
runtime: ruby40
entrypoint: bundle exec rails server Puma -p $PORT

env_variables:
  RAILS_ENV: production
  SECRET_KEY_BASE: your-secret-key

効果: App Engine standard のゼロスケーリングでコスト最小化。Ruby 4.0 のパフォーマンス改善でリクエスト処理高速化。

⚡ ユースケース 2: Cloud Run Functions によるイベント駆動処理

シナリオ: Cloud Storage へのファイルアップロードをトリガーに、Ruby 4.0 で画像処理やデータ変換を実行したい場合。

Ruby app.rb (CloudEvent 関数) 
# app.rb
require "functions_framework"

FunctionsFramework.cloud_event "process_file" do |event|
  file_name = event.data["name"]
  bucket = event.data["bucket"]
  # ファイル処理ロジック
  logger.info "Processing #{file_name} from #{bucket}"
end
Bash デプロイコマンド 
gcloud run deploy process-file \
  --source . \
  --function process_file \
  --base-image ruby40 \
  --region asia-northeast1

効果: ゼロスケーリングでイベントがないときはコスト不要。Ruby 4.0 の処理性能向上で実行時間短縮とコスト削減。
