📋 概要

Gemini Enterprise に Salesforce データフェデレーション接続 (Public Preview) と フェデレーテッドコネクタエラーログの可視化 機能が追加されました。Salesforce に格納された顧客データ、商談情報、ケース履歴などを ETL パイプラインなしで Gemini Enterprise のデータストアに取り込み、AI を活用した検索や回答生成に活用できます。

同時に、コネクタのエラーログを Cloud Logging の Logs Explorer で確認できるようになり、接続問題やデータ変換エラーの迅速な診断が可能になりました。

🔄 Before / After

❌ Before

  • Salesforce データの活用には手動エクスポートやカスタム ETL パイプラインが必要
  • コネクタエラーの詳細な原因特定情報が限定的
  • 接続障害やデータ同期問題の迅速な診断手段が不足

✅ After

  • データフェデレーションで Salesforce を直接接続、ETL パイプライン不要
  • Logs Explorer でエラーログを詳細に確認、原因を迅速に特定
  • Salesforce V2 コネクタで安定した接続とパフォーマンス向上

🚀 主要機能

🔗 Salesforce データフェデレーション

Salesforce V2 コネクタで直接接続。Accounts、Contacts、Cases などの標準オブジェクトに加え、custom_object_name__c 形式のカスタムオブジェクトにも対応。フルシンクとインクリメンタルシンクの両方をサポート。

Public Preview

🔐 柔軟な認証方式

ユーザー名/パスワード + セキュリティトークン認証と、OAuth 2.0 JWT Bearer フロー (RSA 2048 ビット鍵ペア) の 2 つの認証方式をサポート。OAuth 認証では Connected App の設定が必要。

📊 エラーログ可視化

コネクタの接続問題、データ変換エラー、API エラーの詳細ログを Cloud Logging の Logs Explorer で確認可能。JSON メタデータによる高度なクエリフィルタリングにも対応。

🌐 接続タイプの選択

Public (パブリック) と Private (プライベート / Service Attachment 経由) の 2 つの接続先タイプを選択可能。セキュリティ要件に応じた柔軟な構成が可能。

🛠️ 設定方法

💡 前提条件:
  • Google Cloud プロジェクトで Gemini Enterprise API が有効化されていること
  • Salesforce 側で適切な認証設定とユーザー権限が構成されていること
  • エラーログの確認には roles/logging.viewer IAM ロールが必要

Step 1: Salesforce 側の認証設定 (OAuth 2.0 の場合)

Bash 
# RSA 秘密鍵の生成
openssl genrsa -out server.key 2048

# 自己署名証明書の生成
openssl req -new -x509 -sha256 -days 3650 -key server.key -out server.crt

生成した server.crt を Salesforce の Connected App 設定時にアップロード。Callback URL には https://vertexaisearch.cloud.google.com/console/oauth/salesforce_oauth.html を設定。

⚖️ メリット・デメリット

✅ メリット

ビジネス面

  • CRM データの即時活用 -- 営業・サポート部門の情報アクセス向上
  • ETL パイプラインの構築・保守が不要に -- 運用コスト削減
  • エラーログ可視化によるダウンタイム最小化

技術面

  • マネージドコネクタでインフラ保守負荷軽減
  • インクリメンタルシンクで効率的なデータ更新
  • Cloud Logging 統合 + BigQuery へのログルーティング

⚠️ デメリット・制約

制限事項

  • Public Preview 段階 -- SLA 対象外の可能性
  • 旧コネクタから V2 への移行作業 (データストア再作成) が必要
  • Private 接続タイプでは Service Attachment の追加設定が必要

考慮すべき点

  • Salesforce 側のオブジェクトが Private の場合、権限セットの作成が必要
  • OAuth 認証には Connected App と証明書管理の運用が追加
  • データ量によりインジェストに数分~数時間かかる場合あり

💡 ユースケース

🏢 ユースケース 1: 営業チーム向け統合ナレッジ検索

営業担当者が商談前に、Salesforce の過去の商談履歴、顧客の課題、提案内容を Gemini Enterprise で自然言語検索。「A 社の過去 6 ヶ月の商談状況」のような検索が可能に。

設定例 
エンティティ: Accounts, Contacts, Opportunities, Notes
同期頻度:
  フルシンク: 毎日
  インクリメンタルシンク: 1 時間ごと
接続タイプ: Public

🎧 ユースケース 2: カスタマーサポートの問い合わせ対応効率化

サポートエージェントが問い合わせ時に、Salesforce の Cases、Knowledge Articles、顧客プロファイルを横断検索。AI が過去の類似ケースや関連ナレッジ記事を自動提示し、初回解決率の向上と平均対応時間の短縮を実現。

🔍 ユースケース 3: コネクタ運用の監視とアラート設定

インフラ運用チームが Logs Explorer でフェデレーテッドコネクタのエラーログを監視。Cloud Monitoring のアラートポリシーと連携して、データ同期の失敗や API エラーを即座に検知し、ビジネスユーザーへの影響を最小限に抑制。
