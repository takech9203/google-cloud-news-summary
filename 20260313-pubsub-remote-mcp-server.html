📨 Pub/Sub リモート MCP サーバー

Pub/Sub Preview MCP
2026-03-13

📋 概要

Google Cloud Pub/Sub のリモート Model Context Protocol (MCP) サーバーが Preview として利用可能になりました。MCP は Anthropic が開発したオープンソースプロトコルで、LLM や AI アプリケーションが外部データソースやサービスに標準化された方法で接続するための仕組みです。

Gemini CLI、Gemini Code Assist、Claude Code などの AI アプリケーションから、自然言語を通じて Pub/Sub リソースを管理できるようになりました。トピック、サブスクリプション、スナップショットの CRUD 操作に加え、メッセージ発行も MCP ツール経由で実行可能です。

🎯 主な対象ユーザー: AI 駆動の開発ワークフローを構築する開発者、プラットフォームエンジニア、イベント駆動アーキテクチャの構築・運用に携わるソリューションアーキテクト

🔄 Before / After

❌ Before

  • Pub/Sub リソース管理にはコンソール、gcloud CLI、クライアントライブラリが必要
  • AI エージェントからの操作にはカスタムの「グルーコード」が必要 (認証・API 呼び出し・レスポンスパースを自前実装)
  • AI アシスタントとの対話中にコンテキストスイッチが発生しワークフローが中断

✅ After

  • 標準化された MCP プロトコル経由で AI アプリケーションから直接 Pub/Sub を管理
  • 自然言語プロンプトでトピック作成・サブスクリプション設定・メッセージ発行が可能
  • Google Cloud 管理のリモート HTTPS エンドポイント提供でローカルサーバー不要

⚡ 主要機能

📬 トピック管理

トピックの作成・一覧取得・詳細取得・更新・削除に加え、トピックへのメッセージ発行 (publish) が可能

📥 サブスクリプション管理

サブスクリプションの CRUD 操作を自然言語指示で実行。フィルタ設定やメッセージ保持期間の変更も対応

📸 スナップショット管理

スナップショットの CRUD 操作。メッセージ再生 (replay) のためのスナップショット操作を AI ワークフローに統合

🔒 エンタープライズセキュリティ

IAM による細かい粒度の認可制御、Model Armor によるプロンプト/レスポンスのセキュリティスキャン (オプション)、一元化された監査ログ

🌐 マネージドエンドポイント

Google Cloud が管理するグローバルまたはリージョナル HTTPS エンドポイント。ローカルサーバーのセットアップや運用が不要

🔍 自動ツール検出

MCP tools/list による利用可能ツールの自動検出。AI アプリケーションが利用可能な操作を自動的に把握

🛠️ 設定方法

📋 前提条件: Google Cloud プロジェクトが作成済みであること、Pub/Sub API が有効化されていること、gcloud CLI の beta コンポーネントがインストール済みであること、必要な IAM ロールが付与されていること。

Step 1: gcloud CLI beta コンポーネントのインストール

Bash 
gcloud components install beta

Step 2: Pub/Sub MCP サーバーの有効化

Bash 
gcloud beta services mcp enable pubsub.googleapis.com \
  --project=PROJECT_ID
⚠️ 注意: 2026 年 3 月 17 日以降、対応サービスを有効化すると MCP サーバーも自動的に有効化されます。

Step 3: AI アプリケーションでの MCP クライアント設定

AI アプリケーション (Gemini CLI, Claude Code など) の MCP 設定に以下を追加します。

JSON 
{
  "mcpServers": {
    "pubsub": {
      "url": "https://pubsub.googleapis.com/mcp",
      "transport": "http"
    }
  }
}

Step 4: ツールの確認

MCP inspector または tools/list リクエストで利用可能なツールを確認します。

Bash 
# tools/list は認証不要
curl -X POST https://pubsub.googleapis.com/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "method": "tools/list"}'

💼 ビジネス面

開発者生産性の向上

AI アシスタントとの対話中にコンテキストスイッチなく Pub/Sub リソースを管理。イベント駆動アーキテクチャの構築・運用を効率化

オンボーディングの加速

Pub/Sub の操作に不慣れな開発者でも、自然言語で AI エージェントに指示してリソース管理が可能

🔬 技術面

標準化されたプロトコル

MCP により AI アプリケーションと Pub/Sub 間のインテグレーションが標準化。カスタムコードを削減

エンタープライズセキュリティ

IAM、OAuth 2.0、Model Armor、監査ログなどエンタープライズ向けセキュリティ機能が充実

マネージドインフラ

Google Cloud 管理のリモートエンドポイントにより、MCP サーバー自体の運用管理が不要

⚠️ デメリット・制約事項

制限事項
  • Preview 段階 -- SLA は適用されない。本番ワークロードでの利用は慎重に検討が必要
  • Pre-GA のため、仕様変更やサポートが限定的な場合がある
  • 大量のバッチ操作やリアルタイム性が求められる処理には従来の API / CLI が適切
考慮すべき点
  • Pub/Sub リソースの作成・削除権限付与は最小権限の原則に従い、専用のサービスアカウントを使用
  • プロンプトインジェクションなどのセキュリティリスクを考慮し、Model Armor の利用を検討
  • 操作対象と認証のプロジェクトが異なる場合は、両方で MCP サーバーの有効化が必要

💡 ユースケース

🏗️ ユースケース 1: AI エージェントによるイベント駆動アーキテクチャの構築

プラットフォームチームが新しいマイクロサービスのイベントバスを構築する際に、AI エージェント (Gemini CLI) を通じて必要な Pub/Sub リソースを自然言語で作成。

プロンプト例 
orders-events トピックを作成し、メッセージ保持期間を 7 日に設定してください。
また、order-processing-sub と analytics-sub の 2 つのサブスクリプションを作成し、
analytics-sub にはフィルタ 'attributes.priority = "high"' を設定してください。

効果: 複数の gcloud コマンドや API 呼び出しを個別に実行する必要がなくなり、設計意図を自然言語で伝えるだけで構築が完了。

🔧 ユースケース 2: 障害対応時のメッセージ再生操作

サブスクリプションのメッセージ処理でエラーが発生し、スナップショットを使用したメッセージ再生が必要な場合に、AI エージェントと対話しながら復旧操作を実行。

効果: 障害対応手順を自然言語で指示しながらスナップショットの確認・作成・シーク操作を迅速に実行。運用経験の浅いエンジニアでも適切な復旧操作が可能。

🧪 ユースケース 3: データパイプラインのプロトタイピング

AI アプリケーション開発者が、Gemini Code Assist のエージェントモード内でコーディングしながら、テスト用の Pub/Sub トピックやサブスクリプションを作成し、テストメッセージを発行。

効果: IDE を離れることなく Pub/Sub リソースの作成・テストが完了。コード作成とインフラ構成をシームレスに実行。

💰 料金

Pub/Sub リモート MCP サーバー自体の追加料金に関する情報は、本リリース時点では公開されていません。MCP ツール経由で作成・操作される Pub/Sub リソース (トピック、サブスクリプション、メッセージ配信) については、標準の Pub/Sub 料金が適用されます。
