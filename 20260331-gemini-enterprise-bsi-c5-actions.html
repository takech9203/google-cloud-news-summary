🔒 機能 1: BSI C5:2020 コンプライアンス認証

BSI C5:2020 認証の概要

Gemini Enterprise と NotebookLM Enterprise が、ドイツ連邦情報セキュリティ庁 (BSI) が策定した Cloud Computing Compliance Criteria Catalogue (BSI C5:2020) の認証を取得しました。欧州市場でクラウドサービスのセキュリティを評価する上で重要な基準です。

💡 ポイント: Gemini for Google Cloud 全体では既に BSI C5:2020 認証を保持しており、今回 Gemini Enterprise と NotebookLM Enterprise が個別に認証範囲に含まれました。ISO 27001/27017/27018/27701、SOC 1/2/3、HIPAA、FedRAMP などの既存認証に加えて、欧州市場向けのコンプライアンス基盤がさらに強化されています。

📊 機能 3: フェデレーテッドコネクタエラーログ

フェデレーテッドコネクタの詳細なエラーログを Cloud Logging の Logs Explorer で確認できるようになりました。以下の 3 種類のエラーを網羅します。

🔗 接続の問題

Connection problems: データストアへの接続に関するエラー

🔄 データ変換エラー

Data transformation issues: データフォーマットの変換に関するエラー

🌐 API エラー

外部 API への呼び出しに関するエラー

📝 ConnectorRunErrorContext

同期タイプ (FULL / INCREMENTAL)、開始/終了時間、リソース名などの詳細情報

エラーログのクエリ方法

Logs Explorer Query 
resource.type="consumed_api"
resource.labels.service="discoveryengine.googleapis.com"

特定のコネクタ同期エラーを詳細に確認する場合:

Logs Explorer Query (FULL sync) 
resource.type="consumed_api"
resource.labels.service="discoveryengine.googleapis.com"
jsonPayload.connector_run_payload.sync_type="FULL"

ロール説明

🛠️ 設定方法

前提条件

✅ Gemini Enterprise Standard または Plus エディションのライセンスが有効であること
✅ 対象プロジェクトで Gemini Enterprise (Discovery Engine) API が有効化されていること
✅ エラーログ確認には roles/logging.viewer ロールが付与されていること

✅ メリット

ビジネス面
→欧州市場でのコンプライアンス強化: BSI C5:2020 認証で金融、医療、公共セクターでの導入障壁が低減
→業務効率の向上: 自然言語による直接アクション実行でツール間の切り替えが削減
→運用コスト削減: エラーログ可視化でMTTR (平均修復時間) が短縮
技術面
→統合的な操作環境: 5 つのデータストアを単一インターフェースから操作可能
→観測可能性の向上: 3 種類のエラーを体系的に監視、BigQuery へのエクスポートも可能
→セキュリティ監査の簡素化: BSI C5:2020 + ISO + SOC + HIPAA で包括的なコンプライアンス体制

⚠️ デメリット・制約事項

制限事項
→新アクションは Public Preview (Pre-GA Offerings Terms 対象) で、本番利用には注意が必要
→1 つのアプリに同一コネクタタイプのデータストアを複数関連付けることは推奨されない
→HubSpot は検索操作が中心で、書き込みアクションの対応範囲は限定的
→VPC Service Controls の適用は、サードパーティデータストアでは再作成が必要
考慮すべき点
→Gmail/Drive のデータレジデンシー (DRZ) は Google Cloud 内のデータのみ保証。Workspace 側は別途確認が必要
→CMEK は Google Cloud 内のデータのみ適用。サードパーティサービス内のデータには適用されない
→フェデレーテッド検索ではデータがインデックスに取り込まれないため、検索品質が低い場合がある
→サポート対象リージョンは global、us、eu のみ

💻 ユースケース 2: ソフトウェア開発チームの GitHub 連携

シナリオ: 開発チームが Gemini Enterprise のアシスタントを使って GitHub での日常的な開発タスクを効率化。

自然言語コマンド例 
"Issue #234 にバグ修正の進捗コメントを追加して"
→ GitHub の Add comment to an issue アクションが実行

"feature/auth ブランチを作成して"
→ GitHub の Create branch アクションが実行

"PR #567 をマージして"
→ GitHub の Merge pull request アクションが実行

🔍 ユースケース 3: コネクタ障害の迅速な診断

シナリオ: 運用チームが HubSpot コネクタの同期失敗に気づき、原因を特定する必要がある。

Logs Explorer 
resource.type="consumed_api"
resource.labels.service="discoveryengine.googleapis.com"
severity>=ERROR

# jsonPayload.connector_run_payload でコネクタ固有の情報を確認
# - operation: 同期オペレーションのリソース名
# - entity: 同期対象のエンティティ
# - sync_type: FULL または INCREMENTAL
# - start_time / end_time: 同期の開始・終了時間

効果: エラーの種類 (接続問題、データ変換、API エラー) を即座に判別し、MTTR を短縮。
