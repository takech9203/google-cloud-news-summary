🗄️ Memorystore for Valkey

メンテナンスウィンドウ & セルフサービスメンテナンスの Google Cloud コンソール対応

GA Memorystore
2026-03-18

📋 概要

Memorystore for Valkey において、Google Cloud コンソールからメンテナンスウィンドウの検索・設定、およびセルフサービスメンテナンスの実行が可能になりました。これまで gcloud CLI でのみ利用可能だったメンテナンス管理機能が、GUI ベースの操作で完結できるようになり、運用効率が大幅に向上します。

対象ユーザーは、Memorystore for Valkey を利用しているインフラエンジニア、SRE、データベース管理者です。特に FedRAMP コンプライアンス要件に対応する必要がある組織にとって有用な機能です。

⚙️ 主要機能

📅 メンテナンスウィンドウの設定と検索

インスタンスごとにメンテナンスの曜日と開始時刻を設定可能。1 時間の枠で、曜日 (月〜日) と開始時刻 (UTC 0-23 時) を指定。スケジュールされたメンテナンスの開始・終了時刻・期限を GUI で確認。

🔄 メンテナンスの再スケジュール

スケジュールされたメンテナンスを最大 2 週間延期可能。即時実行 (IMMEDIATE) または特定日時への変更 (SPECIFIC_TIME) を選択。バッチサイズは 100 インスタンス以下を推奨。

🛡️ セルフサービスメンテナンス

利用可能なメンテナンスバージョンをコンソール上で確認。次回のスケジュールメンテナンスを待たずに新バージョンを即座に適用。FedRAMP コンプライアンスに必要な CVE パッチの迅速な適用に対応。

🔔 メンテナンス通知

Communication ページからメンテナンス通知をオンに設定可能。メンテナンス実施の少なくとも 1 週間前にメール通知を受信。プロジェクトレベルで設定し、各メールアドレスごとに個別にオプトイン。

📝 設定方法

💡 前提条件: Memorystore for Valkey API が有効化済み / IAM 権限の付与済み / インスタンスが作成済みであること

ステップ 1: メンテナンスウィンドウの設定

コンソールのインスタンス詳細ページからメンテナンスセクションを開き、曜日と開始時刻を設定します。トラフィックが最も少ない時間帯 (例: 日曜深夜) を選択することを推奨します。

Bash gcloud CLI でメンテナンスウィンドウを設定 
# 既存インスタンスにメンテナンスウィンドウを設定
gcloud memorystore instances update my-instance \
  --project=my-project \
  --location=us-central1 \
  --maintenance-policy-weekly-window=day=SUNDAY,startTime=hours=1

ステップ 2: メンテナンス通知の有効化

Google Cloud コンソールの Communication ページから、Memorystore のメール通知をオンに設定します。通知を受け取りたい各ユーザーが個別にオプトインする必要があります。

ステップ 3: セルフサービスメンテナンスの適用 (任意)

利用可能な新しいメンテナンスバージョンがある場合、コンソール上で確認して即座に適用できます。

Bash gcloud CLI でメンテナンスバージョンを更新 
# セルフサービスメンテナンスのバージョン更新
gcloud memorystore instances update my-instance \
  --location=us-central1 \
  --project=my-project \
  --maintenance-version=MAINTENANCE_VERSION

💻 gcloud CLI コマンド例

Bash インスタンス作成時にメンテナンスウィンドウを指定 
# インスタンス作成時にメンテナンスウィンドウを指定
gcloud memorystore instances create my-instance \
  --project=my-project \
  --location=us-central1 \
  --psc-auto-connections=network=projects/my-project/global/networks/default \
  --shard-count=8 \
  --maintenance-policy-weekly-window=day=SUNDAY,startTime=hours=1
Bash メンテナンスの再スケジュール (即時実行) 
# メンテナンスの再スケジュール (即時実行)
gcloud memorystore instances reschedule-maintenance my-instance \
  --project=my-project \
  --location=us-central1 \
  --reschedule-type=IMMEDIATE
Bash セルフサービスメンテナンスのバージョン確認 
# セルフサービスメンテナンスのバージョン確認
gcloud memorystore instances describe my-instance \
  --location=us-central1 \
  --project=my-project

✅ メリット

💰 運用コスト削減

GUI ベースの管理により、CLI スキルに依存せずにメンテナンス管理が可能。チーム全体の運用効率が向上。

⏱️ ダウンタイム最小化

トラフィックの少ない時間帯にメンテナンスをスケジュールすることで、エンドユーザーへの影響を最小限に抑制。

🛡️ コンプライアンス対応

セルフサービスメンテナンスにより、FedRAMP 等のコンプライアンス要件に対応した迅速な脆弱性パッチ適用が可能。

⚡ ゼロダウンタイム

Create-before-destroy 方式と協調フェイルオーバーにより、メンテナンス中もサービス継続が可能。

📅 柔軟なスケジューリング

曜日・時間帯の指定に加え、最大 2 週間の延期が可能。本番環境の運用スケジュールに合わせた調整が容易。

🔄 段階的デプロイ

メンテナンスはフリート全体で段階的にロールアウトされ、問題の早期検出と影響範囲の最小化を実現。

⚠️ 制約事項・注意点

制限事項
✕メンテナンスウィンドウは 1 時間だが、メンテナンスがウィンドウ内に完了しない場合がある
✕メンテナンスのオプトアウトや、インスタンス間のメンテナンス順序の制御は不可
✕複数インスタンスの一括再スケジュール機能は未提供 (プログラマティックなバッチ処理は可能)
✕メンテナンス通知のメールは Google アカウントに紐づくアドレスにのみ送信 (カスタムエイリアス不可)
💡 考慮すべき点
→メンテナンス通知はメンテナンスウィンドウが設定されているインスタンスに対してのみ送信
→再スケジュールの猶予は元のスケジュールから最大 2 週間であり、無期限の延期は不可
→インフライトデータの量やインスタンスサイズによって、フェイルオーバーのレイテンシが増加する可能性

💡 ユースケース

🛒 ユースケース 1: EC サイトのキャッシュ基盤メンテナンス管理

シナリオ: EC サイトでセッション管理やカート情報のキャッシュに Memorystore for Valkey を使用している運用チームが、売上への影響を最小限にメンテナンスを管理したい。

Bash 日本時間の月曜早朝にメンテナンスを設定 
# 日曜深夜 1:00 (UTC) にメンテナンスウィンドウを設定
# UTC 日曜 16:00 = JST 月曜 1:00
gcloud memorystore instances update cart-cache-instance \
  --project=ecommerce-prod \
  --location=asia-northeast1 \
  --maintenance-policy-weekly-window=day=SUNDAY,startTime=hours=16

効果: 日本時間の月曜早朝にメンテナンスがスケジュールされ、ピーク時間帯のサービス影響を回避。セールイベント前に再スケジュールで延期することも可能。

🏦 ユースケース 2: セキュリティパッチの迅速適用

シナリオ: FedRAMP 対応が必要な金融系サービスで、CVE パッチが公開された際にスケジュールメンテナンスを待たずに即座に適用したい。

効果: セルフサービスメンテナンスにより、まずテスト環境のインスタンスに新しいバージョンを適用して影響を確認し、問題がなければ本番環境にも即時適用。コンプライアンス監査への対応期間を短縮可能。

💰 料金

✅ メンテナンスウィンドウの設定およびセルフサービスメンテナンスの利用に追加料金は発生しません。

Memorystore for Valkey の料金はインスタンスのノード数、ノードタイプ、リージョンに基づいて課金されます。

料金例 (us-central1 リージョン)

構成月額料金 (概算)
highmem-medium / 3 シャード / レプリカなし (3 ノード)約 $337

🌐 利用可能リージョン

Memorystore for Valkey がサポートする全リージョンでメンテナンスウィンドウとセルフサービスメンテナンスを利用可能です。主要リージョン:

asia-northeast1 (東京) asia-northeast2 (大阪) us-central1 us-east1 europe-west1
