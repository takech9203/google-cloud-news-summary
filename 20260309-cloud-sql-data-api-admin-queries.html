Cloud SQL for MySQL Cloud SQL for PostgreSQL GA

📋 概要

Cloud SQL for MySQL および Cloud SQL for PostgreSQL において、Cloud SQL Data API を使用して管理クエリ (Administrative Queries) を実行できるようになりました。Data API は Cloud SQL Admin API の一部として提供され、gcloud CLI または REST API 経由で SQL ステートメントをインスタンスに対して直接実行できます。

DML、DDL、DQL のすべての SQL ステートメントタイプをサポートし、パブリック IP、プライベートサービスアクセス、Private Service Connect を使用するインスタンスで利用可能です。PostgreSQL では追加で拡張機能の有効化にも Data API を使用できます。

🔄 Before / After

❌ Before (アップデート前)

  • 管理クエリ実行には Cloud SQL Auth Proxy や直接のデータベース接続設定が必要
  • CI/CD パイプラインからの管理操作に接続ドライバーやプロキシの設定が前提
  • 簡単なスキーマ変更やユーザー管理にもデータベースクライアントツールの準備が必要

✅ After (アップデート後)

  • REST API や gcloud CLI から直接 SQL ステートメントを実行可能
  • IAM ベースの認証でデータベースパスワード管理が不要に
  • インフラ自動化やスクリプトから追加の接続設定なしに管理クエリを発行

🏗️ アーキテクチャ図

Cloud SQL Data API は Cloud SQL Admin API を経由して SQL ステートメントをインスタンスに転送し、IAM 認証による権限チェックを行った上でデータベースエンジンで実行します。

⚡ 主要機能

🌐 REST API による SQL 実行

executeSql エンドポイントで SQL を実行。リクエストボディに DB 名と SQL を JSON で指定し、レスポンスサイズ制御 (partialResultMode) にも対応。

🖥️ gcloud CLI による SQL 実行

gcloud beta sql instances execute-sql コマンドで直接 SQL を実行。--database、--sql オプションで簡単に管理操作を実行可能。

🔐 IAM ベースのアクセス制御

cloudsql.instances.executesql パーミッションによる細かなアクセス制御。Cloud SQL Admin、Instance User、Studio User ロールをサポート。

🐘 PostgreSQL 拡張機能の有効化

PostgreSQL では Data API を通じて CREATE EXTENSION ステートメントを REST API 経由で実行し、拡張機能をプログラマティックに有効化可能。

🛠️ 設定方法

📌 前提条件: Cloud SQL インスタンスで IAM データベース認証が有効化されていること、IAM ユーザーまたはサービスアカウントがインスタンスに追加されていること、適切な IAM ロールが付与されていること。

ステップ 1: Data API アクセスの有効化

Bash Data API を有効化 
gcloud sql instances patch INSTANCE_NAME \
  --data-api-access=ALLOW_DATA_API

ステップ 2: gcloud CLI で SQL を実行

Bash 管理クエリの実行例 
gcloud beta sql instances execute-sql INSTANCE_NAME \
  --database=DATABASE_NAME \
  --sql="CREATE USER 'new_admin'@'%' IDENTIFIED BY 'password'"

ステップ 3: REST API で SQL を実行

Bash REST API でプログラマティックに SQL を実行 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -d '{
    "database": "DATABASE_NAME",
    "sqlStatement": "GRANT SELECT ON *.* TO new_admin",
    "partialResultMode": "FAIL_PARTIAL_RESULT"
  }' \
  "https://sqladmin.googleapis.com/sql/v1beta4/projects/PROJECT_ID/instances/INSTANCE_NAME/executeSql"

✅ メリット

💰 運用コストの削減

データベースクライアントツールやプロキシの設定・管理が不要になり、管理操作の運用負荷が軽減されます。

🔄 自動化の促進

CI/CD パイプラインや IaC ツールからの管理操作をシンプルに統合でき、デプロイプロセスの自動化が進みます。

🔒 セキュリティの向上

IAM 認証を活用することで、データベースパスワードの管理を排除し、最小権限の原則を適用しやすくなります。

🔌 接続管理の簡素化

Cloud SQL Auth Proxy やデータベースドライバーのセットアップなしに SQL を実行でき、サーバーレスワークロードからのアクセスが容易になります。

⚠️ 制約事項・注意点

⚠️ 制限事項:
  • レスポンスサイズが 10 MB を超える場合、結果が切り詰められるかエラーが返される
  • リクエストのタイムアウトは 30 秒に固定され変更不可 (MySQL の SET SESSION MAX_EXECUTION_TIME や PostgreSQL の SET STATEMENT_TIMEOUT は非サポート)
  • 外部サーバーレプリケーションが設定されたインスタンスでは使用不可
  • 同時実行は 1 インスタンス・1 ユーザーあたり最大 10 リクエストに制限
  • MySQL 5.6/5.7 では、長時間実行 DDL がタイムアウトした場合に孤立ファイルやテーブルが発生する可能性
📌 考慮すべき点:
  • Data API は小規模な管理クエリ向けに設計されており、大量データの取得や長時間実行クエリには不向き
  • メモリを大量に消費するステートメントは Out-of-Memory エラーを引き起こす可能性
  • SQL Server では Data API は現時点で非サポート

💡 ユースケース

🚀 ユースケース 1: CI/CD パイプラインでのデータベースマイグレーション

アプリケーションのデプロイ時に、Cloud SQL Auth Proxy の設定なしに GitHub Actions や Cloud Build から直接スキーマ変更を実行。デプロイパイプラインの簡素化とプロキシ設定のメンテナンス負荷を削減。

Bash Cloud Build でのスキーマ変更 
# Cloud Build ステップで直接スキーマ変更を実行
gcloud beta sql instances execute-sql my-instance \
  --database=mydb \
  --sql="ALTER TABLE users ADD COLUMN last_login TIMESTAMP"

☁️ ユースケース 2: サーバーレス環境からの管理操作

Cloud Functions や Cloud Run から、データベースユーザーの作成やロール付与を自動化。REST API を使用してサービスアカウント経由でセキュアに管理操作を実行。接続プールやドライバー管理のオーバーヘッドを回避できます。

🐘 ユースケース 3: PostgreSQL 拡張機能のプログラマティックな有効化

Terraform や Pulumi でインフラをプロビジョニングした後、PostGIS や pg_trgm などの PostgreSQL 拡張機能の有効化を自動化。インフラプロビジョニングからデータベース設定までをエンドツーエンドで自動化。

Bash 拡張機能の有効化 
gcloud beta sql instances execute-sql my-pg-instance \
  --database=mydb \
  --sql="CREATE EXTENSION IF NOT EXISTS postgis"

💲 料金

Data API は Cloud SQL Admin API の一部として提供されます。Data API 自体の追加料金は公式ドキュメントでは明示されていませんが、Cloud SQL の標準料金 (インスタンスの vCPU、メモリ、ストレージ、ネットワーク) は通常通り適用されます。API リクエストに対するレートクォータが設定されているため、大量のリクエストを発行する場合はクォータ制限に注意が必要です。

🌍 利用可能リージョン

Cloud SQL Data API は、Cloud SQL for MySQL および Cloud SQL for PostgreSQL が利用可能なすべてのリージョンで使用できます。パブリック IP、プライベートサービスアクセス、Private Service Connect のいずれのネットワーク構成でも対応しています。

🔗 関連サービス・機能

Cloud SQL Admin API

Data API の基盤となる管理 API。インスタンスの作成・管理・監視に使用。

Cloud SQL Auth Proxy

従来のデータベース接続方式。Data API はプロキシなしでの SQL 実行を可能にする代替手段。

Cloud SQL Studio

Google Cloud コンソール上の SQL エディタ。Data API と同じ cloudsql.studioUser ロールで利用可能。

IAM データベース認証

Data API の前提条件となる認証方式。IAM ユーザー/サービスアカウントでのデータベースアクセスを提供。
