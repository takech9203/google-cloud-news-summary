📝 概要

Vertex AI Vector Search 2.0 が一般提供 (GA) となった。 従来の Vector Search 1.0 が近似最近傍 (ANN) インデックスのサービスであったのに対し、 2.0 ではストレージと検索を統合した包括的なシステムへと進化。 ゼロから設計されたセルフチューニング型のフルマネージド AI ネイティブ検索エンジンであり、 Collection と Data Object を中心としたアーキテクチャを採用している。

主な対象ユーザーは、RAG パイプラインの構築者、セマンティック検索やレコメンデーションシステムの開発者、エンタープライズ AI アプリケーションの構築チームである。

🔄 Before / After 比較

❌ Vector Search 1.0

  • インデックスの管理が主要な操作単位で、VM やレプリカの構成を手動管理
  • ベクトルデータとペイロードデータを別々のストレージで管理 (データ一貫性維持が煩雑)
  • ハイブリッド検索には密ベクトルと疎ベクトルの個別管理が必要
  • エンベディング生成は外部で行い、明示的にベクトルフィールドを設定する必要があった

✅ Vector Search 2.0

  • Collection ベースのアーキテクチャでデータとベクトルを統一的に管理
  • Auto-embeddings でベクトルフィールドを自動生成、管理の手間を大幅削減
  • ハイブリッド検索とランキングを単一の並列クエリで実行可能
  • セルフチューニングにより VM やレプリカの手動構成が不要

⚡ 主要機能

📦 Collections (コレクション)

→データとベクトルを統一管理するコンテナ (RDB のテーブルに類似)
→data_schema + vector_schema を JSON Schema draft-07 で定義
→1 つのデータベース内に複数 Collection を作成可能
→Collection Index で効率的な ANN 検索を実現
→スキーマバリデーションは厳格モード (additionalProperties: false)

🤖 Auto-embeddings

→組み込みモデルでベクトルフィールドを自動生成
→Data Object 作成時に自動でエンベディング付与
→外部でのエンベディング生成・管理が不要に
→Bring Your Own Embeddings (BYOE) にも対応

🔍 Hybrid Search & Ranking

→ベクトル検索 + 全文検索 + セマンティックリランキングを単一クエリで実行
→セマンティック検索とキーワード検索の長所を統合
→「Out of domain」データ (型番、ブランド名、社内コード) にもキーワードで対応

🔒 セキュリティとネットワーキング

→Private Service Connect (PSC): 低レイテンシで安全な接続、複数 VPC 対応
→Private Google Access (PGA): ハイブリッド環境でのプライベートアクセス
→VPC Service Controls: データ漏洩リスクの軽減

Collection Schema の構成

項目説明

💻 設定方法とコードサンプル

Collection の作成 (Python)

Python 
from google.cloud import vectorsearch_v1beta

client = vectorsearch_v1beta.VectorSearchServiceClient()

data_schema = {
    "type": "object",
    "properties": {
        "title": {"type": "string"},
        "genre": {"type": "string"},
        "year": {"type": "number"},
    },
}

vector_schema = {
    "plot_embedding": {"dense_vector": {"dimensions": 3}},
    "sparse_embedding": {"sparse_vector": {}},
}

collection = vectorsearch_v1beta.Collection(
    data_schema=data_schema,
    vector_schema=vector_schema,
)

request = vectorsearch_v1beta.CreateCollectionRequest(
    parent="projects/PROJECT_ID/locations/LOCATION",
    collection_id="COLLECTION_ID",
    collection=collection,
)

operation = client.create_collection(request=request)
operation.result()

Collection の作成 (gcloud CLI)

Bash 
gcloud beta vector-search collections create COLLECTION_ID \
  --location=LOCATION \
  --project=PROJECT_ID \
  --data-schema=data_schema.json \
  --vector-schema=vector_schema.json

Data Object の追加

Bash 
gcloud beta vector-search collections data-objects create DATA_OBJECT_ID \
  --data=data.json \
  --vectors=vectors.json \
  --collection=COLLECTION_ID \
  --location=LOCATION \
  --project=PROJECT_ID
💡 ヒント: Auto-embeddings が有効な場合、ベクトルフィールドは自動的に生成されるため --vectors オプションは省略可能です。
