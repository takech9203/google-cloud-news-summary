🤖 Gemini Enterprise: Data Insights Agent

Gemini Enterprise GA (Allowlist) BigQuery AI Agent
2026-03-24
Google Cloud Release Notes

📋 概要

Gemini Enterprise の Data Insights Agent が GA (allowlist) として提供開始されました。 Google が構築した「Made by Google」エージェントであり、BigQuery に格納されたデータから自然言語でインサイトを取得できる機能を提供します。

ユーザーが自然言語で質問を入力すると、エージェントが自動的に SQL クエリに変換して BigQuery データセットに対して実行し、結果をわかりやすい形式で返します。SQL の知識がないビジネスユーザーでも BigQuery のデータを直接分析できるようになります。

🤖 AI エージェント: Data Insights Agent は自然言語を SQL に変換する NL to SQL エンジンを内蔵しており、スキーマ説明やプロンプトのカスタマイズで変換精度を向上させることが可能です。

🔄 Before / After

❌ Before

  • BigQuery データからインサイトを得るには SQL の知識が必要で、ビジネスユーザーが直接分析できなかった
  • データ分析はエンジニアやアナリストへの依頼が必要で、回答に時間がかかっていた
  • NL to SQL は Preview 段階で、本番利用には信頼性の懸念があった

✅ After

  • Data Insights Agent が GA となり、本番ワークロードでの利用が可能に
  • 自然言語で BigQuery データに質問でき、SQL 不要でデータ分析が可能に
  • Gemini Enterprise アプリまたは REST API からプログラム的に利用可能

🏗️ アーキテクチャ図

ユーザーが自然言語で質問すると、Data Insights Agent が SQL に変換して BigQuery に問い合わせを行い、結果をインサイトとして返します。IAM と OAuth2 による認証・認可でデータアクセスが制御されます。

⚡ 主要機能

💬 自然言語によるデータクエリ

→自然言語の質問を自動で SQL に変換し BigQuery で実行
→スキーマ説明や NL to SQL プロンプトのカスタマイズで精度向上

🔐 テーブルアクセス制御

→allowlist / blocklist でアクセス可能なテーブルを制御
→機密データを含むテーブルへのアクセスを制限しセキュリティ確保

🎯 NL to SQL のカスタマイズ

→自然言語クエリと期待される SQL の例を提供して出力品質を向上
→スキーマの説明を自然言語で記述しエージェントの理解を補助

🔌 マルチインターフェース対応

→Gemini Enterprise アプリの UI からインタラクティブに利用可能
→REST API (streamAssist メソッド) でプログラム的アクセス

🔧 技術仕様

必要な IAM ロール

ロール 説明
roles/bigquery.dataViewer BigQuery データの閲覧権限
roles/bigquery.jobUser BigQuery ジョブの実行権限
roles/bigquery.metadataViewer BigQuery メタデータの閲覧権限
roles/discoveryengine.admin Gemini Enterprise の管理権限 (管理者用)

エージェント設定 (REST API)

JSON Agent Configuration 
{
  "displayName": "Data Insights Agent",
  "description": "BigQuery データからインサイトを提供するエージェント",
  "managed_agent_definition": {
    "data_science_agent_config": {
      "bq_project_id": "PROJECT_ID",
      "bq_dataset_id": "DATASET_ID",
      "allowlistTables": ["table_1", "table_2"],
      "blocklistTables": ["sensitive_table"]
    }
  },
  "authorization_config": {
    "tool_authorizations": ["AUTHORIZATION_RESOURCE_NAME"]
  }
}

Step 1: OAuth2 認証リソースの作成

Bash OAuth2 Authorization Resource 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -H "X-Goog-User-Project: PROJECT_NUMBER" \
  "https://discoveryengine.googleapis.com/v1alpha/projects/PROJECT_NUMBER/locations/LOCATION/authorizations?authorizationId=AUTHORIZATION_ID" \
  -d '{
    "name": "projects/PROJECT_NUMBER/locations/LOCATION/authorizations/AUTHORIZATION_ID",
    "serverSideOauth2": {
      "clientId": "CLIENT_ID",
      "clientSecret": "CLIENT_SECRET",
      "authorizationUri": "AUTHORIZATION_URI",
      "tokenUri": "https://oauth2.googleapis.com/token"
    }
  }'

Step 2: エージェントインスタンスの作成

Bash Create Agent Instance 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -H "X-Goog-User-Project: PROJECT_NUMBER" \
  "https://discoveryengine.googleapis.com/v1alpha/projects/PROJECT_NUMBER/locations/LOCATION/collections/default_collection/engines/APP_ID/assistants/default_assistant/agents" \
  -d '{
    "displayName": "AGENT_NAME",
    "description": "AGENT_DESCRIPTION",
    "managed_agent_definition": {
      "data_science_agent_config": {
        "bq_project_id": "BQ_PROJECT_ID",
        "bq_dataset_id": "BQ_DATASET_ID"
      }
    },
    "authorization_config": {
      "tool_authorizations": ["AUTHORIZATION_RESOURCE_NAME"]
    }
  }'

Step 3: エージェントのデプロイ

Bash Deploy Agent 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -H "X-Goog-User-Project: PROJECT_NUMBER" \
  "https://discoveryengine.googleapis.com/v1alpha/AGENT_RESOURCE_NAME:deploy" \
  -d '{
    "name": "AGENT_RESOURCE_NAME"
  }'

デプロイは Long-Running Operation (LRO) として実行されます。Step 4 では Google Cloud コンソールの Gemini Enterprise ページからユーザー権限を設定します。

✅ メリット

💼 ビジネス面

→データ民主化の推進: SQL 不要で BigQuery データからインサイト取得。データドリブンな意思決定を加速
→分析の待ち時間削減: アナリストへの依頼不要。リアルタイムでデータにアクセスし意思決定スピード向上

🔧 技術面

→API によるプログラム統合: REST API でカスタムアプリケーションにエージェントを組み込み可能
→セキュリティ制御: テーブルの allowlist/blocklist と IAM ロールによる細かなアクセス制御

⚠️ デメリット・制約事項

制限事項:
  • GA with allowlist のため、利用には Google Cloud 営業担当者への連絡が必要 (セルフサービス不可)
  • Gemini Enterprise のライセンスが別途必要

💡 考慮すべき点

→NL to SQL の変換精度はスキーマ説明や例の提供によるチューニングが必要な場合がある
→複雑なクエリや結合を伴う分析では、自然言語からの変換が意図通りにならない可能性
→BigQuery のクエリコストはエージェント利用時にも発生するため、コスト管理が必要

🎯 ユースケース

📈 ユースケース 1: 営業チームによる売上分析

シナリオ: 営業マネージャーが「先月の地域別売上トップ 10 は?」「前四半期比で売上が 20% 以上減少した製品は?」といった質問を自然言語で行い、BigQuery に格納された売上データからリアルタイムにインサイトを取得。

効果: データアナリストへの依頼待ち時間をゼロにし、営業会議の場で即座にデータに基づいた議論が可能に。

🖥️ ユースケース 2: カスタムダッシュボードへの組み込み

シナリオ: 社内ポータルサイトに REST API を通じて Data Insights Agent を組み込み、各部門のユーザーがチャット形式でデータを問い合わせできるインターフェースを構築。

効果: 既存の社内ツールに AI を活用したデータ分析機能を統合し、ユーザーの利便性を向上。

💰 料金

Data Insights Agent の利用には Gemini Enterprise のサブスクリプションとライセンスが必要です。

→月額サブスクリプションまたは年間サブスクリプションから選択可能
→ライセンスはユーザー単位で割り当て
→BigQuery のクエリコストは別途発生

📌 まとめ

Gemini Enterprise の Data Insights Agent が GA (allowlist) となり、BigQuery データに対する自然言語クエリが本番環境で利用可能になりました。 SQL の知識を持たないビジネスユーザーでもデータからインサイトを直接取得できるようになる点で、データ民主化を推進する重要なアップデートです。 利用を希望する場合は、Google Cloud の営業担当者に連絡してアクセスをリクエストすることが推奨されます。
