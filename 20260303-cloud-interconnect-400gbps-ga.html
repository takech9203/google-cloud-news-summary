🌐 Cloud Interconnect: 400 Gbps 接続サポート GA

Cloud Interconnect GA Networking
2026-03-03

📋 概要

Google Cloud は、Cloud Interconnect において 400 Gbps 接続のサポートを一般提供 (GA) として発表しました。 Dedicated Interconnect と Cross-Cloud Interconnect の両方に適用され、 単一接続あたりの最大帯域幅が従来の 200 Gbps から 400 Gbps へと大幅に拡大されました。

同時に、VLAN アタッチメントの最大容量も 400 Gbps に引き上げられ、 単一の VLAN アタッチメントで接続帯域幅をフルに活用できるようになりました。 AI/ML トレーニングデータ転送、大規模データ分析、マルチクラウド連携など、 高帯域幅が求められるワークロードへの対応力が飛躍的に向上します。

🔄 Before / After 比較

❌ アップデート前

  • Dedicated Interconnect の最大接続容量は 200 Gbps (2 x 100 Gbps) に制限
  • VLAN アタッチメント最大容量は 100 Gbps
  • それ以上の帯域幅には複数接続を束ねる必要
  • Cross-Cloud Interconnect も同様の帯域幅制約

✅ アップデート後

  • 単一接続で最大 400 Gbps の帯域幅を確保
  • VLAN アタッチメントも 400 Gbps に拡大
  • より少ない接続数で構成を簡素化
  • Cross-Cloud Interconnect でも 400 Gbps をサポート

🏗️ アーキテクチャ図

Dedicated Interconnect はオンプレミス環境と Google Cloud を直接接続し、Cross-Cloud Interconnect は他のクラウドプロバイダと Google Cloud を接続します。両方の接続タイプで 400 Gbps の帯域幅がサポートされました。

⚡ 主要機能

🔗 400 Gbps Dedicated Interconnect 接続

→単一の Dedicated Interconnect 接続で 400 Gbps の帯域幅を提供
→従来の最大 200 Gbps (2 x 100 Gbps) から 2 倍の帯域幅向上
→コロケーション施設でのオンプレミスと Google ネットワーク間の直接物理接続

☁️ 400 Gbps Cross-Cloud Interconnect 接続

→Google Cloud と他のクラウドプロバイダ (AWS, Azure, OCI, Alibaba Cloud) 間で 400 Gbps を提供
→マルチクラウドアーキテクチャにおける高帯域幅データ転送を実現
→Google がマネージドで提供する物理接続のため、自社でルーティング機器の管理が不要

🔀 400 Gbps VLAN アタッチメント

→単一の VLAN アタッチメントで最大 400 Gbps の帯域幅を設定可能
→Dedicated Interconnect と Cross-Cloud Interconnect の両方で利用可能
→従来必要だった複数 VLAN アタッチメントの構成が不要になるケースが増加

🔧 設定方法

前提条件

✓Google Cloud プロジェクトで Network Connectivity API が有効化されていること
✓400 Gbps 接続に対応したコロケーション施設でネットワーク機器が設置されていること
✓400 Gbps 対応の光トランシーバおよびルーティング機器を所有していること
✓Cloud Router が設定済みであること

ステップ 1: Dedicated Interconnect 接続の発注

Bash Dedicated Interconnect 接続の作成 (400 Gbps) 
# Dedicated Interconnect 接続の作成 (400 Gbps)
gcloud compute interconnects create my-interconnect-400g \
    --interconnect-type=DEDICATED \
    --link-type=LINK_TYPE_ETHERNET_400G_LR4 \
    --requested-link-count=1 \
    --location=COLOCATION_FACILITY \
    --admin-enabled
💡 注意: Google がリソースを割り当て、LOA-CFA (Letter of Authorization and Connecting Facility Assignment) を送付します。ベンダーに LOA-CFA を提出して物理接続をプロビジョニングしてください。

ステップ 2: VLAN アタッチメントの作成

Bash 400 Gbps VLAN アタッチメントの作成 
# 400 Gbps VLAN アタッチメントの作成
gcloud compute interconnects attachments dedicated create my-attachment-400g \
    --region=REGION \
    --router=my-cloud-router \
    --interconnect=my-interconnect-400g \
    --bandwidth=400G

VLAN アタッチメントを Cloud Router に関連付け、BGP セッションを構成します。

✅ メリット

ビジネス面

✓大規模データ転送の高速化: AI/ML モデルのトレーニングデータ、大規模分析データセットの転送時間を大幅に短縮
✓マルチクラウド戦略の強化: Cross-Cloud Interconnect での 400 Gbps サポートにより、クラウド間のデータ連携がより実用的に
✓構成の簡素化: 従来複数の接続で実現していた帯域幅を、より少ない接続数で達成でき、運用管理コストを削減

技術面

✓帯域幅の大幅向上: 単一接続で 400 Gbps を実現し、ネットワークのボトルネックを解消
✓VLAN アタッチメントの統合: 単一アタッチメントで 400 Gbps を利用でき、構成がシンプルに
✓低レイテンシの維持: パブリックインターネットを経由しない専用接続のため、一貫した低レイテンシを実現
✓SLA の維持: 冗長構成で 99.99% の可用性 SLA を引き続きサポート

⚠️ デメリット・制約事項

制限事項:
✗400 Gbps 接続は Dedicated Interconnect と Cross-Cloud Interconnect のみが対象。Partner Interconnect は対象外
✗400 Gbps 接続に対応したコロケーション施設が必要であり、すべてのロケーションで利用可能とは限らない
✗400 Gbps 対応の光トランシーバおよびネットワーク機器の調達が必要
考慮すべき点:
→オンプレミス側のネットワーク機器も 400 Gbps をサポートしている必要がある
→冗長構成 (99.99% SLA) では複数メトロエリアにまたがる接続が必要で、コストが増大する可能性
→Cross-Cloud Interconnect の場合、接続先クラウドプロバイダ側でも 400 Gbps に対応している必要がある

🎯 ユースケース

🤖 ユースケース 1: AI/ML トレーニングデータの高速転送

シナリオ: 大規模な AI モデルのトレーニングにおいて、オンプレミスのデータレイクから Google Cloud の Vertex AI 環境にペタバイト規模のデータセットを転送する。

効果: 400 Gbps の帯域幅により、従来の 200 Gbps 接続と比較してデータ転送時間を最大 50% 短縮。100 TB のデータセットの転送が約 33 分で完了 (理論値)。

🔄 ユースケース 2: マルチクラウドリアルタイムデータ連携

シナリオ: AWS 上のアプリケーションと Google Cloud 上の BigQuery 間でリアルタイムのデータストリーミングを行い、クラウド横断の分析パイプラインを構築する。

効果: Cross-Cloud Interconnect の 400 Gbps 接続により、大量のストリーミングデータを低レイテンシで転送でき、リアルタイム分析の精度とスピードが向上。

🛡️ ユースケース 3: ディザスタリカバリ環境の高速レプリケーション

シナリオ: オンプレミスのプライマリデータセンターから Google Cloud のディザスタリカバリ環境へ、大量のデータベースレプリケーションを行う。

効果: 400 Gbps の帯域幅により、RPO (目標復旧時点) を短縮し、より鮮度の高いバックアップを維持。

💰 料金情報

400 Gbps 接続の料金体系は公式料金ページで確認が必要です。従来の Dedicated Interconnect では以下の課金体系が適用されていました。

→ポート料金: ポートごとの月額料金
→VLAN アタッチメント料金: VLAN アタッチメントごとの料金
→エグレス料金: Cloud Interconnect 経由はインターネット経由より低い料金が適用
→固定ポート料金: アウトバウンドデータ転送の月額料金を固定化するオプションも利用可能
💡 詳細: 最新の料金情報は Cloud Interconnect 料金ページ を参照してください。

🔗 関連サービス・機能

→VPC (Virtual Private Cloud): Cloud Interconnect は VPC ネットワークと外部ネットワークを接続するための基盤サービス
→Cloud Router: BGP を使用した動的ルーティングを提供し、VLAN アタッチメントとオンプレミス/リモートクラウド間でルート交換
→Network Connectivity Center: Cross-Cloud Interconnect と組み合わせて、Google ネットワークを WAN として活用するサイト間データ転送が可能
→HA VPN over Cloud Interconnect: Cloud Interconnect トラフィックの IPsec 暗号化が必要な場合に使用
→MACsec for Cloud Interconnect: Cloud Interconnect 接続のレイヤー 2 暗号化を提供

📝 まとめ

Cloud Interconnect の 400 Gbps 接続サポートの GA は、Google Cloud のネットワーキング能力における重要なマイルストーンです。 Dedicated Interconnect と Cross-Cloud Interconnect の両方で帯域幅が従来の最大 200 Gbps から 400 Gbps に倍増し、 VLAN アタッチメントも 400 Gbps に対応したことで、大規模データ転送やマルチクラウド連携のボトルネックが大幅に解消されます。

推奨: AI/ML ワークロード、リアルタイムデータ分析、ディザスタリカバリなど、高帯域幅を必要とするユースケースを持つ組織は、この新しい接続オプションの導入を検討することを推奨します。
