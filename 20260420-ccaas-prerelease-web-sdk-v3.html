🔄 Before / After 比較

❌ アップデート前

  • コールスケジューリングは基本的なタイムスロット選択のみ。日ごとの閲覧や再スケジューリング・キャンセルが不可
  • HubSpot CRM との連携で SDK カスタムデータのカスタムフィールドマッピング機能がなく、データ連携が限定的
  • Headless SDK にフィーチャーフラグ取得、タイムスロット空き確認、コブラウズ復元、デバッグログ取得メソッドが存在しない
  • WCAG に完全準拠しておらず、RTL 言語 (アラビア語、ヘブライ語等) のサポートが不足

✅ アップデート後

  • 日ごとにタイムスロットを閲覧し、既存予約の再スケジューリング・キャンセルが可能に
  • SDK カスタムデータを HubSpot カスタムフィールドに直接マッピングし、ランタイムで自動書き込み
  • getFeatureFlags, fetchTimeSlotAvailability, restoreCobrowseSession, getLogs の 4 メソッド追加
  • WCAG 準拠、RTL 言語サポート (アラビア語、ヘブライ語、ペルシア語、ウルドゥー語)、スクリーンリーダー互換性改善

🚀 主要機能

🔗 1. HubSpot カスタムフィールドマッピング

→Web / Mobile SDK のカスタムデータフィールドを HubSpot のカスタムフィールドにマッピング可能
→ランタイムで SDK がカスタムデータ値を送信すると、マッピングされた HubSpot カスタムフィールドに自動的に値が書き込まれる

→従来 Zendesk、Salesforce、Kustomer 向けに提供されていた CRM カスタムフィールドマッピングが HubSpot にも拡張

→再スケジューリング: Web SDK を再度開いた際、既存予約があれば再スケジューリングまたはキャンセルを促される
→キャンセル: エンドユーザーが Web SDK から以前にスケジュールしたコールをキャンセル可能

コールスケジューリングのタイムスロット算出

💡 算出ロジック: タイムスロットは静的モデルに基づいており、15 分間隔ごとのスロット数は「キューに割り当てられた総エージェント数 x 0.5」で計算されます。スケジュールコールは Web SDK では PSTN 通話として発信されるため、SmartActions は利用できません。

💡 ユースケース

🛒 ユースケース 1: EC サイトでの VIP 顧客向けコールバック管理

EC サイトで VIP 顧客がサポートに問い合わせる際、高度なコールスケジューリング機能を使って、顧客自身が都合の良い日時を選択してコールバックを予約する。予定が変わった場合は Web SDK から再スケジューリングやキャンセルも可能。

JavaScript VIP 顧客向けコールスケジューリング実装例 
// Web SDK v3 でコールスケジューリングを初期化
var ccaas = new UJET({
  companyId: "VIP_SUPPORT",
  host: "https://support.example.com",
  authenticate: getAuthToken
});

ccaas.config({
  customData: {
    customer_tier: { label: "Customer Tier", value: "VIP" },
    order_id: { label: "Order ID", value: "ORD-98765" }
  }
});

ccaas.mount("#ccaas-widget");
📈 効果: VIP 顧客の待ち時間を解消し、予約管理の自律性を提供することで、顧客満足度と再購入率の向上が見込める。

🌍 ユースケース 2: 多言語対応のグローバルサポートポータル

中東・南アジア地域の顧客向けに、アラビア語・ヘブライ語・ペルシア語・ウルドゥー語対応のサポートウィジェットを Web サイトに組み込む。RTL 言語サポートにより、テキストの方向が自動的に右から左に切り替わる。

📈 効果: 対象地域の顧客がネイティブ言語で快適にサポートを利用でき、WCAG 準拠によりアクセシビリティ要件を満たすことで、規制準拠にも対応できる。

🔗 ユースケース 3: HubSpot を活用したカスタムデータの自動連携

HubSpot を CRM として使用しているサポートチームが、Web SDK から送信される顧客のプランタイプやアカウント ID などのカスタムデータを HubSpot のカスタムフィールドに自動マッピングする。

📈 効果: 手動でのデータ入力が不要になり、コンタクトセンターと CRM のデータ一貫性が確保される。エージェントは HubSpot のレコードで即座にカスタムデータを確認でき、対応品質が向上する。
