📋 概要

Security Command Center の AI Protection が、エージェントワークロード (agentic workloads) をサポートするようになりました。Vertex AI Agent Engine 上にデプロイされた AI エージェントや、Agent Registry にカタログ登録された Model Context Protocol (MCP) サーバーに対するセキュリティの可視化、脆弱性検出、脅威検出が可能になります。

AI エージェントはツールの呼び出し、外部サービスへのアクセス、コード実行などの自律的なアクションを実行するため、攻撃対象面 (Attack Surface) が拡大します。本アップデートは、こうした新しい脅威に対処するための包括的なセキュリティフレームワークを提供します。

💡 対象ユーザー: AI エージェントを本番環境にデプロイしているセキュリティチーム、DevSecOps エンジニア、AI プラットフォーム管理者。Security Command Center の Premium または Enterprise ティアの組織レベルでの有効化が必要です。

  • Agent Engine 上の CVE を一元スキャンする仕組みがない
  • エージェント向けのランタイム脅威検出ルールが存在しない
  • AI ダッシュボードでエージェントを可視化・フィルタリングできない

  • AI ダッシュボードにエージェントリソースのフィルタリングを追加

⚡ 主要機能

1. Agent Vulnerability Scanner (エージェント脆弱性スキャナー)

→Agent Engine でデプロイされたエージェントワークロードのソフトウェア脆弱性 (CVE) を自動的に識別
→新規の AI Protection 有効化時にはデフォルトで有効
→既存の顧客は AI Protection 設定ページの「Vulnerability Assessment for Agent Platform」セクションから手動で有効化が必要
→検出された CVE は AI セキュリティダッシュボードの Finding として表示

2. Agent Platform Threat Detection (エージェントプラットフォーム脅威検出)

→Vertex AI Agent Engine Runtime 上のエージェントに対するランタイム脅威検出
→ウォッチャープロセスがエージェント実行中にイベント情報を収集 (起動まで最大 1 分)
→自然言語処理 (NLP) を使用して Bash/Python スクリプトの悪意あるコードを分析
→Event Threat Detection と連携して監査ログからコントロールプレーンの脅威も検出

検出カテゴリ:

🕵️ Command and Control 🔑 Credential Access 🛡️ Defense Evasion ⚙️ Execution 📤 Exfiltration 💥 Impact ⬆️ Privilege Escalation

3. 拡張されたインベントリとフィルタリング

→AI セキュリティダッシュボードでエージェントと MCP サーバーを可視化
→リソースフィルタリングに aiplatform.googleapis.com/ReasoningEngine を追加
→攻撃パスシミュレーションでエージェントと MCP サーバーを高価値リソースとして設定可能
→過剰な権限を持つ Agent Engine エージェントの検出 (Preview)

🔧 設定方法

⚠️ 前提条件:
  • 1. Security Command Center Premium または Enterprise ティアが組織レベルで有効化されていること
  • 2. 組織レベルでの AI Protection のセットアップが完了していること
  • 3. AI Discovery サービスが有効化されていること

Step 1: AI Protection の有効化

Premium ティア

  1. Console > Security Command Center > Settings
  2. AI Protection カードの Manage Settings をクリック
  3. 画面の指示に従って有効化

Enterprise ティア

  1. Console > Security Command Center > SCC Setup Guide
  2. Review security capabilities summary を展開
  3. AI protection パネルから Set up をクリック

Step 2: Agent Vulnerability Scanner の有効化

  • Google Cloud Console > Security Command Center > Settings
  • AI Protection カードの "Manage Settings" をクリック
  • "Vulnerability Assessment for Agent Platform" セクションで "Manage settings" をクリック
  • 組織に対してサービスを有効化
💡 ヒント: 新規の AI Protection 有効化の場合、Agent Vulnerability Scanner はデフォルトで有効になります。

Step 3: MCP サーバーインベントリ用の App Hub 設定

Bash MCP サーバーをホストする各プロジェクトで App Hub API を有効化 
# MCP サーバーをホストする各プロジェクトで App Hub API を有効化
gcloud services enable apphub.googleapis.com --project=PROJECT_ID

Step 4: 高価値リソースセットへのエージェントリソースの追加 (オプション)

  • Google Cloud Console > Security Command Center > Settings
  • "Attack path simulation" セクションに移動
  • "Create resource value configuration" をクリック
  • リソースタイプに aiplatform.googleapis.com/ReasoningEngine を選択

⚠️ デメリット・制約事項

制限事項

  • ❌ Preview 段階: 本番環境での使用には限定的なサポート (Pre-GA Offerings Terms が適用)
  • ❌ 組織レベル必須: プロジェクトレベルのみの有効化では利用不可
  • ❌ MCP サーバーの手動登録: Agent Registry へのカタログ登録と App Hub API の有効化が必要 (自動検出ではない)
  • ❌ 検出遅延: ウォッチャープロセスの起動に最大 1 分かかり、極めて短時間の攻撃は検出できない可能性

考慮すべき点

  • ⚠️ Premium または Enterprise ティアでのみ利用可能 (Standard ティアでは限定的)
  • ⚠️ Agent Engine 以外のプラットフォーム (Cloud Run 上のカスタムエージェントなど) は Agent Platform Threat Detection の対象外
  • ⚠️ データはメモリ内で処理されインシデント以外は永続化されないが、ログ保持ポリシーとの整合性確認を推奨
  • ⚠️ 過剰権限検出 (Preview) により既存エージェントの権限設定の棚卸しが推奨される

🎯 ユースケース

🏦 金融機関における AI エージェントのセキュリティ管理

大手金融機関が顧客対応の AI エージェントを Vertex AI Agent Engine にデプロイ。エージェントは顧客データベースへのアクセス権を持ち、MCP サーバー経由で外部 API と連携。セキュリティチームはこれらのエージェントに対する脅威を監視する必要がある。

  1. AI Protection を組織レベルで有効化
  2. Agent Vulnerability Scanner で CVE を継続的にスキャン
  3. 顧客データにアクセスするエージェントを高価値リソースセットに追加
  4. App Hub API を有効化して MCP サーバーをインベントリに登録
  5. AI セキュリティダッシュボードで脅威と脆弱性を一元監視
効果: クレデンシャルアクセスやデータ窃取の試行を早期に検出し、PCI DSS や GDPR などの規制要件への対応を強化

🔐 DevSecOps チームによるエージェント開発パイプラインの保護

AI プラットフォームチームが複数の AI エージェントを開発・デプロイ。各エージェントには異なる IAM ロールが付与されており、過剰な権限を持つエージェントがないか確認したい。

  1. AI Protection の過剰権限検出を有効化
  2. Attack Path Simulation でエージェントリソースへの攻撃パスを分析
  3. 検出された Finding に基づいて IAM ポリシーを最小権限に修正
  4. Compliance Manager で AI Protection フレームワークの準拠状況を監視
効果: 過剰な権限を持つエージェントを早期に発見し、最小権限の原則に基づいたセキュリティポスチャを維持。攻撃パスシミュレーションにより侵害時の影響範囲を事前に把握

🏭 マルチエージェントシステムの MCP サーバーセキュリティ

製造業の企業がマルチエージェントシステムを構築。複数の MCP サーバーが生産管理システム、在庫管理 API、品質検査システムと連携。これらの MCP サーバーが攻撃対象になるリスクを評価したい。

効果: MCP サーバーを AI セキュリティダッシュボードで一元管理し、各サーバーに対する脅威検出とリスク評価を統合的に実施。App Hub と連携したインベントリ管理により、シャドー MCP サーバーの検出も可能に
