📋 概要

Google Cloud は 2026 年 4 月 22 日、データ分析およびデータエンジニアリング領域の主要 5 プロダクトについて一斉にリブランディングを実施しました。オープンソースエコシステムへのコミットメント強化と、各プロダクトの機能・役割をより直感的に伝えることが目的です。

特に Apache Spark / Apache Airflow のマネージドサービスは 「Managed Service for Apache XXX」という統一命名規則を採用。データガバナンス領域では AI 駆動コンテキストグラフを中核とする 「Knowledge Catalog」へ進化し、BI ツールは元の 「Data Studio」ブランドに回帰しました。

⚠️ 重要な対応事項: Looker Studio から Data Studio への変更では、ドメインが lookerstudio.google.com から datastudio.google.com に変更されます。プロキシで外部アクセスを制限している組織では、IT 管理者による ACL の更新が必要です。

🔄 Before / After 比較

❌ アップデート前の課題

  • プロダクト名が機能やオープンソース技術を直感的に示していなかった
  • 「Dataproc」「Cloud Composer」では Apache Spark / Airflow のマネージドサービスであることが理解しづらかった
  • 「BigLake」ではレイクハウスアーキテクチャの役割が不明確
  • 「Dataplex Universal Catalog」は AI 駆動のナレッジグラフ機能を反映していなかった
  • Looker Studio は Looker ブランドとの混同が発生

✅ アップデート後の改善

  • Apache Spark / Airflow のマネージドサービスであることが名前から明確に理解可能に
  • 「Google Cloud Lakehouse」でオープンデータレイクハウスの位置づけが明確化
  • 「Knowledge Catalog」で AI 駆動データガバナンスの役割を名称に反映
  • 「Data Studio」への回帰でセルフサービス BI ブランドを再確立
  • データ分析ポートフォリオ全体の一貫性と分かりやすさが向上

🔍 主要機能の詳細

⚡ Dataproc → Managed Service for Apache Spark

  • Dataproc on Compute Engine (クラスタ) と Google Cloud Serverless for Apache Spark (サーバーレス) を統合
  • クラスタモードとサーバーレスモードの 2 つのデプロイメントオプションは引き続き利用可能
  • API 名 (dataproc)、クライアントライブラリ、gcloud dataproc コマンド、IAM リソース名は変更なし
  • 既存のワークロード、設定、スクリプトへの影響なし

🏠 BigLake → Google Cloud Lakehouse

  • BigLake metastore は 「Lakehouse runtime catalog」に名称変更
  • Apache Iceberg オープンテーブルフォーマットとの統合を中核とするオープンデータレイクハウスとして位置づけ
  • API エンドポイント (biglake.googleapis.com)、gcloud biglake コマンド、クライアントライブラリは変更なし
  • Lakehouse runtime catalog は Apache Iceberg REST catalog と カスタム Apache Iceberg catalog for BigQuery の 2 つのインターフェースを提供

📚 Dataplex Universal Catalog → Knowledge Catalog

  • Gemini を活用した AI 駆動のデータカタログとして再定義
  • パッシブなメタデータレジストリから、アクティブな AI パワードコンテキストグラフへの進化を反映
  • AI エージェントがエンタープライズデータの正確なコンテキストを取得するための基盤
  • 自動コンテキストキュレーション、検証済みサンプルクエリ、Model Context Protocol (MCP) 統合などの新機能を反映
  • API (dataplex)、gcloud dataplex コマンド、クライアントライブラリ、IAM 名は変更なし

📊 Looker Studio → Data Studio ドメイン変更あり

  • プロダクト URL が lookerstudio.google.com から datastudio.google.com に変更
  • 自動リダイレクトが設定されており、旧 URL は新ドメインに自動転送
  • レポートの更新は不要
  • プロキシ使用環境: IT 管理者が新ドメイン datastudio.google.com を ACL に追加する必要あり
  • Gemini in Looker は「Gemini in Data Studio」に名称変更
  • Data Studio ホームページからは Data Studio レポート・データソース、BigQuery 会話型エージェント、Colab ノートブックで構築したデータアプリにアクセス可能

🔧 Cloud Composer → Managed Service for Apache Airflow

  • Apache Airflow のマネージドサービスであることを名称で明示
  • Airflow 3 の GA リリースと合わせたブランディング強化
  • Cloud Composer MCP サーバーによる AI アプリケーション連携 (Gemini CLI、ChatGPT、Claude 等) もサポート
  • API (composer)、gcloud composer コマンド、クライアントライブラリ、IAM リソース名は変更なし

⚙️ 設定方法

Looker Studio → Data Studio ドメイン変更への対応

⚠️ 前提条件: 組織でプロキシサーバーまたはファイアウォールを使用して外部サイトへのアクセスを制限しており、IT 管理者権限を持っている場合に実施してください。
  1. 新ドメインの ACL 追加
    組織のプロキシ設定またはファイアウォールルールに datastudio.google.com を許可リストとして追加します。
    Config プロキシ ACL 追加 
    datastudio.google.com
  2. リダイレクトの確認
    旧ドメインから新ドメインへの自動リダイレクトが正常に動作していることを確認します。
    Bash リダイレクト確認コマンド 
    # 旧ドメインからのリダイレクトを確認
curl -I https://lookerstudio.google.com
# Location ヘッダーに https://datastudio.google.com が含まれることを確認
  3. ブックマークの更新 (推奨)
    ユーザーに新しい URL https://datastudio.google.com でのブックマーク更新を案内します。自動リダイレクトにより旧 URL も引き続き利用可能ですが、新ドメインの直接利用を推奨します。

⚖️ メリット・デメリット

✅ メリット

ビジネス面
  • ブランド認知の向上: Apache Spark / Airflow のマネージドサービスであることが名前から即座に理解でき、新規顧客獲得に寄与
  • ポートフォリオの明確化: データ分析製品群の役割分担が名称から直感的に理解可能に
  • OSS コミットメント強化: 「Managed Service for Apache XXX」命名でオープンソース支持を明示
技術面
  • 影響最小化: 大半のプロダクトで API/CLI/IAM 名の変更なし
  • 統一レイクハウス: Google Cloud Lakehouse + Knowledge Catalog の組み合わせで一貫性向上
  • AI 統合の明示化: Knowledge Catalog の名称が AI 駆動コンテキストグラフ機能を反映

⚠️ デメリット・制約事項

制限事項
  • Looker Studio → Data Studio のドメイン変更で、プロキシ使用組織は ACL 更新が必須
  • API/CLI 名は旧名称のままのため、名称の不一致が生じる可能性 (例: gcloud dataproc で「Managed Service for Apache Spark」を操作)
  • 社内ドキュメントやトレーニング資料の更新が必要
考慮すべき点
  • 即時対応: Looker Studio 利用組織では IT 管理者が速やかに新ドメインを許可リストに追加する必要あり
  • 過渡期の混乱: 旧名称と新名称が混在する期間にコミュニケーション上の混乱が発生する可能性
  • サードパーティ互換性: 統合ツールやモニタリングダッシュボードでプロダクト名を参照している場合、表示名不一致の可能性

📝 まとめ

今回の一斉リブランディングは、Google Cloud のデータ分析ポートフォリオ全体の明確化とオープンソースへのコミットメント強化を目的としています。既存の API/CLI 名が維持されるため、大半の環境では技術的な対応は不要です。

⚠️ 唯一の要対応事項: Looker Studio → Data Studio のドメイン変更 (lookerstudio.google.com → datastudio.google.com) について、プロキシを使用している組織では IT 管理者による ACL の更新が必須です。速やかに新ドメインの許可リスト追加を行い、社内ドキュメントやトレーニング資料の名称更新を計画的に進めることを推奨します。
