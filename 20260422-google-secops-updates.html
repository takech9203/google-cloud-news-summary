📋 概要

Google は 2026 年 4 月 22 日、Google Security Operations (SecOps) SIEM のレガシーインフラストラクチャのサポートを 2027 年 4 月 30 日に終了することを正式に発表しました。この期限以降、レガシーインフラストラクチャ上の Google SecOps SIEM インスタンスにはアクセスできなくなります。

同時に、Google SecOps Marketplace では複数の統合アップデートが公開されました。Netskope、Qualys VM、SCC Enterprise、McAfee Mvision EDR など、主要なセキュリティ製品との連携が強化されています。

⚠️ 重要: 該当するインスタンスを運用しているすべての組織は、約 1 年以内に Google Cloud への移行を完了する必要があります。早急な対応計画の策定が求められます。

🔄 Before / After 比較

❌ Before (レガシーインフラ)

  • ❌ Google 所有プロジェクトでホスト - Google Cloud サービスとの直接統合が不可
  • ❌ レガシー SIEM 独自の認証方式 - WIF / Cloud Identity が利用不可
  • ❌ 独自 RBAC - Google Cloud IAM による細かなアクセス制御が未対応
  • ❌ Cloud Audit Logs 非統合 - 監査ログの可視性が限定的

✅ After (Google Cloud インフラ)

  • ✅ 顧客所有 GCP プロジェクト + VPC Service Controls
  • ✅ WIF / Cloud Identity による標準的な Google Cloud 認証
  • ✅ Google Cloud IAM による機能レベル RBAC
  • ✅ Cloud Audit Logs による包括的な監査ログ可視化
  • ✅ Agentic SOC、Dashboards、Security Validation 等の最新機能

⚠️ レガシー SIEM インフラの廃止詳細

🚨 サポート終了日: 2027 年 4 月 30 日 - この日以降、レガシーインフラストラクチャ上の Google SecOps SIEM インスタンスにはアクセスできなくなります。

移行が必要な条件 (いずれかに該当)

❌Google Cloud プロジェクトにデプロイされていない
❌Google Cloud 認証 (WIF / Cloud Identity) を使用していない
❌Google Cloud IAM による Feature RBAC を使用していない

移行が不要な条件 (すべてに該当)

✅Google Cloud プロジェクトにデプロイ済み
✅Workforce Identity Federation または Cloud Identity で認証している
✅Google Cloud IAM でアクセス権限を管理している

🔧 技術仕様: 移行の範囲

項目 レガシースタック モダンスタック
プロジェクトホスティング Google 所有プロジェクト 顧客所有の Google Cloud プロジェクト
認証 レガシー SIEM 認証 Google Cloud Auth: WIF / Cloud Identity
認可 レガシー SIEM RBAC Feature RBAC: Google Cloud IAM
監査ログ 限定的な内部ログ Cloud Audit Logs: 包括的な Google Cloud ログ

🛠️ 移行手順 (3 ステップ)

📝 前提条件: (1) Google Cloud 組織と Google Cloud プロジェクトが存在すること (2) 請求先アカウントにプロジェクトがリンク済みであること (3) レガシー Administrator ロールを持っていること
1 Google Cloud プロジェクトへの移行

Google SecOps にサインインし、Settings > SIEM Settings > Google Cloud Platform に移動。Google Cloud プロジェクト ID を入力し、Generate link をクリック。その後 Connect to Google Cloud Platform を選択。Google Cloud コンソールで Security > Google SecOps に移動し、自動作成されたサービスアカウントを確認して Connect をクリック。

↓
2 Google Cloud 認証への移行

Google Cloud コンソールで Security > Google SecOps に移動。Configure single sign-on で使用する IdP に応じて Google Cloud Identity または Workforce Identity Federation を選択。WIF を選択した場合はテスト SSO セットアップを実行して動作確認。

↓
3 Feature RBAC への移行

Google Cloud コンソールで Security > Google SecOps > Access management に移動。Migrate role bindings で自動生成された gcloud CLI コマンドを確認し、Cloud Shell で実行。Verify access をクリックして検証成功後、Enable IAM をクリックして移行完了。

⚠️ デメリット・制約事項

制限事項

❌移行は不可逆 - 特に RBAC の移行後はレガシー RBAC に戻すことができない
❌複数のフロントエンドパスを持つインスタンスの場合、セルフマイグレーションはサポートされず Google SecOps サポートへの連絡が必要
❌移行期限は 2027 年 4 月 30 日であり、期限超過でインスタンスへのアクセスが不可能に

考慮すべき点

💡移行は 3 ステップの順序を守って実施する必要があり、段階的な計画が必要
💡サードパーティ IdP を使用している場合は Workforce Identity Federation の事前設定が必要
💡IAM ロールとパーミッションの事前レビューが推奨 (レガシー RBAC からの適切なマッピングを確認)
💡移行完了後、ユーザーが適切な IAM ロールでアクセスできることの検証が必要
