📊 Cloud Monitoring: PromQL チャートのレジェンドテンプレート設定

Cloud Monitoring Feature Observability
2026-03-09

☁️ 概要

Cloud Monitoring の PromQL フォーマットチャート で レジェンドテンプレート (Legend Alias) が設定可能になりました。これにより、PromQL クエリで作成したチャートの凡例に表示される時系列の説明をカスタマイズでき、ダッシュボードの可読性が大幅に向上します。

従来は Monitoring フィルターベースのチャートでのみ利用可能であった Legend Alias 機能が、PromQL ベースのチャートでも同様に設定可能になりました。テンプレートにはプレーンテキストとテンプレート変数 (${resource.labels.zone} など) を組み合わせて使用でき、レジェンドの Name 列とツールチップに反映されます。

🔄 Before / After 比較

❌ アップデート前
  • PromQL チャートではレジェンドの表示名をカスタマイズできなかった
  • システムが自動選択したラベル値で構成され、分かりにくい表示になることがあった
  • 複数の時系列を表示するチャートで、各系列の直感的な識別が困難
✅ アップデート後
  • PromQL チャートでも Legend Alias フィールドでレジェンドをカスタマイズ可能に
  • テンプレート変数でリソースラベルやメトリクスラベルの値を動的に反映
  • プレーンテキストとテンプレート変数を組み合わせた柔軟な表示設定が可能

⚡ 主要機能

🔧 1. Legend Alias フィールドの PromQL 対応

PromQL フォーマットのチャートで Legend Alias フィールドが利用可能に。Display ペインから Legend Alias を展開して設定します。テンプレート変数のサジェスト機能を活用して、利用可能なラベルをメニューから選択できます。

🏷️ 2. テンプレート変数によるカスタマイズ

${resource.labels.<ラベル名>} 形式でリソースラベルを参照可能。例えば ${resource.labels.zone} でゾーン名を表示。複数のテンプレート変数とプレーンテキストを組み合わせて使用できます。

📋 3. レジェンド列とツールチップへの反映

テンプレートから生成された値はレジェンドの Name 列に表示されます。同じ内容がチャートのツールチップにも表示されます。ツールチップの表示領域には制限があるため、簡潔なテンプレートが推奨されます。

🛠️ 設定方法

📌 前提条件: Cloud Monitoring へのアクセス権限があり、PromQL クエリを使用したチャートが作成済みであること。
1

PromQL チャートの表示設定を開く

Metrics Explorer またはカスタムダッシュボードのチャート編集画面で、PromQL クエリを入力した後、Display ペインを開きます。

2

Legend Alias の設定

Display ペインで「Legend Alias」セクションを展開し、テンプレートを入力します。「Display template variable suggestions」をクリックすると、メニューからラベルを選択できます。例えば zone を選択すると、${resource.labels.zone} が自動挿入されます。

3

テンプレートの確認

設定したテンプレートがチャートのレジェンド (Name 列) とツールチップに正しく反映されていることを確認します。

✅ メリット

💼 ビジネス面
  • ダッシュボードの可読性向上: チームメンバーが各時系列の意味を即座に理解でき、モニタリング業務の効率が向上
  • インシデント対応の迅速化: 障害発生時にレジェンドが分かりやすく表示されることで、問題のあるリソースの特定が容易に
⚙️ 技術面
  • PromQL ユーザーの利便性向上: Prometheus から移行したユーザーのチャートカスタマイズ性が向上
  • Monitoring フィルターとの機能パリティ: PromQL チャートでも同等のレジェンドカスタマイズが可能に

🎯 ユースケース

📍 ユースケース 1: マルチゾーン環境のリソース監視

シナリオ: 複数のゾーンにまたがる GKE クラスタの CPU 使用率を PromQL で監視している場合、レジェンドにゾーン名を含めることで、どのゾーンのノードに負荷がかかっているかを一目で識別できます。

PromQL + Legend Alias 
# PromQL クエリ
{"kubernetes.io/node/cpu/allocatable_utilization"}

# Legend Alias テンプレート
Node: ${resource.labels.node_name} (${resource.labels.zone})
効果: ゾーンごとのリソース使用状況を直感的に把握でき、特定ゾーンへの負荷集中を迅速に検知できます。

⏱️ ユースケース 2: 複数サービスのレイテンシ比較

シナリオ: 複数のマイクロサービスのレスポンスタイムを 1 つのチャートで比較する場合、サービス名をレジェンドに表示することで、パフォーマンスのボトルネックとなっているサービスを容易に特定できます。

効果: レジェンドの表示が分かりやすくなることで、チーム間でのダッシュボード共有時のコミュニケーションコストが削減されます。

📌 まとめ

Cloud Monitoring の PromQL フォーマットチャートでレジェンドテンプレートが設定可能になったことで、PromQL ユーザーのチャートカスタマイズ性が大幅に向上しました。

Prometheus エコシステムから Cloud Monitoring への移行を進めている組織や、PromQL を主要なクエリ言語として使用しているチームにとって有用なアップデートです。既存の PromQL チャートを使用している場合は、Display ペインの Legend Alias フィールドからテンプレートを設定し、ダッシュボードの可読性向上を検討することを推奨します。
