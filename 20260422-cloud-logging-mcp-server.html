📚 概要

Google Cloud は Cloud Logging API のリモート Model Context Protocol (MCP) サーバーが一般提供 (GA) となったことを発表しました。MCP は Anthropic が開発したオープンソースプロトコルであり、LLM や AI アプリケーションが外部データソースに接続する方法を標準化するものです。

Cloud Logging リモート MCP サーバーにより、Gemini CLI、Claude、ChatGPT、およびカスタム AI アプリケーションから、標準化された MCP インターフェースを通じて Google Cloud のログエントリに直接アクセスできるようになりました。

💡 対象ユーザー: AI エージェントを活用した運用自動化やインシデント対応の効率化を検討している SRE チーム、DevOps エンジニア、セキュリティ運用チーム、AI アプリケーション開発者

🔄 Before / After

❌ Before (Preview)

  • ❌ Preview ステータスで本番利用に SLA やサポートの面で制約
  • ❌ AI エージェントからのログアクセスにカスタム統合コードの実装が必要
  • ❌ API 呼び出しのラッパーを個別に構築し、LLM 向け形式変換が必要
  • ❌ ローカル MCP サーバー (stdio) はセキュリティポリシーの一元管理が困難

✅ After (GA)

  • ✅ 本番環境に適した安定性、SLA、サポートを提供
  • ✅ 標準化された MCP プロトコルで直接接続、カスタムコード不要
  • ✅ IAM + OAuth 2.0 によるエンタープライズ向けセキュリティ
  • ✅ Model Armor + Cloud Audit Logs でプロンプト/レスポンスの保護と監査

⚡ 主要機能

🌐 リモート MCP エンドポイント

→エンドポイント URL: https://logging.googleapis.com/mcp
→HTTP ベースのトランスポート - クライアント側でのサーバー構築・運用が不要
→Google のマネージドインフラストラクチャ上で動作、グローバルまたはリージョナルなエンドポイントを提供

🔧 6 種類の MCP ツール

ツール名 説明 読み取り専用 冪等性
list_log_entries ログエントリの検索・取得 (重大度、リソースタイプ、テキスト内容でのフィルタ) ✅ ✅
list_log_names プロジェクト内のログ名の一覧表示 ✅ ✅
list_buckets ログバケットの一覧表示 ✅ ✅
get_bucket 特定のログバケットの詳細取得 ✅ ✅
list_views ログバケット内のログビューの一覧表示 ✅ ✅
get_view 特定のログビューの詳細取得 ✅ ✅

🔒 エンタープライズ向けセキュリティ

→IAM によるきめ細かいアクセス制御 (roles/mcp.toolUser + roles/logging.admin)
→OAuth 2.0 認証 (全ての Google Cloud ID に対応)
→Model Armor によるプロンプトインジェクション / 機密データ漏洩防止 (オプション)
→IAM 拒否ポリシーによる MCP ツールアクセスの制御
→Cloud Audit Logs によるツール呼び出しの監査記録

📄 技術仕様

OAuth スコープ

スコープ URI 説明
https://www.googleapis.com/auth/logging.admin ログデータの管理 (読み書き)
https://www.googleapis.com/auth/logging.read ログデータの読み取り
https://www.googleapis.com/auth/logging.write ログデータの書き込み

必要な IAM ロール

JSON 必要な IAM ロール 
{
  "required_roles": [
    {
      "role": "roles/mcp.toolUser",
      "description": "MCP ツール呼び出しの実行権限",
      "permission": "mcp.tools.call"
    },
    {
      "role": "roles/logging.admin",
      "description": "Logging MCP ツールの使用権限"
    }
  ]
}

🛠️ 設定方法

📋 前提条件:
✓Google Cloud プロジェクトで Cloud Logging API が有効化されていること
✓適切な IAM ロール (roles/mcp.toolUser および roles/logging.admin) が付与されていること
✓OAuth 2.0 認証の設定が完了していること

Step 1: Cloud Logging API の有効化

Bash API 有効化 
gcloud services enable logging.googleapis.com

Cloud Logging API を有効化すると、リモート MCP サーバーも自動的に有効化されます。

Step 2: IAM ロールの付与

Bash IAM ロール付与 
# MCP ツール呼び出し権限の付与
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/mcp.toolUser"

# Cloud Logging 管理権限の付与
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/logging.admin"

Step 3: MCP クライアントの設定

JSON MCP クライアント設定 
{
  "mcpServers": {
    "cloud-logging": {
      "name": "Cloud Logging MCP server",
      "url": "https://logging.googleapis.com/mcp",
      "transport": "http",
      "authentication": {
        "type": "oauth2",
        "scope": "https://www.googleapis.com/auth/logging.read"
      }
    }
  }
}

Step 4: 動作確認 (ツール一覧の取得)

Bash 接続確認 (認証不要) 
curl --location 'https://logging.googleapis.com/mcp' \
  --header 'content-type: application/json' \
  --header 'accept: application/json, text/event-stream' \
  --data '{
    "jsonrpc": "2.0",
    "method": "tools/list",
    "id": 1
  }'

tools/list メソッドは認証不要で実行できるため、接続確認に利用できます。

🔨 技術面

✓標準化されたインターフェース - MCP 対応の任意の AI アプリケーションからシームレスに利用可能
✓読み取り専用の安全な設計 - 6 つのツール全てが読み取り専用かつ冪等、意図しない変更リスクなし
✓柔軟なフィルタリング - Cloud Logging クエリ言語対応で高度なフィルタリングが可能

⚠️ デメリット・制約事項

🚧 制限事項
✗list_log_entries は一度に単一のリソースプロジェクトのみ対象。複数プロジェクト指定は失敗する
✗提供されるツールは 6 種類のみ。ログの書き込みやシンクの管理はサポート外
✗Model Armor は特定リージョンのみ利用可能。未対応リージョンではルーティング動作が異なる場合あり
💡 考慮すべき点
→エージェント用に専用のサービスアカウントを作成し、最小権限の原則を適用することを推奨
→Data Access 監査ログはデフォルトで無効。完全な監査が必要な場合は明示的に有効化が必要
→ローカル MCP サーバー (stdio 通信) はローカル開発やオフライン利用で引き続き利用可能

🎯 ユースケース

🚨 UC1: AI エージェントによるインシデント対応の自動化

シナリオ: SRE チームが AI エージェントを活用し、アラート発生時にログを自動検索して初期分析を行う。

Text プロンプト例 
「過去 1 時間の重大度が ERROR 以上のログをすべて表示してください。
 特に Compute Engine インスタンス web-server に関連するものを優先してください。」

AI エージェントが list_log_entries ツールを使用し、
フィルタ severity >= ERROR AND resource.type = "gce_instance"
でログエントリを取得・分析する。
✅ 効果: インシデント発生時の初動分析を AI エージェントに委譲し、オンコールエンジニアの負荷軽減と対応時間短縮を実現。

🕵️ UC2: セキュリティ監査ログの日常的な分析

シナリオ: セキュリティチームが AI エージェントを使用して、特定のサービスアカウントに関するアクティビティログを定期的に確認する。

Text プロンプト例 
「過去 7 日間のログから、特定のサービスアカウント
 sa-deploy@my-project.iam.gserviceaccount.com に関連する
 すべてのログを表示してください。」
✅ 効果: セキュリティ監査作業を AI エージェントで自動化し、不正アクセスや異常な動作を早期に検出。

🗃️ UC3: ログバケット構成の可視化と管理

シナリオ: 運用チームが AI エージェントを使用して、プロジェクト内のログバケットとビューの構成状況を把握する。

Text プロンプト例 
「このプロジェクトのログバケットをすべて一覧表示し、
 それぞれの保持期間と分析機能の有効/無効を教えてください。」

AI エージェントが list_buckets ツールを使用して
バケット構成を取得し、人間が理解しやすい形式で報告する。
✅ 効果: ログストレージの構成を AI エージェントが即座にサマリ化し、コスト最適化やコンプライアンス要件の確認が容易に。

💰 料金

Cloud Logging MCP サーバー自体の利用に追加料金は発生しません。料金は Cloud Logging の既存の料金体系に基づきます。

📦 ログ取り込み (月間 50 GiB まで) 無料
📦 ログ取り込み (超過分) $0.50/GiB
🗃️ ログ保持 (デフォルト 30 日間) 無料
🗃️ ログ保持 (延長保持) $0.01/GiB
🔧 MCP ツール呼び出し 追加料金なし
