☁️ 概要

NotebookLM Enterprise において、ノートブックを他のユーザーやグループと共有する際に、メールアドレスおよびグループ名のオートコンプリート機能が一般提供 (GA) となりました。共有ダイアログで入力を開始すると候補が自動的に表示され、正しい宛先を素早く選択できます。

Google Identity Provider を使用する組織はそのまま利用可能です。Microsoft Entra ID + Workforce Identity Federation を使用する組織では、管理者が SCIM テナントのプロビジョニングを行うことで完全な機能を利用できます。

🔄 Before / After 比較

❌ Before

  • メールアドレスを正確に手入力する必要があった
  • Microsoft Entra ID 環境ではグループの UUID を入力する必要があった
  • 入力ミスによる誤共有・共有失敗のリスク

✅ After

  • オートコンプリート候補から正しいアドレスを選択可能
  • 人間が読めるグループ名で共有可能 (SCIM 構成時)
  • 入力ミスのリスクが軽減、操作性が大幅向上

⚡ 主要機能

📧 メールアドレスのオートコンプリート

共有ダイアログでメールアドレスの入力を開始すると、同一プロジェクト内のユーザー候補が表示されます。候補から選択することで正確なアドレスを素早く入力可能です。

👥 グループ名のオートコンプリート

Google グループや Microsoft Entra ID グループの名前を入力すると候補が表示されます。SCIM 構成済みの場合、UUID ではなく人間が読めるグループ名で検索・選択が可能です。

🔐 ID プロバイダー別の動作

→ Google Identity Provider: 追加設定なしでオートコンプリートが利用可能
→ サードパーティ IdP (Microsoft Entra ID): SCIM テナントのプロビジョニングが必要

Step 2: SCIM テナントのプロビジョニング

Gemini Enterprise 管理者が SCIM テナントをプロビジョニングします。ユーザーおよびグループ情報が Google Cloud に同期されます。

Step 3: オートコンプリートの動作確認

SCIM テナントのプロビジョニング完了後、NotebookLM Enterprise の共有ダイアログでグループ名によるオートコンプリートが利用可能になります。

⚠️ 制限事項・注意点

⚠️ 知っておくべき制限:
  • → オートコンプリートは同一プロジェクト内のユーザー・グループに限定
  • → NotebookLM Enterprise と個人版 NotebookLM / NotebookLM Plus 間のノートブック共有は不可
  • → Gemini Enterprise 検索結果からソースを追加したノートブックは共有不可
  • → SCIM フィルタ式は eq (equals) 演算子のみサポート
  • → SCIM のユーザーリスト取得は最大 100 件まで

考慮すべき点

📌 Microsoft Entra ID 環境で SCIM なしの場合、グループ共有にはオブジェクト ID (UUID) の入力が引き続き必要
📌 SCIM テナントのプロビジョニングには管理者権限が必要
📌 SCIM で同期される属性値は不変として扱われるため、変更する場合は IdP 側でユーザー/グループを削除して再作成が必要

💡 ユースケース

🏢 ユースケース 1: チーム内のプロジェクトナレッジ共有

シナリオ: 技術チームのリーダーが、プロジェクトの技術調査結果をまとめた NotebookLM ノートブックをチームメンバー全員と共有したい。

効果: オートコンプリート機能により、チームメンバーのメールアドレスを素早く入力でき、共有操作にかかる時間が短縮される。グループ名での共有を使えば、チーム全体への一括共有も容易に。

🏗️ ユースケース 2: 大規模組織でのグループベース共有

シナリオ: Microsoft Entra ID を ID プロバイダーとして使用する大規模組織で、部門ごとのグループにノートブックを共有したい。

効果: SCIM テナントを構成することで、管理者や一般ユーザーがグループの UUID を調べる必要がなくなり、グループ名を入力するだけで正しいグループを選択して共有できる。
