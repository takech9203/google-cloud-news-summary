📋 概要

Dataplex (Knowledge Catalog) に、非構造化データ向けの Data Insights 機能が Preview として追加されました。Cloud Storage に保存された PDF などの非構造化ファイル (いわゆる「ダークデータ」) を、Vertex AI (Gemini 2.5 Pro) を活用して分析し、構造化されたクエリ可能なアセットへと自動変換します。

🤖 AI によるコンテンツ分析: 従来のファイルレベルのメタデータ取得 (ファイル名、サイズ、タイプ) に留まっていたディスカバリツールが、Gemini 2.5 Pro によるドキュメントの実際のコンテンツ分析へと進化。エンティティの推論やリレーションシップの抽出を自動的に行います。

⚡ 主要機能

🔍
1. 自動ディスカバリスキャン (セマンティック推論)
Cloud Storage 内の非構造化ファイルを自動検出し、BigLake オブジェクトテーブルとしてカタログ化。「Enable semantic inference」チェックボックスで Vertex AI によるコンテンツ分析を実行。オンデマンド/スケジュール実行に対応。
🧠
2. エンティティ推論とリレーションシップ抽出
Generative AI でドキュメントから具体的な属性 (Company, Product, Serial Number) を抽出。エンティティ間の接続関係 (例: Component is_part_of Product) を識別しセマンティックグラフを構築。
📊
3. データ抽出 (SQL / パイプライン)
SQL 抽出: BigQuery リモートモデルを使用したゼロインフラのアドホック分析。
パイプライン抽出: Dataform ベースの大規模処理。リトライロジック、エラーハンドリング、自動オーケストレーション対応。
🕸️
4. インタラクティブなセマンティックグラフビュー
Knowledge Catalog の Insights タブで、エンティティ (ノード) とリレーションシップ (エッジ) をインタラクティブに可視化。AI 生成説明、推論スキーマ (フィールド名、データ型、フィールド説明) を確認可能。

🛠️ 設定方法

前提条件

✓Dataplex API、BigQuery API、Vertex AI API を有効化
✓Dataplex サービスアカウントに必要な IAM ロールを付与
✓非構造化データ (PDF) を Cloud Storage バケットにアップロード
✓BigQuery 接続 (Connection ID) を作成

手順

1
ディスカバリスキャンの作成
Google Cloud コンソール > Metadata curation > Cloud Storage discovery > 「Create」。スキャン名を入力し、対象バケットを選択。Unstructured data options で「Enable semantic inference」を有効にし、BigQuery 接続 ID を指定して実行。
2
インサイトの確認
BigQuery コンソール > Governance > Metadata curation > 対象スキャンを選択 > Published dataset のリンク > BigLake オブジェクトテーブルを選択 > Knowledge Catalog で検索し Insights タブを確認。
3
データの抽出
SQL の場合: Insights タブの「Extract by SQL」で自動生成される SQL を実行。ML.PROCESS_DOCUMENT 関数を使用。
パイプラインの場合: Insights タブで「Extract by pipeline」を選択し、パイプライン名、リージョン、宛先データセットを指定して実行。

✅ メリット

ビジネス面

💡ダークデータの活用
これまでアクセス不可能だった非構造化ファイル内の情報を、分析可能な構造化データとして活用。
💰コスト削減
カスタムパーサーや ETL コードの開発・保守コストが不要。ワンクリックでデータ抽出が可能。
🛡️データガバナンスの強化
Knowledge Catalog への自動登録により、非構造化データも組織全体のデータガバナンス対象に。

技術面

🧠AI ベースの自動スキーマ推論
Vertex AI (Gemini 2.5 Pro) がドキュメントコンテンツからスキーマを自動生成。手動スキーマ設計が不要。
🕸️セマンティックグラフによる関係性の可視化
エンティティ間の関係をグラフ表現。RAG エージェントのグラウンディングにも活用可能。
📊BigQuery との統合
抽出データを既存の構造化データセットと結合して分析。Gemini in BigQuery による会話型分析や Looker Studio でのダッシュボード作成にも対応。

⚠️ デメリット・制約事項

制限事項

✗Preview 段階 - サポートが限定的。Pre-GA Offerings Terms が適用される
✗PDF ファイルに最適化 - 他のファイル形式では同等の精度が保証されない
✗リージョン制限 - Vertex AI Gemini 2.5 Pro がサポートされているリージョンのみ利用可能

考慮すべき点

💸ディスカバリスキャンとデータ抽出には Vertex AI の推論コストが発生
⚙️パイプライン抽出時は Dataform サービスアカウントの追加設定が必要
👤AI 推論結果の精度は human-in-the-loop による検証が推奨される
📍抽出先の BigQuery データセットはソースと同じロケーションに配置する必要がある

💼 ユースケース

🏦 金融サービスにおける請求書の自動分析
金融サービス企業が数千件の PDF 請求書から請求明細、ベンダー名、契約条件を自動抽出し、BigQuery にマテリアライズして支出分析を即座に行う。
✅ カスタムパーシングコードなしで請求書データを構造化テーブルに取り込み、リアルタイムの支出分析が可能に。
⚖️ 法務・コンプライアンス部門における契約書分類
法務部門が大量の過去の契約書リポジトリを自動分類し、主要エンティティ (契約当事者、契約日、条件など) を抽出。データスチュワードが AI 生成メタデータを検証した上で規制報告に使用。
✅ human-in-the-loop の検証プロセスを経て、信頼性の高いメタデータに基づく規制対応が可能に。
🏭 AI エージェントのグラウンディング
製造企業がメンテナンスログから機器の関係性を抽出。会話型 AI エージェントがユーザーの質問に対して、検証済みのリレーションシップグラフに基づいた正確な回答を提供。
✅ RAG エージェントが元のドキュメントまで遡れるトレーサビリティチェーンにより、ハルシネーションが低減。

📝 まとめ

Dataplex (Knowledge Catalog) の非構造化データ向け Data Insights は、Cloud Storage 上の PDF ファイルを Vertex AI (Gemini 2.5 Pro) で自動分析し、構造化されたクエリ可能なアセットに変換する Preview 機能です。手動パーシングやカスタム ETL コードを不要にし、エンティティとリレーションシップの自動抽出、セマンティックグラフの生成、BigQuery へのワンクリック抽出を実現します。

大量の非構造化ドキュメントを保有する組織は、この機能の Preview を評価し、ダークデータの活用可能性を検討することを推奨します。
