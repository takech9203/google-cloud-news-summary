☁️ Go 1.26 ランタイム GA

GA App Engine Cloud Run Cloud Run functions
2026-03-12

📋 概要

Google Cloud は、Go 1.26 ランタイムのサポートを General Availability (GA) として複数のサーバーレス/コンピューティングサービスで提供開始しました。App Engine standard/flexible、Cloud Run、Cloud Run functions の 4 サービスで統一的に Go 1.26 を利用できます。

GA 化により SLA の適用対象となり、エンタープライズワークロードでの安心した採用が可能になりました。

🔄 Before / After

❌ Before (Preview)

  • Go 1.26 は Preview ステータスで SLA が適用されなかった
  • 本番ワークロードでは Go 1.25 以前を使用する必要があった
  • Preview ステータスは機能や安定性が変更される可能性がありリスクがあった

✅ After (GA)

  • GA となり SLA の適用対象として本番環境で安心して利用可能
  • 4 サービスすべてで Go 1.26 が正式サポート
  • Go 1.26 の最新機能とパフォーマンス改善を本番で活用可能

🏗️ アーキテクチャ図

Go 1.26 ランタイムは Google Cloud の 4 つのサーバーレス/コンピューティングサービスで利用可能です。いずれも Google Cloud Buildpacks を基盤とし、Ubuntu 24 ベースイメージ上で動作します。

⚙️ サービスアップデートの詳細

🔹 App Engine standard

app.yaml の runtime に go126 を指定

✓パッチリビジョンを自動更新
✓F1 インスタンスクラス + 自動スケーリング
🔹 App Engine flexible

runtime_config で runtime_version: "1.26" + operating_system: "ubuntu24"

✓Buildpacks ベースのビルド
✓go.mod による依存関係管理
🔹 Cloud Run (サービス)

--base-image フラグで go126 を指定

✓ソースからの自動ビルド
✓自動セキュリティアップデート
🔹 Cloud Run functions

ランタイム ID go126 を使用

✓ベースイメージ: google-24/go126
✓google-24-full/go126 も選択可能

🛠️ 設定方法

💡 前提条件: Google Cloud SDK (gcloud CLI) v420.0.0 以降、Go 1.26 のローカルインストール、go.mod ファイルによる依存関係管理が必要です。

App Engine standard environment

YAML 
# app.yaml
runtime: go126
Bash 
gcloud app deploy

App Engine flexible environment

YAML 
# app.yaml
runtime: go
env: flex
runtime_config:
  operating_system: "ubuntu24"
  runtime_version: "1.26"
Bash 
gcloud app deploy

Cloud Run (サービス)

Bash 
gcloud run deploy SERVICE_NAME \
  --source . \
  --base-image go126

Cloud Run functions

Bash 
gcloud run deploy FUNCTION_NAME \
  --source . \
  --function FUNCTION_ENTRYPOINT \
  --base-image go126

✅ メリット

💼 ビジネス面
✓本番環境での安定運用 - GA により SLA が適用され、エンタープライズワークロードでの採用リスクが低減
✓統一的な開発体験 - 4 つのサービスすべてで同一の Go バージョンを利用でき、チーム間の技術スタック統一が容易
🔧 技術面
✓最新言語機能の活用 - Go 1.26 の新機能やパフォーマンス改善を本番で利用可能
✓自動セキュリティアップデート - Cloud Run で OS レベルのセキュリティパッチが自動適用
✓自動パッチ更新 - App Engine でパッチリビジョンが自動更新 (1.26.x 範囲内)

⚠️ デメリット・制約事項

制限事項
!App Engine ではメジャーバージョンの自動更新は行われない。将来のバージョンアップは手動でのランタイム指定変更が必要
!Cloud Run functions では Go の依存関係は go.mod ファイルまたは vendor ディレクトリで提供する必要がある
💡 考慮すべき点
→Go 1.26 への移行時は、Go リリースノートで破壊的変更がないか確認すること
→Go のリリースポリシーにより、各メジャーリリースは 2 つの新しいメジャーリリースが公開されるまでサポートされる

🎯 ユースケース

🔗 ユースケース 1: マイクロサービスアーキテクチャの統一

シナリオ: 複数の Go マイクロサービスを App Engine standard と Cloud Run で運用しているチームが、すべてのサービスを Go 1.26 に統一する。

YAML 
# App Engine standard の app.yaml
runtime: go126
Bash 
# Cloud Run サービスのデプロイ
gcloud run deploy my-service --source . --base-image go126

効果: サービス間で Go のバージョンが統一され、共通ライブラリの互換性管理が容易になる。

⚡ ユースケース 2: イベント駆動型処理の構築

シナリオ: Cloud Run functions で Go 1.26 を使用し、Cloud Storage や Pub/Sub からのイベントを処理するサーバーレスアプリケーションを構築する。

Bash 
gcloud run deploy my-function \
  --source . \
  --function ProcessEvent \
  --base-image go126

効果: Go 1.26 の最新機能を活用したイベント処理ロジックを、インフラ管理なしで本番運用できる。
