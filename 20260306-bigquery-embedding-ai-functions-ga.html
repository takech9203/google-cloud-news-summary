📝 概要

BigQuery の AI.GENERATE_EMBEDDING 関数と AI.EMBED 関数が一般提供 (GA) になりました。SQL クエリだけで Vertex AI のエンベディングモデルを呼び出し、セマンティック検索や RAG パイプラインを BigQuery 内で完結できるようになります。

⚡ 2 つの AI 関数の比較

🔧 AI.GENERATE_EMBEDDING テーブル値関数

→リモートモデルを参照してエンベディングを生成
→gemini-embedding-001, text-embedding-005, multimodalembedding@001 等に対応
→Model Garden / Hugging Face のオープンモデルにも対応
→task_type や output_dimensionality で柔軟に制御

✨ AI.EMBED スカラー関数

→リモートモデルの事前作成が不要
→endpoint パラメータで直接モデルを指定
→テキスト + 画像 (ObjectRefRuntime) のエンベディングに対応
→STRUCT 型 (result, status) で結果を返却

🔧 設定方法

💡 前提条件: BigQuery API、BigQuery Connection API、Vertex AI API が有効で、課金が有効な Google Cloud プロジェクト、BigQuery 接続 (Cloud リソース接続) が必要です。

ステップ 1: リモートモデルの作成 (AI.GENERATE_EMBEDDING の場合)

SQL リモートモデル作成 
-- gemini-embedding-001 を使用するリモートモデルの作成
CREATE OR REPLACE MODEL `mydataset.embedding_model`
  REMOTE WITH CONNECTION `us.my_connection`
  OPTIONS (ENDPOINT = 'gemini-embedding-001');

ステップ 2: AI.GENERATE_EMBEDDING によるエンベディング生成

SQL テーブル値関数によるエンベディング生成 
-- テーブルのテキストデータからエンベディングを生成
SELECT *
FROM AI.GENERATE_EMBEDDING(
  MODEL `mydataset.embedding_model`,
  (SELECT 'エンベディングしたいテキスト' AS content),
  STRUCT('RETRIEVAL_DOCUMENT' AS task_type, 256 AS output_dimensionality)
);

ステップ 3: AI.EMBED による直接エンベディング生成

SQL リモートモデル不要の直接呼び出し 
-- リモートモデル不要で直接エンベディングを生成
SELECT
  AI.EMBED(
    'エンベディングしたいテキスト',
    endpoint => 'gemini-embedding-001',
    task_type => 'RETRIEVAL_DOCUMENT',
    model_params => JSON '{"outputDimensionality": 256}'
  ) AS embedding;

ステップ 4: VECTOR_SEARCH との組み合わせ

SQL エンベディング保存 + セマンティック検索 
-- エンベディングをテーブルに保存
CREATE OR REPLACE TABLE mydataset.text_embeddings AS
SELECT
  title, body,
  AI.EMBED(body, endpoint => 'gemini-embedding-001').result AS embedding
FROM `mydataset.articles`;

-- セマンティック検索の実行
SELECT base.title, base.body
FROM VECTOR_SEARCH(
  TABLE mydataset.text_embeddings,
  'embedding',
  (SELECT AI.EMBED('検索クエリ', endpoint => 'gemini-embedding-001').result),
  top_k => 5
);

⚠️ 制限事項 / 注意点

制限事項
✗モデルと入力テーブルは同一リージョンに存在する必要がある
✗AI.EMBED の title パラメータは task_type が RETRIEVAL_DOCUMENT の場合のみ使用可能
✗オープンモデル使用時は Vertex AI へのデプロイが必要で、マシン時間単位の課金が発生
💡 考慮すべき点
→大量データのエンベディング生成時は Vertex AI 側の課金に注意
→48 時間以上のクエリジョブでは connection_id によるサービスアカウント認証を推奨
→大規模バッチ処理では Vertex AI のクォータ制限を事前に確認

🎯 ユースケース

🔎 ユースケース 1: エンタープライズセマンティック検索

社内ナレッジベースの大量ドキュメントを BigQuery に格納し、自然言語での類似検索を実現。

SQL ナレッジベース検索の実装例 
-- ドキュメントのエンベディングを生成して保存
CREATE OR REPLACE TABLE knowledge_base.doc_embeddings AS
SELECT
  doc_id, title, content,
  AI.EMBED(content, endpoint => 'gemini-embedding-001',
           task_type => 'RETRIEVAL_DOCUMENT').result AS embedding
FROM `knowledge_base.documents`;

-- ユーザーの質問に関連するドキュメントを検索
SELECT base.title, base.content, distance
FROM VECTOR_SEARCH(
  TABLE knowledge_base.doc_embeddings, 'embedding',
  (SELECT AI.EMBED('BigQuery でコスト最適化するには?',
                    endpoint => 'gemini-embedding-001',
                    task_type => 'RETRIEVAL_QUERY').result),
  top_k => 10
);
効果: 外部ベクトルデータベースなしで、BigQuery 内にエンドツーエンドのセマンティック検索パイプラインを構築可能。

🤖 ユースケース 2: RAG (Retrieval-Augmented Generation) パイプライン

Gemini モデルによるテキスト生成の精度を向上させるため、関連コンテキストをエンベディングベースで検索し、プロンプトに含める。

効果: BigQuery に蓄積された構造化 / 非構造化データを活用し、AI.EMBED + VECTOR_SEARCH + AI.GENERATE を組み合わせた SQL ベースの RAG パイプラインを構築可能。

🌍 利用可能リージョン

AI.EMBED 関数は Vertex AI エンベディングモデルをサポートする全リージョンに加え、US および EU のマルチリージョンで利用可能です。AI.GENERATE_EMBEDDING はリモートモデルとテーブルが同一リージョンにある必要があります。

📌 まとめ

BigQuery のエンベディング生成 AI 関数が GA となり、SQL だけで高品質なエンベディングを生成し、セマンティック検索や RAG パイプラインを構築できる環境が本番対応として整いました。特に AI.EMBED 関数はリモートモデル不要で直接呼び出しが可能であり、開発者の生産性を大幅に向上させます。BigQuery をベクトルデータベースとしても活用したい場合は、まず gemini-embedding-001 + VECTOR_SEARCH の組み合わせから試すことを推奨します。
