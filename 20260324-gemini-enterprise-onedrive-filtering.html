🔒 Gemini Enterprise:
Microsoft OneDrive データストアのフィルタリング強化

Gemini Enterprise Public Preview OneDrive Connector Data Filtering
2026-03-24

📄 概要

Gemini Enterprise において、Microsoft OneDrive データストアに対するフィルタリング機能が強化された。 Google Cloud コンソールまたは API を使用してフィルタを構成できるようになり、 OneDrive パスの包含・除外によってアシスタントがアクセスできるコンテンツを正確に定義できる。

これにより、組織内の機密情報へのアクセスを制御し、セキュリティやコンプライアンスの観点からデータアクセスを細かく管理することが可能になる。 Microsoft OneDrive を外部データソースとして Gemini Enterprise に接続している企業ユーザー、特に大量のファイルが保存された OneDrive 環境の管理者にとって有用な機能である。

🔄 Before / After

❌ Before

  • OneDrive データストアのフィルタ設定は API 経由でのみ可能
  • REST API の知識が必要であり、管理者にとって設定のハードルが高い
  • フィルタ変更時に既存フィルタを含むすべてのフィルタを再指定する必要がある

✅ After

  • Google Cloud コンソールから GUI でフィルタを構成可能に
  • API に加えてコンソールでもフィルタの作成・更新・確認が可能
  • OneDrive パスの包含・除外を柔軟に定義し、直感的に管理可能

🛠️ アーキテクチャ図

管理者が Google Cloud コンソールまたは API を通じてフィルタを設定し、フィルタエンジンが OneDrive から取得するデータの範囲を制御する。エンドユーザーが Gemini Enterprise アシスタントに問い合わせると、フィルタが適用された範囲のデータのみが検索・参照される。

⚡ 主要機能

🖥️ 1. Google Cloud コンソールでのフィルタ構成
  • GUI ベースでフィルタの作成・更新・確認が可能
  • API の知識がなくてもフィルタを管理できる
🔍 2. OneDrive パスベースのフィルタリング
  • structured_search_filter パラメータを使用してフィルタを定義
  • キーと値のペアで OneDrive フィールドに基づくフィルタリングが可能
  • 特定の OneDrive パスを包含または除外してデータアクセス範囲を制御
🔄 3. 既存データストアへのフィルタ追加・更新
  • updateDataConnector メソッドで既存データストアのフィルタを更新可能
  • dataConnector.get メソッドでフィルタの適用状況を確認可能

🛠️ 設定方法

前提条件

  1. Gemini Enterprise のライセンスが有効であること
  2. ID プロバイダの構成が完了していること
  3. Discovery Engine Editor ロール (roles/discoveryengine.editor) が付与されていること
  4. Microsoft Entra ID で OAuth 2.0 アプリケーションとして登録済み (Client ID, Client Secret, Tenant ID が必要)
  5. Microsoft API パーミッションが適切に構成されていること

Step 1: フィルタ付きデータストアの作成

Bash setUpDataConnector API 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -H "X-Goog-User-Project: PROJECT_ID" \
  "https://ENDPOINT_LOCATION-discoveryengine.googleapis.com/v1alpha/projects/PROJECT_ID/locations/LOCATION:setUpDataConnector" \
  -d '{
    "collectionId": "COLLECTION_ID",
    "collectionDisplayName": "COLLECTION_DISPLAY_NAME",
    "dataConnector": {
      "dataSource": "onedrive_federated_search",
      "params": {
        "client_id": "CLIENT_ID",
        "client_secret": "CLIENT_SECRET",
        "tenant_id": "TENANT_ID",
        "structured_search_filter": {
          "FILTER_KEY": ["FILTER_VALUE1", "FILTER_VALUE2"]
        }
      },
      "entities": [{"entityName": "file"}],
      "refreshInterval": "7200s",
      "connectorType": "THIRD_PARTY_FEDERATED",
      "connectorModes": ["FEDERATED"]
    }
  }'

ENDPOINT_LOCATION には us、eu、または global を指定する。

Step 2: フィルタの確認

Bash dataConnector.get API 
curl -X GET \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "X-Goog-User-Project: PROJECT_ID" \
  "https://ENDPOINT_LOCATION-discoveryengine.googleapis.com/v1alpha/projects/PROJECT_ID/locations/LOCATION/collections/COLLECTION_ID/dataConnector"

レスポンスの structured_search_filter フィールドでフィルタが正しく適用されていることを確認する。

⚖️ メリット・デメリット

✅ ビジネス面のメリット

  • データガバナンスの強化: 機密情報を含む OneDrive パスを除外し、コンプライアンス要件に対応
  • 管理の効率化: GUI で設定できるため、管理者の作業負荷が軽減

✅ 技術面のメリット

  • 柔軟なアクセス制御: パスベースの包含・除外フィルタによるきめ細かい制御
  • API と GUI の両対応: 自動化は API、手動管理は GUI と使い分け可能

⚠️ 制限事項

  • Public Preview: Pre-GA Offerings Terms が適用、サポートが限定的
  • フィルタは OneDrive アクション (フォルダ作成、ファイルコピー等) には適用されない
  • フィルタ更新時は全フィルタの再指定が必要

⚠️ 考慮点

  • ID プロバイダ未構成の場合、データソースアクセス制御が機能しない
  • Microsoft Entra ID の OAuth 2.0 設定が必要で、Microsoft 側管理者との連携が不可欠

🎯 ユースケース

🏢 UC1: 部門別データアクセスの制御

シナリオ: 大企業の IT 管理者が、営業部門の Gemini Enterprise アシスタントに対して、営業部門の OneDrive フォルダのみにアクセスを制限したい。

JSON フィルタ設定例 
{
  "structured_search_filter": {
    "Path": [
      "https://contoso.sharepoint.com/personal/sales_team/Documents/Sales/"
    ]
  }
}
⚡ 効果: アシスタントは営業関連のドキュメントのみを参照し、他部門の機密情報にはアクセスしない。

🔒 UC2: 機密フォルダの除外

シナリオ: 法務部門の契約書や人事部門の個人情報を含むフォルダを、全社向けアシスタントの検索対象から除外したい。

⚡ 効果: コンプライアンス要件を満たしつつ、社内ナレッジの活用を促進できる。

📝 まとめ

Gemini Enterprise の Microsoft OneDrive データストアに対するフィルタリング機能が強化され、Google Cloud コンソールと API の両方からフィルタを構成できるようになった。OneDrive を外部データソースとして利用している組織は、パスベースの包含・除外フィルタを活用することで、アシスタントがアクセスするデータの範囲を適切に制御し、セキュリティとコンプライアンスの要件を満たしつつ、社内ナレッジの活用を推進できる。

💡 推奨: 本機能は Public Preview のため、本番環境への導入時には GA への昇格状況を確認することを推奨する。
