📋 概要

Google SecOps がサポートするデフォルトパーサーの大規模アップデートが実施されました。デフォルトパーサーは、各種セキュリティ製品やクラウドサービスから取り込まれた生ログを Unified Data Model (UDM) 形式に正規化するためのプリビルト構成です。今回のアップデートでは、セキュリティ、クラウド、ネットワーク、エンドポイントなど幅広いカテゴリにわたる多数のパーサーが更新されました。

ステップ 1: パーサーの更新状況を確認

Google SecOps コンソールで Settings > SIEM Settings > Parsers に移動し、Filter から Prebuilt と Active を選択。Update 列に「Pending」と表示されているパーサーが更新対象。

ステップ 2: 更新内容の確認 (任意)

自動更新が有効な場合は自動的に適用されるが、事前に確認したい場合は対象パーサーの Menu > View pending update から差分を確認可能。

ステップ 3: 自動更新の管理

操作パス 
Settings > SIEM Settings > Parsers > [対象パーサー] > Menu
  → Turn on auto updates  (自動更新を有効化)
  → Turn off auto updates (自動更新を無効化)

自動更新を無効にした場合は、手動で Update to latest version を選択して更新を適用。

⚖ メリット / デメリット

✔ メリット

  • セキュリティ可視性の向上: 最新のログフォーマットに対応し、脅威検出の精度が向上。インシデントの早期発見に貢献
  • 運用負荷の軽減: プリビルトパーサーが自動更新されるため、カスタムパーサーの開発・メンテナンスコストを削減
  • UDM マッピング精度向上: 検出ルールや UEBA の精度が向上
  • マルチクラウド対応強化: AWS, Azure, GCP の各種サービスのパーサーが更新され、ログ分析が改善

⚠ デメリット / 制約事項

  • 既に取り込み済みのログには遡及適用されない
  • リージョンへの反映に 1~4 日のタイムラグがある
  • パーサー更新により既存の検出ルールの挙動が変わる可能性がある
  • カスタムパーサーを使用している場合、プリビルトパーサーの更新は適用されない
  • パーサー拡張との互換性を確認する必要がある
⚠ 推奨: 重要な検出ルールに影響がないか、更新前にプレビュー機能で確認することを推奨します。ロールアウト完了後に動作確認を行うことが望ましいです。

🎯 ユースケース

🌐 マルチクラウドセキュリティモニタリング

シナリオ: AWS、Azure、GCP を併用する企業が、全クラウド環境のセキュリティログを Google SecOps に集約して統合分析を行っている。

効果: AWS (CloudTrail, VPC Flow 等)、Azure (AD, Front Door 等)、GCP (Cloud SQL 等) のパーサーが最新化され、マルチクラウド環境全体でのログ正規化精度が向上する。

💻 エンドポイントセキュリティの強化

シナリオ: CrowdStrike や Elastic Defend を導入し、エンドポイントの脅威検出を Google SecOps で一元管理している。

効果: エンドポイントセキュリティ製品のパーサーが更新され、最新の脅威イベントや新しいログフィールドが正しく UDM にマッピングされるようになる。
