💡 概要

BigQuery の Conversational Analytics (会話型分析) に 6 つの主要な機能強化が追加されました。ObjectRef を通じた Cloud Storage 上の非構造化データとの統合、BigQuery ML 関数のサポート、SQL エディタでのクエリ結果との会話機能など、自然言語でのデータ分析がより広範囲かつ高度になりました。

SQL に精通していないユーザーでも、予測分析やマルチモーダル分析などの高度な分析タスクを自然言語で実行できるようになります。

🔄 Before / After 比較

❌ Before

  • 構造化データのみが対象で、Cloud Storage 上の画像や PDF は分析不可
  • 予測・異常検知には BigQuery ML の SQL 構文を直接記述が必要
  • クエリ結果に対するフォローアップ会話が SQL エディタ上で不可
  • パーティション列を活用した自動最適化が行えなかった
  • エージェント実行ジョブの識別・フィルタリング手段がなかった
  • 提案される次の質問をクリック選択できなかった

✅ After

  • ObjectRef で Cloud Storage 上の画像・PDF を直接参照・分析可能に
  • 自然言語プロンプトで BigQuery ML 関数が自動呼び出し
  • SQL エディタでクエリ結果に直接会話を開始しフォローアップ分析可能
  • パーティションプルーニングを自動適用しパフォーマンス・コスト改善
  • ca-bq-job: true ラベルでジョブ識別・フィルタリング可能に
  • 提案された次の質問がクリック可能になり分析フロー加速

⚡ 主要機能 (6 つの強化)

📦 1. ObjectRef サポート

Cloud Storage 上の画像・PDF などの非構造化データを会話型分析から参照・操作。OBJ.MAKE_REF や OBJ.FETCH_METADATA 関数でメタデータと認可情報を管理。

🧠 2. BigQuery ML サポート

AI.FORECAST (予測)、AI.DETECT_ANOMALIES (異常検知)、AI.GENERATE (テキスト生成) を自然言語プロンプトから自動呼び出し。

💬 3. クエリ結果との会話

BigQuery Studio の SQL エディタでクエリ実行後、結果に対して直接フォローアップの質問や追加分析を自然言語で実行可能。

🚀 4. パーティション最適化

エージェントがパーティション情報を認識し、日付範囲を使用したパーティションプルーニングを自動適用。スキャンデータ量削減によるコスト最適化。

🏷️ 5. ジョブラベル付与

エージェント生成ジョブに ca-bq-job: true ラベルが自動付与。ジョブ履歴での識別・フィルタリング・分析が可能に。

🖱️ 6. クリック可能な提案

エージェントが提案する次の質問候補が Google Cloud コンソール上で直接クリック可能になり、分析フローを加速。

🎯 ユースケース

📸 ユースケース 1: 非構造化データを含むマルチモーダル分析

シナリオ: EC サイトの運営チームが、商品テーブル (構造化データ) と商品画像 (Cloud Storage 上の画像) を統合して分析したい。
ユーザー: 「商品画像に基づいて、各カテゴリの商品の見た目の特徴を要約して」
エージェント: ObjectRef で商品画像を参照 → AI.GENERATE で特徴をテキスト生成 → カテゴリごとに集計して回答
✅ 効果: SQL を記述せずに構造化データと非構造化データを組み合わせたマルチモーダル分析を実行可能

📈 ユースケース 2: 自然言語による予測分析と異常検知

シナリオ: データアナリストが、過去の売上データから来月の売上を予測し、直近のデータに異常値がないか確認したい。
ユーザー: 「来月の売上を予測して」 → AI.FORECAST を自動呼び出し
ユーザー: 「直近 3 ヶ月の日次売上に外れ値はある?」 → AI.DETECT_ANOMALIES を自動呼び出し
✅ 効果: BigQuery ML の構文を覚えなくても、自然言語で高度な予測・異常検知分析を実行でき分析サイクルが大幅短縮

💰 ユースケース 3: コスト最適化とジョブ監視

シナリオ: データプラットフォームチームが、Conversational Analytics エージェントの利用状況とコストを監視したい。
SQL エージェント生成ジョブの監視クエリ 
-- エージェント生成ジョブの一覧と消費バイト数を確認
SELECT
  job_id,
  user_email,
  total_bytes_processed,
  creation_time
FROM `region-us`.INFORMATION_SCHEMA.JOBS
WHERE JSON_VALUE(labels, '$.ca-bq-job') = 'true'
ORDER BY creation_time DESC;
✅ 効果: パーティション最適化によるスキャン量削減と、ラベルベースのジョブ追跡によりエージェント利用のコストを定量的に把握・最適化可能

⚠️ 制限事項・注意点

制限事項
  • Conversational Analytics は現在 Preview ステータスであり、限定的なサポートのみ提供
  • グローバルに動作し、使用するリージョンを選択できない
  • ObjectRef を使用するクエリは、ObjectRef 値を含むテーブルと同じプロジェクトで実行する必要がある
  • ObjectRef/ObjectRefRuntime を参照するクエリでは、接続数が 20 を超えてはならない
考慮すべき点
  • 大規模テーブル、データ結合、AI 関数の頻繁な呼び出しでは予期しないコストが発生する可能性
  • Gemini による生成出力は事実と異なる可能性があるため、出力結果の検証が推奨
  • AI.GENERATE 関数の使用には Vertex AI の追加コストが発生

💰 料金情報

Conversational Analytics の利用自体に追加料金は発生しません (Preview 期間中)

ただし、以下のコストが発生します:

  • 📊
    BigQuery コンピュート料金 - データエージェントの作成や会話時に実行されるクエリに対して課金
  • 🤖
    Vertex AI 料金 - AI.GENERATE などの生成 AI 関数を使用する場合に発生
  • 📦
    Cloud Storage 料金 - ObjectRef を使用した非構造化データのアクセスに関連

🌍 利用可能リージョン

Conversational Analytics はグローバルに動作し、使用するリージョンを選択できません。
