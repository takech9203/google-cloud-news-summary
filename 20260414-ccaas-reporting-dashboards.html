📊 CCaaS: Advanced Reporting Dashboards の機能強化とバグ修正

CCaaS CCAI Platform リリース済み
2026-04-14
Contact Center as a Service

☁️ 概要

Google Cloud は、Contact Center as a Service (CCaaS) の Advanced Reporting Dashboards に対する包括的なアップデートをリリースした。 Location フィルターの追加、Queue Performance ダッシュボードの改善、各種ダッシュボードのタイル名称変更、 および複数のバグ修正が含まれている。

コンタクトセンターの管理者やスーパーバイザーがリアルタイムおよび履歴データをより正確かつ効率的に分析できるようにすることが目的。 特に、Location フィルターの追加により複数拠点を運用する組織でのデータ分析が容易になり、 データの正確性に関わる重要なバグも修正されている。

🔄 Before / After 比較

❌ Before (アップデート前)

  • Real-time Channel Performance, Transfers, Queue Interval に Location フィルターがない
  • Queue Performance が Landing Page から直接アクセスできなかった
  • 「Queued Now」「Max Queue Time」の名称がリアルタイム値であることを明示していなかった
  • Performance Overview と CSAT ダッシュボード間で CSAT スコアに不一致
  • キューインタラクションの集計が誤った値を表示

✅ After (アップデート後)

  • 3 つのダッシュボードに Location フィルターを追加、拠点別分析が可能に
  • Queue Performance が Landing Page に追加されアクセスが容易に
  • 「Current Queued Now」「Current Max Queue Time」に名称変更
  • CSAT スコアの不一致が修正、データの信頼性が向上
  • 集計誤り、Virtual Agent チャット表示バグなど複数の問題が修正

🚀 主要機能

📍 1. Location フィルターの追加

以下のダッシュボードに Location フィルターが新たに追加された:

  • Real-time Channel Performance
  • Transfers
  • Queue Interval

複数拠点を運用するコンタクトセンターにおいて、拠点ごとのパフォーマンスを個別に確認可能。

📈 2. Queue Performance ダッシュボードの改善
  • Advanced Reporting Landing Page にダッシュボードへのリンクが追加
  • Support Phone Number フィルターが新たに追加
  • 「Total Inbound Handled」が「Total Queue Answered」に改名
  • 「Total Failed」タイルが新規追加 (エージェントへの接続失敗数を可視化)
  • Queue Summary テーブルから「Total Inbound Calls Handled」カラムを削除し、「Total Queue Interactions」と「Total Queue Answered」カラムを追加
🏷️ 3. Performance Overview ダッシュボードの名称変更

リアルタイム値であることを接頭辞「Current」で明示するための変更:

Queued Now → Current Queued Now
Max Queue Time → Current Max Queue Time
📞 4. Real-time Connected ダッシュボードの改善
  • 「Total Connected Calls」/「Total Connected Chats」タイルが追加
  • 「Avg Current Sentiment Score」タイルが追加 -- 接続中インタラクションの平均センチメントスコアをリアルタイム確認可能
🔤 5. Queue Group Performance - All ダッシュボードの改善
Lang → Language

フィルター名がより直感的になり、ダッシュボード間の一貫性が向上。

🐛 バグ修正

✅ CSAT スコアの不一致修正
Performance Overview ダッシュボードと CSAT ダッシュボード間で CSAT スコアが一致しない問題を修正
✅ キューインタラクション集計の修正
キューインタラクションの合計値が不正確に計算されていた問題を修正
✅ Virtual Agent チャットテーブルの修正
Virtual Agents Chats テーブルで誤ったチャットが表示される問題を修正
✅ Failed Interaction カラムの修正
Failed Interaction カラムに関する表示の問題を修正

⚙️ 設定方法

💡 前提条件: CCAI Platform インスタンスが Advanced Reporting 対応リージョンにデプロイされており、 Advanced Reporting 拡張機能が有効化されていること。
1

Advanced Reporting 拡張機能の有効化

  1. Google Cloud コンソールでプロジェクトを選択
  2. ナビゲーションメニューから CCAI Platform をクリック
  3. 対象のインスタンスをクリック → Edit → Configure extensions
  4. Extensions で Advanced reporting チェックボックスを選択し、Save
2

ダッシュボードへのアクセス

  1. CCAI Platform ポータルで Dashboard → Advanced Reporting をクリック
  2. Landing Page から対象のダッシュボード (Queue Performance など) を選択
  3. 新しく追加された Location フィルターや Support Phone Number フィルターでデータを絞り込む
  4. Update または Reload をクリックして結果を表示

⚖️ メリットと制約事項

✅ ビジネス面のメリット

  • 拠点別パフォーマンス分析の強化: Location フィルターにより拠点ごとの比較・分析が容易に
  • データの信頼性向上: CSAT スコア不一致や集計誤りの修正により、経営判断に使用するデータの正確性が向上
  • オペレーション改善の迅速化: Total Connected 数と Avg Current Sentiment Score の追加で、リアルタイム状況把握が可能に

✅ 技術面のメリット

  • 指標名の明確化でリアルタイム値と履歴値の区別が明確に
  • Queue Performance の Landing Page 追加でアクセス改善
  • 「Lang」→「Language」で フィルター名称の一貫性が向上

⚠️ 制限事項

  • Advanced Reporting は 特定の 7 リージョンでのみ利用可能
  • 有効化するとレガシーダッシュボードは利用不可
  • 日付範囲フィルターは最大 45 日間に制限

⚠️ 考慮すべき点

  • 有効化/無効化はカスタムロールの権限設定に影響する可能性あり
  • 名称変更により、既存のレポートやアラートで旧名称を参照している場合は更新が必要
  • Consumer Phone Number の表示は法的補遺の署名が必要

💡 ユースケース

🏢 ユースケース 1: 複数拠点のコンタクトセンター運用管理
シナリオ: 東京、大阪、福岡の 3 拠点でコンタクトセンターを運用する企業のオペレーション管理者が、拠点ごとのチャネルパフォーマンスを比較分析したい。
効果: Real-time Channel Performance ダッシュボードの新しい Location フィルターを使用し、各拠点のリアルタイムパフォーマンスを個別に確認。リソース配分の最適化やスタッフィング調整を迅速に行える。
📈 ユースケース 2: キューパフォーマンスの詳細分析
シナリオ: コンタクトセンターのスーパーバイザーが、特定のキューにおける応答率と失敗率を把握し、SLA 目標の達成状況を評価したい。
効果: Queue Performance ダッシュボードの「Total Queue Interactions」「Total Queue Answered」「Total Failed」タイルで、キュー到達数、応答数、失敗数を一目で把握。ボトルネックの特定とルーティングルールの改善が容易に。
😊 ユースケース 3: リアルタイムの顧客感情モニタリング
シナリオ: カスタマーサクセスチームのリーダーが、現在進行中の通話やチャットの顧客感情をリアルタイムに把握し、問題のあるインタラクションに早期介入したい。
効果: Real-time Connected ダッシュボードの「Avg Current Sentiment Score」タイルにより、接続中インタラクション全体の感情傾向をリアルタイムに確認。スコアの低下を検知した場合、スーパーバイザーがバージやエスカレーションの対応を取ることが可能に。

📝 まとめ

今回の CCaaS Advanced Reporting Dashboards のアップデートは、Location フィルターの追加や タイル名称の明確化、Queue Performance ダッシュボードの大幅な改善など、 コンタクトセンターの運用可視性を向上させる多数の変更を含んでいる。

特に CSAT スコアの不一致やキューインタラクション集計の誤りといったデータ正確性に関わるバグ修正は、 データ駆動型のオペレーション改善を行う組織にとって重要な修正である。

⚠️ アクション推奨: Advanced Reporting を利用中のコンタクトセンター管理者は、名称変更されたタイルやカラムに基づく既存のレポートやアラートの確認を推奨する。
