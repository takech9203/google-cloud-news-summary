📅 2026-04-20

📋 概要

Gemini Enterprise に2 つのエージェント関連機能が追加されました。1 つ目は、エンドユーザーが Agent Gallery から Cloud Marketplace の AI エージェントへのアクセスをリクエストできる機能です。管理者はエージェントの可視性設定や購入リクエストの管理を行えます。2 つ目は、Vertex AI Agent Engine 上の ADK エージェント登録が GA (一般提供) となり、クロスプロジェクト登録にも対応したことです。

これにより、Gemini Enterprise はエージェントエコシステムの統合ハブとしての位置付けを大幅に強化しました。サードパーティエージェントの導入経路と自社 ADK エージェントの全社展開ワークフローが本番環境に対応しています。

🔄 Before / After 比較

❌ アップデート前

  • エンドユーザーは Marketplace エージェントを Agent Gallery から直接リクエストできず、管理者に個別に依頼が必要
  • Marketplace エージェントの可視性制御が限定的で、適切な情報提供が困難
  • ADK エージェント登録は Preview 段階で、本番ワークロードに制約あり
  • クロスプロジェクト ADK エージェント登録の手順が不明確

✅ アップデート後

  • Agent Gallery の Marketplace セクションから直接アクセスリクエスト可能、承認後は「From your organization」に自動表示
  • 4 段階の可視性オプションで管理者が細かく制御可能
  • ADK エージェント登録が GA に昇格、SLA ベースの本番運用が可能
  • クロスプロジェクト登録手順が明確化、異なるプロジェクト間のエージェント共有が容易に

⚡ 主要機能

🏪 1. Cloud Marketplace エージェントのアクセスリクエスト

→エンドユーザーは Agent Gallery の「Marketplace」セクションからエージェントを選択し、「Request access」をクリックするだけでリクエスト可能
→承認されるとベルアイコンで通知が届き、「From your organization」セクションにエージェントが表示
→承認済みエージェントは会話中に @agent_name でメンションして利用可能
→Marketplace エージェントは Agent2Agent (A2A) プロトコルを使用して通信

⚙️ 2. Marketplace エージェントの可視性管理 (管理者向け)

→Google Cloud コンソールから Marketplace エージェントの表示範囲を 4 段階で設定可能
→「アクセス可能なエージェントのみ」「統合済みのみ」「調達済み (デフォルト)」「すべて表示」
→調達担当者の連絡先を設定し、未調達エージェントへのリクエスト時に通知を受け取れる

🧩 3. ADK エージェントの Gemini Enterprise 登録 (GA)

→Vertex AI Agent Engine 上にデプロイされた ADK エージェントを Gemini Enterprise に登録し、Agent Gallery から利用可能に
→VPC Service Controls (VPC-SC) 準拠のセキュアな通信を実現
→エージェントはユーザーのメールアドレスを受け取り、パーソナライズされた応答を提供可能
→Google Cloud コンソールまたは REST API からの登録に対応

🌐 4. クロスプロジェクト ADK エージェントサポート

→Gemini Enterprise アプリとは異なる Google Cloud プロジェクトにホストされた ADK エージェントも登録可能
→Google Cloud のセキュアなプライベートネットワーキングを使用
→サービスエージェントに Discovery Engine Service Agent ロールを付与することで実現

🛠️ 設定方法

前提条件

✅Gemini Enterprise のサブスクリプション (Standard、Plus、または Frontline エディション) が有効
✅Gemini Enterprise Admin ロール (discoveryengine.agentspaceAdmin) が付与済み
✅Discovery Engine API が有効化済み
✅既存の Gemini Enterprise アプリが作成済み

ステップ 1: Marketplace エージェントの可視性設定 (管理者)

1.Google Cloud コンソールで Gemini Enterprise ページに移動
2.設定対象のアプリ名をクリック
3.Agents をクリック → 設定アイコンから「Agent settings & config」パネルを開く
4.Marketplace visibility セクションで可視性オプションを選択
5.必要に応じて Procurement contacts に連絡先メールアドレスを追加 → Done

ステップ 2: ADK エージェントの登録 (管理者)

1.Google Cloud コンソールで Gemini Enterprise ページに移動
2.アプリ名 → Agents → Add Agents → Agent Engine エージェントを選択
3.表示名、説明、Agent Engine のリソースパスを入力
4.リソースパス形式: projects/PROJECT_ID/locations/LOCATION/reasoningEngines/AGENT_ENGINE_ID
5.Save をクリック

技術面

🔒 VPC-SC 準拠セキュリティ

ADK エージェントとの通信は VPC Service Controls に準拠し、エンタープライズグレードのセキュリティ要件を満たす

🔗 クロスプロジェクト対応

エージェント開発プロジェクトと Gemini Enterprise プロジェクトを分離でき、開発と運用の責任分界が明確に

🌐 A2A プロトコル標準化

Marketplace エージェントは A2A オープンプロトコルを使用し、異なるフレームワークやベンダー間でシームレスな通信を実現

✅ GA レベルの信頼性

ADK エージェント登録が GA となり、SLA に基づいた本番環境での安定運用が保証される

⚠️ デメリット・制約事項

制限事項:
  • Model Armor はコンソール設定が Marketplace / ADK エージェントに自動適用されない。エージェントコード内で REST API による設定が必要
  • Gemini Enterprise Frontline エディションでは、ユーザーは管理者がプロビジョニングしたエージェントのみアクセス可能 (Agent Gallery のフル機能は利用不可)
  • Gemini Enterprise Business エディションでは Agent Marketplace へのアクセスが利用不可
  • ADK エージェントのロケーション互換性制約があり、Gemini Enterprise アプリのロケーションと Agent Engine のリージョンが一致する必要あり
⚠️ 考慮すべき点:
  • Marketplace エージェントは A2A プロトコルを使用するため、ベンダーによる Agent Card の維持管理が必要。ベンダーのサポート体制を事前に確認
  • クロスプロジェクト ADK エージェント使用時は、サービスエージェントへの適切な IAM ロール付与が必要。セキュリティポリシーとの整合性を確認
  • Marketplace エージェントの料金モデルはエージェントごとに異なる (無料 / サブスクリプション / 従量課金 / 複合型)。コスト管理に注意

💡 ユースケース

🏢 ユースケース 1: サードパーティ AI エージェントの組織導入

シナリオ: 大規模組織で、営業チームが CRM データ分析エージェントを、カスタマーサポートチームがチケット管理エージェントを必要としている。

効果: 各チームのユーザーが Agent Gallery の Marketplace セクションから必要なエージェントを検索しリクエスト。管理者が承認するだけで、ユーザーは Gemini Enterprise の統一インターフェースから利用開始。個別のツール導入や設定が不要になり、エージェント活用までのリードタイムが大幅に短縮。

🧩 ユースケース 2: 社内 ADK エージェントの全社展開

シナリオ: AI 開発チームが Vertex AI Agent Engine 上に ADK で社内ナレッジ検索エージェントを構築。開発は専用プロジェクトで行っているが、全社の Gemini Enterprise ユーザーに提供したい。

効果: クロスプロジェクト ADK エージェント登録機能を使用し、開発プロジェクトのエージェントを Gemini Enterprise に登録。ユーザーは Agent Gallery の「From your organization」セクションから選択するだけで利用可能。開発環境と本番環境を分離しながら全社展開を実現。

🤖 ユースケース 3: マルチエージェント連携による業務自動化

シナリオ: 財務レポート作成プロセスで、データ収集 (Marketplace エージェント)、分析 (自社 ADK エージェント)、レポート生成 (Gemini Enterprise 組み込み機能) を連携させたい。

効果: Marketplace からデータ収集エージェントを導入し、自社開発の分析エージェントと組み合わせ。A2A プロトコルを通じたマルチエージェント連携により、Gemini Enterprise がオーケストレーション層として機能し、複雑な業務プロセスの自動化を実現。

ただし、Marketplace エージェント自体の料金は個別のエージェントごとに異なります。

料金モデル 説明
Free Google Cloud リソースの利用料金のみ

🔗 関連サービス・機能

→Vertex AI Agent Engine: ADK エージェントのホスティングおよび実行環境。デプロイ、管理、スケーリングを提供するフルマネージドサービス
→Agent Development Kit (ADK): Google 提供のオープンソース エージェント開発フレームワーク。Gemini モデルに最適化、モデル非依存で設計
→Google Cloud Marketplace: AI エージェントの調達・課金プラットフォーム。A2A プロトコル対応エージェントを提供
→Agent2Agent (A2A) プロトコル: エージェント間通信のオープンスタンダード。Marketplace エージェントが使用
→Model Armor: エージェントのセキュリティ保護機能。ADK / Marketplace エージェントでは REST API を通じた設定が必要
→Discovery Engine API: Gemini Enterprise のバックエンド API。エージェントの登録・管理に使用
