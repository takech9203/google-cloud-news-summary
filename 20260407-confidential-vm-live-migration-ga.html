🔒 Confidential VM: C3D マシンタイプでのライブマイグレーションが GA

Confidential VM GA Security
2026-04-07
Compute Engine

📋 概要

Google Cloud は、Confidential VM インスタンスにおけるライブマイグレーション機能を C3D マシンタイプ に拡張し、一般提供 (GA) を開始しました。 AMD EPYC Genoa プロセッサ 搭載の C3D マシンタイプ、AMD SEV Confidential Computing テクノロジー、およびライブマイグレーション対応の OS イメージという構成条件を満たす Confidential VM インスタンスで、ホストメンテナンスイベント時にダウンタイムなしでの VM 移行が可能になりました。

従来、Confidential VM のライブマイグレーションは N2D マシンタイプ (AMD EPYC Milan) 上の AMD SEV 構成のみでサポートされていましたが、今回のアップデートでより高性能なハードウェア上でも機密コンピューティングワークロードの可用性が大幅に向上します。

💡 対象ユーザー: 金融機関、医療機関、政府機関など、厳格なセキュリティ要件と高い可用性の両方を必要とするワークロードを運用する組織にとって特に重要なアップデートです。

🔄 Before / After 比較

❌ Before (アップデート前)

  • ライブマイグレーションは N2D (AMD EPYC Milan) + AMD SEV のみ
  • C3D の Confidential VM は onHostMaintenance を TERMINATE に設定する必要あり
  • メンテナンスイベント時に VM が停止・再起動
  • C3D の高性能と機密コンピューティング + ライブマイグレーションを両立不可

✅ After (アップデート後)

  • C3D (AMD EPYC Genoa) + AMD SEV でもライブマイグレーション対応
  • onHostMaintenance を MIGRATE に設定可能
  • メンテナンスイベント時も VM はシームレスに別ホストへ移行
  • 高性能 + 機密コンピューティング + 高可用性を同時に実現

⚡ 主要機能

🖥️ C3D マシンタイプでのライブマイグレーション

→AMD EPYC Genoa プロセッサ搭載の C3D マシンタイプで Confidential VM のライブマイグレーションがサポート
→ホストメンテナンスイベント中も VM のダウンタイムが発生しない
→メモリ暗号化を維持したまま、安全にホスト間の移行が実行される

🔐 AMD SEV テクノロジーとの統合

→AMD Secure Encrypted Virtualization (SEV) によるハードウェアベースのメモリ暗号化が維持される
→AMD Secure Processor による暗号鍵の生成・管理はハードウェア内に閉じられ、ハイパーバイザーからアクセス不可
→Google の vTPM によるブートタイムアテステーションも引き続き利用可能

💿 対応 OS イメージ

→ライブマイグレーション対応の OS イメージが必要
→SEV_LIVE_MIGRATABLE_V2 タグが付いた OS イメージが対象
→gcloud compute images list --filter="guestOsFeatures[].type:(SEV_LIVE_MIGRATABLE_V2)" で対応イメージを確認可能

🛠️ 設定方法

前提条件: Google Cloud プロジェクトが作成済み / Compute Engine API が有効化済み / C3D マシンタイプが利用可能なゾーンであること

ステップ 1: 対応 OS イメージの確認

Bash ライブマイグレーション対応 OS イメージの一覧表示 
# ライブマイグレーション対応の OS イメージを一覧表示
gcloud compute images list \
  --filter="guestOsFeatures[].type:(SEV_LIVE_MIGRATABLE_V2)"

ステップ 2: Confidential VM インスタンスの作成

Bash C3D + AMD SEV + ライブマイグレーション対応の VM 作成 
# C3D マシンタイプでライブマイグレーション対応の Confidential VM を作成
gcloud compute instances create INSTANCE_NAME \
  --zone=ZONE \
  --machine-type=c3d-standard-4 \
  --confidential-compute-type=SEV \
  --min-cpu-platform="AMD Genoa" \
  --maintenance-policy=MIGRATE \
  --image-family=IMAGE_FAMILY \
  --image-project=IMAGE_PROJECT \
  --network-interface=nic-type=GVNIC

ステップ 3: 設定の確認

Bash インスタンスの Confidential VM 設定を確認 
# インスタンスの設定を確認
gcloud compute instances describe INSTANCE_NAME \
  --zone=ZONE \
  --format="yaml(confidentialInstanceConfig,scheduling)"
✅ 確認ポイント: scheduling.onHostMaintenance が MIGRATE に設定されていることを確認してください。

✨ メリット

💼 ビジネス面

✓可用性の向上 - ホストメンテナンスによるダウンタイムが排除され、SLA の向上に貢献
✓運用コストの削減 - メンテナンスウィンドウの計画や、ワークロード再開の手動対応が不要に
✓コンプライアンス対応の容易化 - 機密データを扱うワークロードの可用性を維持しつつ、セキュリティ要件を充足

⚙️ 技術面

✓シームレスなメンテナンス - VM の停止・再起動を伴わず、アプリケーション状態やネットワーク接続が維持
✓高性能ハードウェアの活用 - C3D (AMD EPYC Genoa) の高性能を機密コンピューティングで活用
✓暗号化の継続性 - ライブマイグレーション中も AMD SEV によるメモリ暗号化が維持

⚠️ 制約事項・注意点

⚠️C3D マシンタイプの AMD SEV 構成のみが対象。AMD SEV-SNP や Intel TDX ではライブマイグレーション不可
⚠️SEV_LIVE_MIGRATABLE_V2 対応 OS イメージが必要。すべての OS イメージが対応しているわけではない
⚠️C3D で 255 vCPU を超える VM およびベアメタルインスタンスは非サポート
⚠️Hyperdisk Balanced および Hyperdisk Throughput は非サポート
⚠️非 Confidential VM と比較してネットワーク帯域幅が低下する場合がある (Tier_1 ネットワーキング有効時でも同様)
⚠️既存の Confidential VM をライブマイグレーション対応に変換することは不可。新規作成が必要
🔍 考慮事項:
  • ライブマイグレーション中は一時的にパフォーマンスが低下する可能性がある
  • rhel-8-4-sap-ha イメージは C3D の 8 vCPU 以上の構成では AMD SEV と互換性がない

🎯 ユースケース

🏦 ユースケース 1: 金融機関の取引処理システム

シナリオ: 金融機関が、顧客の取引データを処理するワークロードを Confidential VM 上で稼働。24/7 の可用性が求められ、ホストメンテナンスによるサービス中断は許容できない。
✅ 効果: C3D でのライブマイグレーションにより、メンテナンスイベント時もトランザクション処理が中断されず、AMD SEV による暗号化でデータの機密性も維持。

🏥 ユースケース 2: 医療データ分析プラットフォーム

シナリオ: 医療機関が、患者の個人健康情報 (PHI) を分析するプラットフォームを Confidential VM で運用。HIPAA 準拠が必要で、データの機密性と処理の継続性の両方が要求される。
✅ 効果: ライブマイグレーションによりメンテナンス時のダウンタイムが排除。AMD EPYC Genoa の高い処理性能と AMD SEV によるハードウェアレベルの暗号化で、コンプライアンス要件を満たしながら高パフォーマンスなデータ分析が可能。

☁️ ユースケース 3: マルチテナント SaaS プラットフォーム

シナリオ: SaaS プロバイダーが、複数のテナントのデータを機密に保ちながら処理するプラットフォームを構築。テナント間の分離と高い可用性が要件。
✅ 効果: Confidential VM による各テナントデータのハードウェアレベルでの隔離と、ライブマイグレーションによるメンテナンス時の無停止運用を両立。
