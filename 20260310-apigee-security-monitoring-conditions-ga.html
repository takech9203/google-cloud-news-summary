🔒 Apigee Advanced API Security:
セキュリティモニタリング条件の GA

GA (一般提供) Apigee API hub Advanced API Security
2026-03-10

📋 概要

Apigee Advanced API Security の Risk Assessment v2 モニタリング条件が一般提供 (GA) になりました。セキュリティプロファイルにゲートウェイリソースをマッピングし、Cloud Monitoring と連携してセキュリティスコアの時系列追跡やメトリクスベースのアラート通知が本番環境で安定的に利用可能になります。

さらに、マルチゲートウェイプロジェクトでのモニタリング条件サポートと、Apigee Edge Private Cloud (OPDK) ゲートウェイタイプのサポートが追加されました。

🔄 Before / After 比較

❌ Before (アップデート前)

  • モニタリング条件機能はプレビュー段階で、本番利用にサポート制限あり
  • マルチゲートウェイプロジェクトでセキュリティスコアの時系列監視やアラート通知が不可
  • OPDK 環境は API hub のマルチゲートウェイリスクアセスメント対象外

✅ After (アップデート後)

  • Risk Assessment v2 モニタリング条件が GA で本番環境で安定利用可能
  • マルチゲートウェイプロジェクトで Cloud Monitoring 連携によるスコア追跡とアラートが可能
  • OPDK ゲートウェイタイプがサポートされ、オンプレミス環境も統合管理対象に

⚡ 主要機能

🛡️ Risk Assessment v2 モニタリング条件の GA

セキュリティプロファイルにリソースをマッピングするモニタリング条件の作成・管理が一般提供。Cloud Monitoring でセキュリティスコアを時系列追跡し、メトリクスベースのアラートを設定可能。Terraform (google_apigee_security_monitoring_condition) にも対応。

🌐 マルチゲートウェイでのモニタリング条件

API hub 経由の Advanced API Security マルチゲートウェイリスクアセスメント機能に、セキュリティモニタリング条件とアラートが追加。複数の Apigee 組織・環境・ゲートウェイにまたがるセキュリティスコアを一元監視可能。

🏢 OPDK ゲートウェイタイプのサポート

マルチゲートウェイのリスクアセスメントセキュリティプロファイルで Apigee Edge Private Cloud (OPDK) が対象に。サポート対象: OPDK バージョン 4.52.02.xx 以降。オンプレミス環境もクラウドベースの統合セキュリティ管理に組み込み可能。

🔍 Abuse Detection のアップデート

Advanced API Security Abuse Detection の更新版がリリース。API 不正利用の検出精度と機能が改善されています。

前提条件

ステップ 1: Advanced API Security アドオンの有効化

Bash 
# Subscription 組織の場合
gcloud alpha apigee organizations update ORG_NAME \
  --api-security-enabled

API hub でマルチゲートウェイを利用する場合は、API hub の Add-on management ページから有効にします。

ステップ 2: セキュリティプロファイルの作成

Bash 
curl -X POST \
  "https://apigee.googleapis.com/v1/organizations/ORG_NAME/securityProfilesV2" \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Custom security profile for monitoring",
    "profileAssessmentConfigs": {
      "authorization": { "enabled": true },
      "cors": { "enabled": true },
      "threat": { "enabled": true },
      "mediation": { "enabled": true }
    }
  }'

ステップ 3: モニタリング条件の作成

Bash 
curl -X POST \
  "https://apigee.googleapis.com/v1/organizations/ORG_NAME/securityMonitoringConditions" \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  -d '{
    "displayName": "Production API Monitoring",
    "securityProfileV2": "organizations/ORG_NAME/securityProfilesV2/PROFILE_ID",
    "scope": {
      "environmentRefs": [
        "organizations/ORG_NAME/environments/prod"
      ]
    }
  }'
💡 ヒント: 作成後、Cloud Monitoring にメトリクスが発行されるまで最大 5 分かかります。

ステップ 4: Cloud Monitoring アラートの設定

Cloud Monitoring コンソールまたは Apigee UI からアラートポリシーを設定し、セキュリティスコアが閾値を下回った場合に通知を受け取ります。

⚠️ デメリット・制約事項

制限事項
  • セキュリティスコアのデータ反映には最大 6 時間の遅延が発生する場合がある
  • モニタリング条件の変更が Cloud Monitoring メトリクスに反映されるまで最大 5 分かかる
  • VPC Service Controls (VPC-SC) が有効な環境では機能に制限がある可能性がある
考慮すべき点
  • OPDK 環境の統合には API hub プラグインインスタンスの作成と Private Cloud コネクタのインストール・設定が必要
  • Advanced API Security の料金はライセンスタイプにより異なるため、事前に Advanced API Security advisor ツールで費用を確認することを推奨
  • マルチゲートウェイ機能のロールアウトは段階的に行われ、全ゾーンへの展開に 5 営業日以上かかる場合がある

💡 ユースケース

🌐 ユースケース 1: マルチクラウド API セキュリティの一元監視

大規模企業で Apigee X (クラウド)、Apigee hybrid (ハイブリッド)、Apigee Edge Private Cloud (オンプレミス) を併用しており、全環境の API セキュリティを統合的に監視したいシナリオ。

HCL 
# Terraform での構成例
resource "google_apigee_security_monitoring_condition" "all_gateways" {
  org_id              = "my-org"
  display_name        = "All Gateways Monitoring"
  security_profile_v2 = google_apigee_security_profile_v2.default.id
}

効果: オンプレミスを含む全 API ゲートウェイのセキュリティスコアを単一のダッシュボードで監視し、セキュリティ基準の逸脱を即座に検知可能。

📉 ユースケース 2: セキュリティスコアの劣化検知と自動アラート

API プロキシのデプロイ時にセキュリティポリシーの設定漏れが発生し、セキュリティスコアが低下した場合に即座に通知を受け取りたいシナリオ。

効果: セキュリティスコアの閾値ベースのアラートにより、設定ミスや不正トラフィックの増加を早期に検知し、MTTR (平均復旧時間) を短縮できる。
