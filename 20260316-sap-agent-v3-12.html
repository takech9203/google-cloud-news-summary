📄 概要

Google Cloud's Agent for SAP v3.12 が一般提供 (GA) となりました。 SAP ワークロードバリデーション機能の強化と、SAP HANA 向けディスクスナップショットベースリカバリの改善が主な変更点です。

Agent for SAP は、Google Cloud 上で SAP システムを運用するための必須エージェントであり、SAP Host Agent メトリクス収集、Workload Manager によるベストプラクティス評価、SAP HANA バックアップ/リカバリ、プロセスモニタリングなどの機能を提供します。

🎯 対象ユーザー: Google Cloud 上で SAP S/4HANA、SAP BW/4HANA、SAP ECC などを運用する Basis 管理者、インフラエンジニア、Solutions Architect

✓HA (高可用性) クラスタ環境および DR (災害復旧) デプロイメントに対応

✓Linux (RHEL、SLES) および Windows 環境での動作安定性が改善

📝 設定方法

前提条件

→Compute Engine インスタンスまたは Bare Metal Solution サーバーで SAP が稼働していること
→Workload Manager API が有効化されていること
→サービスアカウントに必要な IAM ロールが付与されていること

Step 1: エージェントの更新

Bash 
# RHEL の場合
sudo yum update google-cloud-sap-agent

# SLES 15 の場合
sudo zypper update google-cloud-sap-agent

Step 2: 動作確認

Bash 
# エージェントのステータス確認
sudo systemctl status google-cloud-sap-agent

# バージョン確認
sudo /usr/bin/google_cloud_sap_agent version

Step 3: ワークロードバリデーションの有効化

構成ファイル /etc/google-cloud-sap-agent/configuration.json を編集:

JSON 
{
  "provide_sap_host_agent_metrics": true,
  "log_level": "INFO",
  "log_to_cloud": true,
  "collection_configuration": {
    "collect_workload_validation_metrics": true,
    "collect_process_metrics": true,
    "sap_system_discovery": true
  }
}

Step 4: ディスクスナップショット バックアップの実行

Bash 
# SAP HANA ディスクスナップショット バックアップの作成
sudo /usr/bin/google_cloud_sap_agent hanadiskbackup \
  -port=30015 \
  -sid=HDB \
  -hana-db-user=SYSTEM \
  -password-secret=my-hana-password-secret \
  -project=my-project \
  -disk=/dev/sdb

⚠️ 制限事項・注意点

🚫 制限事項
  • ワークロードバリデーション メトリクス収集は Linux のみサポート (Windows 環境では利用不可)
  • ディスクスナップショット機能はスケールアウト構成のホスト自動フェイルオーバー使用時は利用不可
  • sap/validation/hanasecurity メトリクス収集には SAP HANA DB アクセス設定 (DB ユーザー、Secret Manager) が別途必要
⚠️ 考慮すべき点
  • バージョン 2 のサポートは 2025 年 7 月 31 日に終了済み -- 未移行の場合は早急に v3.x へ移行が必要
  • 更新後、SAP Host Agent メトリクス連携を SAP トランザクション ST06 で確認すること
  • HA クラスタ環境では、プライマリ/スタンバイ両方のノードでエージェント更新が必要

💡 ユースケース

🎯 UC1: SAP HANA HA クラスタのベストプラクティス検証

シナリオ: SAP S/4HANA を Pacemaker ベースの HA クラスタで運用し、構成のベストプラクティス準拠状況を定期的に確認したい。

JSON 
{
  "collection_configuration": {
    "collect_workload_validation_metrics": true,
    "workload_validation_db_metrics_config": {
      "hana_db_user": "MONITOR",
      "sid": "HDB",
      "hana_db_password_secret_name": "hdb-monitor-password",
      "hostname": "localhost",
      "port": "30015"
    }
  }
}

効果: Workload Manager ダッシュボードで Pacemaker、Corosync、SAP HANA の構成をベストプラクティスに基づいて一元的に確認でき、構成ドリフトを早期に検知できます。

🎯 UC2: SAP HANA ディスクスナップショットによる迅速なリカバリ

シナリオ: 大規模な SAP BW/4HANA 環境で、Backint ベースのバックアップに加えてディスクスナップショットによる高速リカバリを実装したい。

効果: ディスクスナップショットは Backint ベースのリストアと比較してリカバリ時間を大幅に短縮できます。特に数 TB 規模のデータボリュームでは、スナップショットからの復元が Backint のストリーミングリストアよりも高速に完了します。

💰 料金情報

Agent for SAP 自体の利用は無料です。エージェントが収集するメトリクスに対して Cloud Monitoring の課金が発生します。

機能 インスタンス数 月額料金 (概算)
SAP Host Agent メトリクス収集 100 $209
Process Monitoring メトリクス収集 100 $202.70
SAP HANA Monitoring メトリクス収集 500 $280.30
Workload Manager 評価メトリクス - 追加費用なし (v3.2 以降)
💰 補足: Backint 使用時は Cloud Storage 料金、ディスクスナップショット使用時は Compute Engine スナップショットストレージ料金が別途発生します。

🌎 利用可能リージョン

✅ 全リージョン対応: Agent for SAP は Google Cloud のすべてのリージョンで利用可能です。SAP HANA が認定されている Compute Engine マシンタイプが利用可能なリージョンであれば、エージェントの全機能を使用できます。Bare Metal Solution サーバーでも利用可能です。
