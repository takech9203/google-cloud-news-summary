Dataproc: Iceberg Vacuum 最適化
デフォルト有効化

Dataproc on Compute Engine Feature Security Fix
2026-03-08

📋 概要

Dataproc on Compute Engine において、3 つの重要なアップデートが同時にリリースされました。最も注目すべきは、Premium ティアクラスターで Optimized Iceberg Vacuum がデフォルトで有効化されたことです。これにより、Spark の MERGE クエリで大量のファイルを変更する際に、シャッフルベースのアプローチでデータファイルの書き換えが行われ、パフォーマンスの向上が期待できます。

また、イメージバージョン 2.1、2.2、2.3 の各系統で新しいサブマイナーイメージバージョンがリリースされ、hms-proxy の v0.0.78 へのアップグレードにより複数の CVE が修正されました。

🔄 Before / After

❌ Before

  • Iceberg MERGE クエリで大量ファイル変更時にパフォーマンスが低下
  • Iceberg Vacuum 最適化には明示的なプロパティ設定が必要
  • hms-proxy に複数のセキュリティ脆弱性 (CVE) が存在

✅ After

  • Premium ティアでシャッフルベースの書き換えにより MERGE パフォーマンスが自動改善
  • 追加設定不要で最適化の恩恵を受けられる
  • CVE-2025-58057 など 5 件の CVE が修正済み

⚡ 主要機能

1. Optimized Iceberg Vacuum のデフォルト有効化

→Premium ティアクラスターの Spark で Optimized Iceberg Vacuum がデフォルト有効に
→シャッフルベースのアプローチでデータファイルを書き換え
→MERGE クエリで大量のファイルを変更する場合にパフォーマンスが向上
→無効化するには spark.dataproc.sql.optimizer.lowShuffleMerge.enabled を false に設定

2. 新サブマイナーイメージバージョン

→2.1 系: 2.1.110 (Debian 11, Rocky 8, Ubuntu 20.04, Ubuntu 20.04 ARM)
→2.2 系: 2.2.78 (Debian 12, Rocky 9, Ubuntu 22.04, Ubuntu 22.04 ARM)
→2.3 系: 2.3.25 (Debian 12, Rocky 9, Ubuntu 22.04, Ubuntu 22.04 ARM, ML Ubuntu 22.04)

3. セキュリティ修正 (CVE)

hms-proxy を v0.0.78 にアップグレードし、以下の CVE を修正:

✓ CVE-2025-58057
✓ CVE-2025-53864
✓ CVE-2025-68161
⚠ CVE-2025-48924 (部分)
✓ CVE-2025-33042

🛠️ 設定方法

前提条件

✓Dataproc on Compute Engine クラスターが Premium ティアで構成されていること
✓Apache Iceberg コンポーネントがインストール済み (イメージバージョン 2.2.47 以降)

ステップ 1: 最新イメージバージョンでクラスター作成

Bash 
# Iceberg コンポーネント付きで最新イメージのクラスターを作成
gcloud dataproc clusters create my-cluster \
    --region=us-central1 \
    --image-version=2.3.25-ubuntu22 \
    --optional-components=ICEBERG
💡 ヒント: Premium ティアクラスターでは Optimized Iceberg Vacuum がデフォルトで有効化されます。追加設定は不要です。

ステップ 2: (任意) 無効化する場合

Bash 
# Optimized Iceberg Vacuum を無効化してクラスターを作成
gcloud dataproc clusters create my-cluster \
    --region=us-central1 \
    --image-version=2.3.25-ubuntu22 \
    --optional-components=ICEBERG \
    --properties=spark:spark.dataproc.sql.optimizer.lowShuffleMerge.enabled=false

ステップ 3: (任意) ジョブ単位での制御

Bash 
# 特定のジョブで無効化
gcloud dataproc jobs submit spark-sql \
    --cluster=my-cluster \
    --region=us-central1 \
    --properties=spark.dataproc.sql.optimizer.lowShuffleMerge.enabled=false \
    -e "MERGE INTO catalog.db.target USING catalog.db.source ON ..."

✅ メリット

💼 ビジネス面

→運用負荷の軽減: 追加設定なしで MERGE パフォーマンスが改善。チューニング工数を削減
→処理時間の短縮: 大量ファイル変更を伴う MERGE クエリの高速化でデータパイプラインが効率化

🔨 技術面

→シャッフルベース最適化: データファイル書き換えの I/O 効率が向上
→セキュリティ強化: hms-proxy の CVE 修正で安全性向上
→幅広い OS サポート: 2.3 系で 5 つの OS バリアント対応

⚠️ 制約事項・注意点

⚠ 制限事項:
  • デフォルト有効化は Premium ティアクラスターのみが対象。Standard ティアでは明示的な設定が必要
  • CVE-2025-48924 は部分修正。完全な修正は今後のリリースで提供予定

考慮すべき点

→既存クラスターには自動適用されない。新しいイメージバージョンでクラスターを再作成する必要がある
→変更対象ファイルが少ない場合、シャッフルベースのアプローチはオーバーヘッドとなる可能性がある。パフォーマンス低下時は lowShuffleMerge.enabled=false で無効化を検討
→イメージバージョン更新前に、既存ワークロードとの互換性を十分にテストすること

💡 ユースケース

📊 データレイクハウスでの大規模 MERGE 処理

シナリオ: 毎日数百万件のレコードが到着するデータレイクハウスで、Iceberg テーブルに対して MERGE (UPSERT) 処理を実行。MERGE の対象ファイル数が数千に及び、処理時間が長くなっていた。

SQL 
-- Premium ティアクラスターでは自動的に最適化が適用される
MERGE INTO catalog.db.customer_master AS target
USING catalog.db.daily_updates AS source
ON target.customer_id = source.customer_id
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT *;

効果: シャッフルベースの書き換えにより、大量ファイル変更時の I/O が最適化され、MERGE 処理時間が短縮されます。

🔒 セキュリティコンプライアンスのためのイメージ更新

シナリオ: セキュリティポリシーにより、既知の CVE が修正されたイメージを速やかに適用する必要がある組織で、Dataproc クラスターを運用している。

効果: 新しいサブマイナーイメージバージョン (2.1.110、2.2.78、2.3.25) に更新することで、CVE-2025-58057 をはじめとする 5 つの CVE に対処でき、セキュリティコンプライアンスを維持できます。

📝 まとめ

今回のアップデートにより、Dataproc Premium ティアクラスターでは Iceberg テーブルの MERGE クエリパフォーマンスが追加設定なしで改善されます。大量のデータファイル変更を伴う MERGE ワークロードを持つユーザーは、最新のサブマイナーイメージバージョンへの更新を推奨します。また、複数の CVE が修正されているため、セキュリティの観点からもイメージバージョンの更新を検討すべきです。
