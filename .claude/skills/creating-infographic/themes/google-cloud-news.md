# Google Cloud News インフォグラフィックテーマ

Google Cloud ニュースレポート専用のインフォグラフィックテーマ。Google Cloud ライトテーマ (#[[file:google-cloud-light.md]]) をベースに、Google Cloud サービスアップデートの情報をシングルカラムレイアウトで視覚的に表現します。

## ベーステーマ

このテーマは Google Cloud ライトテーマを継承します。

**参照**: #[[file:google-cloud-light.md]]

- カラーパレット: Google Cloud 公式ブランドカラー (White 背景, Google Blue アクセントなど)
- フォント: Noto Sans JP
- 基本コンポーネント: カード、ハイライト、フロー図など
- レイアウト: シングルカラム、最大幅 1000px (レスポンシブ対応)

## 適用条件

以下の場合にこのテーマを使用します。

- `reports/` フォルダ内の Google Cloud ニュースレポート (.md) からインフォグラフィックを生成する場合
- ユーザーが「Google Cloud ニュースのインフォグラフィック」「レポートを視覚化」などと言った場合

## ファイル名規則

- **形式**: `{YYYYMMDD}-{slug}.html`
- **slug**: レポートファイル名から日付部分を除いた部分を使用
- **例**: `reports/2026/2026-02-14-cloud-run-update.md` → `infographic/20260214-cloud-run-update.html`
- **出力先**: `infographic/` フォルダ

## 情報収集

インフォグラフィック作成時は、レポートの情報だけでなく、Web から関連情報を積極的に収集して内容を充実させる。

### 必須の情報源

1. **Google Cloud News レポート** (`reports/` フォルダ)
   - 基本情報、概要、主要機能、ユースケース

2. **Google Cloud 公式ドキュメント**
   - Web 検索で `site:cloud.google.com {サービス名}` で詳細情報を検索
   - 公式ドキュメントページを読み込み

3. **Google Cloud Blog**
   - 関連する Blog 記事から追加のユースケースや詳細情報を取得

### 推奨の情報源

4. **GitHub サンプルコード** (直接関連するもののみ)
   - `GoogleCloudPlatform`: https://github.com/GoogleCloudPlatform
   - Web 検索で `site:github.com/GoogleCloudPlatform {サービス名}` などで検索
   - サンプルコードがある場合は、インフォグラフィックに「サンプルコード」セクションを追加

5. **Google Cloud Architecture Center**
   - 関連するアーキテクチャパターンやベストプラクティス

6. **Google Cloud Community / Stack Overflow**
   - よくある質問や実装のヒント

### 情報収集の手順

```
1. レポートを読み込み、サービス名と機能キーワードを抽出
2. Google Cloud ドキュメント検索で詳細情報を取得
3. Web 検索で関連する Blog 記事、サンプルコードを検索
4. 直接関連する情報のみを選別してインフォグラフィックに反映
5. 出典を明記
```

## コンテンツ構成

**重要**: 以下は基本構成であり、収集した情報に基づいて有用なコンテンツを積極的に追加すること。読者にとって価値のある情報であれば、セクションの追加・拡張を躊躇しない。

### 必須セクション (この順序で配置)

1. **タイトルエリア**: サービス名、機能名、リリース日
2. **概要セクション**: 要点 + Before/After 比較
3. **アーキテクチャ図**: フロー図、構成図
4. **主要機能セクション**: 3-5 個の機能カード
5. **ユースケースセクション**: 2-3 個のユースケース
6. **フッター**: 出典 URL

### 推奨セクション (情報がある場合は積極的に追加)

- **統計情報**: 数字で見るアップデート効果 (パフォーマンス向上率、コスト削減率など)
- **サンプルコード**: GitHub へのリンク、コードスニペット
- **リージョン情報**: 利用可能リージョン、展開予定
- **関連リソース**: Blog、ドキュメント、チュートリアルへのリンク
- **料金情報**: コスト比較、料金例
- **制限事項・注意点**: 知っておくべき制約
- **関連サービス**: 組み合わせて使うと効果的なサービス
- **移行ガイド**: 既存環境からの移行手順 (該当する場合)

### コンテンツ追加の判断基準

以下に該当する情報は積極的に追加する。

- 読者の意思決定に役立つ情報
- 実装・導入の参考になる情報
- よくある疑問への回答となる情報
- 他のドキュメントでは得にくい洞察

## Google Cloud News 専用コンポーネント

ベーステーマ (#[[file:google-cloud-light.md]]) に加えて、以下のコンポーネントを使用します。

### Before/After 比較

アップデート前後の変化を視覚的に表現。赤 (Before) と緑 (After) で対比。

### サービスバッジ

Google Blue 背景でサービス名を表示。

### 機能カード

左ボーダー付きのカード形式で機能を列挙。アイコン + タイトル + 説明。

### ユースケース

ブルー系の背景でユースケースを表示。タイトル + 説明。

### 統計カード

グラデーション背景で数値を大きく表示。

## 生成ワークフロー

1. **レポートファイルを読み込む**
2. **Web から関連情報を収集** (ドキュメント、Blog、サンプルコード)
3. **情報を整理・選別**
4. **インフォグラフィックを生成** (Google Cloud ライトテーマベース)
5. **`infographic/{YYYYMMDD}-{slug}.html` に保存**
6. **レポートにリンクを追加** (環境変数 `INFOGRAPHIC_BASE_URL` を使用)

## コードブロックの表示ルール

コードブロックはダーク背景 (`#1E1E1E`) で統一する。ライト背景の中でコードブロックだけがダークになることで、コードが視覚的に区別しやすくなる。

### 必須ルール

- 背景色: `#1E1E1E` を使用
- テキスト色: `#E8E8E8` を使用
- 言語ラベル: `#2D2D2D` 背景 + `#9CDCFE` テキスト
- **`pre` タグのスタイル指定は必ず `pre:not(.mermaid)` を使用する** (Mermaid 図は `<pre class="mermaid">` で定義されるため、`pre` に直接ダーク背景を指定すると Mermaid 図も黒背景になる)

### 言語ラベルの表示

コードブロックの上に言語名ラベルを表示する場合は、以下の HTML 構造を使用する。

```html
<span class="code-label">Bash</span>
<pre><code>gcloud compute instances list --project my-project</code></pre>
```

### highlight.js を使用する場合

highlight.js の CDN を読み込む場合は、ダーク系テーマを使用する。

```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/11.9.0/styles/vs2015.min.css">
```

使用してよいテーマ: `vs2015`, `github-dark`, `atom-one-dark`
使用してはいけないテーマ: `github`, `vs`, `xcode` (ライト系はコードブロックのダーク背景と合わない)

## 注意事項

- インフォグラフィックは単一の HTML ファイルで完結させる (ただし Mermaid.js と highlight.js の CDN 読み込みは許可)
- 情報は簡潔に、視覚的に理解しやすく
- 技術的な詳細よりも、ビジネス価値とユースケースを強調
- 収集した情報の出典を明記
- モバイル対応のレスポンシブデザインを維持
