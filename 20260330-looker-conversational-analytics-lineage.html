🔍 概要

Looker に対して、AI 活用、データガバナンス、コンテンツ管理、データベース接続、UX 刷新にわたる大規模なアップデートが一斉にリリースされました。

Looker は単なる BI ツールから、AI 駆動の分析基盤、統合メタデータカタログ、データガバナンスハブとしての役割を大幅に強化しています。

❌ Before (アップデート前)

  • データエージェントは Looker 内でのみ利用可能
  • BigQuery スキーマ変更の影響把握は手動トレースが必要
  • 未使用コンテンツの特定・クリーンアップは管理者の手動作業
  • AlloyDB 接続はプレビュー段階で本番利用に制約
  • Looker と Dataplex のメタデータは分離

✅ After (アップデート後)

  • Gemini Enterprise に公開し組織全体でセルフサービス分析
  • Dataplex リネージで影響分析を自動化
  • Enhanced Content Cleanup で自動クリーンアップ
  • AlloyDB が GA で本番ワークロード正式サポート
  • Dataplex Universal Catalog で統合データディスカバリ

🤖 AI 機能の強化

🗣️ Conversational Analytics データエージェントの Gemini Enterprise 公開 Preview

Looker 内で作成した Conversational Analytics データエージェントを Gemini Enterprise に公開 できるようになりました。

  • 組織全体のユーザーが Gemini Enterprise インターフェースからデータエージェントにアクセス
  • 自然言語でデータに関する質問が可能
  • LookML セマンティックモデリング層に基づいたガバナンスの効いたセルフサービス BI を展開

💬 ダッシュボード内での Conversational Analytics チャット Preview

ユーザー定義ダッシュボードおよび LookML ダッシュボード内でデータエージェントとチャットが可能になりました。

  • ダッシュボードのコンテキストを維持したまま追加分析
  • データの深掘りや仮説検証がダッシュボード上で完結
💡 ポイント: Conversational Analytics により、SQL やダッシュボード操作の知識がなくても、ガバナンスの効いたデータアクセスが可能になります。

📚 Dataplex Universal Catalog 統合 Preview

Looker (Google Cloud core) が Dataplex Universal Catalog と統合され、統一的なメタデータディスカバリが可能になりました。

  • Looker の Explore、ダッシュボード、ビューなどのメタデータが Dataplex Universal Catalog に登録
  • AI によるナチュラルランゲージ検索を含む統合カタログ内で Looker アセットも検索・発見対象

Dataplex リネージ統合の有効化

前提条件:
  • Looker (Google Cloud core) インスタンスであること
  • Data Lineage API がプロジェクトで有効化されていること
  • 必要な IAM ロール: roles/datalineage.viewer
Bash 
gcloud services enable datalineage.googleapis.com --project=PROJECT_ID

有効化後、BigQuery ジョブの完了から最大 24 時間以内にリネージ情報が反映されます。

AlloyDB for PostgreSQL 接続

設定パス 
Admin > Connections > Add Connection
Dialect: Google Cloud AlloyDB for PostgreSQL

接続ホスト、ポート、データベース名、認証情報を入力し、「Test」で接続確認後「Connect」で保存します。

✅ メリット

ビジネス面

✓組織全体のデータ活用促進: Conversational Analytics エージェントを Gemini Enterprise に公開することで、技術的なスキルを持たないビジネスユーザーもデータドリブンな意思決定が可能
✓変更影響の可視化: Dataplex リネージ統合により、データインフラの変更がビジネスレポートに与える影響を事前に把握し障害を防止
✓コンテンツ管理コストの削減: Enhanced Content Cleanup により、肥大化した Looker 環境のコンテンツ整理を自動化し管理者の運用負荷を削減

技術面

✓エンドツーエンドリネージ: BigQuery テーブルから Looker の View、Explore、ダッシュボード、Look までの完全なデータリネージが自動追跡
✓統合メタデータカタログ: Dataplex Universal Catalog との統合により、Looker アセットを含む組織全体のデータカタログが統一
✓データベース接続の拡充: AlloyDB GA により PostgreSQL ワークロードを Google Cloud のフルマネージドデータベースで運用しつつ Looker で分析可能
✓セキュリティ強化: JDBC パラメータ許可リストにより、データベース接続のセキュリティが向上

⚠️ デメリット・制約事項

制限事項:
  • Dataplex リネージ統合は Looker (Google Cloud core) のみで利用可能。Looker (original) では利用不可
  • Dataplex リネージ情報は BigQuery ジョブ完了から最大 24 時間の反映遅延あり
  • データリネージ情報の保持期間は30 日間に限定
  • Conversational Analytics は FedRAMP High / Medium の認可境界に含まれていない
  • 多くの新機能が Preview ステータスであり、本番環境での利用には注意が必要
考慮すべき点:
  • Conversational Analytics の利用には Gemini in Looker の有効化が必要で、追加コストが発生する可能性
  • データリネージ機能は Dataplex Universal Catalog のプレミアム処理 SKU で課金
  • Enhanced Content Cleanup の自動クリーンアップは、重要コンテンツの誤削除防止のためオプトアウト設定の事前検討を推奨
  • AlloyDB 接続でネットワーク暗号化を行う場合、SSL 設定が別途必要

📝 まとめ

今回の Looker アップデートは、AI 駆動の分析、データガバナンス、コンテンツ管理、データベース接続、UX 改善を横断する大規模なリリースです。

特に Conversational Analytics データエージェントの Gemini Enterprise 公開と Dataplex リネージ統合は、Looker を中心としたデータ分析エコシステムの統合度を飛躍的に高めるものであり、データドリブンな組織運営を加速させます。

💡 推奨: Preview 機能が多いため、まずは開発・検証環境で各機能を試用し、GA 移行のタイミングで本番適用を計画することを推奨します。
