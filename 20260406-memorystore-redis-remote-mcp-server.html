Memorystore for Redis

📋 概要

Google Cloud は、Memorystore for Redis のリモート MCP (Model Context Protocol) サーバーを Preview としてリリースしました。 MCP は Anthropic が開発したオープンソースプロトコルで、AI アプリケーションが外部データソースやサービスに接続する方法を標準化するものです。

この MCP サーバーにより、LLM、AI アプリケーション、AI 対応開発プラットフォームから Memorystore for Redis インスタンスに接続し、Redis の管理操作を MCP プロトコルを通じて実行できるようになります。

🤖 対象ユーザー:
  • Memorystore for Redis を運用しているインフラストラクチャエンジニア
  • AI エージェントを活用したキャッシュ/セッション管理の自動化を検討しているプラットフォームエンジニア
  • Gemini CLI、Claude Code、Gemini Code Assist などの AI ツールから Redis を管理したい開発者

🔄 Before / After 比較

❌ Before

  • Redis 管理には Console、gcloud CLI、REST/RPC API のクライアントライブラリを直接使用
  • AI エージェントから管理操作を実行するにはカスタム統合コードが必要
  • 自然言語で Redis インスタンスの操作を実行する手段なし

✅ After

  • AI エージェントが MCP プロトコルで Redis インスタンスを管理可能に
  • MCP 対応 AI アプリから標準化されたインターフェースで操作
  • IAM 認可、OAuth 2.0 認証、Model Armor、監査ログが利用可能

AI アプリケーションからのリクエストは、IAM / OAuth 2.0 認証を経て、Model Armor セキュリティスキャンを通過後、Memorystore for Redis MCP サーバーに到達します。MCP サーバーは受信したリクエストを Memorystore for Redis API の操作に変換し、Redis インスタンスに対する管理操作を実行します。

⚡ 主要機能

🔌 リモート MCP サーバー

Memorystore for Redis の管理機能を MCP プロトコルで公開。HTTPS エンドポイント経由でアクセス可能。tools/list メソッドでツールを自動検出できます。

🗄️ Redis インスタンス管理

インスタンスの作成、取得、一覧表示、更新、削除、フェイルオーバーなどの管理操作を AI エージェントから実行可能。リードレプリカの管理やスケーリングも対象。

🤖 AI アプリケーション接続

Gemini CLI、Claude Code、Gemini Code Assist、ChatGPT などの MCP 対応 AI アプリから直接接続可能。自然言語によるインスタンス管理を実現。

🔒 エンタープライズセキュリティ

IAM による細粒度アクセス制御 (roles/mcp.toolUser)、OAuth 2.0 認証、Model Armor によるセキュリティスキャン、Cloud Audit Logs による監査ログ記録。

🔧 設定方法

Step 1: MCP サーバーの有効化

Bash Memorystore for Redis MCP サーバー有効化 
# gcloud CLI beta コンポーネントのインストール
gcloud components install beta

# Memorystore for Redis の MCP サーバーを有効化
gcloud beta services mcp enable redis.googleapis.com \
  --project=PROJECT_ID

Step 2: IAM ロールの付与

Bash IAM ロール設定 
# MCP Tool User ロールの付与
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/mcp.toolUser"

# Memorystore for Redis 管理者ロールの付与
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/redis.admin"

Step 3: MCP クライアント設定 (Claude Code)

JSON Claude Code MCP 設定 
{
  "mcpServers": {
    "memorystore-redis": {
      "url": "https://redis.googleapis.com/mcp",
      "transport": "http",
      "auth": {
        "type": "google_credentials",
        "scopes": ["https://www.googleapis.com/auth/cloud-platform"]
      }
    }
  }
}

Step 4: MCP クライアント設定 (Gemini CLI)

JSON Gemini CLI MCP 設定 
{
  "name": "memorystore-redis",
  "version": "1.0.0",
  "mcpServers": {
    "memorystore-redis": {
      "httpUrl": "https://redis.googleapis.com/mcp",
      "authProviderType": "google_credentials",
      "oauth": {
        "scopes": ["https://www.googleapis.com/auth/cloud-platform"]
      },
      "timeout": 30000,
      "headers": {
        "x-goog-user-project": "PROJECT_ID"
      }
    }
  }
}

⚠️ 制約事項・注意点

⚠️ Preview 制限: 本番環境での使用は推奨されません。Pre-GA Offerings Terms が適用され、ツール・パラメータ・出力は予告なく変更される可能性があります。
❌ MCP ツールの出力は非決定的 -- 同一の呼び出しでも内容やフォーマットが変わる可能性
❌ Redis のブロックされたコマンド (CONFIG、SHUTDOWN 等) は MCP サーバーからも制限される可能性
💡 コントロールプレーンのみ: 管理操作に焦点を当てており、GET/SET 等のデータプレーン操作とは区別が必要
💡 最小権限の原則: AI エージェントが破壊的操作 (削除、スケールダウン) を実行する可能性があるため、IAM で最小権限を適用し Human-in-the-Middle モードでの運用を推奨
💡 ネットワーク: Memorystore for Redis は VPC 内のプライベート IP で保護されており、MCP サーバーからの管理操作とデータプレーンの接続要件は異なる
