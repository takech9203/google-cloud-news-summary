⭐ 主要機能の詳細

🌐 Akamai v2.0 - Activate Client List アクション

新しい「Activate Client List」アクションが追加されました。SOAR のプレイブック内から Akamai のクライアントリストをアクティベートできるようになり、脅威インテリジェンスに基づく自動ブロックリスト管理が可能になりました。

🔧 Siemplify v103.0 - Response Integration & Connector Upgrade ジョブ

新しいジョブ「Response Integration & Connector Upgrade」が追加されました。インテグレーションとコネクタのアップグレードプロセスを自動化し、運用負荷を軽減します。

📧 Microsoft Graph Mail v36.0 / Delegated v13.0 - フォルダ処理改善

Forward Email、Save Email to File、Send Email アクションにおけるフォルダ処理が更新されました。サブフォルダを含む複雑なフォルダ構成での動作が安定化しています。委任認証版 (v13.0) にも同様の改善が適用されています。

🔍 Google Chronicle v78.0 - 生ログデータ処理の更新

Get Detection Details アクションおよび Execute UDM Query アクションの生ログデータ処理が更新され、クエリ結果の正確性が向上しました。

☁️ AWS GuardDuty v9.0 - 重大度ハンドリングの改善

Findings Connector における重大度 (severity) の処理が更新されました。GuardDuty の重大度スコアがより正確に Google SecOps のアラート優先度にマッピングされるようになりました。

🛡️ Google SCC v15.0 - ミュート状態処理の更新

List Asset Vulnerabilities アクションにおけるミュート状態の処理が更新されました。ミュートされた脆弱性の取り扱いがより正確になり、アラートノイズの削減に貢献します。

🦅 CrowdStrike Falcon v71.0 - IOC 有効期限と隠しホスト対応

IOC (Indicator of Compromise) に有効期限を設定できるようになりました。また、隠しホスト (Hidden Hosts) のサポートが追加され、非アクティブなホストの管理が改善されました。

📊 Splunk v61.0 - Notable Events 入力処理の改善

Update Notable Events アクションにおける入力処理が更新されました。Splunk Enterprise Security との連携における Notable Events の更新操作がより堅牢になりました。

⚙️ 設定方法

💡 前提条件: Google SecOps SOAR コンソールへの管理者アクセス権、各外部サービスの API 認証情報、Marketplace でのインテグレーションアップグレード権限が必要です。
1
コネクタのアップグレード

Google SecOps SOAR コンソールで Marketplace に移動し、対象のインテグレーションを検索して「Upgrade to (バージョン番号)」をクリックします。

2
コネクタの手動更新

インテグレーションにコネクタが設定されている場合、アップグレード後にコネクタの手動更新が必要です。
Settings > Ingestion > Connectors で対象コネクタを選択し「Update」をクリックします。

3
動作確認

各コネクタの Ping アクションを実行して接続性を確認し、実際のデータ取り込みが正常に行われているかをケース画面で確認します。

⚠️ デメリット・制約事項

制限事項
✕コネクタのアップグレード時、コネクタの状態がファイルシステムからコネクタコンテキストに自動移行されないため、初回実行時に重複ケースが発生する可能性があります
✕ステートレスコネクタのデータストレージにはデフォルトで 300 万文字の制限があります
考慮すべき点
✕複数のコネクタを同時にアップグレードする場合、段階的に実施して動作を確認することが推奨されます
✕CrowdStrike Falcon の一部アクションはプレミアムバージョンが必要です
✕アップグレード後はプレイブックのテスト実行を行い、既存ワークフローに影響がないことを確認すべきです

🎯 ユースケース 1: 脅威インテリジェンスに基づく自動ブロック

シナリオ: SOC チームが CrowdStrike Falcon で検出した IOC を Akamai のクライアントリストに自動登録し、有効期限付きでブロックする。

効果: CrowdStrike Falcon v71.0 の IOC 有効期限サポートと Akamai v2.0 の Activate Client List アクションを組み合わせることで、脅威検出からネットワークレベルでのブロックまでを自動化できます。有効期限の設定により、古い IOC が自動的に失効し、誤検知によるビジネス影響を最小限に抑えます。
