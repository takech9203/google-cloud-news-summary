📋 概要

Google Cloud は、Hyperdisk ML の対応マシンシリーズに A3 Ultra、C4D、N4、N4D の 4 つを新たに追加し、一般提供 (GA) としてリリースしました。Hyperdisk ML は全 Hyperdisk タイプの中で最高のスループットを提供するブロックストレージであり、単一ボリュームで最大 2 TiB/s のスループットを実現します。

NVIDIA H200 GPU 搭載の A3 Ultra や、最新世代の汎用マシン (C4D, N4, N4D) で高スループットの読み取り専用ストレージを活用できるようになりました。大規模 AI/ML ワークロードだけでなく、汎用コンピューティング環境でも利用可能です。

🔄 Before / After

❌ Before (アップデート前)

  • A3 Ultra (a3-ultragpu-8g) で Hyperdisk ML 非サポート -- H200 GPU で最高スループットストレージを利用不可
  • C4D, N4, N4D の新世代汎用マシンで Hyperdisk ML を接続不可
  • 大量のモデルデータやデータセットのロードに Hyperdisk Balanced 等の他タイプに限定

✅ After (アップデート後)

  • A3 Ultra で最大 2 TiB/s スループットを活かした高速モデルロードが実現
  • C4D (AMD EPYC Turin) で高スループットストレージを汎用ワークロードに活用可能
  • N4 / N4D でコストパフォーマンスに優れた大規模データ読み取りワークロードが実現

⚡ 主要機能

🎮 A3 Ultra サポート

a3-ultragpu-8g NVIDIA H200 SXM x8

最上位 AI/ML マシンタイプで Hyperdisk ML が利用可能に。インスタンスあたり最大 4,000 MiB/s のスループットをサポート。複数インスタンスからの同時アクセスでボリュームのスループットをフルに活用可能。

🖥️ C4D サポート

AMD EPYC Turin 最大 384 vCPU NVMe

第 5 世代 AMD EPYC Turin プロセッサ搭載の汎用マシン。最大 3,024 GB メモリ。データ分析、Web サービング、AI 推論など幅広いワークロードで高スループットストレージを活用可能。

💻 N4 サポート

Intel Xeon Emerald Rapids 最大 80 vCPU DDR5

第 5 世代 Intel Xeon Scalable プロセッサ搭載。最大 640 GB DDR5 メモリ。コストパフォーマンスに優れた汎用インスタンスで大規模データセットの高速読み取りが可能。

🔧 N4D サポート

AMD EPYC Turin 最大 96 vCPU DDR5

第 5 世代 AMD EPYC Turin プロセッサ搭載。最大 768 GB DDR5 メモリ。AMD プラットフォームを好むユーザーに高スループットストレージの選択肢を追加。

🔧 設定方法

💡 前提条件: 対象マシンシリーズが利用可能なリージョン/ゾーン、Compute Engine API の有効化、必要な IAM 権限 (compute.disks.create, compute.instances.attachDisk 等) が必要です。

ステップ 1: Hyperdisk ML ボリュームを作成

Bash Hyperdisk ML ボリューム作成 
gcloud compute disks create my-hyperdisk-ml \
    --type=hyperdisk-ml \
    --size=1000 \
    --provisioned-throughput=100000 \
    --zone=us-central1-a \
    --access-mode=READ_ONLY_MANY

--access-mode=READ_ONLY_MANY を指定することで、複数インスタンスへの読み取り専用アタッチが可能になります。

ステップ 2: インスタンスにディスクをアタッチ

Bash ディスクのアタッチ 
gcloud compute instances attach-disk my-a3-ultra-instance \
    --disk=my-hyperdisk-ml \
    --mode=ro \
    --zone=us-central1-a

--mode=ro (読み取り専用) で同一ボリュームを複数インスタンスに同時アタッチ可能です。

ステップ 3: ディスクをマウントして使用

Bash 読み取り専用マウント 
# インスタンスに SSH 接続後
sudo mkdir -p /mnt/hyperdisk-ml
sudo mount -o ro /dev/disk/by-id/google-my-hyperdisk-ml /mnt/hyperdisk-ml

✅ メリット

💰 ビジネス面

→コスト効率の向上: 単一ボリュームを複数インスタンスで共有。マルチアタッチに追加料金不要
→柔軟なマシン選択: A3 Ultra の高性能 GPU から N4/N4D の汎用インスタンスまで幅広く対応
→アクセラレータのアイドル時間短縮: 高スループットによりモデルロード時間が短縮、GPU 稼働率が向上

🔬 技術面

→最大 2 TiB/s のスループット: 全 Hyperdisk タイプ中最高
→マルチアタッチ: 最大 2,500 インスタンスから同時読み取り可能 (512 GiB 以下)
→動的パフォーマンス調整: ワークロードに応じてスループットを変更可能 (6 時間ごと)

⚠️ 制限事項・注意点

制限事項
  • ブートディスクとして使用不可
  • 読み取り専用モード設定後、書き込みモードに戻すことは不可
  • スナップショット/ディスクイメージからの読み書きモードでの Hyperdisk ML 作成は不可
  • マルチライターモードは非サポート
  • Storage Pools は非サポート
  • 2026/2/4 以前に作成されたボリュームは第 4 世代マシンにアタッチ不可 (スナップショット経由で回避)
  • マシンイメージ/インスタントスナップショットの作成不可
  • 30 秒ごとに最大 100 VM へのアタッチに制限
考慮すべき点
  • インスタンスレベルのパフォーマンス上限がボリュームの実効スループットを制限する
  • プロビジョニングスループットがインスタンス上限を超える場合、複数インスタンスへのアタッチが必要
  • パフォーマンスはハーフデュプレックス (IOPS/スループット上限は読み書き合計に適用)
  • CUD / SUD は Hyperdisk には適用されない

📝 まとめ

今回のアップデートにより、Hyperdisk ML の対応マシンシリーズが A3 Ultra、C4D、N4、N4D に拡大。AI/ML 向け最上位 GPU マシンから汎用マシンまで、幅広い環境で最大 2 TiB/s の高スループットストレージを活用できるようになりました。

特に A3 Ultra での大規模 LLM ワークロードや、C4D/N4/N4D を使用したコスト効率の高いデータ分析環境の構築が容易になります。Hyperdisk ML を活用した高スループットデータアクセスが必要なワークロードを運用している場合は、新しくサポートされたマシンシリーズへの移行を検討してください。
