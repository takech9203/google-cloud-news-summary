🎬 Veo 3.1 Lite (パブリックプレビュー)

Vertex AI 上で最もコスト効率の高いビデオ生成モデルとして、Veo 3.1 の技術をベースにした軽量バリアントが登場しました。

📌 主な特徴

✓テキストから動画、画像から動画の生成に対応
✓音声付きビデオの出力をサポート
✓テキスト入力は最大 1,024 トークン
✓1 プロンプトあたり 1 本のビデオ出力
✗4K 出力には非対応
✗ビデオ拡張 (Extension) は非対応

💻 設定方法

ステップ 1: Veo 3.1 Lite でテキストからビデオを生成

Python Veo 3.1 Lite によるビデオ生成 
import time
from google import genai
from google.genai.types import GenerateVideosConfig

client = genai.Client()

operation = client.models.generate_videos(
    model="veo-3.1-lite-generate-preview",
    prompt="A cinematic aerial shot of a coastal city at sunset, warm golden light reflecting off glass buildings",
    config=GenerateVideosConfig(
        aspect_ratio="16:9",
        output_gcs_uri="gs://your-bucket/output/",
    ),
)

while not operation.done:
    time.sleep(15)
    operation = client.operations.get(operation)

if operation.response:
    print(operation.result.generated_videos[0].video.uri)

⚠️ デメリット・制約事項

🚧 制限事項

✗Veo 3.1 Lite はプレビュー段階であり、プレビュー利用規約が適用される
✗4K 出力に非対応 (高解像度が必要な場合は Veo 3.1 Generate 001 を使用)
✗ビデオ拡張 (Extension) 機能が非対応 (既存ビデオの延長は不可)
✗プロンプトごとの出力は 1 本に制限 (Veo 3.1 Generate 001 は最大 4 本)

💡 考慮すべき点

→Gemini 2.5 の実質的な移行期限は廃止日の 1 か月前 (2026/9/16 頃) から新規アクセスがブロックされるため、それ以前に完了させる必要がある
→Gemini 3 ファミリーへの移行時にはレスポンスの差異やパフォーマンス特性の変化を事前に検証すること
→Veo 3.1 Lite と Veo 3.1 Generate 001 の品質差を事前評価し、ユースケースに適しているか確認が必要

🎯 ユースケース

🛒 ユースケース 1: 大規模コンテンツ生成パイプライン

シナリオ: EC サイトが商品説明ビデオを数千点の商品に対して自動生成する場合

Python 大量ビデオ生成パイプライン 
products = get_product_list()  # 商品リストを取得

for product in products:
    operation = client.models.generate_videos(
        model="veo-3.1-lite-generate-preview",
        prompt=f"Product showcase video: {product.description}",
        config=GenerateVideosConfig(
            aspect_ratio="16:9",
            output_gcs_uri=f"gs://product-videos/{product.id}/",
        ),
    )

効果: コスト効率の高い Veo 3.1 Lite を使用することで、大量のビデオ生成を経済的に実行でき、商品ページのエンゲージメント向上に貢献
