📋 概要

Cloud Spanner に関する 3 つの機能アップデートが発表された。Spanner リモート MCP サーバーと Database Insights MCP サーバーの GA により、AI アプリケーションから Spanner を直接操作・分析できるようになった。さらに、Spanner Studio のグラフスキーマビジュアルモデリング (Preview) により、DDL を手書きせずにグラフスキーマを設計できる。

🔄 Before / After

❌ Before

✖AI アプリから Spanner を操作するには独自の API 統合やローカル MCP サーバーのセットアップが必要
✖AI エージェントがパフォーマンスメトリクスを取得する標準化インターフェースがなかった
✖Spanner Graph のスキーマ作成に DDL 文の手動記述が必要

✅ After

✔Google 管理のリモート MCP サーバー (spanner.googleapis.com/mcp) で AI アプリから直接操作
✔Database Insights MCP サーバー (databaseinsights.googleapis.com/mcp) で AI からパフォーマンス分析
✔Spanner Studio のビジュアルモデリングで DDL なしにグラフスキーマ設計
✔外部キー制約やリバースエッジインデックスが自動生成

⭐ 主要機能

🌐 Spanner リモート MCP サーバー (GA)

→MCP プロトコルに準拠したリモートサーバー: https://spanner.googleapis.com/mcp
→Spanner API を有効化するだけで自動的に利用可能
→OAuth 2.0 による認証・認可、IAM によるきめ細かなアクセス制御
→Gemini CLI、Claude.ai、ChatGPT、カスタム AI アプリとの接続をサポート
→15 種類の MCP ツールを提供 (インスタンス管理、データベース操作、SQL 実行、スキーマ更新など)

📊 Database Insights リモート MCP サーバー (GA)

→エンドポイント: https://databaseinsights.googleapis.com/mcp
→get_system_metrics ツールで PromQL クエリによるシステムメトリクス取得
→CPU 使用率、オペレーション数、スループット、ストレージ使用量などを AI アプリから分析
→Spanner だけでなく Cloud SQL、AlloyDB、Bigtable など他のデータベースでも利用可能

🎨 Spanner Studio グラフスキーマビジュアルモデリング (Preview)

→直感的なインターフェースでグラフスキーマを作成・管理
→ゼロからの設計、既存テーブルからのマッピング、ハイブリッドアプローチに対応
→ノード・エッジの追加、プロパティ定義、ラベル設定をビジュアルに操作
→CREATE TABLE、CREATE OR REPLACE PROPERTY GRAPH、外部キー制約、リバースエッジインデックスを自動生成
→Enterprise / Enterprise Plus エディションで利用可能

⚙️ 設定方法

前提条件

1.Google Cloud プロジェクトで Spanner API が有効化されていること
2.必要な IAM ロール (roles/mcp.toolUser、roles/spanner.admin) が付与されていること
3.OAuth 2.0 の認証情報が設定されていること

💻 Gemini CLI での設定

Bash Gemini CLI 拡張機能の設定 
# 拡張機能ファイルを作成
mkdir -p ~/.gemini/extensions/spanner
cat > ~/.gemini/extensions/spanner/gemini-extension.json << 'EOF'
{
  "name": "spanner",
  "version": "1.0.0",
  "mcpServers": {
    "Spanner MCP Server": {
      "httpUrl": "https://spanner.googleapis.com/mcp",
      "authProviderType": "google_credentials",
      "oauth": {
        "scopes": ["https://www.googleapis.com/auth/cloud-platform"]
      },
      "timeout": 30000,
      "headers": {
        "x-goog-user-project": "YOUR_PROJECT_ID"
      }
    }
  }
}
EOF

🤖 Claude.ai での設定

1.Google Cloud Console で OAuth 2.0 クライアント ID を作成 (リダイレクト URI: https://claude.ai/api/mcp/auth_callback)
2.Claude.ai の Settings > Connectors で「Add custom connector」をクリック
3.Server name、Remote MCP server URL (https://spanner.googleapis.com/mcp)、OAuth クライアント ID/シークレットを入力

⚖️ メリット・デメリット

✅ ビジネス面のメリット

✔AI 統合の加速: MCP プロトコルにより AI アプリから Spanner を直接操作でき、AI 駆動のデータベース管理ワークフローを迅速に構築
✔運用コストの削減: AI エージェントがパフォーマンスメトリクスを自動分析し、問題検出・診断を効率化
✔グラフ DB 導入障壁の低下: ビジュアルモデリングにより DDL の専門知識なしでグラフスキーマ設計が可能

✅ 技術面のメリット

✔標準プロトコル対応: MCP という業界標準プロトコルで特定の AI プラットフォームに依存しない柔軟な統合
✔セキュアなアクセス制御: OAuth 2.0 + IAM でエージェントごとにきめ細かなアクセス制御
✔自動最適化: ビジュアルモデリングでエッジトラバーサル最適化 (外部キー制約、リバースエッジインデックス) が自動適用
✔オブザーバビリティ: MCP サーバー経由クエリにリクエストタグ (mcp_execute_sql など) が自動付与

⚠️ 制限事項

✖Spanner MCP サーバーは API キーによる認証をサポートしていない (OAuth 2.0 が必須)
✖グラフスキーマビジュアルモデリングは Preview 段階 (Pre-GA Offerings Terms 適用)
✖ビジュアルモデリングは Enterprise / Enterprise Plus エディションのみ (Standard は不可)

⚠️ ビジュアルモデリングの制約

✖DROP TABLE や DROP COLUMN の DDL は安全性のため生成されない
✖一部のスキーマオブジェクト、列データ型、修飾子は自動生成の対象外
✖下書き保存機能が未対応 (ページを閉じると進捗が失われる)
✖スキーマレスデータには対応していない
✖ビューおよび名前付きスキーマテーブルをデータソースとして選択不可
✖既存テーブルからマッピングしたエッジには外部キー制約やリバースエッジインデックスが自動生成されない

💡 ユースケース

🤖 ユースケース 1: AI エージェントによるデータベース管理の自動化

シナリオ: 開発チームが AI エージェントを使って Spanner データベースのスキーマ確認、データクエリ、パフォーマンス分析を自動化する。

JSON MCP サーバー設定例 
{
  "mcpServers": {
    "Spanner MCP Server": {
      "httpUrl": "https://spanner.googleapis.com/mcp",
      "authProviderType": "google_credentials",
      "oauth": {
        "scopes": ["https://www.googleapis.com/auth/cloud-platform"]
      },
      "headers": {
        "x-goog-user-project": "my-project-id"
      }
    },
    "Database Insights MCP Server": {
      "httpUrl": "https://databaseinsights.googleapis.com/mcp",
      "authProviderType": "google_credentials",
      "oauth": {
        "scopes": ["https://www.googleapis.com/auth/cloud-platform"]
      },
      "headers": {
        "x-goog-user-project": "my-project-id"
      }
    }
  }
}

効果: AI エージェントが自然言語での指示に基づいてデータベースの状態確認、クエリ実行、パフォーマンス問題の診断を一貫して実行でき、DBA の負荷を大幅に軽減。

🔍 ユースケース 2: グラフデータベースの迅速なプロトタイピング

シナリオ: ソーシャルネットワーク分析や不正検知のために、Spanner Graph を使ったグラフデータベースを新規に設計する。ビジュアルモデリングを使い、ユーザーノード、アカウントノード、トランザクションエッジを直感的に定義してプロトタイプを作成。

効果: DDL の専門知識がなくても、ドラッグ&ドロップ操作でグラフスキーマを設計でき、ベストプラクティスが自動適用されるため、最適化されたスキーマをすばやく構築してプロトタイピングを加速。

💰 料金

Spanner MCP サーバーおよび Database Insights MCP サーバーの利用自体に追加料金は発生しない。料金は通常の Spanner リソース使用量 (コンピュート、ストレージ) に基づく。

Spanner Graph のビジュアルモデリング機能は Enterprise エディション以上で利用可能であり、エディションごとの料金体系が適用される。

🔗 関連サービス・機能

→Google Cloud MCP サーバー: Cloud SQL、AlloyDB、Bigtable、Firestore など複数サービスが MCP サーバーを提供。統一インターフェースで AI アプリから操作可能
→Cloud Monitoring: Database Insights MCP サーバーは内部的に Cloud Monitoring のメトリクスを PromQL 経由で取得。既存のモニタリング基盤と連携
→Spanner Query Insights: MCP サーバーの get_system_metrics と補完的に使用可能。クエリパフォーマンスの詳細分析 (追加料金なし)
→Model Armor: Google Cloud MCP サーバーではオプションで Model Armor によるプロンプト/レスポンスのセキュリティ保護が利用可能
→Spanner Graph: ビジュアルモデリングは Spanner Graph のスキーマ管理を拡張。GQL によるクエリ、クエリ結果の可視化などの既存機能と組み合わせて利用
