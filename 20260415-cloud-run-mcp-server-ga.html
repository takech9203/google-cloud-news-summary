☁️ Cloud Run: リモート MCP サーバーが GA に昇格

Cloud Run MCP GA Model Context Protocol
2026-04-15
Google Cloud

📋 概要

Cloud Run リモート MCP (Model Context Protocol) サーバーが General Availability (GA) となり、本番環境での利用が正式にサポートされるようになりました。MCP は AI エージェントが外部データソースやツールと標準化された方法でやり取りするためのオープンプロトコルです。

Cloud Run 上で MCP サーバーをホストすることで、Gemini CLI、Claude、ChatGPT などの主要な AI アプリケーションから Cloud Run サービスのデプロイや管理を自然言語で実行できるようになります。エンドポイント (https://run.googleapis.com/mcp) を通じて、AI 駆動のインフラストラクチャ管理が可能です。

✅ GA リリースのポイント: Google Cloud SLA に基づくサポート対象となり、エンタープライズグレードの本番ワークロードで安心して利用可能に。Cloud Run Admin API を有効化するだけで利用開始できます。

🔄 Before / After 比較

❌ Before (プレビュー段階)

  • 🔴 MCP サーバーはプレビュー段階で本番環境に SLA が適用されなかった
  • 🔴 MCP サーバーのデプロイ手順やベストプラクティスが限定的
  • 🔴 Cloud Run 操作には手動で gcloud CLI やコンソールを使用する必要があった
  • 🔴 MCP クライアントの認証方法が確立されておらずセキュリティ面で不安

✅ After (GA リリース)

  • 🟢 GA として正式リリース、Google Cloud SLA の対象で本番環境利用可能
  • 🟢 Streamable HTTP トランスポートによるリモートホスティングが完全サポート
  • 🟢 AI エージェントから自然言語で Cloud Run のデプロイ・管理が可能
  • 🟢 OAuth 2.0 と IAM による堅牢な認証・認可フレームワークが確立

⚡ 主要機能

🌐 MCP サーバーエンドポイント

マネージド HTTP エンドポイント (https://run.googleapis.com/mcp) を提供。Cloud Run Admin API を有効化するだけで利用可能。追加インフラ不要。

🔧 MCP ツール群

Cloud Run サービスの一覧取得、詳細確認、デプロイ (コンテナイメージ / アーカイブ / ソースコード) を標準化された MCP ツールで操作。

list_services get_service deploy_service_from_image deploy_service_from_archive deploy_service_from_file_contents

🛠️ カスタム MCP サーバーのホスティング

独自開発した MCP サーバーを Cloud Run 上にデプロイ可能。FastMCP や公式 MCP SDK (TypeScript, Python, Go, Kotlin, Java, C#, Ruby, Rust) をサポート。

📦 ツールセット (Toolsets)

MCP ツール群からサブセットを定義し、個別エンドポイントとして公開可能。エージェントの過負荷を防ぎ、必要なツールだけを選択的に提供。

必要な IAM ロール

ロール 説明
roles/run.developerCloud Run サービスの作成
roles/iam.serviceAccountUserサービスアカウントとしてのオペレーション実行
roles/artifactregistry.readerデプロイ済みコンテナイメージへのアクセス
roles/mcp.toolUserMCP ツール呼び出しの実行
roles/iam.serviceAccountTokenCreatorクロスプロジェクトのサービスアカウント使用 (必要な場合)

OAuth スコープ

JSON OAuth スコープ設定 
{
  "scopes": [
    "https://www.googleapis.com/auth/run.readonly",
    "https://www.googleapis.com/auth/run"
  ]
}

run.readonly: 読み取り専用アクセス (サービス一覧、詳細取得) / run: 読み取りおよび書き込みアクセス (デプロイ、更新)

🛠️ 設定方法

前提条件

✓Google Cloud プロジェクトの作成と Cloud Run Admin API の有効化
✓gcloud CLI のインストールと初期化
✓必要な IAM ロールの付与 (roles/run.developer, roles/mcp.toolUser など)

ステップ 1: Cloud Run Admin API の有効化

Bash API の有効化 
gcloud services enable run.googleapis.com \
  --project=PROJECT_ID

ステップ 2: IAM ロールの付与

Bash 必要な IAM ロールを付与 
# MCP ツールの呼び出し権限
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/mcp.toolUser"

# Cloud Run サービスの作成権限
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/run.developer"

# サービスアカウントの使用権限
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="user:USER_EMAIL" \
  --role="roles/iam.serviceAccountUser"

ステップ 3: Gemini CLI での設定例

JSON ~/.gemini/extensions/cloud-run-mcp/gemini-extension.json 
{
  "name": "cloud-run-mcp",
  "version": "1.0.0",
  "mcpServers": {
    "cloud_run": {
      "httpUrl": "https://run.googleapis.com/mcp",
      "authProviderType": "google_credentials",
      "oauth": {
        "scopes": ["https://www.googleapis.com/auth/run"]
      },
      "timeout": 30000,
      "headers": {
        "x-goog-user-project": "PROJECT_ID"
      }
    }
  }
}

ステップ 4: Claude Desktop での設定例

JSON Claude Desktop MCP 設定 
{
  "mcpServers": {
    "cloud_run": {
      "url": "https://run.googleapis.com/mcp",
      "transport": "http"
    }
  }
}

ステップ 5: カスタム MCP サーバーのデプロイ (オプション)

Bash Cloud Run へのデプロイ 
# ソースコードからのデプロイ
gcloud run deploy mcp-server \
  --no-allow-unauthenticated \
  --region=us-central1 \
  --source .

# コンテナイメージからのデプロイ
gcloud run deploy mcp-server \
  --image us-central1-docker.pkg.dev/PROJECT_ID/remote-mcp-servers/mcp-server:latest \
  --region=us-central1 \
  --no-allow-unauthenticated

💻 カスタム MCP サーバー実装例

FastMCP を使用した Python のカスタム MCP サーバー実装例です。Cloud Run にデプロイして AI エージェントから利用できます。

Python FastMCP によるカスタム MCP サーバー 
from fastmcp import FastMCP
import os

mcp = FastMCP("Custom Business MCP Server")

@mcp.tool()
def query_inventory(product_id: str) -> dict:
    """在庫情報を取得する"""
    # ビジネスロジックの実装
    return {"product_id": product_id, "stock": 150}

if __name__ == "__main__":
    import asyncio
    asyncio.run(
        mcp.run_async(
            transport="streamable-http",
            host="0.0.0.0",
            port=os.getenv("PORT", 8080),
        )
    )

🚀 ユースケース 1: AI エージェントによる自動デプロイ

開発者が Gemini CLI を使って新しい Web サービスを Cloud Run にデプロイ。Docker Hub のコンテナイメージを指定するだけで、AI エージェントが MCP ツールを使用して自動的にデプロイを実行。

Text Gemini CLI でのプロンプト例 
"Deploy a private Cloud Run service named my-api
 from the Docker image us-docker.pkg.dev/cloudrun/container/hello
 to project my-project in us-central1 region"

効果: gcloud CLI のコマンド構文を覚えることなく、自然言語でインフラ操作が完了。デプロイ時間の短縮と人的ミスの削減を実現。

🏢 ユースケース 2: カスタム MCP サーバーのホスティング

社内のデータベースや外部 API と連携するカスタム MCP サーバーを開発し、Cloud Run 上にホスティング。AI エージェントがこの MCP サーバーを通じてビジネスロジックを実行。

効果: AI エージェントが社内システムのデータにアクセスし、業務の自動化を実現。Cloud Run のスケーラブルな基盤により、高負荷時にも安定した応答を提供。

⚡ ユースケース 3: バイブコーディングによる高速プロトタイピング

AI エージェントとの対話を通じて Web アプリケーションのコードを生成し、deploy_service_from_file_contents ツールを使って即座に Cloud Run にデプロイ。開発と検証のサイクルを極めて短縮。

効果: コンテナイメージのビルド工程をスキップして数秒でデプロイが完了し、アイデアの検証スピードが飛躍的に向上。

⚠️ デメリット・制約事項

制限事項
  • stdio トランスポート非対応 (ローカル MCP サーバーの直接ホスティングは不可)
  • deploy_service_from_archive のアーカイブサイズは最大 250MiB
  • deploy_service_from_file_contents のリクエストサイズは最大 50MiB
  • リージョナルエンドポイントはプレビュー段階 (GA はグローバルエンドポイントのみ)
  • API キー認証は非対応 (OAuth 2.0 + IAM が必須)
💡 考慮すべき点
  • MCP クライアントの認証設定は AI アプリケーションごとに異なる
  • 過剰なツール公開はエージェント性能劣化の原因 -- ツールセット機能で最小限に
  • エージェント用アイデンティティはユーザーアカウントと分離して管理推奨
  • ヘッドレス環境では OAuth ブラウザリダイレクトが動作しない -- サービスアカウント認証を検討

💲 料金情報

Cloud Run MCP サーバーのエンドポイント (run.googleapis.com/mcp) の利用自体に追加料金は発生しません。Cloud Run Admin API の一部として提供されます。

項目 月額料金 (概算)
Cloud Run MCP エンドポイント利用無料 (Cloud Run Admin API に含まれる)
Cloud Run vCPU (インスタンスベース課金)$0.0648 / vCPU 時間
Cloud Run メモリ (インスタンスベース課金)$0.0072 / GiB 時間
Cloud Run vCPU (リクエストベース課金)$0.0526 / vCPU 時間 + リクエスト料
Artifact Registry ストレージ$0.10 / GB 月
✅ 無料枠: Cloud Run には毎月 180,000 vCPU 秒と 360,000 GiB 秒の無料利用分があります。MCP サーバーの利用頻度が低い開発・テスト用途であれば、無料枠内で運用可能です。
