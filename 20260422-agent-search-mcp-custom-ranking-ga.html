📝 概要

Google Cloud は Agent Search (旧 Vertex AI Search) に対して 5 つの重要なアップデートを同時に発表しました。MCP サーバーの GA 昇格、カスタムランキングにおける Dense Reciprocal Rank (DRR) と Geodistance の GA、ドキュメントレベル関連性フィルタの GA、そして Vertex AI Search から Agent Search への製品名変更です。

🤖 対象ユーザー: AI エージェント開発者、検索アプリケーション構築者、RAG システム運用エンジニア、位置情報を活用した検索サービス事業者

🔄 Before / After 比較

❌ Before

  • AI エージェントから Agent Search へのアクセスには独自の統合コードが必要
  • rr 関数では同一スコアのドキュメントのランキング品質が低下
  • 位置情報ベースのランキング調整は別途計算ロジックが必要
  • ドキュメントレベルの関連性フィルタリングが GA 未提供

✅ After

  • MCP 標準プロトコルで Claude / Gemini CLI / ADK から直接アクセス可能
  • drr 関数で重複スコア問題を解消し、高品質ランキングを実現
  • geo_distance 関数で距離ベースランキングを式に直接組み込み可能
  • キーワード / セマンティック検索それぞれに独立した関連性閾値を設定可能

⚡ 主要機能 1: MCP Server (GA)

🌐 MCP サーバーによるエージェント統合

Agent Search のデータストアに対して Model Context Protocol (MCP) 経由で検索を実行可能に。2026 年 2 月の Public Preview を経て今回 GA に昇格しました。

→エンドポイント: https://discoveryengine.googleapis.com/mcp
→プロトコル: JSON-RPC 2.0 over HTTPS
→認証: OAuth 2.0 / OIDC
→MCP メソッド: tools/list, tools/call

🔎 search ツール

データストアに取り込まれたデータに対する検索を実行

💬 conversational_search ツール

データストアに対する会話型検索を実行

JSON MCP クライアント設定例 
{
  "mcpServers": {
    "agent-search": {
      "httpUrl": "https://discoveryengine.googleapis.com/mcp",
      "headers": {
        "Authorization": "Bearer $(gcloud auth print-access-token)"
      }
    }
  }
}

📈 主要機能 2: Dense Reciprocal Rank (DRR) によるカスタムランキング (GA)

🎯 drr() - 重複シグナル値の問題を解消

従来の rr (Reciprocal Rank) 関数の改良版。同一のシグナル値を持つドキュメントが複数存在する場合に、rr では任意の順序が割り当てられていた問題を解消し、より高品質なランキングを実現します。

→カスタムランキング式内で他のシグナルや関数と組み合わせて使用可能
→構文: drr(expression, k)

📍 主要機能 3: Geodistance 関数によるカスタムランキング (GA)

🗺️ geo_distance() - 距離ベースランキング

geo_distance 関数をカスタムランキング式で使用して、ソース位置とデスティネーション位置の間の距離をメートル単位で計算します。

ソース位置として 3 種類をサポート:

💬 query_loc

自然言語クエリから自動抽出。ポイント、ポリライン、サークル、ポリゴンに対応

📌 request_loc (座標)

明示的に提供された緯度・経度の座標

🏠 request_loc (住所)

明示的に提供された住所文字列

💡 デスティネーション位置: カスタム取得可能フィールド (例: c.hotel_location, c.office_location) を使用。rr や drr 関数と組み合わせて距離ベースのランキング重み付けが可能。

🎚️ 主要機能 4: ドキュメントレベル関連性フィルタ (GA)

検索リクエストにドキュメントレベルの関連性フィルタを指定し、閾値を満たすドキュメントのみを返却。2025 年 12 月に Public Preview として提供開始、今回 GA に昇格しました。

閾値タイプ 検索タイプ 値の範囲 粒度
relevanceThreshold キーワード / セマンティック HIGH, MEDIUM, LOW, LOWEST 粗粒度
semanticRelevanceThreshold キーワード / セマンティック 0.0 - 1.0 (浮動小数点) 細粒度
💡 対応データストア: Advanced Website Indexing によるウェブサイトデータ、カスタム非構造化データ、カスタム構造化データ

💻 設定方法・実装例

ステップ 1: MCP サーバーのツール一覧を取得

Bash MCP ツール一覧取得 
curl --location 'https://discoveryengine.googleapis.com/mcp' \
  --header 'content-type: application/json' \
  --header 'accept: application/json, text/event-stream' \
  --header "Authorization: Bearer $(gcloud auth print-access-token)" \
  --data '{
    "method": "tools/list",
    "jsonrpc": "2.0",
    "id": 1
  }'

ステップ 2: DRR + Geodistance を使用したカスタムランキング検索

Bash DRR + Geodistance によるランキング検索 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  "https://discoveryengine.googleapis.com/v1/projects/PROJECT_ID/locations/global/collections/default_collection/engines/APP_ID/servingConfigs/default_search:search" \
  -d '{
    "query": "東京駅近くのホテル",
    "rankingExpression": "drr(semantic_similarity_score, 32) * 0.4 + drr(keyword_similarity_score, 32) * 0.3 + drr(geo_distance(query_loc, c.hotel_location) * -1, 32) * 0.3",
    "rankingExpressionBackend": "RANK_BY_FORMULA",
    "userInfo": {
      "preciseLocation": {
        "point": { "lat": 35.6812, "lon": 139.7671 }
      }
    }
  }'
💡 ポイント: 距離の乗算係数を -1 にすることで、近い場所ほど高いランクが付与されます。DRR でセマンティック類似度・キーワード類似度を正規化しつつ、Geodistance で距離調整を行います。

ステップ 3: ドキュメントレベル関連性フィルタの適用

Bash 関連性フィルタの適用 
curl -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  "https://discoveryengine.googleapis.com/v1/projects/PROJECT_ID/locations/global/collections/default_collection/engines/APP_ID/servingConfigs/default_search:search" \
  -d '{
    "query": "検索クエリ",
    "relevanceFilterSpec": {
      "keywordSearchThreshold": {
        "relevanceThreshold": "MEDIUM"
      },
      "semanticSearchThreshold": {
        "semanticRelevanceThreshold": 0.75
      }
    }
  }'

✅ メリット

💼 ビジネス面

→エージェント統合の簡素化: MCP 標準プロトコルにより、カスタム統合コードの開発・保守コストを削減
→検索品質の向上: DRR + 関連性フィルタで検索離脱率低下とコンバージョン率改善
→位置情報サービス強化: 不動産、旅行、小売、物流で距離を考慮した検索体験を迅速構築

⚙️ 技術面

→MCP 標準準拠: Claude / Gemini CLI / ADK との相互運用性を確保
→ランキング精度改善: 同一スコアの大量ドキュメントでも適切なランキング
→柔軟な関連性制御: キーワード / セマンティック検索に独立した閾値
→自然言語位置抽出: ユーザーが座標を指定不要、クエリから自動抽出

⚠️ デメリット・制約事項

制限事項:
  • 関連性フィルタは Basic Website Indexing、メディアデータ、ヘルスケアデータには未対応
  • 関連性フィルタはブレンド検索アプリ (複数データストア接続) では使用不可
  • コンソール UI は引き続き「Vertex AI Search and AI Applications」と表示 (名称不一致)
  • API エンドポイントは引き続き Discovery Engine API (API レベルの名称変更なし)
考慮すべき点:
  • MCP サーバー利用には事前に認証設定 (OAuth 2.0 / OIDC) と IAM 権限が必要
  • 既存の rr 使用箇所は drr への移行を検討
  • 関連性フィルタの閾値は本番適用前に複数クエリでテストを推奨
  • 名称変更に伴い、既存ドキュメント・社内ガイド・自動化スクリプトの参照更新が必要

🎯 ユースケース

📚 ユースケース 1: AI エージェントによる社内ナレッジベース検索

社内の FAQ、マニュアル、ポリシー文書を Agent Search に格納し、MCP サーバー経由で AI エージェント (Claude Desktop / ADK) が直接検索。関連性フィルタで HIGH 閾値を設定し、高精度な回答のみを返却。

JSON MCP ツール呼び出し例 
{
  "method": "tools/call",
  "jsonrpc": "2.0",
  "id": 2,
  "params": {
    "name": "search",
    "arguments": {
      "query": "リモートワークの経費精算ルール",
      "app_id": "internal-knowledge-base",
      "relevanceFilterSpec": {
        "keywordSearchThreshold": {
          "relevanceThreshold": "HIGH"
        }
      }
    }
  }
}
✅ 効果: AI エージェントが関連性の高い社内文書のみを参照して回答 → 回答精度向上、ハルシネーションリスク低減

🏢 ユースケース 2: 不動産・旅行サイトの距離ベース検索ランキング

「渋谷駅近くのオフィス」というクエリに対して、Geodistance + DRR で距離・セマンティック類似度・キーワード一致度を総合的にランキング。

Text カスタムランキング式 
drr(semantic_similarity_score, 32) * 0.3
+ drr(keyword_similarity_score, 32) * 0.2
+ drr(geo_distance(query_loc, c.office_location) * -1, 32) * 0.5
✅ 効果: 自然言語クエリから位置情報を自動抽出、距離重視 + テキスト関連性考慮 → ユーザー満足度・コンバージョン率向上

🛒 ユースケース 3: E コマースの高精度商品検索

大量の類似商品が存在するカタログで DRR 関数を使用して重複スコア問題を解消。セマンティック関連性 0.8 以上のドキュメントのみを返却し、検索結果の質を向上。

✅ 効果: 類似商品が多数あるカタログでもユーザーの検索意図に最も合致する商品が上位表示 → 検索離脱率低下
