📋 概要

GKE のクラスタバージョンが 2026-R14 として更新されました。Rapid、Regular、Stable、Extended、およびチャネル未登録の全チャネルにおいて、Kubernetes 1.30 から 1.35 までの新しいパッチバージョンが利用可能です。

本アップデートには Container-Optimized OS (COS) イメージの累積セキュリティ修正 も含まれており、GKE ノードの基盤となる COS イメージが更新され、既知の脆弱性に対するパッチが適用されています。

❌ アップデート前

  • COS に未修正のセキュリティ脆弱性が存在
  • 一部の Kubernetes コンポーネントにバグや安定性の問題
  • ノードイメージに最新パッチが未適用

✅ アップデート後

  • COS の累積セキュリティ修正でノードセキュリティ強化
  • 全チャネルで最新パッチバージョンが利用可能
  • Kubernetes 1.35 系が Rapid / チャネル未登録で利用可能

🖥️ リリースチャネル構造とバージョン伝播

Kubernetes のオープンソースリリースから Rapid チャネルに最初に提供され、その後 Regular、Stable、Extended の順に段階的に展開されます。全チャネルに対して Container-Optimized OS のセキュリティ修正が適用されています。

⭐ 主要機能

📦 全チャネルでのパッチバージョン更新

→Rapid、Regular、Stable、Extended、チャネル未登録の 5 つのチャネル全て で新しいパッチバージョンが利用可能
→マイナーバージョン 1.30 から 1.35 までの広範囲をカバー
→各チャネルの特性に応じた適切なバージョンが配布

→GKE ノードで使用される COS イメージが更新
→累積的なセキュリティ修正により既知の脆弱性に対処
→Google のセキュリティチームによるメンテナンスで迅速なパッチ提供

🚀 Kubernetes 1.35 系の提供拡大

→Rapid チャネルで 1.35.3-gke.1234000 が利用可能
→Regular、Stable、Extended チャネルでも 1.35.2 系が利用可能
→チャネル未登録クラスタでも 1.35.3 系が利用可能

✅ 状態確認

Step 1: 利用可能なバージョンの確認

Bash 利用可能バージョンの取得 
# クラスタのコントロールプレーンで利用可能なバージョンを確認
gcloud container get-server-config \
  --location=CONTROL_PLANE_LOCATION

Step 2: コントロールプレーンのアップグレード

Bash コントロールプレーンのアップグレード 
# 特定のバージョンにアップグレード
gcloud container clusters upgrade CLUSTER_NAME \
  --master \
  --location=CONTROL_PLANE_LOCATION \
  --cluster-version=VERSION

Step 3: ノードプールのアップグレード

Bash ノードプールのアップグレード 
# ノードプールを特定のバージョンにアップグレード
gcloud container clusters upgrade CLUSTER_NAME \
  --node-pool=NODE_POOL_NAME \
  --location=CONTROL_PLANE_LOCATION \
  --cluster-version=VERSION

Step 4: アップグレード状態の確認

Bash バージョン確認 
# クラスタのバージョンを確認
gcloud container clusters describe CLUSTER_NAME \
  --location=CONTROL_PLANE_LOCATION \
  --format="value(currentMasterVersion)"

# ノードプールのバージョンを確認
gcloud container clusters describe CLUSTER_NAME \
  --location=CONTROL_PLANE_LOCATION \
  --format="value(nodePools[].version)"

⚖️ メリットと制約事項

📈 メリット

ビジネス面

✓セキュリティコンプライアンスの維持 - PCI DSS や SOC 2 などの要件を満たす最新パッチ
✓ダウンタイムリスクの低減 - 定期更新により既知の脆弱性を放置しない

技術面

✓最新の Kubernetes 機能 - 1.35 系を含む最新 API へのアクセス
✓ノードイメージのセキュリティ強化 - カーネルレベルのセキュリティ修正
✓チャネル間のパッチ適用 - より新しいチャネルのパッチを手動で適用可能

🚨 制約事項・注意点

制限事項

⚠️Extended チャネルで延長サポート期間のマイナーバージョンは自動パッチが一時停止される場合あり
⚠️Extended チャネルは Autopilot、アルファクラスタ、一部マルチクラスタ機能では利用不可
⚠️チャネル未登録クラスタは GKE SLA 対象外の場合あり

考慮すべき点

💡ノードプールアップグレード時にノードが再作成される (永続ディスク以外のデータは消失)
💡小規模クラスタではリソース不足の可能性 (PDB の設定確認を推奨)
💡メンテナンスウィンドウの設定を推奨
💡バージョンスキューポリシー: CP とノードの差は 2 マイナーバージョン以内

🎯 ユースケース

📊 ユースケース 1: セキュリティパッチの計画的適用

シナリオ: 金融系サービスを運用する企業で、月次パッチ適用が求められている。Regular チャネルのクラスタに今回のセキュリティ修正を速やかに適用したい。
Bash メンテナンスウィンドウ設定 + アップグレード 
# メンテナンスウィンドウを設定 (土曜 2:00-6:00 JST)
gcloud container clusters update my-cluster \
  --location=asia-northeast1 \
  --maintenance-window-start=2026-04-12T17:00:00Z \
  --maintenance-window-end=2026-04-12T21:00:00Z \
  --maintenance-window-recurrence="FREQ=WEEKLY;BYDAY=SA"

# コントロールプレーンをアップグレード
gcloud container clusters upgrade my-cluster \
  --master \
  --location=asia-northeast1 \
  --cluster-version=1.34.5-gke.1208000

# ノードプールをアップグレード
gcloud container clusters upgrade my-cluster \
  --node-pool=default-pool \
  --location=asia-northeast1 \
  --cluster-version=1.34.5-gke.1208000
✨ 効果: COS の累積セキュリティ修正が適用され、コンプライアンス要件を満たしつつ、メンテナンスウィンドウにより業務時間外にアップグレードが実行されます。

🏥 ユースケース 2: Extended チャネルによる長期安定運用

シナリオ: 医療系システムで Kubernetes 1.32 を長期間使用する必要がある。セキュリティパッチは適用しつつ、マイナーバージョンのアップグレードは避けたい。
Bash Extended チャネル登録 + マイナーバージョン固定 
# Extended チャネルにクラスタを登録
gcloud container clusters update my-healthcare-cluster \
  --location=asia-northeast1 \
  --release-channel=extended

# マイナーバージョンアップグレードを除外
gcloud container clusters update my-healthcare-cluster \
  --location=asia-northeast1 \
  --add-maintenance-exclusion-name=no-minor-upgrades \
  --add-maintenance-exclusion-start=2026-04-09T00:00:00Z \
  --add-maintenance-exclusion-end=2027-04-09T00:00:00Z \
  --add-maintenance-exclusion-scope=no_minor_upgrades
✨ 効果: Extended チャネルにより最大 24 ヶ月間同一マイナーバージョンを維持しつつ、1.32.13-gke.1147000 のようなセキュリティパッチが自動的に適用されます。

💰 料金情報

💡 ポイント: GKE のバージョン更新自体に追加料金は発生しません。
→GKE クラスタ管理費: GKE Standard および Autopilot の通常のクラスタ管理費が適用
→Extended サポート料金: Extended チャネルで延長サポート期間に入ったマイナーバージョンを使用する場合、従量課金制の追加料金が発生

🌐 利用可能リージョン

GKE のバージョン更新は 全ての GKE 対応リージョンおよびゾーン で利用可能です。バージョンの展開は段階的に行われるため、一部のリージョンでは利用可能になるまでに時間差が生じる場合があります。

⚠️ 注意: 段階的ロールアウトのため、お使いのリージョンで最新バージョンが表示されるまでに数日かかる場合があります。gcloud container get-server-config コマンドで確認してください。
