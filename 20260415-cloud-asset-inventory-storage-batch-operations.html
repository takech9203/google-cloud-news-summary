🗂️ Cloud Asset Inventory: Storage Batch Operations API リソースタイプのサポート追加

Cloud Asset Inventory Storage Batch Operations 機能追加
2026-04-15
Google Cloud

📋 概要

Cloud Asset Inventory (CAI) において Storage Batch Operations API のリソースタイプ storagebatchoperations.googleapis.com/Job の一般公開サポートが開始されました。Cloud Storage のバッチ操作ジョブを CAI の主要な 7 つの API を通じて検索、エクスポート、監視できるようになります。

Storage Batch Operations は、Cloud Storage オブジェクトに対する大規模な一括操作 (ストレージクラスの変更、削除、メタデータ更新など) をジョブとして管理するサービスです。今回のアップデートにより、組織全体のストレージバッチ操作の可視性とガバナンスが大幅に向上します。

🔄 Before / After 比較

❌ Before

  • 🔴 各プロジェクトの Storage Batch Operations API を個別に呼び出す必要があった
  • 🔴 CAI のアセットインベントリにストレージバッチ操作ジョブが含まれなかった
  • 🔴 バッチ操作ジョブの IAM ポリシー監査や変更履歴追跡を CAI 経由で実施不可

✅ After

  • 🟢 SearchAllResources API で組織全体のバッチ操作ジョブを一括検索可能
  • 🟢 ExportAssets API で BigQuery / Cloud Storage にエクスポートし大規模分析可能
  • 🟢 Feed を設定してバッチ操作ジョブの作成・変更・削除をリアルタイム監視

⚡ 主要機能

🔍 7 つの CAI API での完全サポート

ExportAssets、ListAssets、BatchGetAssetsHistory、QueryAssets、Feed、SearchAllResources、SearchAllIamPolicies の全 API で Storage Batch Operations ジョブを操作可能。

ExportAssets ListAssets BatchGetAssetsHistory QueryAssets Feed SearchAllResources SearchAllIamPolicies

🌐 組織横断的な可視性

組織、フォルダ、プロジェクトの各レベルでスコープを指定してバッチ操作ジョブを検索・エクスポート可能。複数プロジェクトに分散するジョブを一元管理。タグやラベルによるフィルタリングにも対応。

📡 リアルタイム監視

Feed を使用して、バッチ操作ジョブの作成・完了・キャンセルをリアルタイムで検知。Pub/Sub 経由で Cloud Functions や Cloud Run と連携し、自動通知やアラートを構築可能。

🔒 コンプライアンス監査

BatchGetAssetsHistory で過去のジョブ状態の変遷を監査。SearchAllIamPolicies でバッチ操作に対するアクセス権限を組織全体で可視化し、最小権限の原則を徹底。

🛠️ 設定方法

前提条件

✓Cloud Asset API (cloudasset.googleapis.com) が有効化されていること
✓Storage Batch Operations API (storagebatchoperations.googleapis.com) が有効化されていること
✓適切な IAM ロール (roles/cloudasset.viewer など) が付与されていること

ステップ 1: Storage Batch Operations ジョブの検索

Bash 組織全体でジョブを検索 
# 組織全体で Storage Batch Operations ジョブを検索
gcloud asset search-all-resources \
  --scope="organizations/ORGANIZATION_ID" \
  --asset-types="storagebatchoperations.googleapis.com/Job" \
  --format="table(name, assetType, project, state)"

ステップ 2: BigQuery へのエクスポート

Bash BigQuery にエクスポート 
# Storage Batch Operations ジョブを BigQuery にエクスポート
gcloud asset export \
  --organization=ORGANIZATION_ID \
  --asset-types="storagebatchoperations.googleapis.com/Job" \
  --content-type="resource" \
  --bigquery-table="projects/PROJECT_ID/datasets/DATASET_ID/tables/TABLE_NAME" \
  --output-bigquery-force

ステップ 3: リアルタイム監視用 Feed の作成

Bash Pub/Sub トピックと CAI Feed の作成 
# Pub/Sub トピックの作成
gcloud pubsub topics create storage-batch-ops-feed

# CAI Feed の作成
gcloud asset feeds create storage-batch-ops-monitor \
  --organization=ORGANIZATION_ID \
  --asset-types="storagebatchoperations.googleapis.com/Job" \
  --content-type="resource" \
  --pubsub-topic="projects/PROJECT_ID/topics/storage-batch-ops-feed"

ステップ 4: IAM ポリシーの監査

Bash IAM ポリシーの検索 
# Storage Batch Operations に関連する IAM ポリシーを検索
gcloud asset search-all-iam-policies \
  --scope="organizations/ORGANIZATION_ID" \
  --query="policy:storagebatchoperations" \
  --format="table(resource, policy.bindings.role, policy.bindings.members)"

💡 ユースケース

🔍 ユースケース 1: 組織全体のバッチ操作の監査

シナリオ: セキュリティチームが、組織全体で実行された Storage Batch Operations ジョブを定期的に監査し、不正なオブジェクト削除操作が行われていないか確認したい。

Bash BigQuery へのエクスポートと削除ジョブの分析 
# 過去のバッチ操作ジョブを BigQuery にエクスポート
gcloud asset export \
  --organization=123456789 \
  --asset-types="storagebatchoperations.googleapis.com/Job" \
  --content-type="resource" \
  --snapshot-time="$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
  --bigquery-table="projects/audit-project/datasets/asset_audit/tables/batch_ops_jobs" \
  --output-bigquery-force
SQL BigQuery で削除ジョブを分析 
SELECT
    name,
    resource.data.state AS job_state,
    resource.data.createTime AS created,
    resource.data.description AS description
FROM `audit-project.asset_audit.batch_ops_jobs`
WHERE resource.data.deleteObject IS NOT NULL
ORDER BY resource.data.createTime DESC

効果: 組織全体のバッチ削除操作を一覧で把握でき、ポリシーに反する操作を迅速に検知して対応できる。

📡 ユースケース 2: バッチ操作ジョブのリアルタイムアラート

シナリオ: 運用チームが、大規模なバッチ操作ジョブが作成された際に即座に通知を受け取り、意図しない操作を早期に検知したい。

Bash 条件付き Feed の作成 
# Feed を作成して Pub/Sub に変更通知を送信
gcloud asset feeds create batch-ops-alert \
  --organization=123456789 \
  --asset-types="storagebatchoperations.googleapis.com/Job" \
  --content-type="resource" \
  --pubsub-topic="projects/ops-project/topics/batch-ops-alerts" \
  --condition-expression="temporal_asset.asset.resource.data.state == 'RUNNING'"

効果: バッチ操作ジョブが RUNNING 状態に遷移した時点で自動通知が発行され、Cloud Functions や Cloud Run と連携して Slack 通知やチケット作成を自動化できる。

🔒 ユースケース 3: IAM ポリシーの棚卸し

シナリオ: コンプライアンスチームが、Storage Batch Operations に対するアクセス権限を持つユーザーやサービスアカウントを組織全体で棚卸しし、最小権限の原則に従っているかを確認したい。

Bash IAM ポリシーの棚卸し 
# Storage Batch Operations に関連する IAM ポリシーを検索
gcloud asset search-all-iam-policies \
  --scope="organizations/123456789" \
  --query="policy:roles/storage.admin" \
  --format="json" | jq '[.[] | select(.resource | contains("storagebatchoperations"))]'

効果: 過剰な権限が付与されているアカウントを特定し、最小権限への是正を実施できる。

✅ メリット

📊 ビジネス面

→ガバナンスの強化: 組織全体のバッチ操作ジョブを一元的に把握し、不正操作やポリシー違反を迅速に検出
→コンプライアンス対応の効率化: BigQuery エクスポートで監査レポートの自動生成が実現し、規制対応の工数を削減
→運用コストの削減: 個別プロジェクトごとの手動確認が不要になり、運用チームの負荷が大幅に軽減

🔧 技術面

→統合的なアセット管理: 既存の CAI パイプラインに自動的に含まれるため、追加開発が不要
→リアルタイム検知: Feed + Cloud Functions で大規模バッチ削除操作の自動検知・アラート機能を構築可能
→SQL ベースの分析: QueryAssets API で BigQuery 互換 SQL によるジョブ分析やトレンド把握が容易

⚠️ デメリット・制約事項

制限事項
  • CAI の API クォータ (SearchAllResources: 100 QPS、ExportAssets: 1 回/プロジェクト/同時実行) に従う。大量ジョブはエクスポート方式を推奨
  • BatchGetAssetsHistory のタイムウィンドウに制限あり。非常に古い履歴は取得できない場合がある
  • 対象は storagebatchoperations.googleapis.com/Job のみ。個別の operations リソースは現時点で対象外
💡 考慮すべき点
  • Storage Batch Operations API と CAI の両方の有効化が必要。事前にサービス有効化状況を確認すること
  • エクスポートデータにはジョブのメタデータが含まれるが、個別オブジェクトの詳細は含まれない
  • 大規模組織では SearchAllResources のレスポンスサイズが大きくなる可能性があるため、フィルタ条件やページサイズの設定を検討すること

💲 料金情報

Cloud Asset Inventory の利用料金は API 呼び出し回数に基づきます。Storage Batch Operations ジョブのアセットタイプが追加されたことによる追加のサービス料金は発生しません。

API 料金 (概算)
ExportAssets無料 (BigQuery / GCS の保存料金は別途)
ListAssets無料
SearchAllResources無料
SearchAllIamPolicies無料
QueryAssets無料 (BigQuery スロットの利用料金は別途)
FeedPub/Sub のメッセージ料金が別途発生
✅ ポイント: CAI の主要 API は無料で利用可能。ただし、エクスポート先の BigQuery テーブルのストレージ料金、Pub/Sub のメッセージ料金、Cloud Storage のストレージ料金は通常通り発生します。

🌍 利用可能リージョン

Cloud Asset Inventory はグローバルサービスであり、すべての Google Cloud リージョンで利用可能です。Storage Batch Operations API のリソースはグローバルロケーション (locations/global) として管理され、CAI からのアクセスもリージョンの制約なく利用できます。

グローバル (全リージョン対応)

📝 まとめ

今回のアップデートにより、Storage Batch Operations API の Job リソースが Cloud Asset Inventory の 7 つの主要 API で完全にサポートされるようになりました。組織全体のストレージバッチ操作ジョブの可視性、監査、リアルタイム監視が CAI のエコシステム内で統合的に実現できます。

ストレージ管理やセキュリティガバナンスを担当するチームは、既存の CAI ベースの運用パイプラインにこの新しいリソースタイプを追加し、バッチ操作の監視体制を強化することを推奨します。
