# Cloud SQL for MySQL: MySQL 8.0.44 がデフォルトマイナーバージョンに

**リリース日**: 2026-03-26

**サービス**: Cloud SQL for MySQL

**機能**: MySQL 8.0.44 is now the default minor version

**ステータス**: Feature

[このアップデートのインフォグラフィックを見る](https://takech9203.github.io/google-cloud-news-summary/20260326-cloud-sql-mysql-8-0-44-default.html)

## 概要

Cloud SQL for MySQL 8.0 において、デフォルトのマイナーバージョンが MySQL 8.0.44 に更新されました。Cloud SQL では、MySQL 8.0 のデフォルトマイナーバージョンは最新マイナーバージョンの1つ前のバージョンに設定されるポリシーを採用しており、最新バージョンである 8.0.45 のリリースに伴い、8.0.44 が新たなデフォルトとなりました。

この変更により、今後マイナーバージョンを指定せずに新しい MySQL 8.0 インスタンスを作成した場合、MySQL 8.0.44 でプロビジョニングされます。また、自動マイナーバージョンアップグレードが有効なインスタンスは、次回の定期メンテナンス時に 8.0.44 へ自動的にアップグレードされます。

MySQL 8.0.44 には、セキュリティパッチやバグ修正、パフォーマンス改善が含まれており、Cloud SQL を利用する全ての MySQL 8.0 ユーザーに影響するアップデートです。

**アップデート前の課題**

- 以前のデフォルトマイナーバージョン(8.0.43 以前)では、最新のセキュリティパッチが適用されていなかった
- 新規インスタンス作成時に古いマイナーバージョンでプロビジョニングされる可能性があった
- MySQL コミュニティが修正済みのバグが Cloud SQL 環境で残存していた

**アップデート後の改善**

- 新規作成される MySQL 8.0 インスタンスが自動的に 8.0.44 でプロビジョニングされるようになった
- 自動マイナーバージョンアップグレードが有効なインスタンスは定期メンテナンス時に 8.0.44 へ自動アップグレードされる
- 最新のセキュリティパッチとバグ修正が適用された環境でデータベースを運用できる

## サービスアップデートの詳細

### 主要機能

1. **デフォルトマイナーバージョンの更新**
   - MySQL 8.0 の新規インスタンス作成時、バージョンを指定しない場合は 8.0.44 が適用される
   - Cloud SQL のポリシーにより、デフォルトは最新バージョン(8.0.45)の1つ前のバージョンに設定される

2. **自動マイナーバージョンアップグレード**
   - MySQL 8.0.35 以降を実行中で自動マイナーバージョンアップグレードが有効なインスタンスは、次回の定期メンテナンス時に 8.0.44 へ自動アップグレードされる
   - 自動アップグレードは定期メンテナンスイベント中にのみ実行され、セルフサービスメンテナンスでは実行されない

3. **手動アップグレードオプション**
   - 8.0.34 以前のバージョンを実行中のインスタンスは手動でのアップグレードが必要
   - 手動で 8.0.35 以降にアップグレード後、自動マイナーバージョンアップグレードを有効化可能

## 設定方法

### 前提条件

1. Google Cloud プロジェクトが作成済みであること
2. Cloud SQL Admin API が有効化されていること
3. 適切な IAM 権限（`cloudsql.instances.update`）を持っていること

### 手順

#### ステップ 1: 現在のバージョンを確認

```bash
gcloud sql instances describe INSTANCE_NAME --format="value(databaseVersion)"
```

インスタンスの現在の MySQL バージョンを確認します。

#### ステップ 2: 自動マイナーバージョンアップグレードの状態を確認

```bash
gcloud sql instances describe INSTANCE_NAME --format="json(settings.databaseFlags)"
```

自動マイナーバージョンアップグレードが有効かどうかを確認します。

#### ステップ 3: 手動でマイナーバージョンをアップグレード（必要な場合）

```bash
gcloud sql instances patch INSTANCE_NAME --database-version=MYSQL_8_0_44
```

自動アップグレードが無効な場合や、即座にアップグレードしたい場合に手動でバージョンを指定します。

#### ステップ 4: 自動マイナーバージョンアップグレードを有効化（オプション）

Google Cloud コンソールでインスタンスの編集画面を開き、「インスタンスのカスタマイズ」セクションで「自動マイナーバージョンアップグレードを有効にする」を選択して保存します。

## メリット

### ビジネス面

- **運用負荷の軽減**: 自動アップグレード機能により、DBA の手動作業を削減し、運用コストを低減できる
- **セキュリティコンプライアンスの維持**: 最新のセキュリティパッチが適用されたバージョンが自動的に展開されるため、コンプライアンス要件を満たしやすくなる

### 技術面

- **セキュリティ向上**: MySQL コミュニティによる最新のセキュリティ修正が含まれており、脆弱性リスクが低減される
- **安定性向上**: バグ修正とパフォーマンス改善により、データベースの安定性が向上する
- **Enterprise Plus でのほぼゼロダウンタイム**: Enterprise Plus エディションではマイナーバージョンアップグレード時のダウンタイムがほぼゼロ

## デメリット・制約事項

### 制限事項

- MySQL 8.0 ではマイナーバージョンのダウングレードはサポートされていない
- 自動マイナーバージョンアップグレードを一度有効にすると、無効化できない
- 自動アップグレードは定期メンテナンスイベント中のみ実行され、セルフサービスメンテナンスでは実行されない

### 考慮すべき点

- アップグレード前に MySQL 8.0.44 のリリースノートを確認し、アプリケーションへの影響がないか検証することを推奨
- 8.0.34 以前のインスタンスは手動アップグレードが必要であり、事前にテスト環境での検証が重要
- 本番環境へのアップグレード前に、メンテナンスウィンドウを適切に設定しておくことを推奨

## ユースケース

### ユースケース 1: 新規プロジェクトでのデータベース構築

**シナリオ**: 新しいアプリケーション開発プロジェクトで Cloud SQL for MySQL 8.0 インスタンスを新規作成する場合

**実装例**:
```bash
gcloud sql instances create my-new-instance \
  --database-version=MYSQL_8_0 \
  --tier=db-n1-standard-2 \
  --region=asia-northeast1
```

**効果**: バージョン指定なしで作成すると、自動的に MySQL 8.0.44 でプロビジョニングされ、最新のセキュリティパッチとバグ修正が適用された状態で運用を開始できる

### ユースケース 2: 既存インスタンスの自動アップグレード

**シナリオ**: 複数の Cloud SQL for MySQL 8.0 インスタンスを運用しており、手動でのバージョン管理の負荷を減らしたい場合

**効果**: 自動マイナーバージョンアップグレードを有効にすることで、次回の定期メンテナンス時に自動的に 8.0.44 へアップグレードされ、運用チームの負荷を大幅に軽減できる

## 料金

このマイナーバージョンの変更自体に追加費用は発生しません。Cloud SQL for MySQL の料金は、インスタンスの構成（vCPU、メモリ、ストレージ）やリージョンに基づいて課金されます。

なお、MySQL 8.0 のメジャーバージョンについては、2027年1月1日から Extended Support 期間に入り、追加料金が発生する可能性があります。Extended Support の追加料金を避けるためには、MySQL 8.4 へのメジャーバージョンアップグレードを検討してください。

## 利用可能リージョン

Cloud SQL for MySQL 8.0.44 は、Cloud SQL がサポートする全てのリージョンで利用可能です。日本では `asia-northeast1`（東京）および `asia-northeast2`（大阪）が利用できます。

## 関連サービス・機能

- **Cloud SQL for MySQL 8.4**: 2024年10月に GA となった最新メジャーバージョン。Enterprise Plus エディションがデフォルト
- **Cloud SQL Enterprise Plus エディション**: マイナーバージョンアップグレード時にほぼゼロダウンタイムを実現
- **Cloud SQL メンテナンスウィンドウ**: アップグレードが実行されるメンテナンスの日時を制御可能
- **Cloud SQL Auth Proxy**: インスタンスへの安全な接続を提供するプロキシツール
