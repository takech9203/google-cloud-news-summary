📋 概要

Database Migration Service (DMS) にQuick-Start Migrations機能が Preview として追加されました。この機能は、PostgreSQL の同種 (ホモジニアス) マイグレーションにおいて、VPC ネットワーク内でプライベート IP を持つソースデータベースから Cloud SQL for PostgreSQL および AlloyDB for PostgreSQL への移行を、軽量かつ継続的なマイグレーションフローで実現します。

最大の特徴は、マイグレーションに必要なセットアップ (接続プロファイル、ネットワーク構成、レプリケーション設定) を DMS が自動的に行う点です。従来の複数の手動ステップが不要になり、移行の初期設定にかかる時間と労力を大幅に削減できます。

❌ Before: 従来の DMS マイグレーション

  • ❌ソース接続プロファイルの手動作成が必要
  • ❌VPC ピアリングやプライベート接続構成の手動設定
  • ❌ファイアウォールルールの手動設定が必要

  • ✅ネットワーク構成を DMS が自動化
  • ✅プライベート IP 接続の自動確立
  • ✅レプリケーション設定の自動構成
  • ✅専門知識不要で開始可能
  • ✅AlloyDB / Cloud SQL 両方のコンソールから直接利用

⚡ 主要機能

🔧 自動セットアップ

→マイグレーションに必要な接続プロファイル、ネットワーク構成、レプリケーション設定を DMS が自動的に構成
→VPC ネットワーク内のプライベート IP を持つソースへの接続を自動的に確立
→従来必要だったプライベート接続構成の手動セットアップが不要

🔄 軽量な継続的マイグレーションフロー

→フルダンプによる初期データロード後、Change Data Capture (CDC) による継続的レプリケーションを実行
→ソースデータベースを稼働させたまま、最小限のダウンタイムでの移行が可能
→移行先の準備ができたタイミングで、マイグレーションジョブをプロモートして切り替え

🔗 AlloyDB / Cloud SQL for PostgreSQL への統合

→AlloyDB for PostgreSQL のコンソール/ワークフローから直接 Quick-Start Migrations を開始可能
→Cloud SQL for PostgreSQL のコンソール/ワークフローからも同様に利用可能
→ホモジニアス (同種) PostgreSQL マイグレーションに特化

Step 1: DMS コンソールにアクセス

Google Cloud コンソール > Database Migration Service > マイグレーションジョブ

AlloyDB または Cloud SQL for PostgreSQL のコンソールからも開始可能

Step 2: Quick-Start Migration の作成

Quick-Start Migrations を選択すると、DMS がソースデータベースの検出とネットワーク接続の自動構成を実行。接続プロファイルやプライベート接続構成を手動で作成する必要はありません。

Step 3: マイグレーションの実行と監視

Bash マイグレーションジョブのステータス確認 
gcloud database-migration migration-jobs describe MIGRATION_JOB_ID \
  --region=REGION

DMS がフルダンプを完了した後、CDC フェーズに移行し、ソースデータベースの変更を継続的にレプリケートします。

Step 4: マイグレーションのプロモート

ソースとデスティネーションが同期した状態で、マイグレーションジョブをプロモートしてアプリケーションを切り替えます。

Bash マイグレーションジョブのプロモート 
gcloud database-migration migration-jobs promote MIGRATION_JOB_ID \
  --region=REGION

⚖️ メリットとデメリット

✅ メリット

ビジネス面

  • ✅セットアップ自動化でマイグレーション計画から実行までの時間を大幅短縮
  • ✅手動作業が不要になり運用コストを削減
  • ✅移行ハードルが下がり、より多くのチームが自律的に実行可能

  • ✅VPC ピアリングやプライベート接続構成を DMS が自動処理
  • ✅CDC ベースの継続的レプリケーションで最小ダウンタイム移行
  • ✅AlloyDB / Cloud SQL 両方のコンソールから直接利用可能

⚠️ デメリット・制約事項

制限事項

  • ⚠️Preview 段階のため、SLA 対象外。機能変更・廃止の可能性あり
  • ⚠️ホモジニアス (PostgreSQL to PostgreSQL) のみ対応
  • ⚠️ソースは VPC 内プライベート IP 必須。パブリック IP のみのソースは非対応
  • ⚠️対象は Cloud SQL / セルフマネージド PostgreSQL (Compute Engine) に限定

考慮すべき点

  • 💡Pre-GA Offerings Terms が適用。本番利用前に利用規約を確認
  • 💡従来フローとの機能差分を事前に評価
  • 💡大規模 DB ではフルダンプの所要時間と帯域幅への影響を見積もり
  • 💡自動構成のネットワーク設定が組織のセキュリティポリシーに準拠しているか確認

💰 料金

Database Migration Service 自体のマイグレーションジョブ実行に対する追加料金はありません (ホモジニアスマイグレーションの場合)。ただし、関連リソースの費用が発生します。

項目 料金
DMS マイグレーションジョブ (ホモジニアス) 無料
Cloud SQL for PostgreSQL (デスティネーション) インスタンスタイプ・ストレージに応じた通常料金
AlloyDB for PostgreSQL (デスティネーション) vCPU・メモリ・ストレージに応じた通常料金
ネットワーク Egress リージョン間通信の場合、標準のネットワーク料金
Compute Engine (セルフマネージドの場合) VM インスタンスの通常料金

🌍 利用可能リージョン

Database Migration Service は完全にリージョナルなサービスです。マイグレーションに関連するすべてのエンティティ (ソース・デスティネーション接続プロファイル、マイグレーションジョブ) は単一のリージョンに保存される必要があります。

⚠️ 注意: Quick-Start Migrations は Preview 段階のため、利用可能リージョンに制限がある可能性があります。最新のリージョン情報は公式ドキュメントを参照してください。
