Overview

Cloud Database Migration Service に Model Context Protocol (MCP) Server が Preview として追加されました。AI Agent や AI Application から実行中の Migration Job の表示と管理が可能になります。

MCP は Anthropic が開発したオープンソースプロトコルで、LLM や AI Application が外部データソースに接続する方法を標準化しています。Google Cloud はリモート MCP Server としてエンタープライズレベルのガバナンス、セキュリティ、アクセス制御を備えた形で提供しています。

AI Integration: Gemini CLI、Claude、ChatGPT、その他カスタム AI Application から直接 Database Migration Service の機能にアクセスできるようになります。

Before / After

Before

  • Migration Job の監視・管理には Console / gcloud CLI / REST API を直接操作する必要
  • AI Agent やチャットボットから Migration 状況を確認する標準的な方法がなかった
  • 複数の Migration Job を横断的に管理する際、手動でのステータス確認が必要

After

  • AI Agent / AI Application から MCP 経由で Migration Job の表示・管理が可能
  • Gemini CLI、Claude、ChatGPT などから自然言語で Migration 状況を問い合わせ可能
  • カスタム AI Application に DMS の機能を容易に組み込み可能

Architecture

AI Client が MCP Protocol を通じて Database Migration Service の Remote MCP Server に接続し、IAM による認証と Model Armor によるセキュリティ検査を経て、Migration Job の表示・管理操作を実行します。

Setup

Step 1: Enable API

Bash Enable Database Migration Service API 
# Database Migration Service API を有効化
gcloud services enable datamigration.googleapis.com

Step 2: Grant IAM Roles

Bash IAM Role Assignment 
# MCP ツール呼び出し権限を付与
gcloud projects add-iam-policy-binding PROJECT_ID \
    --member="user:USER_EMAIL" \
    --role="roles/mcp.toolUser"

# Database Migration Service 管理者ロールを付与
gcloud projects add-iam-policy-binding PROJECT_ID \
    --member="user:USER_EMAIL" \
    --role="roles/datamigration.admin"

Step 3: Configure MCP Client

JSON MCP Client Configuration 
{
  "mcpServers": {
    "database-migration-service": {
      "url": "https://datamigration.googleapis.com/mcp",
      "transport": "http",
      "authentication": {
        "type": "oauth2",
        "credentials": "google-cloud-credentials"
      }
    }
  }
}

Step 4: Verify Available Tools

Bash List MCP Tools 
# MCP Server の利用可能なツールを確認
curl --location 'https://datamigration.googleapis.com/mcp' \
    --header 'content-type: application/json' \
    --header 'accept: application/json, text/event-stream' \
    --data '{
        "method": "tools/list",
        "jsonrpc": "2.0",
        "id": 1
    }'

Benefits

Business

->Operation Efficiency: AI Agent を活用した自然言語での Migration 管理により、専門知識がなくても Job の状況確認が可能
->Incident Response: AI Chatbot からリアルタイムで Migration 状況を把握し、問題の早期発見・対応が可能
->Developer Experience: IDE 内の AI Assistant から Migration 状況を確認でき、コンテキストスイッチが不要

Technical

->Standard Protocol: MCP というオープンスタンダードに準拠しており、特定のベンダーに依存しない
->Security: IAM ベースのきめ細やかなアクセス制御と Model Armor による Prompt / Response の保護
->Extensibility: カスタム AI Application に容易に統合でき、自動化ワークフローの構築が可能

Limitations / Considerations

Preview Limitations:
  • Preview 段階のため、本番環境での利用は推奨されない
  • Pre-GA 機能は「現状のまま」提供され、サポートが限定的
  • SLA の対象外であり、機能の変更や廃止の可能性がある
  • API Key による認証は非対応 (OAuth 2.0 が必須)
Considerations:
  • Preview から GA までの間に API やツールの仕様が変更される可能性がある
  • MCP Protocol 自体がまだ発展途上であり、仕様の変更に追従が必要
  • AI Agent からの操作にはセキュリティポリシーの慎重な設計が必要 (最小権限の原則の適用)
  • Model Armor の設定を適切に行わないと、意図しない操作がリクエストされるリスクがある

Use Cases

Use Case 1: AI Chatbot による Migration 監視

大規模な DB Migration Project において、チーム全員が Migration の進捗を把握する必要がある場合。Slack や Teams に統合された AI Chatbot から Migration 状況を問い合わせる。

User: 「現在実行中の Migration Job の状態を教えて」
AI Agent: (DMS MCP Server に接続して Job 一覧を取得)
「現在 3 つの Migration Job が実行中です:
1. mysql-to-cloudsql-prod: Progress 87% - CDC Replication
2. postgres-to-alloydb-staging: Progress 45% - Initial Snapshot
3. sqlserver-to-cloudsql-dev: Ready - Promotion Available」

Effect: 技術チーム以外のステークホルダーも Migration 進捗をリアルタイムで把握可能

Use Case 2: IDE 統合による開発者向け Migration 管理

開発者が VS Code や JetBrains IDE 内の AI Assistant (Gemini Code Assist、Claude) から Migration Job の状態確認や簡単な管理操作を行う。

Effect: コンテキストスイッチなしで Migration 管理が可能になり、開発効率が向上

Use Case 3: 自動化された Migration 運用パイプライン

カスタム AI Agent が MCP Server を通じて Migration Job を監視し、条件に基づいて自動的にアクション (通知、プロモーション提案など) を実行する。

Effect: Migration 運用の自動化により、人的ミスの削減と対応速度の向上を実現

MCP Server 自体の利用料金については、Preview 段階では明示的な料金体系は公開されていません。GA リリース時に料金体系が更新される可能性があります。

Region Availability

Database Migration Service MCP Server の具体的なリージョン制限は Preview 時点では公開されていません。Google Cloud の Remote MCP Server は Global / Regional Endpoint として提供されるため、Database Migration Service が利用可能なリージョンで利用できる可能性が高いです。
