📋 概要

Gemini Code Assist の VS Code 拡張機能において、File Outline (ファイルアウトライン) と Finish Changes (変更の完了) の 2 つの機能が一般提供 (GA) となった。IntelliJ では既に GA だったこれらの機能が VS Code でも利用可能になり、マルチ IDE 対応がさらに強化された。

🤖 AI コーディング支援の進化: File Outline はコードブロックの英語サマリーを AI で自動生成し、Finish Changes は擬似コードや TODO から残りの実装を自動補完する。複雑なプロンプトなしで AI ペアプログラマーの支援を受けられる。

🔄 Before / After 比較

❌ Before

  • VS Code では File Outline と Finish Changes が利用不可
  • コード全体構造の把握に手動スクロールが必要
  • 作業途中のコードは全て手動で記述
  • IntelliJ ユーザーとの機能格差

✅ After

  • VS Code でも両機能が GA として利用可能
  • AI 生成サマリーで構造を瞬時に把握
  • 擬似コードや TODO から AI が自動補完
  • IntelliJ と VS Code で同等の AI 支援

🔑 主要機能

🗂️ File Outline

  • コードブロックごとに AI が短い英語サマリーを自動生成
  • 抽象レベルでファイル構造を理解
  • 自動生成の無効化・手動モードへの切替が可能
  • コードレビューやオンボーディング時に特に有用

✨ Finish Changes

  • AI ペアプログラマーとして作業中のコードを観察
  • 擬似コード、#TODO、半分書きかけのコードに対応
  • 複雑なプロンプト不要で高レベル設計に集中
  • コードの意図を理解し一貫性のある補完を提供

🚀 設定方法

📋 前提条件: Gemini Code Assist のサブスクリプション (Standard / Enterprise / 個人向け) が設定済みであること

ステップ 1: 拡張機能の更新

VS Code の拡張機能マーケットプレイスから Gemini Code Assist を最新バージョンに更新する。

ステップ 2: File Outline の使用

サイドバーの Gemini Code Assist パネルから Outline タブを開く。コードファイルを開くと自動的に各コードブロックのサマリーが生成される。

ステップ 3: Finish Changes の使用

コードファイル内で変更を開始し、擬似コード、TODO コメント、または部分的なコードを記述。Gemini Code Assist が残りの変更を補完する。

⚠️ 制限事項・注意点

⚠️ 制限事項:
  • File Outline のサマリーは英語のみで生成される
  • AI による生成結果は非決定的であり、他プラグインとの同時使用で挙動が変わる可能性あり
  • Gemini Code Assist のライセンスが必要 (無料プランではクォータに制限あり)
💡 考慮すべき点:
  • AI が生成したコードや補完結果は必ず開発者自身で検証すること
  • Finish Changes の精度はコードのコンテキストや記述スタイルに依存
  • 機密性の高いコードを扱う場合はデータガバナンスポリシーを確認
