⚙️ Google Distributed Cloud 1.34.200-gke.68

VMware / bare metal 向けパッチリリース & バグ修正

2026-03-18
GA GKE Enterprise

📄 概要

Google Distributed Cloud (software only) for VMware および for bare metal の バージョン 1.34.200-gke.68 がリリースされた。本バージョンは Kubernetes v1.34.3-gke.400 上で動作し、VMware 版では 2 つの重要なバグ修正が含まれている。

🔄 Before / After 比較

🔌 Bug Fix 1: Docker ブリッジ IP レンジ

❌ Before

VMware V2 (Advanced Clusters) v1.31 以前で Docker デフォルトブリッジ IP の設定が欠落

IP レンジ: 172.17.0.0/16 (デフォルト)

顧客ネットワークと重複 → 接続障害発生の可能性

✔️ After

Docker デフォルトブリッジ IP を明示的に設定

IP レンジ: 169.254.123.1/24 (リンクローカル)

顧客ネットワークとの IP 競合リスクを根本的に解消

🔌 Bug Fix 2: 管理クラスタアップグレードのスタック

❌ Before

管理クラスタアップグレード時に VSphereMachine が Creating フェーズで無期限にスタック

アクションにつながるエラーメッセージが表示されない

✔️ After

VSphereMachine のスタック問題を修正

アップグレードプロセスの信頼性が向上し、メンテナンスウィンドウ内での作業完了が確実に

⚡ 主要機能

📦 VMware 版リリース

GDC for VMware 1.34.200-gke.68 がダウンロード可能。Kubernetes v1.34.3-gke.400 上で動作。

→ サードパーティストレージベンダー使用時は GDC Ready パートナーの認定状況を確認

📦 bare metal 版リリース

GDC for bare metal 1.34.200-gke.68 も同時にダウンロード可能。VMware 版と同じ Kubernetes バージョン上で動作。

🔧 Docker ブリッジ IP 修正

VMware V2 (Advanced Clusters) v1.31 以前で欠落していた Docker デフォルトブリッジ IP レンジの設定を修正。169.254.123.1/24 を明示的に設定。

🔧 アップグレードスタック修正

管理クラスタアップグレード時の VSphereMachine Creating フェーズ停止問題を修正。エラーメッセージなく無期限スタックする問題を解消。

🛠️ アップグレード手順

📋 前提確認
→
📤 バンドル準備
→
⚙️ 管理クラスタ
→
🔧 ユーザークラスタ
💡 前提条件:
  • gkectl のバージョンがターゲットバージョンと一致していること
  • 管理ワークステーションが適切にセットアップされていること
  • サードパーティストレージベンダー使用時は GDC Ready パートナーの認定状況を確認

Step 1: バンドルのダウンロードと準備 (VMware)

Bash 
gkectl prepare \
  --bundle-path /var/lib/gke/bundles/gke-onprem-vsphere-1.34.200-gke.68.tgz \
  --kubeconfig ADMIN_CLUSTER_KUBECONFIG

Step 2: 管理クラスタのアップグレード (VMware)

Bash 
gkectl upgrade admin \
  --kubeconfig ADMIN_CLUSTER_KUBECONFIG \
  --config ADMIN_CLUSTER_CONFIG_FILE

Step 3: ユーザークラスタのアップグレード (VMware)

Bash 
# まず dry-run でプリフライトチェックを実行
gkectl upgrade cluster \
  --kubeconfig ADMIN_CLUSTER_KUBECONFIG \
  --config USER_CLUSTER_CONFIG \
  --dry-run

# 問題がなければ本番アップグレード実行
gkectl upgrade cluster \
  --kubeconfig ADMIN_CLUSTER_KUBECONFIG \
  --config USER_CLUSTER_CONFIG

Step 4: bare metal クラスタのアップグレード

bare metal 環境では bmctl を使用してアップグレードを実行する。詳細は公式ドキュメントの「Upgrade clusters」を参照。

🎯 メリット

💼 ビジネス面

✓ 運用リスクの低減 - Docker ブリッジ IP 競合による予期しない接続障害のリスクが解消
✓ アップグレード作業の効率化 - 管理クラスタアップグレード時のスタック問題が解消され、メンテナンスウィンドウ内での作業完了の確実性が向上

💻 技術面

✓ ネットワーク競合の根本的解決 - リンクローカルアドレス (169.254.123.1/24) で顧客ネットワークとの IP 空間競合を根本的に回避
✓ 最新 Kubernetes 基盤 - Kubernetes v1.34.3-gke.400 ベースにより最新のセキュリティ修正と機能改善を含む

⚠️ 制約事項・注意点

⚠️ 制限事項:
  • GKE On-Prem API クライアント (Google Cloud コンソール、gcloud CLI、Terraform) で利用可能になるまでに約 7-14 日の遅延がある
  • サードパーティストレージベンダー使用時はベンダーの認定取得状況を事前に確認する必要がある
💡 考慮すべき点:
  • 管理クラスタを先にアップグレードし、その後ユーザークラスタをアップグレードする順序を守ること
  • Advanced Clusters へのアップグレードでは gkectl のバージョンがターゲットバージョンと完全に一致している必要がある

📝 まとめ

Google Distributed Cloud (software only) 1.34.200-gke.68 は、VMware 環境と bare metal 環境の両方を対象としたパッチリリース。特に VMware 版では Docker ブリッジ IP 競合と管理クラスタアップグレードのスタックという 2 つの重要なバグ修正が含まれている。

オンプレミスや edge 環境で Google Distributed Cloud を運用しているユーザーは、リリースノートの内容を確認のうえ、計画的なアップグレードを検討することを推奨する。
