Gemini Enterprise: Dropbox フェデレーテッドデータストア

Gemini Enterprise GA Federated Search
2026-04-07
Google Cloud Release Notes

☁️ 概要

Gemini Enterprise の Dropbox フェデレーテッドデータストアが一般提供 (GA) となりました。 Dropbox に保存されたファイルやフォルダを、データコピーなしでリアルタイムに検索し、Google ドライブや SharePoint などの他のデータソースと統合した包括的な回答を提供できます。

さらに、フォルダ作成やファイルのアップロード・ダウンロード・コピーといった操作も、Gemini Enterprise のアシスタントから自然言語で実行可能です。

⭐ 主要機能

🔍 1. フェデレーテッド検索

→Dropbox に保存されたファイルやフォルダをリアルタイムに検索
→データを Gemini Enterprise 側にコピーせず、Dropbox API を直接呼び出して結果を取得
→他の接続済みデータソースの結果と統合 (ブレンド) して包括的な検索結果を提供

⚡ 2. アクション実行

Gemini Enterprise アシスタントから自然言語で Dropbox 操作を実行可能:

📁
フォルダ作成
Dropbox にフォルダを新規作成
📋
ファイル / フォルダのコピー
Dropbox 内のアイテムをコピー
⬆️
ファイルのアップロード
Dropbox にファイルを送信
⬇️
ファイルのダウンロード
Dropbox からファイルを取得

🤖 3. LLM によるクエリリライト

→検索精度向上のため、LLM がユーザーのクエリを Dropbox API に送信する前にリライト
→セッション内のクエリ履歴を考慮したコンテキスト対応のリライトを実行

🔒 4. セキュリティとアクセス制御

→パーミッションアウェア検索: ユーザーがアクセス権を持つデータのみを結果に表示
→Cloud KMS によるカスタマー管理暗号鍵 (CMEK) のサポート
→VPC Service Controls によるネットワークセキュリティ (制約あり)

Step 1: Dropbox App Console でアプリを作成

1.Dropbox App Console にアクセスし Create app をクリック
2.Scoped access を選択し、Full Dropbox アクセスを指定
3.アプリ名を入力し、利用規約に同意して作成

Step 2: Dropbox アプリの認証設定

1.Settings タブで Enable additional users をクリック
2.App key と App secret をコピー
3.OAuth 2 Redirect URI を追加:
Redirect URIs 
https://vertexaisearch.cloud.google.com/console/oauth/default_oauth.html
https://vertexaisearch.cloud.google.com/oauth-redirect
4.Permissions タブで必要なスコープを選択し Submit

Step 3: Gemini Enterprise でデータストアを作成

1.Google Cloud Console で Gemini Enterprise ページに移動
2.Data stores > Create data store > Dropbox を選択
3.コネクタモードで Federated search を選択
4.App key / App secret を入力してログイン
5.エンティティタイプ、アクション、リージョン、暗号化、課金プランを設定して作成

Step 4: アプリの作成と接続

1.Gemini Enterprise でアプリを作成
2.作成したアプリに Dropbox データストアを接続
3.ユーザーに Dropbox への認可を設定

⚖️ メリット・デメリット

✅ メリット

💼 ビジネス面

  • ✔️ 分散データの統合検索 -- Dropbox, Google ドライブ, SharePoint を一元検索
  • ✔️ 業務効率の向上 -- 自然言語でファイル操作、ツール切替を削減
  • ✔️ 本番運用の信頼性 -- GA により SLA とエンタープライズサポートが適用

🔧 技術面

  • ✔️ データコピー不要 -- ストレージコスト抑制しつつ最新データにアクセス
  • ✔️ セキュリティ -- パーミッションアウェア検索、CMEK、OAuth 2.0
  • ✔️ 柔軟な暗号化 -- Google 管理鍵 + Cloud KMS を選択可能

⚠️ デメリット・制約事項

⛔ 制限事項

  • • global / us / eu の 3 リージョンのみサポート
  • • 既存データストアへの VPC Service Controls ペリメーター適用は非サポート (再作成が必要)
  • • 1 データストアに 1 コネクタタイプのアクションのみ推奨

💭 考慮すべき点

  • • インデックスなしのため検索品質がインジェスト方式より低い可能性
  • • クエリ文字列が Dropbox API に送信 (サードパーティプライバシーポリシー適用)
  • • LLM リライトでセッション履歴の一部が Dropbox API に送信される可能性
  • • Dropbox 側でクエリがユーザー ID と関連付けられる可能性

💡 ユースケース

🌐 ユースケース 1: マルチプラットフォームのナレッジ統合検索
シナリオ: 大企業で部門ごとに Google ドライブ、SharePoint、Dropbox を併用。社員が業務に必要な情報を探す際に各プラットフォームを個別に検索する必要がある。
✅ Gemini Enterprise に全データソースを接続し、1 つの検索インターフェースから組織全体のナレッジを横断検索。情報検索時間を大幅に削減し、部門間の情報共有を促進。
🤖 ユースケース 2: AI アシスタントによるファイル操作の自動化
シナリオ: 営業チームが Dropbox 上のプロジェクトフォルダに対して、自然言語で「今月の提案書を新しいフォルダにまとめて」といった指示を出す。
✅ アクション機能でフォルダ作成やファイルコピーを自然言語実行。Dropbox UI に切り替えることなく AI アシスタントからファイル管理を完結。
🔒 ユースケース 3: コンプライアンス対応の情報探索
シナリオ: コンプライアンス部門が複数のクラウドストレージに分散した規程文書や監査証跡を、パーミッションアウェアな検索で安全に参照。
✅ 各ユーザーのアクセス権限に基づいた検索結果のみを返却。情報漏洩リスクを最小化しつつ必要な文書に迅速アクセス。CMEK で追加セキュリティ要件にも対応。

Dropbox データストアは以下のリージョンでサポートされています。

🌏 global (推奨)
🇺🇸 us (米国)
🇪🇺 eu (欧州)
⚠️ 注意: us または eu を選択した場合は、暗号化設定 (Google 管理暗号鍵 または Cloud KMS) の構成が必要です。
