📋 概要

Security Command Center に、AI エージェント (Agent Platform / Vertex AI Agent Engine) のセキュリティを強化するための新しい事前定義ルールとコントロールが追加されました。4 つの検出レイヤーで Agent Runtime 向けのサポートが拡充され、さらに Gemini Enterprise Agent Platform の「セキュリティ」タブから AI セキュリティ Findings を直接確認できるようになりました。

🔄 Before / After 比較

❌ Before (アップデート前)

  • AI エージェント固有の脅威に対する専用検出ルールが限定的
  • Agent Runtime の CVE リスクや過剰権限を網羅的に検出不可
  • AI セキュリティ Findings の確認に SCC コンソールへの移動が必要
  • 複数の検出サービスにまたがる AI 脅威の一元把握が困難

✅ After (アップデート後)

  • 4 層の検出レイヤーで AI エージェントの脅威をプロアクティブに検出
  • CVE リスク、過剰権限、SA 偽装などを自動検出
  • Agent Platform のセキュリティタブから直接 Findings を確認可能
  • 相関脅威分析で多段階攻撃を一元的に検出・対応

🔍 4 つの検出レイヤー

多層防御により AI エージェントのセキュリティをリアルタイムで保護します。

📊

1. セキュリティグラフルール

Agent Runtime 対応

  • CVE リスクの自動検出
  • サービスアカウントの過剰権限検出
  • SA 偽装によるアクセス検出
  • 高価値リソースへの直接アクセスリスク評価
  • 公開 Cloud Storage バケットの検出
🔗

2. 相関脅威ルール

Agent Runtime 対応

  • マルウェア URL + リバースシェルの相関分析
  • 複数シグナルの統合による多段階攻撃検出
  • 単一シグナルでは困難な攻撃パターンを識別
⚡

3. Agent Platform Threat Detection

ランタイム検出器の拡充

  • 悪意あるバイナリ/ライブラリの実行検出
  • コンテナエスケープ、リバースシェル検出
  • NLP ベースの Python/Bash 悪意検出
  • Kubernetes 攻撃ツール検出
  • ファイルレス実行 (/memfd:) 検出
  • 暗号通貨マイニング検出
📝

4. Event Threat Detection

AI エージェント専用ルール

  • エージェント ID への異常な権限付与
  • エージェント認証情報の外部利用
  • AI エージェントによる機密ロール付与
  • Token Creator ロール付与の検出
  • データ流出 (BigQuery, CloudSQL, GCS)
  • 権限昇格 (signJwt, 暗黙的委任)

Step 1: Agent Platform Threat Detection の有効化確認

デフォルトで有効化されていますが、無効の場合は以下のコマンドで有効化します。

Bash Agent Platform Threat Detection の有効化 
# Agent Platform Threat Detection の状態を確認
gcloud scc manage services update agent-engine-threat-detection \
  --organization=ORGANIZATION_ID \
  --enablement-state=ENABLED

Step 2: AI Protection の有効化

Google Cloud コンソールで AI Protection の Service Enablement ページに移動し、「Activate」をクリック。

Step 3: Findings の確認

SCC の Findings ページで「Source display name」フィルタから「Agent Platform Threat Detection」を選択。

Step 4: Agent Platform セキュリティタブの確認

Gemini Enterprise Agent Platform のセキュリティタブにアクセスし、AI セキュリティ Findings、アクティブな脅威、攻撃パスシミュレーションを確認。

💡 ユースケース

🗂️ ユースケース 1: AI エージェントのデータ流出検出

シナリオ: 企業が Vertex AI Agent Engine で顧客データにアクセスする AI エージェントを運用。悪意ある第三者がエージェントを悪用し、BigQuery の機密データを組織外の Cloud Storage バケットにエクスポートしようとする。

Bash Findings の確認コマンド 
# Agent Platform Threat Detection が自動検出する Finding の例
# Category: Exfiltration: AI Agent Initiated BigQuery Data Extraction
# API Name: AGENT_ENGINE_BIG_QUERY_EXFIL_TO_CLOUD_STORAGE

# Findings の確認コマンド
gcloud scc findings list ORGANIZATION_ID \
  --source=AGENT_PLATFORM_THREAT_DETECTION_SOURCE_ID \
  --filter="category=\"AGENT_ENGINE_BIG_QUERY_EXFIL_TO_CLOUD_STORAGE\""

→ 効果: AI エージェント経由のデータ流出をニアリアルタイムで検出し、迅速なインシデント対応が可能に。

🔑 ユースケース 2: AI エージェントの権限昇格検出

シナリオ: AI エージェントに関連付けられたサービスアカウントが、signJwt メソッドを使用して別のサービスアカウントのアクセストークンを不正に生成しようとするケース。

→ 効果: ETD の AGENT_ENGINE_SUSPICIOUS_TOKEN_GENERATION_SIGN_JWT ルールが権限昇格の試みを即座に検出。攻撃パスシミュレーションにより侵害の影響範囲を事前に把握可能。

🔗 ユースケース 3: 多段階攻撃の相関分析

シナリオ: AI エージェントで悪意ある URL が観測された後、同じエージェントからリバースシェルが検出されるケース。

→ 効果: 相関脅威ルール「Multiple correlated threat signals of malicious software」がシグナルを統合し、単一の高優先度 Finding として報告。個別には低重要度でも、相関分析により深刻な多段階攻撃として適切に優先順位付け。

✅ メリット

ビジネス面

✓ AI ワークロードのリスク低減 - エージェント型 AI の脅威をプロアクティブに検出・対応。データ流出や不正アクセスのリスクを低減。
✓ コンプライアンス対応の強化 - AI セキュリティの Google 推奨ベストプラクティスへの準拠状況を可視化し、監査対応を効率化。

技術面

✓ 多層防御の実現 - ランタイム検出、コントロールプレーン検出、セキュリティグラフ分析、相関脅威分析の 4 層で AI エージェントを保護。
✓ NLP ベースの高度な検出 - Bash/Python スクリプトの悪意あるコードを NLP で分析し、難読化された攻撃も検出可能。
✓ リアルタイム検出 - watcher プロセスでリアルタイムにテレメトリを収集・分析し、ニアリアルタイムで Finding を生成。
✓ 攻撃パスの可視化 - セキュリティグラフを活用した攻撃パスシミュレーションにより、潜在的なリスクの伝播経路を可視化。

⚠️ デメリット・制約事項

制限事項
  • Security Command Center Premium または Enterprise ティアが必須 (無料の Standard ティアでは利用不可)
  • Agent Platform Threat Detection は現在 Preview ステータスの検出器が多く、Pre-GA の制約が適用される
  • watcher プロセスの起動に最大 1 分の遅延が発生する可能性がある
  • プロジェクトレベルのアクティベーションでは一部の Finding (クロスプロジェクトトークン生成など) が利用不可
  • 収集データはメモリ内処理のため、インシデント未報告時の事後フォレンジック分析が制限される
考慮すべき点
  • 組織レベルでの SCC アクティベーションが推奨 (プロジェクトレベルでは機能制限あり)
  • ETD コントロールプレーン検出器は Cloud Audit Logs の有効化が前提
  • 大量の AI エージェント環境では Finding 量が増大するため、適切なミュートルール/フィルタリング設定が重要
