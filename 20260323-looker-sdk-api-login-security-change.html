🔒 Looker SDK / API /login クレデンシャル送信方法の変更

Looker Breaking Change 2026年10月適用
2026-03-23 発表

📋 概要

Looker は、/login エンドポイントにおけるクレデンシャル (client_id / client_secret) の送信方法を変更します。今後、クレデンシャルは HTTP リクエストボディでのみ受け付けられ、URL クエリパラメータでの送信は廃止されます。

この変更は Looker 26.18 リリース (2026年10月予定) で適用されます。現在 URL クエリパラメータを使用しているスクリプトやアプリケーションは、この期限までに移行が必要です。

⚠️ 重要: 影響を受けるのは、Looker SDK を使用しているすべての顧客、カスタムスクリプト、および /login API エンドポイントを直接呼び出しているアプリケーションです。公式 SDK (Python, TypeScript, Ruby) に加え、コミュニティ SDK (Kotlin, Swift, R) も対象です。

Before / After

❌ Before (非推奨 - 26.18 で廃止)

✗ クレデンシャルが URL に含まれる
✗ サーバーログ/ブラウザ履歴に記録されるリスク
✗ リファラヘッダーで第三者に漏洩するリスク
✗ プロキシログで平文記録の危険

✅ After (SDK 26.4 以降 / 推奨)

✓ クレデンシャルはボディ内に格納
✓ URL を通じた漏洩リスクが排除
✓ ログへのクレデンシャル記録リスクが低減
✓ OAuth 2.0 セキュリティベストプラクティスに準拠

🔑 主要な変更点

🚫 URL クエリパラメータでの送信廃止

/login エンドポイントへの client_id と client_secret の URL クエリパラメータ経由の送信が無効化されます。Looker 26.18 (2026年10月) 適用後、URL クエリパラメータを使用した認証リクエストは失敗します。

📦 HTTP リクエストボディへの一本化

Content-Type: application/x-www-form-urlencoded 形式でボディにクレデンシャルを送信する方式に統一されます。SDK バージョン 26.4 以降ではこの方式がデフォルトで採用済みです。

🔄 Looker SDK の更新

公式 SDK (Python, TypeScript, Ruby) のバージョン 26.4 以降で対応済み。コミュニティ SDK (Kotlin, Swift, R, C#, Go) も sdk-codegen 経由で更新が提供されます。

Step 3: カスタムスクリプトの修正

Python 
# 変更前 (非推奨)
import requests
response = requests.post(
    f"{base_url}/api/4.0/login?client_id={client_id}&client_secret={client_secret}"
)

# 変更後 (推奨)
import requests
response = requests.post(
    f"{base_url}/api/4.0/login",

        "client_secret": client_secret
    }
)

⚠️ デメリット・制約事項

制限事項:
  • Looker 26.18 適用後、URL クエリパラメータを使用した認証は完全に動作しなくなる (後方互換性なし)
  • SDK 26.4 より前のバージョンを使用し続ける場合、26.18 リリース後に認証が失敗する
考慮すべき点:
  • 本番環境のすべてのスクリプト・アプリケーションの棚卸しが必要
  • サードパーティ製ツールや連携アプリが URL クエリパラメータ方式を使用している場合、ベンダーへの確認が必要
  • CI/CD パイプラインで Looker API を呼び出している場合、パイプライン設定の見直しが必要

💻 ユースケース 2: curl を使用した直接 API 呼び出しの修正

シナリオ: インフラチームが cron ジョブで curl コマンドを使用し、Looker API の /login エンドポイントを直接呼び出してトークンを取得。

Bash 
# 変更後の curl コマンド
ACCESS_TOKEN=$(curl -s -X POST \
  "https://your-instance.looker.com/api/4.0/login" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "client_id=${LOOKER_CLIENT_ID}&client_secret=${LOOKER_CLIENT_SECRET}" \
  | jq -r '.access_token')

# 取得したトークンを使用して API を呼び出し
curl -H "Authorization: token ${ACCESS_TOKEN}" \
  "https://your-instance.looker.com/api/4.0/user"
✅ 効果: スクリプトの修正箇所は /login の呼び出し部分のみ。取得したアクセストークンの使用方法に変更はなし。
