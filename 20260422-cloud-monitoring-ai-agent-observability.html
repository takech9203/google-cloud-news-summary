📋 概要

Google Cloud の Application Monitoring に、AI エージェントのオブザーバビリティ機能と AI リソース監視機能が GA (一般提供) として追加されました。AI エージェントや MCP サーバーを含むアプリケーションを運用するチームに向けた重要なアップデートです。

🤖 ポイント: App Hub でサービスやワークロードを登録する際に「Agent」や「MCP server」といった Functional Type を設定するだけで、Application Monitoring が自動的に AI エージェント固有のダッシュボードとメトリクスを生成します。

対象ユーザーは、Vertex AI Agent Engine、ADK (Agent Development Kit) ベースのエージェント、MCP サーバーなど、AI リソースを本番環境で運用しているプラットフォームチーム、SRE チーム、および AI/ML エンジニアです。

🔄 Before / After

❌ Before

  • AI エージェントのパフォーマンスメトリクス (トークン使用量、エラーレート) を統合的に確認できる専用ダッシュボードが存在しなかった
  • エージェントや MCP サーバーを他のワークロードと区別して管理・監視する仕組みがなく、Functional Type による分類ができなかった
  • AI リソースの健全性を把握するには、Metrics Explorer で個別にメトリクスを組み合わせて確認する必要があった

✅ After

  • Application Monitoring ダッシュボードで AI リソースのエラーレート、トークン使用量などのパフォーマンスメトリクスを一元的に確認可能
  • App Hub の Functional Type (Agent / MCP server) によるフィルタリングで、AI エージェントと MCP サーバーを区別して監視可能
  • AI エージェントの推論トレース、ツール呼び出し、モデル出力などの内部状態を外部テレメトリから測定可能

💻 コードサンプル

Cloud Monitoring API を使用して、AI エージェントのメトリクスを取得する例です。

Bash エージェントメトリクスの定義を取得 
# エージェントメトリクスの定義を取得
gcurl https://monitoring.googleapis.com/v3/projects/PROJECT_ID/metricDescriptors?filter='metric.type=starts_with("aiplatform.googleapis.com/reasoning_engine")'
Bash 特定のエージェントインスタンスのリクエスト数を取得 
# 特定のエージェントインスタンスのリクエスト数を取得
gcurl "https://monitoring.googleapis.com/v3/projects/PROJECT_ID/timeSeries?filter=metric.type%3D%22aiplatform.googleapis.com/reasoning_engine/request_count%22%20AND%20resource.labels.reasoning_engine_id%3D%22RESOURCE_ID%22&interval.endTime=2026-04-22T12:00:00Z&interval.startTime=2026-04-22T11:00:00Z"

⚙️ 設定方法

📝 前提条件:
  • • App Hub のアプリケーション管理境界 (Application Management Boundary) が設定済み
  • • Cloud Monitoring API が有効化済み
  • • 適切な IAM ロール (roles/monitoring.viewer 以上) が付与済み

Step 1: App Hub でアプリケーションを定義

App Hub または Application Design Center を使用して、アプリケーションを定義し、AI リソースをサービスまたはワークロードとして登録します。登録時に Functional Type を「Agent」または「MCP server」に設定します。

Bash gcloud CLI でアプリケーションを作成 
# gcloud CLI でアプリケーションを作成 (例)
gcloud app-hub applications create my-ai-app \
    --location=us-central1 \
    --display-name="My AI Application"

→自動ダッシュボード生成: App Hub でリソース登録するだけで、追加設定なしにダッシュボードが自動生成
→OpenTelemetry 対応: 標準的な計装と互換性があり、既存の監視パイプラインと統合可能
→統合テレメトリ: ログ、メトリクス、トレースを一つのビューで相関分析

⚠️ 制約事項・注意点

⚠️ 制限事項:
  • ✖ Application Monitoring を利用するには App Hub でのアプリケーション定義とリソース登録が前提
  • ✖ サポート対象リージョンごとの Discovered サービス/ワークロードは各 100 件が上限
  • ✖ Vertex AI Studio からのコンソール利用分のメトリクスはダッシュボードに含まれない (API 呼び出し経由のメトリクスのみ)
📝 考慮すべき点:
  • • 既存のインフラストラクチャを App Hub に登録する移行作業が必要な場合がある
  • • Functional Type (Agent / MCP server) の設定は手動で行う必要があり、自動分類ではない
  • • ビルトインメトリクスは Vertex AI Agent Engine のマネージドリソースが対象 (セルフホストのモデルは含まれない)

💰 料金

✅ Application Monitoring 自体には追加料金は発生しません。

ただし、テレメトリデータの収集・保存に Cloud Monitoring の標準料金が適用されます。

→システムメトリクス: 無料
→カスタムメトリクス / 外部メトリクス: バイト数またはサンプル数に基づく課金
→アラートポリシー: メトリクス参照ごとの料金が発生
