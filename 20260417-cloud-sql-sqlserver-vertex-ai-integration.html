🗄️ Cloud SQL for SQL Server: Vertex AI 統合によるベクトルエンベディング生成

Cloud SQL for SQL Server Preview Vertex AI 統合
2026-04-17

📋 概要

Cloud SQL for SQL Server と Vertex AI およびサードパーティモデルとの統合が Preview として利用可能になりました。SQL Server 2025 インスタンスから直接、Vertex AI でホストされているモデルを使用してベクトルエンベディングを生成できます。

🤖 ポイント: SQL Server 2025 で導入されたネイティブベクトルデータ型と AI_GENERATE_EMBEDDINGS 関数を活用し、SQL クエリのみでエンベディング生成が完結します。これにより Cloud SQL の全データベースエンジン (PostgreSQL, MySQL, SQL Server) で Vertex AI 統合がサポートされました。

🔄 Before / After

❌ Before (アップデート前)

  • アプリケーション層で Vertex AI API / OpenAI API を個別に呼び出し、結果をDBに格納する必要があった
  • SQL Server からデータベースから直接 ML モデルにアクセスする手段がなく、データ移動やコードの複雑化が発生
  • PostgreSQL / MySQL の Cloud SQL では Vertex AI 統合が利用可能だったが、SQL Server ではサポートされていなかった

✅ After (アップデート後)

  • AI_GENERATE_EMBEDDINGS 関数で SQL Server から直接ベクトルエンベディングを生成可能
  • Vertex AI / Hugging Face / OpenAI のモデルを外部モデルオブジェクトとして登録し、統一的な SQL インターフェースで利用可能
  • ストアドプロシージャでモデルの登録・変更・削除をDB管理者が直接管理可能

🏗️ アーキテクチャ図

💡 認証方式: Vertex AI モデルは IAM サービスアカウントで認証し (Secret Manager 不要)、サードパーティモデル (OpenAI / Hugging Face) は Secret Manager に格納した API キーで認証します。

⚡ 主要機能

📋 外部モデルオブジェクト管理

3 つのストアドプロシージャ (gcloudsql_ml_create_external_model、gcloudsql_ml_alter_external_model、gcloudsql_ml_drop_external_model) で AI モデルのエンドポイントを SQL Server 内に登録・変更・削除できます。ストアドプロシージャは msdb データベースの dbo スキーマに配置されています。

🧠 AI_GENERATE_EMBEDDINGS 関数

SQL Server の AI_GENERATE_EMBEDDINGS 関数でテキストをベクトルエンベディングに変換します。登録済みの外部モデルを USE MODEL 句で指定して呼び出します。実行にはデータベースユーザーに EXECUTE ON EXTERNAL MODEL 権限が必要です。

🌐 マルチプロバイダ対応

Vertex AI: Cloud SQL サービスアカウントの IAM 認証 (Secret Manager 不要)
Hugging Face / OpenAI: API キーを Secret Manager に格納し、シークレット URL を外部モデル作成時に指定。パブリック IP が必要 (プライベート IP インスタンスからはアクセス不可)

🔧 設定方法

前提条件: Cloud SQL for SQL Server 2025 インスタンス (Enterprise Plus / Enterprise)、Cloud SQL Admin API / Vertex AI API / Compute Engine API が有効化されていること。サードパーティモデル使用時は Secret Manager API も有効化が必要。

Step 1: 必要な API の有効化

Bash API の有効化 
gcloud services enable sqladmin.googleapis.com \
    aiplatform.googleapis.com \
    compute.googleapis.com \
    secretmanager.googleapis.com

Step 2: Vertex AI 統合を有効にしたインスタンスの作成

Bash 新規インスタンス作成 
# 新規インスタンス作成時
gcloud sql instances create INSTANCE_NAME \
    --database-version=SQLSERVER_2025_ENTERPRISE \
    --tier=MACHINE_TYPE \
    --region=REGION_NAME \
    --enable-google-ml-integration

# 既存インスタンスの更新
gcloud sql instances patch INSTANCE_NAME \
    --enable-google-ml-integration

Step 3: IAM 権限の付与

Bash Vertex AI アクセス権限 
gcloud projects add-iam-policy-binding PROJECT_ID \
    --member="serviceAccount:SERVICE_ACCOUNT_EMAIL" \
    --role="roles/aiplatform.user"

Step 4: Secret Manager 権限の付与 (サードパーティモデル使用時)

Bash Secret Manager アクセス権限 
SA_NAME=$(gcloud sql instances describe INSTANCE_NAME \
    --format="value(serviceAccountEmailAddress)")

gcloud secrets add-iam-policy-binding SECRET_NAME \
    --member="serviceAccount:${SA_NAME}" \
    --role="roles/secretmanager.secretAccessor"

Step 5: 外部モデルオブジェクトの作成

SQL Vertex AI モデルの登録例 
-- Vertex AI モデルの登録例
EXECUTE [msdb].[dbo].[gcloudsql_ml_create_external_model]
    @db = [mydb],
    @model_name = 'gemini-embedding',
    @model_provider = 'Vertex AI',
    @model = 'gemini-embedding-002',
    @model_url = 'https://us-central1-aiplatform.googleapis.com/v1/projects/PROJECT_ID/locations/us-central1/publishers/google/models/gemini-embedding-002',
    @secret_url = ''

-- OpenAI モデルの登録例
EXECUTE [msdb].[dbo].[gcloudsql_ml_create_external_model]
    @db = [mydb],
    @model_name = 'openai-embedding',
    @model_provider = 'OpenAI',
    @model = 'text-embedding-3-small',
    @model_url = 'https://api.openai.com/v1/embeddings',
    @secret_url = 'https://secretmanager.googleapis.com/v1/projects/PROJECT_ID/secrets/openai-api-key/versions/1'

Step 6: エンベディングの生成

SQL ベクトルエンベディングの生成 
-- ベクトルエンベディングの生成
SELECT AI_GENERATE_EMBEDDINGS(
    N'Cloud SQL for SQL Server は Google Cloud のマネージドデータベースです'
    USE MODEL gemini-embedding
)

-- 次元数を指定する場合
DECLARE @params JSON = N'{"dimensions": "1536"}';
SELECT AI_GENERATE_EMBEDDINGS(
    'This article introduces AI concepts.'
    USE MODEL gemini-embedding
    PARAMETERS @params
)

✅ メリット

💼 ビジネス面

✓ 開発効率の向上: SQL クエリのみでエンベディング生成が完結。API 統合コードが不要になり開発サイクルが短縮
✓ SQL Server エコシステムの強化: 既存 SQL Server ワークロードに AI 機能を追加。DB プラットフォーム移行なしに生成 AI アプリを構築可能
✓ マルチプロバイダの柔軟性: Vertex AI + OpenAI + Hugging Face から要件に応じた最適なモデルを選択可能

🔧 技術面

✓ データ移動の最小化: DB 内で直接エンベディング生成を実行。大量テキストデータのアプリ層転送が不要
✓ ネイティブベクトル型の活用: SQL Server 2025 のベクトルデータ型 + ベクトルインデックスで DB 内ベクトル検索が可能
✓ 統一的な管理: ストアドプロシージャによるモデル管理 + IAM/Secret Manager 認証を一元的に運用

⚠️ デメリット・制約事項

制限事項
  • Preview 機能 - 本番ワークロードでの利用は SLA の対象外。「Pre-GA Offerings Terms」が適用
  • SQL Server 2025 のみ対応 - SQL Server 2022, 2019, 2017 では利用不可
  • float32 ベクトルの最大次元数は 1,998 - 高次元モデルでは dimensions パラメータで削減するか float16 を使用
  • サードパーティモデルにはパブリック IP が必要 - プライベート IP のみのインスタンスからはアクセス不可
  • Vertex AI 統合を無効にしても外部モデルは自動削除されない。DROP EXTERNAL MODEL で手動削除が必要
考慮すべき点
  • Private Service Connect / プライベートサービスアクセスは Vertex AI 統合のみでサポート (サードパーティモデルには非対応)
  • 実行ユーザーには EXECUTE ON EXTERNAL MODEL 権限が必要
  • Preview から GA への移行時に仕様変更が発生する可能性あり

🎯 ユースケース

🔍 ユースケース 1: SQL Server データベースでのセマンティック検索

SQL Server に蓄積された商品カタログに対して、自然言語による類似性検索を実装。ユーザーの検索クエリをエンベディングに変換し、事前生成した商品説明のエンベディングとベクトル類似性検索を行います。

SQL セマンティック検索の実装例 
-- 商品テーブルにベクトル列を追加
ALTER TABLE products
ADD description_embedding VECTOR(768);

-- 商品説明のエンベディングを生成
UPDATE products
SET description_embedding = AI_GENERATE_EMBEDDINGS(
    description USE MODEL gemini-embedding
);

-- ベクトルインデックスの作成
CREATE VECTOR INDEX idx_product_embedding
ON products(description_embedding);

-- 類似商品の検索
DECLARE @query_vec VECTOR(768);
SET @query_vec = AI_GENERATE_EMBEDDINGS(
    N'軽量で防水のランニングシューズ' USE MODEL gemini-embedding
);

SELECT TOP 5 product_name, description,
    VECTOR_DISTANCE('cosine', description_embedding, @query_vec) AS distance
FROM products
ORDER BY distance;

効果: アプリケーション層でのベクトル検索ロジックが不要になり、SQL のみでセマンティック検索を実現。既存 SQL Server アプリケーションに最小限の変更で AI 機能を追加できます。

📚 ユースケース 2: RAG (検索拡張生成) パイプラインの構築

社内ナレッジベースのドキュメントをエンベディング化し、ユーザーの質問に対して関連するドキュメントを検索した上で、外部の LLM に回答を生成させる RAG パイプラインを構築します。

効果: SQL Server 内にナレッジベースのベクトルストアを構築することで、別途ベクトルデータベースを運用する必要がなくなります。既存の SQL Server のセキュリティモデルやバックアップ体制をそのまま活用できます。

💰 料金情報

Cloud SQL for SQL Server の料金はインスタンスの構成設定 (CPU, メモリ, ストレージ, リージョン) に依存します。Vertex AI 統合自体に追加料金は発生しませんが、Vertex AI モデルの呼び出しには Vertex AI の料金が別途適用されます。サードパーティモデル使用時は各プロバイダの API 利用料金も別途発生します。

🔗 関連サービス・機能

🧠 Vertex AI

Cloud SQL から直接アクセスする ML モデルのホスティング基盤。Gemini エンベディングモデルなどを提供し、IAM ベースの認証でシームレスに統合。

🔑 Secret Manager

サードパーティモデルプロバイダ (OpenAI, Hugging Face) の API キーを安全に保管・管理。Cloud SQL インスタンスのサービスアカウントにアクセス権を付与して使用。

🗄️ Cloud SQL for PostgreSQL / MySQL

PostgreSQL / MySQL では既に GA で Vertex AI 統合が提供されており、google_ml_integration 拡張 (PostgreSQL) や mysql.ml_embedding() 関数 (MySQL) を通じてアクセス可能。

🗃️ AlloyDB AI

AlloyDB でも Model Endpoint Management を通じた Vertex AI 統合が提供されており、Cloud SQL と類似のアーキテクチャで AI 機能を利用可能。
