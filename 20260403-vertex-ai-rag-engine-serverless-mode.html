📋 概要

Vertex AI RAG Engine に新たな Serverless モード が Public Preview として追加されました。 Serverless モードは、RAG リソースの保存に使用されるデータベースのプロビジョニングとスケーリングを 完全に抽象化 したフルマネージドなデータベースを提供します。これにより、ユーザーはインフラストラクチャの管理を意識することなく RAG アプリケーションの構築に集中できるようになります。

従来の Spanner モード (専用インスタンス) に加え、Serverless モード (フルマネージド) の 2 つのデプロイメントモードが選択可能になり、両モード間の シームレスな切り替え もサポートされています。

🔄 Before / After 比較

❌ Before (従来)

  • Spanner インスタンスのプロビジョニングが必要で、Basic ティアまたは Scaled ティアを明示的に選択・管理
  • データベースのスケーリング設定を自分で管理する必要があり、ワークロード変動への調整が運用負担に
  • 小規模な検証やプロトタイピングの段階でも Spanner インスタンスの費用が発生

✅ After (改善後)

  • Serverless モードにより、データベースのプロビジョニングとスケーリングが完全に自動化
  • Serverless モードと Spanner モード間でシームレスに切り替え可能、開発から本番への移行が柔軟に
  • フルマネージドなサーバーレス DB により初期設定の手間が大幅に削減

⚡ 主要機能

☁️ Serverless モード

RAG リソースの保存用にフルマネージドなデータベースを提供。データベースのプロビジョニングとスケーリングを完全に抽象化し、ユーザーはインフラ管理を意識せずに RAG アプリケーションの開発に集中できます。

🗄️ Spanner モード

専用で分離されたデータベースインスタンスを提供。Basic ティア (100 処理ユニット、コスト効率重視) と Scaled ティア (最大 10,000 処理ユニット、オートスケーリング) の 2 構成から選択可能。

🔄 モード間のシームレス切り替え

Serverless モードと Spanner モードの間で、データを保持したまま切り替えが可能。開発・検証フェーズでは Serverless モード、本番環境では Spanner モードの Scaled ティアといった使い分けが可能です。

🔧 技術仕様: デプロイメントモード比較

項目 Serverless モード Spanner (Basic) Spanner (Scaled)
ステータス Public Preview GA GA
データベース管理 フルマネージド ユーザー管理 (固定構成) ユーザー管理 (オートスケーリング)
プロビジョニング 不要 自動 (100 処理ユニット) 自動 (1,000〜10,000 処理ユニット)
スケーリング 自動 固定 オートスケーリング
推奨用途 プロトタイピング、小〜中規模 実験、小規模データ 本番環境、大規模データ

📝 設定方法・コードサンプル

前提条件: Google Cloud プロジェクトで Vertex AI API が有効化済み、適切な IAM 権限 (Vertex AI ユーザーロール以上) が付与済み、RAG Engine がサポートされているリージョンを使用すること。

Serverless モードの設定 (Python SDK)

Python Serverless モードの設定例 
from vertexai import rag
import vertexai

PROJECT_ID = "YOUR_PROJECT_ID"
LOCATION = "YOUR_RAG_ENGINE_LOCATION"

vertexai.init(project=PROJECT_ID, location=LOCATION)

# Serverless モードの設定例
rag_engine_config_name = f"projects/{PROJECT_ID}/locations/{LOCATION}/ragEngineConfig"
new_rag_engine_config = rag.RagEngineConfig(
    name=rag_engine_config_name,
    rag_managed_db_config=rag.RagManagedDbConfig(
        serverless=rag.Serverless()
    ),
)
updated_config = rag.rag_data.update_rag_engine_config(
    rag_engine_config=new_rag_engine_config
)

Serverless モードへの切り替え (REST API)

Bash 
# REST API を使用した Serverless モードの設定
curl -X PATCH \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  https://${LOCATION}-aiplatform.googleapis.com/v1beta1/projects/${PROJECT_ID}/locations/${LOCATION}/ragEngineConfig \
  -d '{"ragManagedDbConfig": {"serverless": {}}}'

Spanner モード (Basic ティア) への切り替え

Bash 
# Spanner モード (Basic ティア) への切り替え
curl -X PATCH \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  https://${LOCATION}-aiplatform.googleapis.com/v1beta1/projects/${PROJECT_ID}/locations/${LOCATION}/ragEngineConfig \
  -d '{"ragManagedDbConfig": {"spanner": {"basic": {}}}}'

💡 ユースケース

🚀 ユースケース 1: RAG アプリケーションの迅速なプロトタイピング

シナリオ: 新しい社内ナレッジベースの RAG アプリケーションを検証したい。データベースの構成管理に時間をかけずに、すぐに動作確認を始めたい。

Python プロトタイプ用の Serverless モード設定 
from vertexai import rag
import vertexai

vertexai.init(project="my-project", location="us-central1")

# Serverless モードで RAG Engine を構成
config = rag.RagEngineConfig(
    name="projects/my-project/locations/us-central1/ragEngineConfig",
    rag_managed_db_config=rag.RagManagedDbConfig(
        serverless=rag.Serverless()
    ),
)
rag.rag_data.update_rag_engine_config(rag_engine_config=config)

# RAG コーパスの作成とドキュメントのインポートをすぐに開始可能
corpus = rag.create_corpus(display_name="knowledge-base-prototype")

効果: データベースのプロビジョニング待ちなしで即座に RAG アプリケーションの開発・検証を開始でき、プロトタイピング期間を短縮できます。

📈 ユースケース 2: 開発環境から本番環境への段階的移行

シナリオ: Serverless モードで開発・検証を行った RAG アプリケーションを、本番環境に移行する際に Spanner モードの Scaled ティアに切り替えたい。

⚖️ メリット・デメリット

✅ メリット

ビジネス面

  • ✓ 初期導入コストの削減: DB プロビジョニング不要で開始時間・コストを大幅削減
  • ✓ 運用負担の軽減: インフラ管理が完全自動化され、アプリケーションロジックに集中可能
  • ✓ 柔軟なスケーリング戦略: モード切り替えで段階的にスケールアップ可能

技術面

  • ✓ インフラ抽象化: 宣言的な構成のみで利用可能
  • ✓ シームレスなモード切り替え: データを保持したまま切り替え可能
  • ✓ 既存機能との互換性: KNN/ANN 検索、VPC-SC、CMEK はモードに関係なく利用可能

⚠️ デメリット・制約

制限事項

  • ! Public Preview であり、本番環境での利用には SLA の確認が必要
  • ! Serverless モードのパフォーマンス特性や処理ユニット上限は公式ドキュメントで要確認
  • ! Data Residency および AXT セキュリティコントロールは RAG Engine 全体で未サポート

考慮すべき点

  • ● Preview 段階のため GA に向けて仕様変更の可能性
  • ● 大規模本番ワークロードでは Spanner Scaled ティアの方が適切な場合がある
  • ● モード切り替え時のダウンタイムや影響範囲について事前検証を推奨

💰 料金情報

Vertex AI RAG Engine の料金はデプロイメントモードにより異なります。Spanner モードでは、バックエンドの Spanner インスタンスに対する標準的な Spanner SKU 料金が発生します。

🔹 Spanner Basic ティア

100 処理ユニット (0.1 ノード相当) の固定料金

🔸 Spanner Scaled ティア

1〜10 ノードのオートスケーリング、使用量に応じた料金

☁️ Serverless モード

Public Preview 時点の料金体系については公式ドキュメントを参照。データ取り込み、エンベディング生成、LLM による応答生成にはそれぞれ利用コストが別途発生します。

📌 まとめ

Vertex AI RAG Engine の Serverless モードは、RAG アプリケーション開発における最大の摩擦の一つであったデータベースのプロビジョニングとスケーリング管理を解消する重要なアップデートです。Serverless モードと Spanner モードのシームレスな切り替えにより、プロトタイピングから本番環境まで一貫した開発体験が実現されます。

🎯 推奨アクション: RAG アプリケーションを構築・運用しているチームは、Serverless モードの Preview 機能を評価し、開発ワークフローへの組み込みを検討してください。
