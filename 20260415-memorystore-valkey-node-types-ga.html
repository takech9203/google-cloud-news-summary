🗄️ Memorystore for Valkey: 新しいノードタイプが GA

Memorystore for Valkey GA In-Memory Database
2026-04-15

📋 概要

Memorystore for Valkey で新しいノードタイプが一般提供 (GA) となり、インスタンス作成時に選択できるノードタイプのラインナップが大幅に拡充されました。従来の限られた選択肢に加えて、10 種類のノードタイプが GA として利用可能になり、1.25 GB から 110 GB までの幅広い容量帯をカバーできるようになりました。

ワークロードの特性に応じて、より細かいリソースサイジングが可能となり、コストパフォーマンスの最適化が実現できます。GA ステータスにより、すべてのノードタイプが SLA の対象となり (shared-core-nano を除く)、本番環境での利用に適した信頼性が保証されます。

Before / After 比較

❌ Before (アップデート前)
✗利用可能なノードタイプが限られ、過剰または不足するリソースを選択せざるを得ないケースがあった
✗SLA 付きで 1 GB - 3 GB の小規模容量帯を利用するオプションが不足
✗CPU 集約型ワークロードに特化したノードタイプがなく、処理性能が不足するケースがあった
✗Memorystore for Redis からの移行時にサイズ選択に制約
✅ After (アップデート後)
✓10 種類のノードタイプから最適なものを選択可能 (1.25 GB - 110 GB)
✓custom-pico / micro / mini により SLA 付きの小規模インスタンス (1-3 GB) を作成可能
✓highcpu-medium により CPU 集約型ワークロードに最適なリソース配分を選択可能
✓Memorystore for Redis からの移行が容易に

⚡ 主要機能

📦 1. 10 種類のノードタイプ

→shared-core-nano から highmem-2xlarge まで、幅広い容量帯をカバーする 10 種類のノードタイプが GA
→メモリ集約型、CPU 集約型、小規模開発用など、ワークロードの特性に応じた最適な選択が可能
→shared-core-nano を除くすべてのノードタイプで SLA が適用され、本番環境での利用に適した可用性を保証

🔧 2. Custom ノードタイプ (Cluster Mode Disabled 専用)

→custom-pico (1.25 GB)、custom-micro (2.5 GB)、custom-mini (3.75 GB) の 3 種類
→Memorystore for Redis からの移行時に、1 GB - 3 GB の小規模インスタンスを SLA 付きで作成可能
→Cluster Mode Disabled インスタンスでのみ利用可能

⚙️ 3. High-CPU ノードタイプ

→highcpu-medium は 8 vCPU / 13 GB で、メモリ容量に対して高い処理性能を提供
→CPU 集約型ワークロード (複雑なデータ構造操作、Lua スクリプト処理など) に最適
→同じメモリ容量の highmem-medium (2 vCPU) と比較して 4 倍の処理能力

🎯 ユースケース

🔄 ユースケース 1: Memorystore for Redis から Valkey への移行

シナリオ: 現在 2 GB の Memorystore for Redis インスタンスを使用しているアプリケーションを、Memorystore for Valkey に移行。SLA の保証が必要で、Cluster Mode Disabled で運用したい。

Bash custom-micro (2.5 GB) で Cluster Mode Disabled インスタンスを作成 
gcloud memorystore instances create my-migrated-cache \
  --location=asia-northeast1 \
  --project=my-project \
  --node-type=custom-micro \
  --shard-count=1 \
  --mode=cluster-disabled \
  --replica-count=1

効果: SLA 付きで既存の Redis インスタンスと同等のサイズ帯を利用でき、ダウンタイムを最小限にした段階的な移行が可能。

🛒 ユースケース 2: 大規模リアルタイムレコメンデーションエンジン

シナリオ: EC サイトで数百万ユーザーのセッションデータとレコメンデーションモデルをリアルタイムで処理。高いメモリ容量と水平スケーリングが求められる。

Bash highmem-xlarge (58 GB) で大規模クラスターを作成 
gcloud memorystore instances create recommendation-cache \
  --location=asia-northeast1 \
  --project=my-project \
  --node-type=highmem-xlarge \
  --shard-count=10 \
  --replica-count=2

効果: 10 シャード x 58 GB = 580 GB の大容量キャッシュにより、数百万ユーザーのリアルタイムデータ処理が可能。レプリカにより読み取りスループットも確保。

🎮 ユースケース 3: CPU 集約型の Lua スクリプト処理

シナリオ: ゲームアプリケーションのランキングシステムで、複雑な Lua スクリプトによるアトミックなスコア集計処理が頻繁に発生。

効果: highcpu-medium (8 vCPU / 13 GB) を使用することで、Lua スクリプトの実行性能が向上。同じメモリ容量の highmem-medium (2 vCPU / 13 GB) と比較して 4 倍の処理能力で複雑な計算ロジックを実行できる。

✅ メリット

💼 ビジネス面

✓コスト最適化: ワークロード規模に合った最適なノードタイプで過剰プロビジョニングを回避し、コスト削減
✓移行の容易さ: custom ノードタイプにより Redis からの移行時に同等サイズ帯 (1-3 GB) を SLA 付きで利用可能
✓スケーラビリティ: 小規模から始めて段階的にスケールアップできる柔軟性

🔧 技術面

✓パフォーマンス最適化: highcpu-medium により CPU 集約型ワークロードに最適な処理性能を選択
✓柔軟なスケーリング: 垂直 (ノードタイプ変更) と水平 (シャード数変更) の両方を組み合わせたリソース管理
✓SLA 保証: shared-core-nano を除く全ノードタイプで SLA 提供。本番ワークロードに適した可用性

⚠️ デメリット・制約事項

制限事項

✗shared-core-nano ノードタイプには SLA が適用されないため、本番環境での利用は非推奨
✗custom-pico / custom-micro / custom-mini は Cluster Mode Disabled インスタンスでのみ利用可能。クラスター構成への移行時にはインスタンスの再作成が必要
✗ノードサイズが大きくなるほど vCPU の追加に対する性能向上が線形にスケールしない場合がある

考慮すべき点

→最適なコストパフォーマンスには、大きなノードへのスケールアップよりも小さなノードを追加するスケールアウト戦略が推奨
→ノードタイプ変更 (垂直スケーリング) 中はインスタンスへの影響がある場合があるため、メンテナンスウィンドウでの実施を推奨
→maxmemory 設定の変更はデフォルト値からの変更による OOM エラーリスクを考慮

💰 料金

Memorystore for Valkey の料金はノードタイプ、リージョン、およびノード数に基づいて課金されます。各ノードはノードタイプに応じた時間単位の料金が適用されます。

料金例 (us-central1 リージョン概算)

構成例 ノード数 概算月額料金
highmem-medium x 3 シャード (レプリカなし) 3 約 $337/月
highmem-medium x 10 シャード (レプリカ 2) 30 約 $3,370/月
standard-small x 5 シャード (レプリカ 1) 10 約 $560/月

コスト削減オプション

💡 確約利用割引 (CUD): 1 年間の確約で 20% 割引、3 年間で 40% 割引。CUD は Memorystore for Redis Cluster、Redis、Memcached の利用にも適用可能。スケールアウト戦略 (小さなノードを追加) でコストパフォーマンスが向上する場合もあります。

🔗 関連サービス・機能

🔄 Memorystore for Redis

Memorystore for Valkey の前身サービス。custom ノードタイプにより Redis からの移行パスが改善された。

🖥️ Compute Engine

Memorystore for Valkey への接続元となる仮想マシン。ノードタイプの vCPU 数は Compute Engine のマシンタイプに類似した概念。

Memorystore for Valkey のメトリクスをモニタリングし、ノードタイプ変更のタイミングを判断するために使用。

Memorystore for Valkey インスタンスへのネットワーク接続に必要な基盤サービス。
