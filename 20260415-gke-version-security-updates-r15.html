☸️ GKE 2026-R15 バージョンアップデート & セキュリティアップデート

Google Kubernetes Engine (GKE) | 2026-04-15
GKE Change Security

📋 概要

GKE の 2026-R15 リリースでは、Rapid、Regular、Extended の各リリースチャネルに新しいクラスタバージョンが配信されました。Kubernetes のバグ修正やパフォーマンス改善に加え、Container-Optimized OS (COS) イメージの累積セキュリティ修正が含まれています。

Rapid チャネルには最新の 1.35.3 系が、Regular チャネルには 1.35.2 系が提供され、Extended チャネルでは長期サポート対象の 1.30 および 1.31 マイナーバージョンの新しいパッチが利用可能になりました。Stable チャネルについては今回のリリースサイクルでは新バージョンの配信はありません。

❌ Before (R15 以前)

  • 以前のパッチバージョンのセキュリティ脆弱性が未修正
  • COS イメージに累積セキュリティ修正が未適用
  • カーネルやシステムコンポーネントの既知の問題が残存

✅ After (R15 適用後)

  • 各チャネルで最新セキュリティパッチ適用済みバージョンが利用可能
  • COS イメージの累積セキュリティ修正でノードのセキュリティ強化
  • Rapid チャネルで 1.35.3 系の新機能を早期検証可能

⭐ 主要機能

🔥 Rapid チャネル バージョン更新

1.32.13、1.33.10、1.34.6、1.35.3 の各 GKE パッチが利用可能。最新の Kubernetes パッチリリースに基づくセキュリティ修正とバグ修正を含みます。新規クラスタの作成および既存クラスタの手動アップグレードで利用可能です。

✅ Regular チャネル バージョン更新

1.32.13、1.33.10、1.34.6、1.35.2 の各 GKE パッチが利用可能。Rapid チャネルで検証済みのバージョンが展開されており、本番環境に推奨です。デフォルトのリリースチャネルとして、機能の利用可能性と安定性のバランスを提供します。

🕑 Extended チャネル バージョン更新

1.30.14、1.31.14 の GKE パッチが利用可能。長期サポートが必要なクラスタ向けに、同一マイナーバージョンのパッチ更新を継続提供します。

⚠️ 注意: 1.30 の標準サポート終了は 2026 年 Q3 予定、Extended サポートは 2026-07-30 まで。計画的なアップグレードを検討してください。

🔒 Container-Optimized OS セキュリティ更新

累積的なセキュリティ修正を含む更新された COS イメージが各バージョンに適用されています。

🛠️ 設定方法: アップグレード手順

前提条件

💡 必要な準備:
  • Google Cloud プロジェクトで GKE API が有効であること
  • gcloud CLI がインストールされ、適切な認証が設定されていること
  • container.clusterAdmin または同等の IAM ロールが付与されていること

✅ 確認

Step 1: 利用可能なバージョンの確認

Bash チャネルで利用可能なバージョンを確認 
# 特定のチャネルで利用可能なバージョンを確認
gcloud container get-server-config \
  --zone=asia-northeast1-a \
  --flatten="channels" \
  --filter="channels.channel=RAPID" \
  --format="yaml(channels.channel,channels.validVersions)"

→ channels.channel を REGULAR、STABLE、EXTENDED に変更して各チャネルを確認可能。

Step 2: コントロールプレーンをアップグレード

Bash コントロールプレーンのアップグレード 
# コントロールプレーンを指定バージョンにアップグレード
gcloud container clusters upgrade CLUSTER_NAME \
  --zone=asia-northeast1-a \
  --master \
  --cluster-version=1.35.2-gke.1962000

→ 数分~数十分かかります。アップグレード中もワークロードは実行されます。

Step 3: ノードプールのアップグレード

Bash ノードプールのアップグレード 
# ノードプールを指定バージョンにアップグレード
gcloud container clusters upgrade CLUSTER_NAME \
  --zone=asia-northeast1-a \
  --node-pool=NODE_POOL_NAME \
  --cluster-version=1.35.2-gke.1962000

→ ノードがローリング方式で更新されます。Surge upgrade / Blue-Green upgrade で影響を最小化可能。

Step 4: アップグレードの確認

Bash バージョン確認 
# クラスタのバージョンを確認
gcloud container clusters describe CLUSTER_NAME \
  --zone=asia-northeast1-a \
  --format="table(name,currentMasterVersion,currentNodeVersion)"

⚖️ メリット / デメリット

✅ メリット

💼 ビジネス面
  • セキュリティコンプライアンスの維持: COS の累積セキュリティ修正でセキュリティ要件への継続的な準拠が可能
  • サービス継続性の確保: リリースチャネルを活用した段階的アップグレードでビジネス影響を最小化
  • 長期運用の安定性: Extended チャネルで大規模メジャーアップグレードなしにセキュリティパッチを継続受領
🔨 技術面
  • 脆弱性の迅速な修正: COS イメージでノードレベルのセキュリティ強化
  • 段階的な検証: Rapid から Stable まで段階的な品質検証プロセス
  • 柔軟なバージョン管理: 複数マイナーバージョンが同時提供、互換性に応じた選択が可能

⚠️ デメリット / 制約

🚫 制限事項
  • 全ゾーンでの利用可能化まで数日を要する場合がある
  • Stable チャネルは今回のリリースで新バージョンの配信なし
  • 自動アップグレードのタイミングはチャネルごとに異なる
💡 考慮すべき点
  • CP とノードプールのバージョン skew ポリシーに注意 (CP は 2 マイナーバージョン以上新しい必要)
  • Extended の 1.30 は標準サポート終了が近い (2026 Q3 予定)
  • メンテナンスウィンドウ / 除外設定によりアップデート適用が遅延する可能性
  • UEFI Secure Boot 有効な VM では COS インプレースアップデート非対応

🎯 ユースケース

💼 ユースケース 1: 本番環境のセキュリティ維持
シナリオ: Regular チャネルに登録された本番 GKE クラスタを運用しており、セキュリティコンプライアンス要件として最新のセキュリティパッチの適用が求められている。
Bash メンテナンスウィンドウの設定 
# メンテナンスウィンドウを設定して業務時間外にアップグレード
gcloud container clusters update my-prod-cluster \
  --zone=asia-northeast1-a \
  --maintenance-window-start=2026-04-16T18:00:00Z \
  --maintenance-window-end=2026-04-17T06:00:00Z \
  --maintenance-window-recurrence="FREQ=WEEKLY;BYDAY=SA"
✔ 効果: メンテナンスウィンドウ内で自動的にパッチが適用され、業務影響を最小限にコンプライアンス要件を充足
💰 ユースケース 2: Extended チャネルによる長期安定運用
シナリオ: 金融系システムで GKE 1.31 を利用しており、アプリケーションの互換性テストに時間を要するため、頻繁なマイナーバージョンアップグレードを避けたい。
Bash Extended チャネルへの登録 
# Extended チャネルにクラスタを登録
gcloud container clusters update my-finance-cluster \
  --zone=asia-northeast1-a \
  --release-channel=extended
✔ 効果: 1.31.14-gke.1790000 のパッチを継続受領しつつ、マイナーバージョンアップは計画的に。Extended サポートは 2026-10-22 まで継続
🚀 ユースケース 3: 新バージョンの早期検証
シナリオ: 開発チームが Kubernetes 1.35 の新機能 (Gateway API の拡張、PodDisruptionBudget の改善など) を本番導入前に検証したい。
Bash Rapid チャネルでステージング環境を作成 
# Rapid チャネルでステージング環境のクラスタを作成
gcloud container clusters create staging-cluster \
  --zone=asia-northeast1-a \
  --release-channel=rapid \
  --cluster-version=1.35.3-gke.1389000 \
  --num-nodes=3
✔ 効果: Rapid チャネルの 1.35.3 を使用して、最新機能を本番導入前に安全に検証可能

💰 料金情報

GKE のバージョンアップデート自体に追加料金は発生しません。

項目 詳細
バージョンアップグレード 無料 (追加料金なし)
GKE Standard クラスタ管理料金 $0.10/時間/クラスタ
GKE Autopilot Pod のリソース使用量に応じた従量課金
Extended チャネル (Extended サポート期間) 標準サポート終了後はクラスタごとに追加料金が発生
⚠️ 注意: Extended チャネルを利用して標準サポート期間を超えて運用する場合は、追加のクラスタ管理料金が発生します。

🌎 利用可能リージョン

GKE バージョンアップデートはすべての Google Cloud リージョンおよびゾーンで提供されます。ただし、ロールアウトは段階的に進行し、すべてのゾーンで利用可能になるまで数日を要する場合があります。

Bash 特定ゾーンでの利用可能状況を確認 
# 特定ゾーンでの利用可能バージョンを確認
gcloud container get-server-config \
  --zone=asia-northeast1-a
