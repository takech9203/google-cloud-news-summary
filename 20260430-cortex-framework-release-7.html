📌 概要

Google Cloud Cortex Framework バージョン 7 がパブリックプレビューとしてリリースされました。エンタープライズシステムからの生データを信頼性の高いデータアセットに変換するデータプロダクトアクセラレータが大幅に進化しています。

💡 ポイント: モジュラーデプロイ、Dataform によるサーバーレス実行、AI 対応データプロダクトのサポート強化が主な特徴です。SAP ECC / S/4HANA からのデータ活用に最適化されています。

Before / After 比較

❌ Before (旧バージョン)

  • Airflow VM や専用コンピュートクラスタが必要
  • 不要なテーブルも含めた全処理が必要
  • カスタムフィールド追加時に標準モデルと競合
  • 依存関係を手動で管理
  • アップグレード時の互換性問題

✅ After (Release 7)

  • Dataform でサーバーレス実行 (VM 不要)
  • 必要なデータプロダクトのみ選択デプロイ
  • 名前空間ベースでカスタムと標準を分離
  • ネイティブ依存グラフで自動解決
  • インクリメンタルローディングで高速化

⚡ 主要機能

📦
モジュラーデプロイ
必要なデータプロダクトを選択するだけで、フレームワークが自動的に必要なテーブルを特定・取得・変換。不要なデータ処理を排除。
🔗
ネイティブ依存グラフ生成
複雑なデータモデルの処理順序を自動管理。前提テーブルがデプロイ前に準備されることを保証。
🔄
Bring Your Own CDC
組み込み CDC をバイパスし、既存の CDC パイプラインをファンデーションレイヤーに直接接続可能。
☁️
サーバーレス実行
Dataform によるバージョン管理された SQL 変換。Airflow VM やコンピュートクラスタが不要。
⚡
インクリメンタルローディング
前回実行以降の新規・変更データのみを処理。大規模データセットの処理時間とコストを大幅削減。
🤖
AI 対応メタデータ
セマンティックマッピングと AI 対応メタデータの自動付与。暗号的テーブル名をビジネス用語に変換。
🌐
マルチシステム SAP
SAP ECC と S/4HANA の両方に対する動的依存解決とロジック差分。複数 ERP からのシームレス統合。
🛠
拡張性フレームワーク
名前空間による分離で、カスタムデータプロダクトと Cortex 標準を明確に分離。アップデートの影響なし。

🔧 設定方法

前提条件

✅ Google Cloud プロジェクトの作成と請求先アカウントの設定
✅ BigQuery API、Dataform API、Cloud Resource Manager API の有効化
✅ 適切な IAM ロールの付与 (roles/bigquery.jobUser, roles/bigquery.dataViewer, roles/bigquery.dataEditor, roles/dataform.admin)
✅ GitHub リポジトリへのアクセス申請 (Request Access フォーム経由)

ステップ 1: 構成ファイルの設定

YAML config/config.yaml 
data:
  bigQueryLocation: US
  sources:
    - id: sap_raw
      projectId: my-source-project
      datasetId: cortex_sap_raw
  targets:
    - id: sap_foundation
      projectId: my-target-project
      datasetId: cortex7_sap_data_foundation
  modules:
    foundation:
      - moduleId: sap_erp
        type: cortex.sap_erp
        dataSourceId: sap_raw
        dataTargetId: sap_foundation
    product:
      - moduleId: sap_purchasing_organizations
        type: cortex.purchasing_organizations
        dependsOn:
          sapModule: erp
        dataTargetId: product_target

ステップ 2: ビルドとデプロイの実行

Bash 一括実行コマンド 
# ビルドとデプロイを一括実行
uv run cortex-build-and-deploy --config "config/config.yaml"

# 個別実行 (オプション)
uv run cortex-build --config "config/config.yaml"    # コンパイルのみ
uv run cortex-deploy --config "config/config.yaml"   # デプロイのみ
💡 補足: cortex-build-and-deploy コマンドにより、前提条件の検証、スキーマ情報に基づく .sqlx スクリプトのコンパイル、Dataform リポジトリへのアーティファクト同期が自動実行されます。

⚠️ 制約事項・注意点

🚧 制限事項
  • 現在パブリックプレビュー段階 (GA ではない)
  • サポート対象はSAP ECC と SAP S/4HANA に限定 (v7 時点)
  • GitHub リポジトリへのアクセスには事前申請が必要
💡 考慮すべき点
  • 既存 Cortex Framework (旧バージョン) からの移行計画が必要
  • Dataform 利用が前提のため、既存 Airflow ワークフローからの移行検討が必要
  • Dataform リポジトリのリージョンと BigQuery データセットのリージョンを一致させる必要がある

📝 まとめ

Cortex Framework Release 7 は、エンタープライズデータパイプラインの構築と運用を根本的に変革するアップデートです。Dataform によるサーバーレス実行、モジュラーデプロイ、そして AI 対応メタデータの自動付与により、SAP データから AI/エージェントユースケースまでの Time-to-Value を大幅に短縮します。

SAP ERP データを BigQuery で活用している、またはこれから活用を検討している企業は、早期にパブリックプレビューへのアクセスを申請し、バージョン 7 の新機能を評価することを推奨します。
