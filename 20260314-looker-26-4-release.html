📋 概要

Looker 26.4 は、埋め込み Conversational Analytics の一般提供 (GA)、Gemini in Looker 管理ページのリデザイン、Preview Features ページの追加、およびセキュリティに関する破壊的変更とバグ修正を含む包括的なリリースです。

Looker (Google Cloud core) へのデプロイは 2026 年 3 月 16 日 (月) に開始され、最終デプロイは 2026 年 3 月 23 日 (月) に予定されています。

🤖 Gemini 活用: Embed Conversational Analytics の GA により、Gemini を活用した自然言語データ分析機能を外部アプリケーションやウェブサイトに正式に埋め込めるようになりました。

🔄 Before / After 比較

❌ Before (26.4 以前)

  • Embed Conversational Analytics は Preview 段階
  • Looker (Google Cloud core) では利用不可
  • Embed SDK が Conversational Analytics 非対応
  • 埋め込み時のテーマ適用がサポートされていなかった
  • Gemini 機能は一括で有効/無効のみ
  • Preview Features ページが利用不可

✅ After (26.4)

  • Embed Conversational Analytics が GA に
  • Looker (Google Cloud core) でも利用可能に
  • Embed SDK が Conversational Analytics をサポート
  • 埋め込みテーマの General オプションが部分的にサポート
  • Gemini 機能を個別に有効/無効化可能に
  • Preview Features ページで 4 機能を管理可能に

ステップ 2: Conversational Analytics の埋め込み (Embed SDK)

JavaScript Embed SDK の初期化と埋め込み 
// Embed SDK の初期化
import { LookerEmbedSDK } from '@looker/embed-sdk'

LookerEmbedSDK.init('your-instance.looker.com', '/auth')

// Conversational Analytics の埋め込み
LookerEmbedSDK.createConversationWithId('conversation_id')
  .appendTo('#conversation-container')
  .on('conversation:message', (event) => {
    console.log('New message:', event)
  })
  .build()
  .connect()

⚠️ デメリット・制約事項

制限事項

❌埋め込みテーマは General オプションのみ。チャート色、軸スタイルなどは非対応
❌Conversational Analytics は FedRAMP High/Medium の認証境界には含まれていない
❌Labs カテゴリのプレビュー機能は Looker (Google Cloud core) では利用不可
❌Gemini の出力は事実と異なる情報を含む可能性あり。使用前の検証を推奨

考慮すべき点

破壊的変更への対応: カスタマーホスト型インスタンスでプロジェクトレベルのローカル Git フックを使用している場合、26.4 へのアップグレード前に代替手段への移行が必要
SDK アップグレード推奨: Looker SDK バージョン 26.4 以降へのアップグレードが推奨。26.18 リリース (2026 年 10 月) で API 認証情報の URL クエリパラメータ渡しが廃止予定
権限管理: 埋め込みユーザーに chat_with_explore などの適切な権限付与が必要
データガバナンス: Gemini for Google Cloud のデータ処理ポリシーを確認し、組織の要件との整合性を検証すること

🎯 ユースケース

ユースケース 1: SaaS プロダクトへの AI データ分析機能の組み込み

シナリオ: B2B SaaS 企業が自社プロダクトに Conversational Analytics を埋め込み、顧客がダッシュボードデータについて自然言語で質問できる機能を提供。

HTML iframe による埋め込み例 
<iframe
  src="https://my-instance.looker.com/embed/agents/agent_id"
  width="100%"
  height="600"
  frameborder="0">
</iframe>

効果: 顧客のデータリテラシーに依存せず、誰でも自然言語でビジネスデータにアクセスできるようになり、プロダクトの利用率と顧客満足度が向上。

ユースケース 2: 段階的な Gemini 機能導入

シナリオ: 大規模企業の Looker 管理者が、まず Conversational Analytics のみを限定的なユーザーグループに有効化し、ガバナンスとセキュリティの検証後に Code Interpreter などの Trusted Tester 機能を展開。

効果: リスクを最小化しながら AI 分析機能を段階的に導入でき、各機能の影響を個別に評価可能。
