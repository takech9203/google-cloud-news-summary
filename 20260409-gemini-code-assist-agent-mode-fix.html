🔧 Gemini Code Assist: エージェントモードのログ帰属先の修正

Gemini Code Assist Bug Fix VS Code 拡張機能
2026-04-09
Google Cloud Update

🔍 概要

VS Code 向け Gemini Code Assist 拡張機能のバージョン 2.77.1 において、エージェントモードのログ帰属先に関するバグが修正されました。

以前のバージョンでは、エージェントモードで生成されたログが Gemini Code Assist ではなく Gemini CLI に誤って帰属されるという不具合が存在していました。この修正により、Cloud Monitoring 上の使用状況メトリクスが正しく報告されるようになりました。

組織における Gemini Code Assist の利用状況を Cloud Monitoring で追跡している管理者にとって特に影響が大きい修正です。Google は、正確な使用状況メトリクスを確保するために、バージョン 2.77.1 以上へのアップデートを推奨しています。

⚠️ 推奨アクション: VS Code で Gemini Code Assist を使用しているすべてのユーザーは、バージョン 2.77.1 以上へのアップデートを行ってください。

🔄 Before / After 比較

❌ Before (v2.77.0 以前)

  • エージェントモードのログが Gemini CLI に誤って帰属されていた
  • Cloud Monitoring で Gemini Code Assist の使用状況メトリクス (日次アクティブユーザー数、チャット応答数、コード提案承認数など) が実際よりも少なく表示
  • Gemini CLI のメトリクスが過大に計上
  • 組織内のツール利用状況分析に不正確なデータが使用されるリスク

✔️ After (v2.77.1 以降)

  • エージェントモードのログが正しく Gemini Code Assist に帰属
  • Cloud Monitoring のメトリクスが正確な使用状況を反映
  • Gemini Code Assist と Gemini CLI それぞれの利用状況を正しく区別して把握可能
  • labels.product フィールドが正しく code_assist として記録

📊 影響を受けるメトリクス

今回のバグにより、以下の Cloud Monitoring メトリクスが不正確に計上されていました。修正後はすべて正しい値が記録されます。

👤 日次アクティブユーザー code_assist/daily_active_users
📅 28 日間アクティブユーザー code_assist/twenty_eight_day_active_users
💬 日次チャット応答数 chat_responses_count
✔️ 日次コード提案承認数 code_assist/code_suggestions_accepted_count
📝 日次コード行承認数 code_assist/code_lines_accepted_count
💡 ポイント: Cloud Logging に記録されるメタデータの labels.product フィールドが、誤って gemini_cli と記録されていたものが、正しく code_assist として記録されるようになりました。

🤖 エージェントモードとは

Gemini Code Assist のエージェントモードは、VS Code および IntelliJ で利用可能な AI ペアプログラミング機能です。通常のチャットモードとは異なり、以下の操作が可能です。

🛠️ 組み込みツールの使用

ファイルの読み書きやターミナル操作などの組み込みツールを直接使用可能

🔌 MCP サーバー連携

MCP サーバーを介した外部ツールとの連携が可能

⚡ 複数ステップの実行

複数ステップにわたる複雑なタスクの実行に対応

📝 コード生成

設計ドキュメントや Issue からのコード生成が可能

🔧 設定方法 (アップデート手順)

前提条件: VS Code がインストールされていること、Gemini Code Assist 拡張機能がインストールされていること。

ステップ 1: 拡張機能のバージョン確認

VS Code のコマンドパレットを開き、インストール済み拡張機能のバージョンを確認します。

ステップ 2: 拡張機能のアップデート

バージョンが 2.77.1 未満の場合、以下のいずれかの方法でアップデートします。

方法 A: 拡張機能の管理画面から「更新」ボタンをクリック
方法 B: コマンドパレットから Extensions: Check for Extension Updates を実行

Google Cloud コンソールの Gemini Code Assist 概要ページでメトリクスを確認し、アップデート後にログの帰属先が正しくなっていることを検証します。

📌 まとめ

今回のバグ修正は、Gemini Code Assist のエージェントモードにおけるログ帰属先の不具合を解消するものです。 特に組織の管理者が Cloud Monitoring を通じて使用状況を追跡している場合、このバグにより不正確なデータが記録されていた可能性があります。

VS Code で Gemini Code Assist を使用しているすべてのユーザーに対し、バージョン 2.77.1 以上へのアップデートを推奨します。 アップデート後は、Cloud Monitoring のメトリクスが正確な使用状況を反映するようになり、ライセンス管理やツール利用状況の分析が改善されます。

✔️ 対応手順: VS Code のコマンドパレットから Extensions: Check for Extension Updates を実行し、Gemini Code Assist をバージョン 2.77.1 以上にアップデートしてください。
