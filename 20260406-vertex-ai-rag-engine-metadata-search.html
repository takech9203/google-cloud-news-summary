🤖 Vertex AI RAG Engine: メタデータ検索機能

Generative AI on Vertex AI Feature RAG Engine
リリース日: 2026-04-06

📋 概要

Vertex AI RAG Engine にスキーマベースのメタデータ検索機能が追加されました。RAG コーパスにメタデータスキーマを定義し、ファイルにメタデータを付与することで、検索時にメタデータベースのフィルタリングが可能になります。

ベクトル類似度だけでは十分な精度が得られない場合に、ドキュメントのカテゴリ、作成日、部署、言語などの構造化属性でフィルタリングし、より的確なコンテキストを LLM に渡すことが可能になります。

🎯 対象ユーザー: Vertex AI RAG Engine で RAG アプリケーションを構築しているデベロッパー、データサイエンティスト、企業のナレッジマネジメントシステム運用チーム

🔄 Before / After 比較

❌ Before

  • 検索はベクトル類似度とベクトル距離閾値のみ
  • カテゴリ別検索にはコーパス分割やアプリ側の後処理が必要
  • ファイルの属性情報を体系的に管理する仕組みがなかった

✅ After

  • コーパスレベルでメタデータスキーマを定義可能
  • 検索時に metadata_filter でフィルタリング
  • スキーマベースでメタデータの一貫性と構造化を保証

⚙️ 主要機能

📐 メタデータスキーマの定義

RAG コーパスレベルで JSON 形式のメタデータスキーマを定義。メタデータフィールドの名前・型・構造を統一的に管理できます。

  • スキーマソース: Cloud Storage (GCS)、Google Drive、インライン JSON の 3 通り
  • GCS ファイル命名規則: metadata_schema.json

🏷️ ファイルへのメタデータ付与

RagFileMetadataConfig を使用して、ファイルインポート時にメタデータを付与。ディレクトリ単位での一括適用にも対応。

  • メタデータソース: GCS、Google Drive、インライン JSON
  • ディレクトリ単位: metadata.json ファイルで一括適用

🔍 メタデータフィルタリングによる検索

RagRetrievalConfig.Filter の metadata_filter フィールドでフィルタリング条件を指定。

  • ベクトル距離/類似度閾値との組み合わせが可能
  • Retrieval Query API と GenerateContent API の両方で利用可能

📊 技術仕様

RagFileMetadataConfig

項目 詳細
メタデータスキーマソース gcs_metadata_schema_source / google_drive_metadata_schema_source / inline_metadata_schema_source
メタデータソース gcs_metadata_source / google_drive_metadata_source / inline_metadata_source
スキーマ形式 JSON 形式
GCS ファイル命名規則 スキーマ: metadata_schema.json / メタデータ: metadata.json

RagRetrievalConfig.Filter

項目 詳細
metadata_filter メタデータフィルタリング用の文字列 (Optional)
vector_distance_threshold ベクトル距離閾値によるフィルタ (Optional)
vector_similarity_threshold ベクトル類似度閾値によるフィルタ (Optional)

🛠️ 設定方法

📌 前提条件:
  • Google Cloud プロジェクトで Vertex AI API が有効化されていること
  • RAG Engine を利用可能なリージョン (us-central1 など) を使用すること
  • 適切な IAM 権限 (Vertex AI ユーザーロール以上) が付与されていること

Step 1: メタデータスキーマ付きの RAG コーパスを作成

Python RAG コーパスの作成 
from vertexai import rag
import vertexai

PROJECT_ID = "your-project-id"
LOCATION = "us-central1"

vertexai.init(project=PROJECT_ID, location=LOCATION)

# インラインでメタデータスキーマを指定してコーパスを作成
rag_corpus = rag.create_corpus(
    display_name="my-corpus-with-metadata",
    backend_config=rag.RagVectorDbConfig(
        vector_db=rag.RagManagedVertexVectorSearch()
    )
)

Step 2: メタデータ付きでファイルをインポート

Python メタデータ付きファイルインポート 
# メタデータ設定を含めてファイルをインポート
response = rag.import_files(
    corpus_name=rag_corpus.name,
    paths=["gs://my-bucket/documents/"],
    rag_file_metadata_config=rag.RagFileMetadataConfig(
        inline_metadata_schema_source='{"properties": {"category": {"type": "string"}}}',
        gcs_metadata_source=rag.GcsSource(
            uris=["gs://my-bucket/metadata/"]
        )
    )
)

Step 3: メタデータフィルタを使用して検索

Python メタデータフィルタ付き検索クエリ 
response = rag.retrieval_query(
    rag_resources=[
        rag.RagResource(
            rag_corpus=rag_corpus.name,
        )
    ],
    text="プロジェクト管理のベストプラクティスについて教えてください",
    rag_retrieval_config=rag.RagRetrievalConfig(
        top_k=10,
        filter=rag.utils.resources.Filter(
            metadata_filter='category = "project_management"',
            vector_distance_threshold=0.5,
        ),
    ),
)
print(response)

⚖️ メリット・デメリット

✅ ビジネス面のメリット

  • → 検索精度の向上: メタデータ事前フィルタリングでビジネスコンテキストに合致した回答
  • → マルチテナント対応: 単一コーパスで部署・プロジェクト単位のスコープ分離
  • → 運用コスト削減: コーパス分割せずメタデータで論理的にデータ整理

✅ 技術面のメリット

  • → 検索パフォーマンス最適化: フィルタでベクトル検索対象を絞り込み
  • → 柔軟なスキーマ設計: JSON Schema ベースで自由な属性設計
  • → 既存機能との併用: ベクトル閾値やリランキングと多層フィルタリング
⚠️ 制限事項・考慮点:
  • メタデータスキーマはコーパスレベルで定義 - 全ファイルが同一スキーマに従う必要がある
  • v1beta1 API で提供 - GA 昇格時に仕様変更の可能性あり
  • メタデータの品質・一貫性がフィルタリング精度に直接影響
  • フィルタ条件式の構文を正確に把握する必要がある
  • 大規模コーパスのメタデータ更新にはファイル再インポートが必要な場合あり

💡 ユースケース

🏢 企業内ナレッジベースの部署別検索

全社横断のナレッジベースに部署名・ドキュメント種別・機密レベルなどのメタデータを付与。ユーザーは自分の部署に関連するドキュメントのみを対象に質問を実行。無関係な部署のドキュメントの混入を防ぎ、回答精度と応答速度が向上。

Python 人事部ドキュメントに限定した検索 
# 人事部のドキュメントのみを対象に検索
response = rag.retrieval_query(
    rag_resources=[rag.RagResource(rag_corpus=corpus_name)],
    text="育児休業の取得手続きについて",
    rag_retrieval_config=rag.RagRetrievalConfig(
        top_k=5,
        filter=rag.utils.resources.Filter(
            metadata_filter='department = "hr"',
        ),
    ),
)

🌐 多言語ドキュメントの言語別フィルタリング

多言語技術ドキュメントをコーパスに格納し、ユーザーの使用言語に応じて対象を絞り込み。言語の不一致によるノイズを排除し、最適化されたコンテキストを提供。

Python 日本語ドキュメントに限定した検索 
# 日本語ドキュメントのみを対象に検索
response = rag.retrieval_query(
    rag_resources=[rag.RagResource(rag_corpus=corpus_name)],
    text="Kubernetes クラスタのスケーリング方法",
    rag_retrieval_config=rag.RagRetrievalConfig(
        top_k=10,
        filter=rag.utils.resources.Filter(
            metadata_filter='language = "ja"',
            vector_distance_threshold=0.5,
        ),
    ),
)

Vertex AI RAG Engine のメタデータ検索機能は、RAG アプリケーションの検索精度を大幅に向上させる重要なアップデートです。スキーマベースのメタデータ管理と検索時のフィルタリングにより、大規模なナレッジベースでも的確なコンテキスト取得が可能になります。RAG アプリケーションを運用中の場合は、既存のコーパスにメタデータスキーマを追加し、フィルタリング機能を活用することを推奨します。
