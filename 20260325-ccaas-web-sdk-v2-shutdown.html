📋 概要

Google Cloud は、Contact Center AI Platform (CCAI Platform) の Web SDK v2 を 2026 年 6 月 26 日に完全シャットダウンすることを発表しました。Web SDK v2 は、ウェブアプリケーションにコンタクトセンター機能 (チャット、音声通話など) を統合するためのクライアントサイド SDK です。

2025 年 6 月 26 日に Web SDK v3 が GA としてリリースされており、約 1 年間の移行期間が設けられています。本アナウンスは 2025 年 12 月 23 日の初回通知に続く再通知であり、シャットダウン期限まで残り約 3 か月のタイミングでの注意喚起です。

⚠️ 重要: 2026 年 6 月 26 日以降、Web SDK v2 は完全に動作しなくなります。期限までに v3 への移行を完了しなければ、ウェブサイトのコンタクトセンター統合が機能停止します。

🔄 Before / After 比較

❌ Before (Web SDK v2)

✗Headless SDK との統合が限定的
✗チャット履歴の閲覧・トランスクリプトダウンロード不可
✗プロアクティブチャットに OR/AND 条件が使用不可
✗エージェントからのファイル添付不可
✗外部ホスト (websdk.ujet.co) からのスクリプト配信

✅ After (Web SDK v3)

✓Headless SDK ベースで全メソッド利用可能
✓チャット履歴の閲覧・トランスクリプトダウンロード対応
✓OR/AND 条件演算子によるプロアクティブチャット
✓エージェントによるチャット中ファイル添付
✓自社 CCAI Platform インスタンスからの配信

🏗️ アーキテクチャ図

Web SDK v2 は外部ホストから loader.js を読み込む構成でしたが、v3 では自社の CCAI Platform インスタンスから widget.js を読み込む構成に変更されています。v3 は Headless SDK を基盤としており、より柔軟なカスタマイズと機能拡張が可能です。

✨ Web SDK v3 の主要機能

🧩 Headless SDK ベース

Headless SDK の全メソッドにアクセス可能。独自 UI の構築や高度なカスタマイズが容易に。

📜 チャット履歴・トランスクリプト

以前のチャット履歴の閲覧とトランスクリプトのダウンロードが可能に。

📝 HTML ウェブフォーム

カスタマイズされたデータ収集フローを HTML ウェブフォームで実装可能。

🔍 プロアクティブチャット強化

OR/AND 条件演算子によるプロアクティブチャットトリガーのサポート。

📎 エージェントファイル添付

エージェントがチャット中にファイルを添付可能。チャットオーディオの無効化機能も追加。

🎯 システムメッセージ分類

standard, confirmation, error のカテゴリ分類。セッション終了時のポストセッション転送にも対応。

🛠️ 移行手順

前提条件

✓CCAI Platform の管理者権限を持つアカウント
✓対象ウェブサイトのソースコードへのアクセス権
✓CCAI Platform インスタンスの Company Key と Host 情報

Step 1: 既存の v2 スクリプトの特定

現在のウェブサイトのソースコードから v2 スクリプトタグを探します。

HTML v2 スクリプト (削除対象) 
<script type="module" src="https://websdk.ujet.co/v2/loader.js"></script>

Step 2: v3 スクリプトへの置き換え

HTML v3 スクリプト (置き換え後) 
<script src="https://{your_ccaas_host}/web-sdk/v3/widget.js"></script>

Step 3: マウントポイントの追加と初期化コード更新

HTML v3 初期化コード 
<!-- マウントポイント用の空要素を追加 -->
<div id="ccaas-widget"></div>

<script>
  var ccaas = new UJET({
    companyId: "{COMPANY_KEY}",
    host: "https://{your_ccaas_host}",
    authenticate: getAuthToken
  });

  // v2 で初期化時に指定していたオプションを config() に移行
  ccaas.config({
    disableAttachment: true,
    customData: {
      version: { label: 'Version', value: '1.0.0' }
    },
    hideNewConversation: true,
    hideDownloadTranscriptOptions: false
  });

  ccaas.mount('#ccaas-widget');
</script>

Step 4: イベントリスナーの更新

JavaScript イベントリスナーの移行 
// v2 のイベントリスナー (変更前)
// ccaas.on('chat:update', (chat) => { console.log(chat) })

// v3 のイベントリスナー (変更後)
const client = ccaas.client;
client.on('chat.updated', (chat) => { console.log(chat) });

📈 メリット

💼 ビジネス面

→顧客体験の向上: チャット履歴の閲覧やトランスクリプトダウンロードなど、エンドユーザー向け機能が充実しサポート品質が向上
→柔軟なデータ収集: HTML ウェブフォームにより、カスタマイズされたデータ収集フローを実装。顧客情報の事前取得が効率化

🔧 技術面

→Headless SDK 統合: 全メソッドが利用可能になり、高度なカスタマイズや独自 UI 構築が容易に
→自社インスタンス配信: 外部依存が減少し、セキュリティとパフォーマンスが改善
→npm パッケージ対応: @ujet/websdk-headless によりモダンな開発ワークフローに統合可能

⚠️ デメリット・制約事項

制限事項
✗2026/6/26 以降、v2 は完全に動作停止。期限までの移行完了が必須
✗v2 と v3 でイベント名・メソッドの互換性がないため、カスタム実装部分の書き換えが必要
💡 考慮すべき点
→customData や disableAttachment を初期化オプションに含めていた場合、v3 では ccaas.config() に移動が必要
→CSP 設定時は https://{your_ccaas_host}/ を script-src と frame-src に追加が必要
→Private Preview 版 v3 使用時は、自社インスタンスから widget.js にアクセスするよう更新が必要
→移行作業には開発チームのサポートが必要 (特に Headless SDK 使用時)

💡 ユースケース

🌐 ユースケース 1: 基本的なチャットウィジェットの移行

シナリオ: 既存のウェブサイトで Web SDK v2 を使用してチャットウィジェットを提供している企業が v3 に移行する。

HTML v3 基本実装例 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>CCAI Platform Web SDK v3</title>
</head>
<body>
  <div id="ccaas-widget"></div>
  <script src="https://{your_ccaas_host}/web-sdk/v3/widget.js"></script>
  <script>
    var ccaas = new UJET({
      companyId: "$COMPANY_ID",
      host: "$HOST",
      authenticate: getAuthToken
    });
    ccaas.mount("#ccaas-widget");

    function getAuthToken() {
      return fetch('/auth/token').then(function(resp) {
        return resp.json();
      });
    }
  </script>
</body>
</html>

効果: 最小限のコード変更で v3 への移行が完了し、Headless SDK の全機能にアクセス可能に。

🎨 ユースケース 2: Headless SDK を活用した独自 UI の構築

シナリオ: 企業独自のチャット UI を構築したい場合に、Headless SDK をベースにカスタムウィジェットを開発する。

効果: v3 の Headless SDK を直接使用することで、ブランドに合わせた完全カスタム UI を構築でき、標準ウィジェットでは実現できないユーザー体験を提供可能。

📝 まとめ

Web SDK v2 は 2026 年 6 月 26 日に完全シャットダウンされます。現在 v2 を使用している企業は早急に v3 への移行計画を策定し実施する必要があります。

移行ガイドと実装例が公式に提供されているため、これらを活用して移行作業を進めることを推奨します。v3 では Headless SDK ベースのアーキテクチャにより機能が大幅に強化されており、移行は単なる必須対応ではなく、コンタクトセンター統合の品質向上の機会でもあります。
