📊 Looker: Table Visualization Improvements

テーブルビジュアライゼーション改善
Looker Preview 2026-04-07

☁️ 概要

Looker のテーブルビジュアライゼーションに 5 つの改善機能 がプレビューとして追加されました。 ページネーション、ヘッダー表示制御、ボーダーカスタマイズ、セルハイライト、文字列フィールドの条件付き書式設定により、 テーブルの表現力と操作性が大幅に向上します。

この機能はデフォルトで無効化されており、Looker 管理者が Admin Settings の Preview Features ページから有効化する必要があります。 プレビュー段階のため「Pre-GA Offerings Terms」が適用されます。

❌ アップデート前の課題

  • 大量行が一括表示され閲覧が困難
  • ヘッダー表示/非表示の制御不可
  • ボーダースタイルのカスタマイズ手段が限定的
  • セルホバー時のハイライト機能なし
  • 条件付き書式は数値フィールドのみ対応

✅ アップデート後の改善

  • ページネーションで 50/100/250/500/1,000 行ずつ表示
  • ヘッダー表示/非表示をスイッチで切替
  • ボーダーの幅・色・スタイルをカスタマイズ
  • セルホバーで行と列をハイライト表示
  • 文字列フィールドにも条件付き書式を適用可能

⚡ 主要機能 (5つの新機能)

1. 📄 ページネーション (Pagination)

50 行以上を含むテーブルでページネーションを有効化できます。

  • 「Paginate Rows」チェックボックスで有効化
  • 1 ページあたりの行数: 50 / 100 / 250 / 500 / 1,000 行から選択
  • ダッシュボード上での変更にはダッシュボード編集モードが必要

2. 👁️ テーブルヘッダーの表示/非表示 (Show/Hide Headers)

「Show Headers」スイッチでテーブルヘッダーの表示/非表示を切り替え可能です。

  • ダッシュボード内でテーブルをよりコンパクトに表示したい場合に有用

3. 🎨 テーブルボーダー (Borders)

「Borders」スイッチでテーブルボーダーの表示/非表示を切り替え、詳細をカスタマイズできます。

  • ボーダー幅: ピクセル単位で指定
  • ボーダー色: カラーピッカーで任意の色を指定
  • ボーダースタイル: solid (実線) / dotted (点線) / dashed (破線)

4. 🔦 セルハイライト (Cell Highlighting)

テーブル内のセルにマウスオーバーした際に、該当する行と列がハイライト表示されます。

  • 「Cell Highlighting」スイッチで有効化
  • 「Highlight Color」カラーピッカーでハイライト色をカスタマイズ可能
⚠️ 注意: セルハイライトが有効な場合、条件付き書式設定は使用できません。

5. 🏷️ 文字列フィールドの条件付き書式設定 (Conditional Formatting)

従来の数値フィールドに加え、文字列 (string) フィールドにも条件付き書式設定を適用できるようになりました。

  • ディメンションとメジャーの両方に対してルールを適用可能
  • 最大 3 つ のルールを定義可能
  • スケールベース (グラデーション) または特定の値に基づく書式設定
  • 背景色、フォント色、フォントスタイルをルールごとに指定可能

🛠️ 設定方法

📋 前提条件: Looker 管理者権限を持つアカウントと Looker インスタンスへのアクセスが必要です。
1

プレビュー機能の有効化

Looker の管理パネル (Admin) にアクセスし、General セクション配下の Preview Features ページを開きます。 Table Visualization Improvements を有効化します。

2

テーブルビジュアライゼーションの編集

Explore または Look でクエリを実行し、ビジュアライゼーションバーの Edit メニューを開きます。 Plot タブでページネーションを設定し、Formatting タブでヘッダー、ボーダー、セルハイライト、条件付き書式を設定します。

3

ダッシュボードでの利用

ダッシュボードに配置したテーブルでページネーション設定を変更する場合は、ダッシュボードの編集モードに入る必要があります。

⚖️ メリット / デメリット

✅ メリット

  • ビジネス面: ページネーションでデータ閲覧性が向上し、分析効率がアップ
  • ビジネス面: ボーダーやハイライトで経営層向けダッシュボードの見栄えを改善
  • ビジネス面: 文字列フィールドの条件付き書式でステータスの視覚的識別が可能に
  • 技術面: ヘッダー制御やボーダーカスタマイズで柔軟なテーブル設計が可能
  • 技術面: セルハイライトで大きなテーブルでのデータ追跡が容易に
  • 技術面: Looker ネイティブ機能のため追加開発が不要

⚠️ デメリット / 制約

  • プレビュー段階のためサポートが限定的
  • セルハイライトと条件付き書式は排他的 (同時使用不可)
  • Cell Visualization が有効な列には条件付き書式不可
  • 小計 (subtotals) があるテーブルでは条件付き書式不可
  • 条件付き書式ルールは最大 3 つまで
  • ページネーションは 50 行未満では利用不可
  • GA 時に仕様が変更される可能性あり
  • 管理者による明示的な有効化が必要

💡 ユースケース

📋 ユースケース 1: 大量の取引データの閲覧

シナリオ: 数千件の取引データをテーブルで表示し、ビジネスアナリストがページごとにデータを確認したい場合

効果: ページネーション機能により、250 行ずつ表示するなど適切な単位でデータを閲覧でき、ブラウザのパフォーマンスも向上

🏷️ ユースケース 2: ステータス別の色分け表示

シナリオ: 注文ステータス (「処理中」「出荷済み」「キャンセル」など) の文字列フィールドに対して色分けを行い、一目でステータスを把握したい場合

効果: 文字列フィールドへの条件付き書式設定により、ステータスごとに背景色やフォント色を変更でき、視覚的な識別が容易に

🎨 ユースケース 3: 経営ダッシュボードのデザイン最適化

シナリオ: 経営層向けダッシュボードでテーブルのデザインをブランドカラーに合わせてカスタマイズしたい場合

効果: ボーダーの色・スタイルのカスタマイズとヘッダーの表示制御により、統一感のあるダッシュボードデザインを実現
